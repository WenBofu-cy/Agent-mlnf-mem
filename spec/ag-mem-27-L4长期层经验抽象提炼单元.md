# ag-mem-27-L4长期层经验抽象提炼单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-27 |
| 模块名称 | L4长期层经验抽象提炼单元 |
| 所属分区 | 三、漏斗二：任务经验漏斗 / 五层存储 |
| 核心职责 | 接收 ag-mem-26（L4长期层存储单元）发起的抽象提炼触发指令，对L4层中来自同一场景分槽的同类经验条目集进行通用规则提取。通过聚类分析、频繁模式挖掘与共性特征抽取，将多条相似但含有个性化差异的经验提炼为一条不依赖特定用户、可跨场景复用的通用规则。提炼后的规则存入L4对应条目的关联规则字段，同时返回给ag-mem-26更新条目标记。为L5核心层的安全底线固化提供规则候选。不参与经验写入或晋升决策，仅执行规则提取与抽象化处理 |
| 依赖模块 | ag-mem-26（L4长期层存储单元，发送抽象提炼触发指令与同类经验条目集） |
| 被依赖模块 | ag-mem-26（返回抽象提炼完成回执与提炼后的规则列表）、ag-mem-28（L5核心层存储单元，可选地将高置信度通用规则推送至L5） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 无提炼任务，等待指令 | 系统初始化完成，无待处理请求 |
| 特征提取 | `FEATURE_EXTRACT` | 正在从同类经验条目集中提取共性特征 | 收到 ag-mem-26 的抽象提炼触发指令 |
| 规则生成 | `RULE_GENERATE` | 正在基于共性特征生成通用规则 | 特征提取完成，存在有效共性特征 |
| 结果输出 | `OUTPUTTING` | 提炼完成，正在返回规则列表 | 规则生成完成 |
| 数据不足 | `INSUFFICIENT_DATA` | 同类经验条目数不足以支持有效提炼 | 条目数 < 3 或共性特征不显著 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |

## 提炼规则

### 一、提炼前提条件

| 条件 | 阈值 | 说明 |
|------|:---:|------|
| 最小同类经验条目数 | **≥ 3条** | 少于3条无法提取统计显著的共性特征 |
| 共性特征显著度 | **≥ 0.6** | 工具调用序列或任务特征向量的聚类中心与各条目的平均相似度 |
| 结果标签一致性 | **≥ 70%相同** | 同一组内经验的结果标签应有较高一致性 |
| 单次最大处理条目数 | **50条** | 超过此数量分批处理 |

### 二、提炼方法

1. **工具调用序列模式挖掘**：
   - 提取所有条目的工具调用序列，去除序列中的个性化参数
   - 使用序列对齐算法找出最长公共子序列（LCS）
   - 公共子序列长度占比超过60%即认定为通用工具调用模式

2. **任务特征向量聚类**：
   - 将所有条目的任务特征向量进行聚类（K-Means，K=1）
   - 计算聚类中心与各条目的余弦相似度均值
   - 均值 ≥ 0.6 即认定为共性显著

3. **通用规则生成**：
   - 以聚类中心作为通用规则的任务特征
   - 以公共工具调用序列作为通用规则的执行模板
   - 以条目中最高重要度的经验作为规则描述参考

### 三、规则置信度计算

```
规则置信度 = 0.4 × 共性特征显著度 + 0.3 × 结果标签一致性 + 0.2 × 条目重要度均值 + 0.1 × 条目数归一化因子

其中：
- 条目数归一化因子 = MIN(同类经验条目数 / 20, 1.0)
- 规则置信度范围 [0, 1]
```


## 核心处理逻辑

```
FUNCTION l4_abstraction_main_loop():
    STATE_IDLE = IDLE
    STATE_FEATURE = FEATURE_EXTRACT
    STATE_RULE = RULE_GENERATE
    STATE_OUTPUT = OUTPUTTING
    STATE_INSUFF = INSUFFICIENT_DATA
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_IDLE
    初始化提炼统计计数器

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_IDLE

        // 第2步：接收抽象提炼触发指令
        IF 收到 ag-mem-26 抽象提炼触发指令:
            SET internal_state = STATE_FEATURE
            同类经验条目集 = 指令.同类经验分组
            来源槽位编号 = 指令.来源槽位编号
            触发原因 = 指令.触发原因

            // 2a. 校验最小条目数
            IF 同类经验条目集.size < 3:
                SET internal_state = STATE_INSUFF
                向 ag-mem-26 返回数据不足通知(
                    原因="同类经验条目数不足",
                    当前条目数=同类经验条目集.size,
                    最小需求=3,
                    共性特征评分=0.0
                )
                SET internal_state = STATE_IDLE
                CONTINUE

            // 2b. 提取工具调用序列公共子序列
            所有序列 = 提取所有条目的工具调用序列(同类经验条目集)
            公共子序列 = 计算最长公共子序列(所有序列)
            序列一致性 = 公共子序列.长度 / 所有序列.平均长度

            IF 序列一致性 < 0.60:
                SET internal_state = STATE_INSUFF
                向 ag-mem-26 返回数据不足通知(
                    原因="工具调用序列一致性不足",
                    当前条目数=同类经验条目集.size,
                    最小需求=3,
                    共性特征评分=序列一致性
                )
                SET internal_state = STATE_IDLE
                CONTINUE

            // 2c. 任务特征向量聚类分析
            所有特征向量 = 提取所有条目的任务特征向量(同类经验条目集)
            聚类中心 = 计算K均值聚类中心(所有特征向量, K=1)
            各条目相似度 = 计算余弦相似度(聚类中心, 所有特征向量)
            共性显著度 = MEAN(各条目相似度)

            IF 共性显著度 < 0.6:
                SET internal_state = STATE_INSUFF
                向 ag-mem-26 返回数据不足通知(
                    原因="任务特征共性不显著",
                    当前条目数=同类经验条目集.size,
                    最小需求=3,
                    共性特征评分=共性显著度
                )
                SET internal_state = STATE_IDLE
                CONTINUE

            // 2d. 结果标签一致性
            标签分布 = 统计结果标签分布(同类经验条目集)
            主导标签比例 = MAX(标签分布.values()) / 同类经验条目集.size
            结果标签 = 主导标签对应的标签名 IF 主导标签比例 >= 0.7 ELSE "混合"

            IF 主导标签比例 < 0.7:
                // 标签一致性不足，但不阻止提炼，仅降低置信度
                标签一致性 = 主导标签比例
            ELSE:
                标签一致性 = 1.0

            // 第3步：生成通用规则
            SET internal_state = STATE_RULE

            // 3a. 计算规则置信度
            条目重要度均值 = MEAN(同类经验条目集中各条目的重要度I值)
            条目数归一化 = MIN(同类经验条目集.size / 20.0, 1.0)
            规则置信度 = 0.4 × 共性显著度 + 0.3 × 标签一致性 + 0.2 × 条目重要度均值 + 0.1 × 条目数归一化

            // 3b. 选择规则描述参考（取重要度最高的条目）
            最高重要度条目 = 按I值降序排列(同类经验条目集)[0]

            // 3c. 生成通用规则
            通用规则 = {
                规则ID: "RULE-L4-" + 生成UUID(),
                规则描述: 基于最高重要度条目生成规则描述(最高重要度条目, 公共子序列),
                适用范围: {
                    场景类别: 来源槽位编号对应的场景类别,
                    工具类型: 提取公共工具类型(公共子序列),
                    任务类型: 基于聚类中心推断任务类型(聚类中心)
                },
                置信度: 规则置信度,
                来源条目ID列表: 同类经验条目集中所有条目ID,
                规则类型: IF 规则置信度 >= 0.80 THEN "高置信度规则" ELSE "一般规则",
                生成时间戳: NOW()
            }

            // 第4步：输出结果
            SET internal_state = STATE_OUTPUT

            // 4a. 发送通用规则列表至L4
            向 ag-mem-26 发送通用规则列表([通用规则])

            // 4b. 若为高置信度规则，可选推送至L5
            IF 规则置信度 >= 0.85:
                向 ag-mem-28 发送高置信度通用规则(通用规则)

            // 4c. 返回完成回执
            向 ag-mem-26 返回抽象提炼完成回执(
                提炼范围=来源槽位编号,
                生成规则数=1,
                提炼耗时=NOW() - 提炼开始时间,
                置信度=规则置信度,
                所依据条目ID列表=同类经验条目集中所有条目ID
            )

            // 4d. 更新统计与上报
            更新提炼统计计数器
            向 ag-mem-03 上报提炼状态(提炼统计)

            SET internal_state = STATE_IDLE

        SLEEP 50ms


FUNCTION 计算最长公共子序列(序列列表):
    IF 序列列表为空:
        RETURN {序列: [], 长度: 0}

    // 取最短序列作为基准，计算与其他序列的LCS
    基准序列 = 按长度升序排列(序列列表)[0]
    公共序列 = 基准序列

    FOR EACH 序列 IN 序列列表[1:]:
        公共序列 = LCS(公共序列, 序列)

    RETURN {序列: 公共序列, 长度: LEN(公共序列)}
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 同类经验条目数 < 3 | 返回数据不足通知，不执行提炼 | 条目数积累到3条以上 |
| 工具调用序列一致性 < 0.60 | 返回数据不足通知，标记“序列不一致” | 积累更相似的经验条目 |
| 任务特征共性显著度 < 0.6 | 返回数据不足通知，标记“特征不显著” | 积累更相似的经验条目 |
| 提炼结果置信度 < 0.5 | 规则仍生成并返回L4，但不推送L5，标记低置信度 | 后续条目补充后重新提炼 |
| 提炼过程中同类条目被删除 | 使用快照数据继续提炼，不受并发删除影响 | — |
| 单批次条目数 > 50 | 分批处理，每批最多50条，返回每批的提炼结果 | — |
| 紧急熔断 | 中断当前提炼，未完成的结果丢弃 | 紧急解除后恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 提炼过程中仅读取条目的特征向量和工具调用序列，不得访问原始用户输入文本 |
| S-02 | 生成的通用规则必须完全去除个性化参数，仅保留结构化的工具调用模板与任务特征 |
| S-03 | 数据不足时必须明确告知下游模块，不得使用低于3条的数据强行生成规则 |
| S-04 | 高置信度规则（≥0.85）推送至L5前必须经过安全规则库（ag-mem-45）的合规校验 |
| S-05 | 提炼操作的输入数据采用快照机制，提炼过程中原条目的并发修改不影响当前提炼结果 |
