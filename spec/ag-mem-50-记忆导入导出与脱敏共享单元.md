# ag-mem-50-记忆导入导出与脱敏共享单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-50 |
| 模块名称 | 记忆导入导出与脱敏共享单元 |
| 所属分区 | 五、存储与系统运维 |
| 核心职责 | 管理漏斗二中 L4/L5 层泛化经验的导出、脱敏处理，以及外部经验包的合规导入。在导出过程中，对经验数据执行强制脱敏——剔除所有用户个人身份信息（用户ID、会话ID、设备指纹、地理位置、原始输入文本），仅保留结构化的通用任务模板（任务特征向量、工具调用序列、成功/失败标签、泛化规则）。脱敏后的经验包可跨用户、跨实例共享，用于新系统的经验预载或开发者工具库建设。同时支持从外部导入经过安全审计的经验包，导入前必须通过格式校验与安全规则库的合规检查，防止恶意经验注入。本模块不参与经验内容的生成或修改，仅执行安全的导入导出操作 |
| 依赖模块 | ag-mem-26/28（L4/L5 存储单元，提供待导出的经验数据）、ag-mem-45（安全规则库，执行导入前的合规检查）、ag-mem-48（全局容量配额管控单元，确认导入后的容量是否充足）、ag-mem-51（记忆变更日志追溯单元，记录导入导出事件） |
| 被依赖模块 | 系统管理接口、开发者工具、其他 EM-Core 实例（通过外部文件或 API 使用导出的经验包） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 无任务，等待指令 | 系统初始化完成，无待处理请求 |
| 导出处理中 | `EXPORTING` | 正在查询经验条目并执行脱敏导出 | 收到导出请求 |
| 导入校验中 | `IMPORTING_VALIDATE` | 正在对导入经验包进行格式与安全合规校验 | 收到导入请求 |
| 导入写入中 | `IMPORTING_WRITE` | 校验通过，正在将经验条目写入漏斗二 | 导入校验通过 |
| 操作失败 | `OPERATION_FAILED` | 导入导出过程中发生不可恢复的异常 | 存储异常或校验失败 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|

| 经验导入请求 | Struct（导入经验包数据 + 导入来源 + 导入范围 + 是否覆盖已有经验 + 校验签名） | 系统管理接口、OTA 系统、其他 EM-Core 实例 | 系统部署、升级或迁移时 | 普通 |
| L4/L5 经验查询回执 | List（条目ID + 经验数据 + 元数据：来源分槽/I值/S值/V值/C值/时间戳） | ag-mem-26（L4存储）、ag-mem-28（L5存储） | 导出时查询 | **高** |
| 安全合规校验回执 | Struct（是否合规 + 违规条目列表 + 违规原因 + 建议处理） | ag-mem-45 安全规则库 | 导入校验时 | **高** |
| 容量确认回执 | Struct（当前使用率 + 可用空间 + 能否容纳导入条目数） | ag-mem-48 全局容量配额管控单元 | 导入写入前 | **高** |
| 导入写入确认 | Struct（写入条目数 + 成功数 + 失败列表 + 分配层级） | ag-mem-15~19（各场景分槽） | 导入写入完成时 | **高** |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-mem-01 总控漏斗 F₀ | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 脱敏经验包 | Struct（经验包版本 + 导出时间戳 + 来源实例ID + 脱敏经验条目列表 + 各条目元数据 + 校验签名） | 请求来源（管理接口/开发者工具） | 导出完成时 | 普通 |

## 脱敏规则与导出格式

### 一、强制脱敏字段清单（导出时必须剔除或替换）

| 字段 | 处理方式 |
|------|------|
| 用户ID | **删除**，替换为空字符串 |
| 会话ID | **删除**，替换为空字符串 |
| 设备指纹 | **删除**，替换为 "ANONYMOUS" |
| 地理位置信息 | **删除**，替换为 "UNKNOWN" |
| 用户原始输入文本 | **删除**，仅保留任务特征向量与意图标签 |
| 用户个性化偏好参数 | **删除**，替换为默认参数 |
| 用户显式反馈标记 | **删除**，替换为 "UNKNOWN" |
| 任何可关联到特定用户的其他字段 | **删除** |

### 二、保留字段（经验的核心价值）

| 字段 | 说明 |
|------|------|
| 条目ID（重新生成） | 导出时分配新的唯一ID，避免与导入实例冲突 |
| 任务特征向量 | 任务的通用特征表示 |
| 工具调用序列 | 工具名称与调用顺序 |
| 成功/失败标签 | 经验的结果标签 |
| 泛化规则（L4） | ag-mem-27 抽象提炼的通用规则 |
| S值 / V值 / C值 / I值 | 重新计算（去除V值中与用户偏好相关的部分） |
| 来源分槽类别 | 场景类别标签 |
| 时间戳 | 保留原始时间戳（仅用于时序分析） |

### 三、导出经验包格式

```
脱敏经验包 = {
    经验包版本: "1.0",
    导出时间戳: UTC时间戳,
    来源实例ID: "ANONYMIZED",
    校验签名: SHA-256(经验条目列表),
    脱敏经验条目列表: [
        {
            条目ID: 重新生成的UUID,
            任务特征向量: {...},
            工具调用序列: [...],
            结果标签: "成功 | 失败",
            泛化规则: {...} (仅L4条目),
            重要度分量: {S值, V值(重置), C值, I值(重算)},
            场景类别: "对话交互 | 工具调用 | ...",
            原始时间戳: UTC时间戳
        },
        ...
    ]
}
```


## 核心处理逻辑

```
FUNCTION memory_import_export_main_loop():
    STATE_IDLE = IDLE
    STATE_EXPORT = EXPORTING
    STATE_IMPORT_CHECK = IMPORTING_VALIDATE
    STATE_IMPORT_WRITE = IMPORTING_WRITE
    STATE_FAIL = OPERATION_FAILED
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_IDLE

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_IDLE

        // ==================== 导出流程 ====================
        // 第2步：接收经验导出请求
        IF 收到经验导出请求:
            SET internal_state = STATE_EXPORT
            导出范围 = 请求.导出范围
            导出格式 = 请求.导出格式
            最大条目数 = 请求.最大导出条目数

            // 2a. 查询指定层级的经验条目
            经验条目列表 = []
            IF 导出范围 IN ["L4", "全部"]:
                L4条目 = 向 ag-mem-26 查询L4全量条目(最大条目数)
                经验条目列表.合并(L4条目)
            IF 导出范围 IN ["L5", "全部"]:
                L5条目 = 向 ag-mem-28 查询L5全量条目(最大条目数)
                经验条目列表.合并(L5条目)

            IF 经验条目列表为空:
                向请求来源返回导出完成回执(0, 0, 0, 0, "无可导出条目")
                SET internal_state = STATE_IDLE
                CONTINUE

            // 2b. 执行强制脱敏
            脱敏条目列表 = []
            FOR EACH 条目 IN 经验条目列表:
                脱敏条目 = 执行强制脱敏(条目)
                IF 脱敏条目 != None:
                    脱敏条目列表.append(脱敏条目)

            // 2c. 构建脱敏经验包
            经验包 = 构建脱敏经验包(脱敏条目列表, 导出格式)
            经验包大小 = 估算经验包大小(经验包)

            // 2d. 生成校验签名
            经验包.校验签名 = SHA256(经验包.脱敏经验条目列表)

            // 2e. 返回结果
            向请求来源返回脱敏经验包(经验包)
            向请求来源返回导出完成回执(
                导出条目总数=经验条目列表.size,
                脱敏条目数=脱敏条目列表.size,
                导出耗时=NOW() - 导出开始时间,
                经验包大小=经验包大小,
                经验包校验签名=经验包.校验签名
            )

            // 2f. 记录导出事件日志
            向 ag-mem-51 发送导入导出事件日志("导出", 脱敏条目列表.size, 请求来源)

            SET internal_state = STATE_IDLE

        // ==================== 导入流程 ====================
        // 第3步：接收经验导入请求
        IF 收到经验导入请求:
            SET internal_state = STATE_IMPORT_CHECK
            导入经验包 = 请求.导入经验包数据
            导入来源 = 请求.导入来源

            // 3a. 校验经验包格式
            格式校验结果 = 校验经验包格式(导入经验包)
            IF NOT 格式校验结果.通过:
                向请求来源返回导入完成回执(0, 0, [], "经验包格式校验失败")
                SET internal_state = STATE_IDLE
                CONTINUE

            // 3b. 校验经验包签名
            IF NOT 校验签名(导入经验包.校验签名, 导入经验包.脱敏经验条目列表):
                向请求来源返回导入完成回执(0, 0, [], "经验包签名校验失败")
                SET internal_state = STATE_IDLE
                CONTINUE

            // 3c. 向安全规则库发起合规检查
            合规检查结果 = 向 ag-mem-45 发送安全合规校验请求(导入经验包.脱敏经验条目列表)
            IF 合规检查结果.违规条目数 > 0:
                向请求来源返回导入完成回执(
                    导入条目总数=导入经验包.脱敏经验条目列表.size,
                    成功导入数=0,
                    拒绝条目列表=合规检查结果.违规条目列表,
                    拒绝原因="安全合规校验不通过"
                )
                SET internal_state = STATE_IDLE
                CONTINUE

            // 3d. 容量确认
            容量确认 = 向 ag-mem-48 查询容量(导入经验包.脱敏经验条目列表.size)
            IF NOT 容量确认.能否容纳:
                向请求来源返回导入完成回执(0, 0, [], "存储容量不足")
                SET internal_state = STATE_IDLE
                CONTINUE

            // 3e. 开始写入（合规条目）
            SET internal_state = STATE_IMPORT_WRITE
            成功导入数 = 0
            失败条目列表 = []
            FOR EACH 脱敏条目 IN 合规检查结果.合规条目列表:
                // 根据场景类别路由到对应分槽
                目标分槽 = 根据场景类别匹配分槽(脱敏条目.场景类别)
                写入结果 = 向 目标分槽 发送经验写入请求(脱敏条目, 来源="外部导入")
                IF 写入结果.成功:
                    成功导入数++
                ELSE:
                    失败条目列表.append({条目ID: 脱敏条目.条目ID, 失败原因: 写入结果.错误信息})

            // 3f. 返回结果
            向请求来源返回导入完成回执(
                导入条目总数=导入经验包.脱敏经验条目列表.size,
                成功导入数=成功导入数,
                拒绝条目列表=失败条目列表,
                导入耗时=NOW() - 导入开始时间
            )

            // 3g. 记录导入事件日志
            向 ag-mem-51 发送导入导出事件日志("导入", 成功导入数, 导入来源)

            SET internal_state = STATE_IDLE

        SLEEP 50ms


FUNCTION 执行强制脱敏(原始条目):
    IF 原始条目.层级 == "L5":
        // L5 为永久记忆，默认不导出，除非明确授权
        IF NOT 请求.L5导出授权:
            RETURN None

    脱敏条目 = 原始条目.clone()
    脱敏条目.条目ID = 重新生成UUID()
    脱敏条目.用户ID = ""
    脱敏条目.会话ID = ""
    脱敏条目.设备指纹 = "ANONYMOUS"
    脱敏条目.地理位置信息 = "UNKNOWN"
    脱敏条目.用户原始输入文本 = None
    脱敏条目.用户个性化偏好参数 = 默认参数
    脱敏条目.用户显式反馈标记 = "UNKNOWN"
    脱敏条目.V值 = 重新计算V值(去除用户偏好部分)
    脱敏条目.I值 = 重新计算I值(脱敏条目.I₀, 脱敏条目.S值, 脱敏条目.V值, 脱敏条目.C值)

    // 保留的核心字段
    保留: 任务特征向量, 工具调用序列, 结果标签, 泛化规则(L4), S值, C值, 场景类别, 原始时间戳

    RETURN 脱敏条目
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| L5 条目导出未获授权 | 跳过 L5 条目，仅导出 L4 条目 | 请求中明确 L5 导出授权 |
| 导出条目数为 0 | 返回完成回执（全 0），不报错 | — |
| 导入经验包格式校验失败 | 拒绝导入，返回错误详情 | 提供格式正确的经验包 |
| 导入经验包签名校验失败 | 拒绝导入，记录安全事件 | 提供签名正确的经验包 |
| 导入合规检查不通过 | 拒绝导入，返回违规条目列表 | 修正违规条目后重试 |
| 导入时容量不足 | 拒绝导入，提示清理存储 | 容量释放后重试 |
| 导入写入部分失败 | 已成功写入的条目保留，仅失败的条目回滚 | — |
| 紧急熔断 | 中断当前操作，未完成的导入导出任务丢弃 | 紧急解除后恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| E-01 | 导出时必须执行强制脱敏，禁止任何包含用户个人身份信息的字段出现在经验包中 |
| E-02 | L5 核心层经验默认不导出，仅在请求明确授权且通过双重确认后方可导出 |
| E-03 | 导入经验包必须通过格式校验、签名校验与安全合规校验三道关卡，防止恶意经验注入 |
| E-04 | 导入的经验条目 V 值重置为基线值，确保不与新用户的偏好数据混淆 |
| E-05 | 所有导入导出操作必须记录完整的不可变事件日志，包括操作来源、条目数、校验签名 |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-M50-01 | `IDLE`，请求导出 L4 经验 | 导出请求（导出范围=L4，格式=JSON，最大条目=100） | 返回脱敏经验包，所有隐私字段已剔除 |
| TC-M50-02 | `IDLE`，请求导出 L5 经验（未授权） | 导出请求（导出范围=L5，L5导出授权=False） | L5 条目被跳过，仅导出 L4 或返回空 |
| TC-M50-03 | `IDLE`，导入有效经验包 | 导入请求（经验包格式正确、签名有效、合规通过） | 校验通过，写入漏斗二，返回成功导入数 |
| TC-M50-04 | `IDLE`，导入签名无效的经验包 | 导入请求（经验包签名校验失败） | 拒绝导入，返回“签名校验失败” |
| TC-M50-05 | `IDLE`，导入合规检查不通过 | 导入请求（经验包包含违规工具调用） | 拒绝导入，返回违规条目列表 |
| TC-M50-06 | `IDLE`，导入时容量不足 | 导入请求 + 容量确认返回“不能容纳” | 拒绝导入，返回“存储容量不足” |
