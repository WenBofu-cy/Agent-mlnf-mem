# ag-mem-44-独立知识库 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-44 |
| 模块名称 | 独立知识库 |
| 所属分区 | 四、漏斗外挂扩展区（物理隔离） |
| 核心职责 | 作为漏斗外挂扩展区的结构化知识存储库，对接工具使用手册、API 文档、常见问题库、系统运维手册等结构化知识的统一管理与查询。为 ECC 认知大脑的工具选择模块（ag-ecc-03）提供工具描述与参数约束查询，为安全仲裁模块（ag-ecc-04）提供操作安全等级参考，为意图解析模块（ag-ecc-01）提供领域术语与实体词典。本库完全独立于双漏斗记忆系统运行，不参与记忆沉淀、筛选、晋升与遗忘机制。知识内容在系统部署前预置，支持定期离线更新。仅提供只读查询服务，不参与任何认知决策 |
| 依赖模块 | 无（作为独立知识服务，不依赖记忆系统内部模块） |
| 被依赖模块 | ag-mem-01（总控漏斗 F₀，可转发来自 ECC 的知识查询请求）、ag-ecc-01（意图解析模块）、ag-ecc-03（工具选择模块）、ag-ecc-04（安全仲裁模块） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常服务 | `NORMAL_SERVICE` | 知识库加载完成，正常响应查询 | 系统初始化完成，知识索引构建成功 |
| 知识加载中 | `LOADING` | 正在加载或更新知识库数据 | 系统启动时或收到知识更新指令 |
| 知识库降级 | `DEGRADED` | 部分知识索引损坏或不可用，仅返回未受损部分 | 知识库完整性校验失败 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 知识查询请求 | Struct（请求来源模块 + 查询类型：工具描述/API文档/FAQ/操作安全等级/领域术语 + 查询关键词列表 + 最大返回条数 + 匹配模式：精确/模糊/语义） | ag-mem-01（转发 ECC 请求）、ag-ecc-01/03/04（直接查询） | ECC 需要获取结构化知识辅助决策时 | **高** |
| 知识库更新指令 | Struct（更新范围：全量/增量 + 更新数据包 + 版本号 + 校验签名） | 系统管理接口或 OTA 升级系统 | 知识库需要更新时（低频操作） | 普通 |
| 知识库完整性校验指令 | Struct（校验范围 + 校验深度） | ag-mem-01 总控漏斗 F₀（定期触发） | 系统启动时或定期（每24小时） | 普通 |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-mem-01 总控漏斗 F₀ | 模式切换或紧急事件时 | **紧急** |

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 知识查询结果 | Struct（匹配知识条目列表 + 各条目置信度 + 来源版本 + 查询耗时） | 请求来源模块 | 查询完成时 | **高** |
| 知识库状态上报 | Struct（当前状态 + 知识条目总数 + 各分类条目数 + 最近更新时间 + 版本号） | ag-mem-01（周期性）、ag-mem-03（调度单元） | 初始化完成或状态变更时 | 普通 |
| 知识库完整性报告 | Struct（总条目数 + 校验通过数 + 损坏条目列表 + 校验耗时） | ag-mem-01 总控漏斗 F₀ | 完整性校验完成时 | 普通 |
| 知识更新确认回执 | Struct（更新结果 + 新增条目数 + 更新条目数 + 删除条目数 + 新版本号） | 更新发起方（管理接口/OTA） | 知识更新完成时 | 普通 |


## 知识分类与条目结构

### 一、知识分类

| 分类ID | 分类名称 | 内容说明 | 典型条目数 |
|:---:|------|------|:---:|
| TOOL | 工具使用手册 | 各注册工具的名称、功能描述、参数约束、返回值格式、调用示例 | 50-200 |
| API | API 参考文档 | 外部 API 的端点、请求方法、认证方式、速率限制、错误码 | 30-100 |
| FAQ | 常见问题库 | 常见任务的标准处理流程、已知问题的解决方案、最佳实践 | 20-80 |
| SAFETY | 操作安全等级 | 工具与操作的安全等级定义、权限要求、审计规则 | 30-50 |
| TERM | 领域术语词典 | 各业务领域的专有名词、同义词映射、实体识别规则 | 100-500 |
| MANUAL | 系统运维手册 | 系统配置参数、故障排查指南、日志说明 | 20-50 |

### 二、知识条目结构

```
知识条目 = {
    条目ID: "KB-{分类}-{UUID}",
    分类: "TOOL | API | FAQ | SAFETY | TERM | MANUAL",
    标题: 知识条目标题,
    关键词列表: [关键词1, 关键词2, ...],
    内容: 结构化知识内容（JSON或Markdown格式）,
    关联条目ID列表: [相关条目ID],
    安全等级: "公开 | 内部 | 敏感",
    版本: 知识版本号,
    更新时间: 最后更新时间戳,
    来源: "预置 | 更新 | 人工录入"
}
```


## 核心处理逻辑

```
FUNCTION knowledge_base_main_loop():
    STATE_LOADING = LOADING
    STATE_NORMAL = NORMAL_SERVICE
    STATE_DEGRADED = DEGRADED
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_LOADING
    初始化知识索引（倒排索引 + 分类索引 + 关键词索引）
    初始化查询缓存（LRU，最大500条）

    // 系统启动时加载知识库
    加载结果 = 加载知识库数据()
    IF 加载结果.完整性校验通过:
        SET internal_state = STATE_NORMAL
        构建全量知识索引()
    ELSE:
        SET internal_state = STATE_DEGRADED
        构建部分知识索引（跳过损坏条目）

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_NORMAL

        // 第2步：接收知识查询请求
        IF 收到知识查询请求:
            IF internal_state NOT IN [STATE_NORMAL, STATE_DEGRADED]:
                向请求来源返回查询失败("知识库当前不可用")
                CONTINUE

            查询类型 = 请求.查询类型
            关键词列表 = 请求.查询关键词列表
            匹配模式 = 请求.匹配模式

            // 2a. 检查查询缓存
            缓存键 = 生成缓存键(查询类型, 关键词列表, 匹配模式)
            IF 查询缓存.存在(缓存键):
                向请求来源返回知识查询结果(查询缓存[缓存键], 来源="缓存")
                CONTINUE

            // 2b. 按查询类型限定搜索范围
            IF 查询类型 != None:
                搜索范围 = 知识索引.按分类筛选(查询类型)
            ELSE:
                搜索范围 = 知识索引.全量

            // 2c. 按匹配模式执行检索
            CASE 匹配模式 OF:
                "精确":
                    匹配条目 = 精确匹配(搜索范围, 关键词列表)
                "模糊":
                    匹配条目 = 模糊匹配(搜索范围, 关键词列表)
                "语义":
                    匹配条目 = 语义相似度匹配(搜索范围, 关键词列表)

            // 2d. 排序与截断
            匹配条目 = 按匹配置信度降序排列(匹配条目)
            匹配条目 = 截断到最大返回条数(匹配条目, 请求.最大返回条数)

            // 2e. 构建查询结果
            查询结果 = 构建知识查询结果(匹配条目, 当前版本号)

            // 2f. 写入缓存
            查询缓存[缓存键] = 查询结果

            // 2g. 返回结果
            向请求来源返回知识查询结果(查询结果)

        // 第3步：接收知识库更新指令
        IF 收到知识库更新指令:
            // 3a. 校验更新包签名
            IF NOT 校验签名(指令.校验签名):
                向更新发起方返回知识更新确认回执(失败, "签名校验失败")
                CONTINUE

            SET internal_state = STATE_LOADING

            // 3b. 备份当前知识库
            旧知识库快照 = 当前知识库数据.clone()

            // 3c. 执行更新
            更新结果 = 执行知识更新(指令.更新数据包, 指令.更新范围)
            IF 更新结果.成功:
                重新构建知识索引()
                清空查询缓存()
                更新版本号 = 指令.版本号
                SET internal_state = STATE_NORMAL
            ELSE:
                恢复旧知识库快照
                SET internal_state = STATE_NORMAL
                向更新发起方返回知识更新确认回执(失败, 更新结果.错误信息)
                CONTINUE

            // 3d. 返回确认
            向更新发起方返回知识更新确认回执(成功, 更新统计, 新版本号)

        // 第4步：接收完整性校验指令
        IF 收到完整性校验指令:
            校验结果 = 执行知识库完整性校验()
            向 ag-mem-01 返回知识库完整性报告(校验结果)
            IF 校验结果.损坏条目数 > 0:
                SET internal_state = STATE_DEGRADED
            ELSE IF internal_state == STATE_DEGRADED:
                SET internal_state = STATE_NORMAL

        // 第5步：周期性状态上报
        IF 距上次状态上报 >= 60秒:
            向 ag-mem-01 上报知识库状态(当前状态, 条目统计, 版本号)

        SLEEP 10ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 知识库数据文件缺失或损坏 | 进入 DEGRADED 状态，仅加载未损坏部分，上报告警 | 重新加载有效数据文件 |
| 查询关键词列表为空 | 返回空结果 + 提示“请输入查询关键词” | — |
| 知识更新签名校验失败 | 拒绝更新，记录安全事件，不修改知识库 | 使用正确签名重试 |
| 知识更新过程中系统断电 | 重启后自动回滚至更新前旧版本 | 重新尝试更新 |
| 查询缓存满（达500条上限） | 按 LRU 策略淘汰最久未访问的缓存条目 | — |
| 紧急熔断 | 暂停服务，保持当前知识索引不变 | 紧急解除后恢复 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| 内部调度总线 / MemoryBus | 读 | 知识查询请求 | 只读 | ag-mem-01（转发 ECC）、ag-ecc-01/03/04 发送 |
| 管理接口 / OTA 总线 | 读 | 知识库更新指令 | 只读 | 管理接口 / OTA 系统 |
| 内部调度总线 | 读 | 知识库完整性校验指令 | 只读 | ag-mem-01 发送 |
| 内部调度总线 / MemoryBus | 写 | 知识查询结果 | 专属写入 | 向请求来源返回 |
| 内部调度总线 | 写 | 知识库状态上报 | 周期性写入 | 向 ag-mem-01/03 发送 |
| 内部调度总线 | 写 | 知识库完整性报告 | 事件触发写入 | 向 ag-mem-01 发送 |
| 管理接口 / OTA 总线 | 写 | 知识更新确认回执 | 事件触发写入 | 向更新发起方返回 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 知识库为只读数据，运行时不得被任何控制模块或自动化流程修改 |
| S-02 | 知识更新必须通过签名校验与完整性验证，防止恶意篡改 |
| S-03 | 知识库更新前必须完整备份当前版本，更新失败时原子回滚 |
| S-04 | 安全等级为“敏感”的知识条目仅向授权模块（ag-ecc-04 安全仲裁模块）开放查询 |
| S-05 | 本库完全独立于漏斗记忆系统，不参与记忆沉淀、晋升、遗忘任何环节 |
