# ag-mem-34-基础重要度I₀赋值单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-34 |
| 模块名称 | 基础重要度I₀赋值单元 |
| 所属分区 | 三、漏斗二：任务经验漏斗 / 三维重要度计算引擎 |
| 核心职责 | 在每条新经验写入漏斗二时，基于经验的生成来源、任务类型、场景标签与语义特征，赋予基础重要度初始值I₀（0.0–1.0）。I₀是三维重要度I值的计算起点，反映该经验在未被任何S/V/C信号增强或衰减前的“先天重要性”。不同来源和类型的经验拥有不同的I₀基线：ECC主动请求记录的经验高于被动观测经验，工具调用类高于对话交互类，高风险操作高于普通操作。不参与后续重要度聚合计算或晋升决策，仅负责I₀的初始赋值 |
| 依赖模块 | ag-mem-15~19（各场景分槽，在写入新经验时附带经验元数据供I₀赋值）、ag-mem-31（安全显著性S值计算单元，可参考S值辅助确定高安全经验的I₀） |
| 被依赖模块 | ag-mem-36（综合重要度I值聚合计算单元，消费I₀值）、ag-mem-35（三维权重系数配置单元，提供I₀赋值规则配置） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 无赋值请求，等待输入 | 系统初始化完成，无待处理经验 |
| 赋值进行中 | `ASSIGNING` | 正在基于经验元数据计算I₀值 | 收到各场景分槽的I₀赋值请求 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| I₀赋值请求 | Struct（条目ID + 经验生成来源 + 任务类型 + 场景标签 + 语义特征向量 + 工具调用列表 + 是否涉及敏感操作 + S值参考） | ag-mem-15~19 各场景分槽（通过内部调度总线） | 每次新经验写入L1前 | **高** |
| I₀赋值规则配置 | Struct（各来源类型的基线I₀ + 各任务类型的调整系数 + 敏感操作额外加分表） | ag-mem-35 三维权重系数配置单元 | 系统初始化加载，配置更新时 | 普通 |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-mem-01 总控漏斗F0 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| I₀赋值结果 | Struct（条目ID + I₀值 + 赋值依据摘要 + 调整因子列表） | 请求来源分槽（ag-mem-15~19）、ag-mem-36（重要度聚合） | 赋值完成时 | **高** |
| 赋值状态上报 | Struct（当前状态 + 最近I₀分布统计 + 各类来源I₀均值） | ag-mem-03 漏斗二专属调度单元 | 周期性（每60秒）或状态变更时 | 普通 |


## I₀赋值规则

### 一、经验生成来源基线

| 生成来源 | 基线I₀ | 说明 |
|----------|:---:|------|
| ECC主动请求记录（显式记忆） | 0.60 | ECC认知大脑明确要求记录的经验，重要度最高 |
| 任务执行自动记录（成功） | 0.45 | 任务成功完成后的自动经验沉淀 |
| 任务执行自动记录（失败） | 0.50 | 失败经验略高于成功自动记录，以便从错误中学习 |
| 用户显式反馈触发记录 | 0.55 | 用户点赞/收藏等行为触发的经验记录 |
| 被动观测记录（背景行为） | 0.30 | 系统自动观测到的用户行为模式 |
| 系统安全事件触发记录 | 0.70 | 涉及安全告警、异常检测等系统级事件 |

### 二、任务类型调整系数

| 任务类型 | 调整系数 | 说明 |
|----------|:---:|------|
| 工具调用 | ×1.0 | 标准 |
| 信息检索 | ×0.9 | 检索经验重要度略低于工具调用 |
| 对话交互 | ×0.8 | 对话经验重要度相对较低 |
| 创作生成 | ×0.95 | 创作经验接近工具调用 |
| 通用任务 | ×0.85 | 未分类经验默认偏低 |

### 三、额外加分项

| 加分条件 | 加分值 | 说明 |
|----------|:---:|------|
| 涉及敏感操作（删除、权限修改等） | +0.20 | 安全相关经验强制提高I₀ |
| 涉及用户隐私数据访问 | +0.15 | 隐私相关经验需要更高留存优先级 |
| 任务执行耗时超过历史均值200% | +0.10 | 异常耗时的任务可能需要更多关注 |
| 任务结果被用户主动忽略/跳过 | -0.10 | 低价值信号 |
| 经验来自新场景（该分槽中首次出现） | +0.05 | 新场景经验有一定探索价值 |

### 四、快速赋值规则

| 条件 | I₀直接赋值 | 说明 |
|------|:---:|------|
| 系统安全事件触发记录 | 0.70 + 加分项 | 安全事件有最高基础重要度 |
| ECC显式请求 + 涉及敏感操作 | 0.80 | 同时满足最高优先级条件 |
| 被动观测 + 对话交互 + 无加分项 | 0.24 | 最低I₀基线组合 |


## 核心处理逻辑

```
FUNCTION i0_assignment_main_loop():
    STATE_IDLE = IDLE
    STATE_ASSIGN = ASSIGNING
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_IDLE
    加载I₀赋值规则配置（基线表、调整系数表、加分项表）

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_IDLE

        // 第2步：接收I₀赋值请求
        IF 收到来自 ag-mem-15~19 的I₀赋值请求:
            SET internal_state = STATE_ASSIGN
            条目ID = 请求.条目ID
            生成来源 = 请求.经验生成来源
            任务类型 = 请求.任务类型
            场景标签 = 请求.场景标签
            是否敏感操作 = 请求.是否涉及敏感操作
            是否隐私访问 = 请求.是否涉及隐私数据
            执行耗时比 = 请求.执行耗时 / 历史平均耗时
            是否被用户跳过 = 请求.是否被用户忽略
            是否新场景 = 请求.是否该分槽首次出现

            // 2a. 获取来源基线I₀
            基线I₀ = 获取来源基线(生成来源)
            IF 基线I₀ == None:
                基线I₀ = 0.40  // 默认基线

            // 2b. 应用任务类型调整系数
            调整系数 = 获取任务类型调整系数(任务类型)
            IF 调整系数 == None:
                调整系数 = 0.85  // 默认通用任务系数

            // 2c. 计算基础I₀
            I₀值 = 基线I₀ × 调整系数
            赋值依据 = ["来源基线=" + 基线I₀, "任务系数=" + 调整系数]
            调整因子列表 = []

            // 2d. 应用额外加分项
            IF 是否敏感操作:
                I₀值 += 0.20
                调整因子列表.append("敏感操作+0.20")

            IF 是否隐私访问:
                I₀值 += 0.15
                调整因子列表.append("隐私访问+0.15")

            IF 执行耗时比 > 2.0:
                I₀值 += 0.10
                调整因子列表.append("异常耗时+0.10")

            IF 是否被用户跳过:
                I₀值 -= 0.10
                调整因子列表.append("用户跳过-0.10")

            IF 是否新场景:
                I₀值 += 0.05
                调整因子列表.append("新场景+0.05")

            // 2e. 边界裁剪
            IF I₀值 > 1.0:
                I₀值 = 1.0
            IF I₀值 < 0.05:
                I₀值 = 0.05

            // 2f. 构建赋值结果
            赋值结果 = {
                条目ID: 条目ID,
                I₀值: ROUND(I₀值, 2),
                赋值依据摘要: JOIN(赋值依据, ", "),
                调整因子列表: 调整因子列表
            }

            // 2g. 发送至请求来源分槽和重要度聚合单元
            向请求来源分槽返回I₀赋值结果
            向 ag-mem-36 发送I₀赋值结果

            // 2h. 更新统计
            更新I₀分布统计(I₀值)
            SET internal_state = STATE_IDLE

        // 第3步：周期性状态上报
        IF 距上次状态上报 >= 60秒:
            向 ag-mem-03 上报赋值状态(
                当前状态=internal_state,
                最近I₀分布=统计分布,
                各类来源I₀均值=按来源分类的均值统计
            )
            重置统计计数器

        SLEEP 10ms


FUNCTION 获取来源基线(生成来源):
    基线表 = {
        "ECC主动请求记录": 0.60,
        "任务执行自动记录-成功": 0.45,
        "任务执行自动记录-失败": 0.50,
        "用户显式反馈触发记录": 0.55,
        "被动观测记录": 0.30,
        "系统安全事件触发记录": 0.70
    }
    RETURN 基线表.GET(生成来源, 0.40)


FUNCTION 获取任务类型调整系数(任务类型):
    系数表 = {
        "工具调用": 1.0,
        "信息检索": 0.9,
        "对话交互": 0.8,
        "创作生成": 0.95,
        "通用任务": 0.85
    }
    RETURN 系数表.GET(任务类型, 0.85)
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 生成来源不在已知分类中 | 使用默认基线0.40，标记“未知来源” | 来源类型被注册到规则表 |
| 任务类型不在已知分类中 | 使用默认调整系数0.85，标记“未知任务类型” | 任务类型被注册到规则表 |
| I₀赋值规则配置加载失败 | 使用编译期内置默认规则（基线0.40, 系数0.85），标记降级 | 规则配置恢复 |
| 计算出的I₀值超出[0.05, 1.0] | 强制裁剪至边界，记录异常日志 | — |
| 紧急熔断 | 暂停赋值，未完成的请求丢弃 | 紧急解除后恢复 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| 内部调度总线 | 读 | I₀赋值请求 | 只读 | ag-mem-15~19 发送 |
| 内部调度总线 | 读 | I₀赋值规则配置 | 只读 | ag-mem-35 提供 |
| 内部调度总线 | 写 | I₀赋值结果 | 专属写入 | 向请求来源分槽与ag-mem-36发送 |
| 内部调度总线 | 写 | 赋值状态上报 | 周期性写入 | 向 ag-mem-03 发送 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| I-01 | I₀赋值规则（基线表、系数表、加分项表）为只读配置，运行时不得被任何模块修改 |
| I-02 | I₀值下限为0.05，确保每条经验至少有极低的基础重要度，不会被立即遗忘 |
| I-03 | I₀值上限为1.0，防止初始赋值溢出导致后续聚合计算失真 |
| I-04 | I₀赋值结果必须可追溯，每个I₀值均附带赋值依据摘要与调整因子列表 |
