# ag-mem-28-L5核心层存储单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-28 |
| 模块名称 | L5核心层存储单元 |
| 所属分区 | 三、漏斗二：任务经验漏斗 / 五层存储 |
| 核心职责 | 作为漏斗二五层记忆存储架构的第五层（最高层），专门存储终身不可遗忘的安全底线经验、不可抗力事件及经人工锁定或高安全显著性（S≥0.9）直达写入的关键策略。L5是整个记忆中枢的“永久记忆区”，占漏斗二总容量的0.5%，默认情况下物理锁定，写入权限由ag-mem-29管控，访问权限由ag-mem-30管控。所有L5层条目默认不可删除、不可修改，仅提供只读查询服务。不参与晋升判定或遗忘策略，仅执行高安全经验的接收、存储与只读检索 |
| 依赖模块 | ag-mem-16（工具调用槽，S值直达L5的写入请求）、ag-mem-27（L4抽象提炼单元，高置信度通用规则推送）、ag-mem-29（L5核心层安全规则硬锁定单元，管控写入权限的临时解除）、ag-mem-30（L5核心层防篡改与只读管控单元，管控查询令牌签发）、ag-mem-48（全局容量配额管控单元，查询L5当前容量） |
| 被依赖模块 | ag-mem-29（接收锁定状态变更通知）、ag-mem-30（接收查询令牌验证请求）、ag-mem-15~19（各场景分槽，通过ag-mem-30授权的只读查询接口检索L5经验） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常锁定 | `LOCKED_NORMAL` | L5存储分区处于写保护锁定状态，仅允许只读查询 | 系统初始化完成，默认状态 |
| 临时解锁 | `TEMP_UNLOCKED` | 写入权限被ag-mem-29临时解除，允许写入新条目 | ag-mem-29下发写入权限临时解除令牌 |
| 容量已满 | `CAPACITY_FULL` | L5存储使用率已达上限，拒绝新写入 | L5使用率 ≥ 95% |
| 锁定异常 | `LOCK_FAULT` | 写入保护机制出现异常，强制只读 | 锁定机制校验失败 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |

## L5存储配置

| 配置项 | 默认值 | 说明 |
|--------|:---:|------|
| L5占漏斗二总容量比例 | **0.5%** | L5是最小的存储层，仅承载最核心的永久经验 |
| L5最大条目数 | **200条** | 严格限制，确保L5只保留真正关键的经验 |
| L5条目留存时长 | **永久（不可自动删除）** | L5层经验不受遗忘策略约束，仅人工可删除 |
| L5单条目最大大小 | 30KB | 允许存储较丰富的安全规则与不可抗力事件数据 |
| 写入超时 | 500ms | 单次写入操作的最大等待时间 |
| 容量紧急阈值 | 95% | 触发拒绝写入 |
| 默认锁定状态 | **LOCKED（写保护）** | 系统启动后默认锁定，写入需临时解除令牌 |
| 临时解锁最大持续时长 | **30秒** | 超时自动恢复锁定 |


## L5写入准入条件

| 写入来源 | 触发条件 | 所需令牌 |
|----------|------|:---:|
| S值直达（ag-mem-16） | 工具调用经验条目S值 ≥ 0.9 且 结果标签=成功 | ag-mem-29签发的临时解锁令牌 |
| L4推送（ag-mem-27） | 抽象提炼生成的高置信度通用规则（置信度 ≥ 0.85）且通过安全规则库校验 | ag-mem-29签发的临时解锁令牌 |
| 人工锁定（ag-mem-29） | 人工审查后标记为永久保留的关键经验 | 人工锁定的安全令牌 |

**写入拒绝条件**：
- L5未处于临时解锁状态
- 写入来源不属于上述三种
- S值不满足条件（< 0.9）
- L5容量已满（≥ 95%）且无可淘汰条目（L5条目不可自动淘汰）
- 安全令牌校验失败


## 核心处理逻辑

```
FUNCTION l5_storage_main_loop():
    STATE_LOCKED = LOCKED_NORMAL
    STATE_UNLOCKED = TEMP_UNLOCKED
    STATE_FULL = CAPACITY_FULL
    STATE_FAULT = LOCK_FAULT
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_LOCKED
    初始化L5存储分区（内存索引 + 持久化存储 + 写保护锁）
    初始化条目计数器 = 0
    初始化临时解锁令牌 = None
    初始化临时解锁超时计时器 = 0

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            // 立即恢复锁定状态
            IF 临时解锁令牌 != None:
                吊销临时解锁令牌
                临时解锁令牌 = None
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_LOCKED

        // 第2步：处理锁定状态变更通知
        IF 收到 ag-mem-29 锁定状态变更通知:
            IF 通知.新锁定状态 == "LOCKED":
                SET internal_state = STATE_LOCKED
                吊销临时解锁令牌
                临时解锁令牌 = None
            ELSE IF 通知.新锁定状态 == "UNLOCKED" AND 通知.变更原因 == "人工授权":
                临时解锁令牌 = 通知.令牌
                临时解锁超时计时器 = NOW()
                SET internal_state = STATE_UNLOCKED

        // 第3步：检查临时解锁超时
        IF internal_state == STATE_UNLOCKED:
            IF NOW() - 临时解锁超时计时器 > 30秒:
                // 超时自动恢复锁定
                SET internal_state = STATE_LOCKED
                临时解锁令牌 = None
                向 ag-mem-29 发送超时自动锁定通知

        // 第4步：接收L5写入请求
        IF 收到 L5写入请求:
            // 4a. 校验L5是否可写入
            IF internal_state != STATE_UNLOCKED:
                向写入来源返回L5写入拒绝通知("L5处于锁定状态，写入需临时解锁令牌")
                CONTINUE

            // 4b. 校验安全令牌
            IF 请求.安全令牌 != 临时解锁令牌.令牌ID:
                向写入来源返回L5写入拒绝通知("安全令牌校验失败")
                CONTINUE

            // 4c. 校验写入来源合法性
            IF 请求.写入来源 NOT IN ["S值直达", "L4推送", "人工锁定"]:
                向写入来源返回L5写入拒绝通知("非法写入来源")
                CONTINUE

            // 4d. 校验S值条件（S值直达来源必须满足S≥0.9）
            IF 请求.写入来源 == "S值直达":
                IF 请求.S值 < 0.9:
                    向写入来源返回L5写入拒绝通知("S值不满足L5直达条件（S<0.9）")
                    CONTINUE

            // 4e. 查询当前L5容量
            容量回执 = 向 ag-mem-48 查询L5当前容量()
            L5使用率 = 容量回执.当前使用率

            IF L5使用率 >= 95%:
                SET internal_state = STATE_FULL
                向写入来源返回L5写入拒绝通知("L5容量已满，无法写入")
                CONTINUE

            IF 条目计数器 >= L5最大条目数:
                SET internal_state = STATE_FULL
                向写入来源返回L5写入拒绝通知("L5已达到最大条目数上限（200条）")
                CONTINUE

            // 4f. 校验条目大小
            IF 请求.经验数据.大小 > 30KB:
                向写入来源返回L5写入拒绝通知("条目大小超过L5上限（30KB）")
                CONTINUE

            // 4g. 写入L5存储（物理锁定分区，写入需验证写保护已解除）
            存储位置 = L5存储分区.安全写入(
                条目ID=请求.条目ID,
                经验数据=请求.经验数据,
                重要度I值=请求.重要度I值,
                S值=请求.S值,
                写入来源=请求.写入来源,
                锁定时间=NOW(),
                安全令牌=请求.安全令牌
            )
            IF 存储位置 == None:
                向写入来源返回L5写入拒绝通知("存储写入异常")
                CONTINUE

            // 4h. 更新元数据
            条目计数器++
            更新L5索引(请求.条目ID, 请求.写入来源, NOW())

            // 4i. 返回确认
            向写入来源返回L5写入确认回执(
                条目ID=请求.条目ID,
                写入状态="成功",
                L5当前使用率=L5使用率,
                锁定状态恢复确认="写入完成后保持临时解锁状态"
            )

        // 第5步：响应L5查询请求
        IF 收到 L5查询请求:
            // 5a. 向ag-mem-30验证查询令牌
            向 ag-mem-30 发送查询令牌验证请求(请求.查询令牌)
            验证回执 = 等待 ag-mem-30 返回()

            IF 验证回执 == None OR NOT 验证回执.令牌有效:
                向来源分槽返回L5查询拒绝通知("查询令牌无效")
                CONTINUE

            // 5b. 检索匹配条目（仅返回来源分槽匹配的条目）
            匹配条目 = L5存储.查询(请求.查询条件, 验证回执.授权槽位编号, 请求.最大返回条数)

            // 5c. L5条目均为只读，返回时标记不可修改
            FOR EACH 条目 IN 匹配条目:
                条目.readonly = True
                条目.editable = False
                条目.deletable = False

            // 5d. 返回结果
            向来源分槽返回L5查询结果列表(匹配条目, 层级="L5")

        // 第6步：周期性状态上报
        IF 距上次状态上报 >= 120秒:
            向 ag-mem-48、ag-mem-03、ag-mem-29 上报L5状态(
                当前状态=internal_state,
                条目总数=条目计数器,
                使用率=L5使用率,
                锁定状态=IF internal_state == STATE_LOCKED THEN "锁定" ELSE "临时解锁",
                最近写入时间=最近写入时间
            )

        SLEEP 10ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 未处于临时解锁状态收到写入请求 | 拒绝写入，提示“L5处于锁定状态” | ag-mem-29下发临时解锁令牌 |
| 安全令牌校验失败 | 拒绝写入，记录告警 | 使用合法令牌重试 |
| S值直达写入但S值<0.9 | 拒绝写入，提示S值不满足条件 | — |
| L5容量已满 | 拒绝写入，提示容量已满 | 人工清理L5条目或扩容 |
| 临时解锁超时（>30秒） | 自动恢复锁定，未完成的写入回滚 | 重新申请临时解锁令牌 |
| 写入过程中锁定状态被强制恢复 | 写入回滚，返回写入失败 | 重新申请解锁后重试 |
| 紧急熔断 | 立即恢复锁定状态，暂停所有操作 | 紧急解除后恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | L5层默认处于物理写保护锁定状态，任何写入操作必须持有ag-mem-29签发的有效临时解锁令牌 |
| S-02 | L5层条目永久保留，不受任何遗忘策略约束，不得通过ag-mem-40或ag-mem-42自动删除 |
| S-03 | L5层条目仅可人工删除（需ag-mem-29双重确认），任何自动化模块无权修改或删除L5数据 |
| S-04 | L5层查询必须通过ag-mem-30的令牌验证，禁止无令牌访问 |
| S-05 | 临时解锁令牌有效期30秒，超时自动作废，L5自动恢复锁定状态 |
| S-06 | L5写入来源仅限“S值直达”、“L4推送”、“人工锁定”三种，其他来源一律拒绝 |
| S-07 | S值直达写入必须满足S ≥ 0.9且结果标签为成功，失败经验不得通过S值直达进入L5 |
