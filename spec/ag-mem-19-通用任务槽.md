# ag-mem-19-通用任务槽 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-19 |
| 模块名称 | 通用任务槽 |
| 所属分区 | 三、漏斗二：任务经验漏斗 / 场景分槽管理 |
| 核心职责 | 作为漏斗二中承载“通用任务”及混合/未分类任务经验的场景分槽。接收 ag-mem-03 路由的通用任务场景经验条目，以及当其他四个场景分槽无法明确判定时作为默认兜底槽接收经验。管理该场景下的五层记忆存储（L1-L5）。本槽采用标准权重配置（无偏向上调），但拥有最强的遗忘保护参数，确保跨场景通用策略和不常见但可能有价值的混合经验不被过早清理。不参与认知决策，仅执行通用任务类经验的存储、检索与生命周期管理 |
| 依赖模块 | ag-mem-03（漏斗二专属调度单元，下发经验路由）、ag-mem-20~30（五层存储单元，实际存储操作）、ag-mem-35（三维权重系数配置单元，读取并应用本槽专属权重） |
| 被依赖模块 | ag-mem-03（返回查询/写入结果）、ag-mem-25（相似经验归并单元，可扫描本槽内的重复经验） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 无操作请求，等待指令 | 系统初始化完成，槽位已激活 |
| 写入处理 | `WRITING` | 正在将新经验条目写入五层存储 | 收到 ag-mem-03 的经验写入请求 |
| 查询处理 | `QUERYING` | 正在检索匹配的历史通用任务经验 | 收到 ag-mem-03 的经验查询请求 |
| 维护整理 | `MAINTENANCE` | 正在执行槽内经验归并或遗忘扫描 | 收到维护指令或定时触发 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 经验写入请求 | Struct（经验条目数据 + 用户ID + 重要度预估值 + 场景标签确认 + 路由原因） | ag-mem-03 漏斗二专属调度单元（通过内部调度总线） | 通用任务执行完成或无法明确分类时 | **高** |
| 经验查询请求 | Struct（查询条件 + 用户ID + 最大返回条数 + 时间窗口） | ag-mem-03 漏斗二专属调度单元（通过内部调度总线） | ECC 需要检索历史通用任务经验时 | **高** |
| 维护扫描指令 | Struct（扫描类型：归并/遗忘 + 扫描深度） | ag-mem-03 漏斗二专属调度单元 | 定时触发或容量告警时 | 普通 |
| 专属权重配置参数 | Struct（权重系数 + 各层晋升阈值 + 遗忘阈值 + 强遗忘保护标记） | ag-mem-35 三维权重系数配置单元 | 槽位激活或配置更新时 | 普通 |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-mem-01 总控漏斗F0 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 写入确认回执 | Struct（条目ID + 分配层级 + 预估晋升时间 + 写入耗时 + 兜底标记） | ag-mem-03 漏斗二专属调度单元 | 写入完成时 | **高** |
| 查询结果列表 | Struct（匹配经验列表 + 各条目层级 + 重要度 + 查询耗时） | ag-mem-03 漏斗二专属调度单元 | 查询完成时 | **高** |
| 维护完成回执 | Struct（归并条目数 + 遗忘条目数 + 释放空间量） | ag-mem-03 漏斗二专属调度单元 | 维护扫描完成时 | 普通 |
| 槽位状态上报 | Struct（当前条目总数 + 各层级分布 + 存储使用量 + 兜底接收比例） | ag-mem-03 漏斗二专属调度单元 | 周期性（每60秒）或状态变更时 | 普通 |


## 专属参数配置

| 参数名称 | 默认值 | 说明 |
|----------|:---:|------|
| 权重系数（α/β/γ） | **1.0 / 1.0 / 1.0**（标准权重，无偏向上调） | 通用任务经验保持原始重要度计算，不做偏向增强 |
| L1→L2晋升阈值 | I ≥ 0.42 | 略高于标准0.40，仅晋升确实有留存价值的经验 |
| L2→L3晋升阈值 | I ≥ 0.62 | 略高于标准0.60 |
| L3→L4晋升阈值 | I ≥ 0.82 | 略高于标准0.80 |
| L4→L5晋升阈值 | I ≥ 0.92 | 略高于标准0.90 |
| L1遗忘阈值 | I < 0.06 | 显著低于标准0.10，强保护 |
| L2遗忘阈值 | I < 0.15 | 显著低于标准0.20，强保护 |
| L3遗忘阈值 | I < 0.22 | 显著低于标准0.30，强保护 |
| 遗忘保护标记 | **强保护** | 所有层级遗忘阈值大幅下调，确保通用经验不易丢失 |
| 兜底接收标记 | **是** | 当其他四个场景分槽均无法判定时，本槽作为默认接收槽 |


## 核心处理逻辑

```
FUNCTION general_slot_main_loop():
    STATE_IDLE = IDLE
    STATE_WRITE = WRITING
    STATE_QUERY = QUERYING
    STATE_MAINT = MAINTENANCE
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_IDLE
    初始化槽位编号 = "ag-mem-19"
    加载专属权重配置（标准权重，强遗忘保护，兜底标记=True）
    初始化兜底接收计数器 = 0

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_IDLE

        // 第2步：接收经验写入请求
        IF 收到 ag-mem-03 经验写入请求:
            // 本槽接受“通用任务”标签或作为兜底接收未分类经验
            IF 请求.场景标签 != "通用任务" AND 请求.路由原因 != "兜底":
                // 如果是明确属于其他场景的经验，不应路由到本槽
                向 ag-mem-03 返回错误("本槽仅接受通用任务或兜底经验")
                CONTINUE

            SET internal_state = STATE_WRITE
            经验条目 = 请求.经验条目数据
            是否兜底 = 请求.路由原因 == "兜底"

            // 2a. 不做权重上调，保持原始重要度
            调整后重要度 = 计算重要度(经验条目.i0, 经验条目.s_value, 经验条目.v_value, 经验条目.c_value)
            // 注意：使用标准权重系数 1.0/1.0/1.0

            // 2b. 写入L1临时层
            写入结果 = 向 ag-mem-20 发起L1写入(经验条目, 调整后重要度)

            IF 是否兜底:
                兜底接收计数器++

            // 2c. 返回确认
            向 ag-mem-03 返回写入确认回执(
                条目ID=写入结果.条目ID,
                分配层级="L1",
                预估晋升时间=计算预估晋升时间(调整后重要度, 当前时间),
                兜底标记=是否兜底
            )
            SET internal_state = STATE_IDLE

        // 第3步：接收经验查询请求
        IF 收到 ag-mem-03 经验查询请求:
            SET internal_state = STATE_QUERY
            查询条件 = 请求.查询条件

            // 3a. 从五层存储中检索匹配经验
            结果集 = []
            FOR EACH 层级 IN [L5, L4, L3, L2, L1]:
                层级结果 = 向对应存储单元发起查询(层级, 查询条件, 请求.最大返回条数)
                结果集.合并(层级结果)
                IF 结果集.size >= 请求.最大返回条数:
                    BREAK

            // 3b. 按重要度排序
            结果集 = 按重要度降序排列(结果集)
            结果集 = 截断到最大返回条数(结果集, 请求.最大返回条数)

            // 3c. 返回结果
            向 ag-mem-03 返回查询结果列表(结果集, 各条目层级, 重要度)
            SET internal_state = STATE_IDLE

        // 第4步：执行维护扫描（使用强遗忘保护阈值）
        IF 收到维护扫描指令:
            SET internal_state = STATE_MAINT

            // 4a. 归并相似经验
            归并结果 = 向 ag-mem-25 发起槽内归并扫描(槽位编号=ag-mem-19)
            
            // 4b. 遗忘低重要度条目（使用强保护阈值，遗忘标准远低于其他槽）
            遗忘结果 = 向 ag-mem-40 发起槽内遗忘扫描(槽位编号=ag-mem-19, 专属遗忘阈值)

            // 4c. 返回维护结果
            向 ag-mem-03 返回维护完成回执(
                归并条目数=归并结果.归并数,
                遗忘条目数=遗忘结果.遗忘数,
                释放空间量=遗忘结果.释放空间
            )
            SET internal_state = STATE_IDLE

        // 第5步：周期性状态上报
        IF 距上次状态上报 >= 60秒:
            统计各层级条目数、存储使用量、兜底接收比例
            向 ag-mem-03 上报槽位状态

        SLEEP 20ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 收到明确属于其他场景的经验 | 拒绝写入，返回错误提示，建议路由至正确分槽 | — |
| 作为兜底槽接收低置信度经验 | 正常写入，标记兜底来源，不额外处理 | — |
| 写入L1失败（存储满） | 触发L1清理（使用强保护遗忘阈值，清理更保守），重试写入 | 清理完成且有足够空间 |
| 查询结果为空 | 返回空列表 + 标记“无匹配经验” | — |
| 维护扫描期间收到写入请求 | 写入请求排队等待，维护完成后处理 | 维护完成 |
| 紧急熔断 | 暂停所有操作，保持槽位状态 | 紧急解除后恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 本槽接受“通用任务”标签经验及兜底路由的未分类经验，但不得主动接收明确属于其他四个场景的经验 |
| S-02 | 兜底接收的经验在写入时不做任何权重调整，保持原始重要度值 |
| S-03 | 强遗忘保护仅在本槽内生效，不得影响其他分槽的遗忘策略 |
| S-04 | 槽内经验的跨槽查询必须通过 ag-mem-03 统一路由，不得直接响应外部模块的查询 |
| S-05 | 兜底接收计数器仅用于统计，不得作为经验质量判断依据 |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-M19-01 | `IDLE`，收到通用任务经验写入 | 写入请求（场景标签=“通用任务”） | 标准权重计算，写入L1，返回确认（兜底标记=False） |
| TC-M19-02 | `IDLE`，收到兜底路由经验 | 写入请求（路由原因=“兜底”，场景标签=“通用任务”） | 标准权重计算，写入L1，返回确认（兜底标记=True） |
| TC-M19-03 | `IDLE`，收到非通用任务场景的写入 | 写入请求（场景标签=“工具调用”） | 拒绝写入，返回错误 |
| TC-M19-04 | `WRITING`，L1存储满 | 写入请求 + L1返回存储满 | 触发L1清理（使用强保护阈值），重试写入成功 |
| TC-M19-05 | `IDLE`，收到维护扫描指令 | 维护指令 | 执行归并与遗忘（使用强保护阈值），遗忘数量显著低于其他槽 |
| TC-M19-06 | `SYSTEM_PAUSED`，收到查询请求 | 查询请求 | 拒绝处理，保持暂停状态 |
