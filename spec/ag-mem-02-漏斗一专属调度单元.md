# ag-mem-02-漏斗一专属调度单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-02 |
| 模块名称 | 漏斗一专属调度单元 |
| 所属分区 | 一、顶层总控中枢 |
| 核心职责 | 作为漏斗一（用户画像漏斗）的专属调度单元，负责用户画像槽的全生命周期管理。接收 ag-mem-01 转发的用户身份识别结果，匹配已有画像槽或触发新槽创建。管理长期槽、临时槽和访客槽的激活、休眠与过期清理。确保不同用户的画像槽之间严格物理隔离，禁止跨槽数据访问。不参与任何认知决策，仅执行漏斗一内部资源的调度与管理 |
| 依赖模块 | ag-mem-01（总控漏斗F0，接收调度指令）、ag-mem-04（用户身份识别单元，提供用户ID与识别结果）、ag-mem-05（画像槽创建与初始化单元，触发新槽创建）、ag-mem-06（画像槽数据隔离管控单元，确保槽间隔离）、ag-mem-12（临时画像槽自动清除单元，触发过期清理） |
| 被依赖模块 | ag-mem-01（返回槽位匹配结果）、ag-mem-04（接收槽位激活信号）、ag-mem-07（用户行为观测记录单元，接收活跃槽位号） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 无调度任务，等待指令 | 系统初始化完成，无待处理请求 |
| 槽位查询 | `LOOKUP` | 正在根据用户ID查询已有画像槽 | 收到 ag-mem-01 的用户画像查询/写入请求 |
| 槽位创建 | `CREATING` | 为新用户创建画像槽 | 查询无匹配槽位且用户类型允许创建 |
| 槽位清理 | `CLEANING` | 正在清理过期临时槽或释放长期槽资源 | 收到定时清理信号或手动清理请求 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 用户画像操作请求 | Struct（操作类型：查询/写入 + 用户ID + 画像槽类型 + 数据负载） | ag-mem-01 总控漏斗F0（通过内部调度总线） | ECC 发起用户画像查询或漏斗一经验写入时 | **高** |
| 用户身份识别结果 | Struct（用户ID + 识别方式 + 置信度 + 是否新用户标记） | ag-mem-04 用户身份识别单元 | 每次用户会话开始时 | **高** |
| 槽位创建完成回执 | Struct（槽位编号 + 存储分区指针 + 初始化状态） | ag-mem-05 画像槽创建与初始化单元 | 新槽创建完成时 | **高** |
| 槽位清理完成回执 | Struct（槽位编号 + 清理类型 + 释放空间量） | ag-mem-12 临时画像槽自动清除单元 | 临时槽过期清理完成时 | 普通 |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-mem-01 总控漏斗F0 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 槽位匹配结果 | Struct（槽位编号 + 槽位类型 + 用户ID + 激活时间戳） | ag-mem-01 总控漏斗F0（通过内部调度总线） | 槽位查询成功或创建完成时 | **高** |
| 新槽创建指令 | Struct（用户ID + 槽位类型 + 初始存储配额） | ag-mem-05 画像槽创建与初始化单元 | 查询无匹配槽位时 | **高** |
| 槽位激活信号 | Struct（槽位编号 + 用户ID + 活跃状态） | ag-mem-04（身份识别单元）、ag-mem-07（行为观测单元） | 槽位匹配或创建成功后 | **高** |
| 槽位清理指令 | Struct（槽位编号 + 清理原因 + 清理策略） | ag-mem-12 临时画像槽自动清除单元 | 临时槽过期或长期槽主动释放时 | 普通 |
| 槽位状态上报 | Struct（活跃槽位列表 + 各槽使用率 + 长期槽数量 + 临时槽数量） | ag-mem-01 总控漏斗F0 | 周期性（每5秒）或状态变更时 | 普通 |


## 画像槽管理策略

| 槽位类型 | 创建条件 | 最大数量 | 生命周期 | 清理策略 |
|----------|----------|:---:|------|------|
| **长期槽** | 用户首次登录并确认身份 | 6个（可配置） | 永久保留 | 用户手动删除或90天未活跃提醒 |
| **临时槽** | 用户身份未确认或选择访客模式 | 2个 | 7天 | 到期自动覆写擦除 |
| **访客槽** | 一次性会话，无身份关联 | 1个 | 当前会话 | 会话结束即时清除 |


## 核心处理逻辑

```
FUNCTION funnel_one_dispatcher_main_loop():
    STATE_IDLE = IDLE
    STATE_LOOKUP = LOOKUP
    STATE_CREATING = CREATING
    STATE_CLEANING = CLEANING
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_IDLE
    初始化槽位映射表（用户ID → 槽位编号）
    初始化槽位计数器（长期槽/临时槽/访客槽）

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_IDLE

        // 第2步：接收用户画像操作请求
        IF 收到 ag-mem-01 转发请求:
            SET internal_state = STATE_LOOKUP
            请求 = 操作请求
            用户ID = 请求.用户ID
            操作类型 = 请求.操作类型

            // 2a. 查询槽位映射表
            已有槽位 = 槽位映射表.查找(用户ID)

            IF 已有槽位 != None:
                // 槽位已存在，直接激活
                槽位编号 = 已有槽位.槽位编号
                更新槽位最后活跃时间
                SET internal_state = STATE_IDLE
                向 ag-mem-01 返回槽位匹配结果(槽位编号, 已有槽位.类型, 用户ID)
                向 ag-mem-04 发送槽位激活信号(槽位编号, 用户ID)
                CONTINUE

            // 2b. 槽位不存在，判定是否允许创建
            IF 操作类型 == "写入":
                // 写入请求但无槽位，需要创建
                槽位类型 = 判定槽位类型(请求.画像槽类型, 用户ID)
                IF 槽位类型 == "长期槽" AND 长期槽计数 >= 长期槽上限:
                    向 ag-mem-01 返回错误(原因="长期槽数量已达上限")
                    SET internal_state = STATE_IDLE
                    CONTINUE

                SET internal_state = STATE_CREATING
                向 ag-mem-05 发送新槽创建指令(用户ID, 槽位类型, 默认配额)
                // 等待创建完成回执
            ELSE IF 操作类型 == "查询":
                // 查询但无槽位，返回空结果
                向 ag-mem-01 返回空结果(原因="用户画像槽不存在")
                SET internal_state = STATE_IDLE
                CONTINUE

        // 第3步：接收槽位创建完成回执
        IF 收到 ag-mem-05 创建完成回执:
            槽位编号 = 回执.槽位编号
            用户ID = 回执关联用户ID
            槽位类型 = 回执关联槽位类型

            更新槽位映射表(用户ID → 槽位编号)
            递增对应槽位类型计数器
            SET internal_state = STATE_IDLE

            向 ag-mem-01 返回槽位匹配结果(槽位编号, 槽位类型, 用户ID)
            向 ag-mem-04 发送槽位激活信号(槽位编号, 用户ID)
            CONTINUE

        // 第4步：定时清理扫描
        IF 距上次清理扫描 >= 3600秒（1小时）:
            SET internal_state = STATE_CLEANING
            FOR EACH 临时槽 IN 槽位映射表:
                IF NOW() - 临时槽.创建时间 > 7天:
                    向 ag-mem-12 发送槽位清理指令(临时槽.槽位编号, "过期清理")
            FOR EACH 访客槽 IN 槽位映射表:
                IF 访客槽.关联会话已结束:
                    向 ag-mem-12 发送槽位清理指令(访客槽.槽位编号, "会话结束清理")
            SET internal_state = STATE_IDLE

        // 第5步：周期性状态上报
        IF 距上次状态上报 >= 5秒:
            统计活跃槽位数量（长期/临时/访客）
            向 ag-mem-01 上报槽位状态

        SLEEP 20ms


FUNCTION 判定槽位类型(请求类型, 用户ID):
    IF 请求类型 == "访客":
        RETURN "访客槽"
    ELSE IF 用户ID 已认证:
        RETURN "长期槽"
    ELSE:
        RETURN "临时槽"
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 长期槽数量达上限 | 拒绝新建，返回错误提示，建议用户手动释放旧槽 | 用户释放长期槽或管理员扩容 |
| 临时槽创建时用户身份未确认 | 正常创建临时槽，待用户确认身份后可升级为长期槽 | 用户完成身份认证 |
| 槽位创建超时（ag-mem-05 无响应） | 重试一次，仍失败则返回错误 | ag-mem-05 恢复正常 |
| 同一用户重复创建请求 | 查询已有槽位直接返回，不重复创建 | — |
| 紧急熔断 | 暂停所有操作，保持当前槽位状态 | 紧急解除后恢复 |

| 内部调度总线 | 读 | 用户身份识别结果 | 只读 | ag-mem-04 发送 |
| 内部调度总线 | 读 | 槽位创建完成回执 | 只读 | ag-mem-05 返回 |
| 内部调度总线 | 读 | 槽位清理完成回执 | 只读 | ag-mem-12 返回 |
| 内部调度总线 | 写 | 槽位匹配结果 | 专属写入 | 向 ag-mem-01 返回 |
| 内部调度总线 | 写 | 新槽创建指令 | 专属写入 | 向 ag-mem-05 发送 |
| 内部调度总线 | 写 | 槽位激活信号 | 广播 | 向 ag-mem-04/07 发送 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 不同用户的画像槽之间物理存储隔离，禁止任何形式的跨槽数据访问 |
| S-02 | 临时槽和访客槽的清理必须执行安全擦除（覆写后删除），不可直接回收存储 |
| S-03 | 长期槽的用户画像数据编译期禁止接入 Agent 自学习链路，确保隐私数据不污染任务经验 |
| S-04 | 本模块仅负责槽位调度，不直接操作用户画像数据内容 |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-M02-01 | `IDLE`，已存在用户U001的长期槽 | 查询请求（用户ID=U001） | 返回槽位编号，激活该槽，状态保持IDLE |
| TC-M02-02 | `IDLE`，新用户U002首次登录 | 写入请求（用户ID=U002，类型=长期） | 进入CREATING，向ag-mem-05发送创建指令 |

| TC-M02-04 | `IDLE`，长期槽已达上限6个 | 写入请求（新用户，类型=长期） | 拒绝创建，返回错误“长期槽数量已达上限” |

| TC-M02-06 | `SYSTEM_PAUSED`，收到查询请求 | 用户画像查询请求 | 拒绝处理，保持暂停状态 |
