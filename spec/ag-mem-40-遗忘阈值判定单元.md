# ag-mem-40-遗忘阈值判定单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-40 |
| 模块名称 | 遗忘阈值判定单元 |
| 所属分区 | 三、漏斗二：任务经验漏斗 / 晋升与遗忘执行机制 |
| 核心职责 | 接收来自 ag-mem-20~26（各层级存储单元）、ag-mem-37（重要度增量定时刷新单元）以及各场景分槽（ag-mem-15~19）发起的遗忘扫描请求，对指定范围内的经验条目进行遗忘判定。基于条目的综合重要度I值、当前层级、来源分槽的专属遗忘阈值以及最低复用次数保护条件，逐条判定是否满足遗忘条件。满足遗忘条件的条目被整理为“遗忘候选清单”，输出至 ag-mem-42（冗余记忆删除与归档单元）执行实际的删除或归档操作。L4/L5层条目默认受遗忘保护，仅在I值极低且复用验证不足时方予评估；L5层条目永不参与遗忘判定。不参与遗忘执行或数据修改，仅负责筛选与判定 |
| 依赖模块 | ag-mem-20~26（各层级存储单元，提供待判定条目的当前I值与元数据）、ag-mem-35（三维权重系数配置单元，获取各分槽专属的遗忘阈值）、ag-mem-41（最低复用次数校验单元，可选地对遗忘候选条目进行复用次数保护验证） |
| 被依赖模块 | ag-mem-42（冗余记忆删除与归档单元，接收遗忘候选清单执行删除或归档）、ag-mem-20~26（返回遗忘判定完成回执与释放空间量预估）、ag-mem-51（记忆变更日志追溯单元，记录遗忘事件） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 无判定任务，等待扫描指令 | 系统初始化完成，无待处理请求 |
| 条目扫描中 | `SCANNING` | 正在遍历指定范围内的经验条目，逐条读取I值与元数据 | 收到遗忘扫描请求 |
| 判定执行中 | `JUDGING` | 正在对扫描到的条目逐一进行遗忘阈值比对 | 条目扫描完成或分批扫描中 |
| 结果输出 | `OUTPUTTING` | 判定完成，正在向 ag-mem-42 输出遗忘候选清单 | 所有条目判定完成 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |

## 遗忘判定规则

### 一、各层级遗忘策略

| 层级 | 是否参与遗忘判定 | 遗忘阈值来源 | 说明 |
|:---:|:---:|------|------|
| L1（临时层） | **是** | 各分槽专属L1遗忘阈值 | 容量压力大时首批清理对象 |
| L2（近期层） | **是** | 各分槽专属L2遗忘阈值 | 超过7天未晋升且I值低的条目 |
| L3（中期层） | **是** | 各分槽专属L3遗忘阈值 | 超过30天未晋升且I值低的条目 |
| L4（长期层） | **默认不参与** | 各分槽专属L4遗忘阈值（仅强制扫描时） | 仅当ag-mem-26明确要求“强制扫描L4”时才评估 |
| L5（核心层） | **永不参与** | — | 编译期跳过，任何扫描请求均不包含L5条目 |

### 二、遗忘判定条件

```
满足以下任一条件即标记为遗忘候选:

条件1: I值 < 当前层级遗忘阈值（来自ag-mem-35分槽配置）
条件2: I值 < 遗忘阈值 + 0.05 且 复用次数 < 最低保护次数
条件3: 触发原因为"容量紧急"且I值在该层级中排名后20%

条件1为基本条件，条件2为边界保护条件（接近阈值但复用不足时仍遗忘），条件3为容量紧急时的强制清理条件。
```

### 三、遗忘保护规则

| 保护条件 | 说明 |
|------|------|
| 条目层级为L5 | 永不遗忘 |
| 条目层级为L4且未强制扫描 | 默认跳过，不参与遗忘判定 |
| 条目复用次数 ≥ 最低保护次数且I值 > 遗忘阈值×0.8 | 即使I值低于阈值也暂时保留 |
| 条目标记为“用户收藏”或“人工锁定” | 跳过遗忘判定 |
| 条目在最近24小时内被访问过 | 跳过遗忘判定（给予最近使用的经验一次保护周期） |


## 核心处理逻辑

```
FUNCTION forget_threshold_judge_main_loop():
    STATE_IDLE = IDLE
    STATE_SCAN = SCANNING
    STATE_JUDGE = JUDGING
    STATE_OUTPUT = OUTPUTTING
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_IDLE
    加载各分槽专属遗忘阈值配置（从 ag-mem-35 获取）
    初始化遗忘统计计数器

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_IDLE

        // 第2步：接收遗忘扫描请求
        IF 收到遗忘扫描请求:
            请求来源 = 请求.来源模块ID
            来源层级 = 请求.来源层级
            扫描范围 = 请求.扫描范围
            触发原因 = 请求.触发原因
            是否强制扫描L4 = 请求.是否强制扫描L4
            来源分槽 = 请求.来源分槽编号

            // 2a. L5层级直接跳过
            IF 来源层级 == "L5":
                向请求来源返回遗忘判定完成回执(0, 0, 0, 0, 0, "L5层不参与遗忘判定")
                CONTINUE

            // 2b. L4层级默认跳过（除非强制扫描）
            IF 来源层级 == "L4" AND NOT 是否强制扫描L4:
                向请求来源返回遗忘判定完成回执(0, 0, 0, 0, 0, "L4层默认受遗忘保护")
                CONTINUE

            // 2c. 获取待判定条目列表
            SET internal_state = STATE_SCAN
            待判定条目列表 = 获取待判定条目(请求来源, 来源层级, 扫描范围)
            IF 待判定条目列表为空:
                向请求来源返回遗忘判定完成回执(0, 0, 0, 0, 0, "无待判定条目")
                SET internal_state = STATE_IDLE
                CONTINUE

            // 第3步：逐条判定
            SET internal_state = STATE_JUDGE
            遗忘候选清单 = []
            保留计数 = 0
            受保护跳过计数 = 0
            当前时间 = NOW()

            FOR EACH 条目 IN 待判定条目列表:
                // 3a. 获取该条目的分槽专属遗忘阈值
                分槽配置 = 遗忘阈值配置表[条目.来源分槽编号]
                IF 分槽配置 == None:
                    分槽配置 = 遗忘阈值配置表["通用任务槽"]  // 默认使用通用任务槽配置

                // 根据层级选择对应阈值
                CASE 条目.当前层级 OF:
                    "L1": 遗忘阈值 = 分槽配置.L1遗忘I值阈值
                    "L2": 遗忘阈值 = 分槽配置.L2遗忘I值阈值
                    "L3": 遗忘阈值 = 分槽配置.L3遗忘I值阈值
                    "L4": 遗忘阈值 = 分槽配置.L4遗忘I值阈值

                // 3b. 检查遗忘保护条件
                受保护 = False
                保护原因 = ""

                IF 条目.标记 IN ["用户收藏", "人工锁定"]:
                    受保护 = True
                    保护原因 = "用户收藏或人工锁定"

                ELSE IF (当前时间 - 条目.最近访问时间) < 24小时:
                    受保护 = True
                    保护原因 = "最近24小时内被访问"

                ELSE IF 条目.复用次数 >= 最低保护次数 AND 条目.I值 > 遗忘阈值 × 0.8:
                    受保护 = True
                    保护原因 = "复用次数满足保护条件"

                IF 受保护:
                    受保护跳过计数++
                    向 ag-mem-41 发送遗忘保护条目通知(条目.条目ID, 保护原因, 条目.I值, 遗忘阈值)
                    CONTINUE

                // 3c. 执行遗忘判定
                应遗忘 = False
                遗忘原因 = ""
                建议处理方式 = "删除"

                // 条件1: I值低于遗忘阈值
                IF 条目.I值 < 遗忘阈值:
                    应遗忘 = True
                    遗忘原因 = "I值低于遗忘阈值"
                    建议处理方式 = IF 条目.当前层级 IN ["L3", "L4"] THEN "归档" ELSE "删除"

                // 条件2: I值接近阈值且复用不足
                ELSE IF 条目.I值 < 遗忘阈值 + 0.05 AND 条目.复用次数 < 最低保护次数:
                    应遗忘 = True
                    遗忘原因 = "I值接近阈值且复用次数不足"
                    建议处理方式 = "删除"

                // 条件3: 容量紧急时的强制清理
                ELSE IF 触发原因 == "容量紧急" AND 条目.I值 < 该层级I值排名后20%阈值:
                    应遗忘 = True
                    遗忘原因 = "容量紧急强制清理"
                    建议处理方式 = "删除"

                // 3d. 记录判定结果
                IF 应遗忘:
                    遗忘候选清单.append({
                        条目ID: 条目.条目ID,
                        遗忘原因: 遗忘原因,
                        当前I值: 条目.I值,
                        遗忘阈值: 遗忘阈值,
                        建议处理方式: 建议处理方式,
                        来源层级: 条目.当前层级,
                        来源分槽: 条目.来源分槽编号
                    })
                ELSE:
                    保留计数++

            // 第4步：输出结果
            SET internal_state = STATE_OUTPUT

            // 4a. 发送遗忘候选清单至 ag-mem-42
            IF 遗忘候选清单 非空:
                向 ag-mem-42 发送遗忘候选清单

            // 4b. 发送遗忘事件日志
            向 ag-mem-51 发送遗忘事件日志(遗忘候选清单摘要, 触发原因)

            // 4c. 返回完成回执
            总判定数 = 待判定条目列表.size
            释放空间预估 = 遗忘候选清单.size × 平均条目大小
            向请求来源返回遗忘判定完成回执(
                扫描条目总数=总判定数,
                遗忘候选数=遗忘候选清单.size,
                保留数=保留计数,
                受保护跳过数=受保护跳过计数,
                释放空间量预估=释放空间预估,
                判定耗时=NOW() - 判定开始时间
            )

            // 4d. 更新统计计数器
            更新遗忘统计(遗忘候选清单.size, 保留计数, 受保护跳过计数)
            SET internal_state = STATE_IDLE

        // 第5步：周期性状态上报
        IF 距上次状态上报 >= 180秒:
            向 ag-mem-03 上报遗忘判定状态(遗忘统计)
            重置统计计数器

        SLEEP 10ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 待判定条目列表为空 | 直接返回完成回执（全0），不报错 | — |
| 某条目的分槽编号在配置中不存在 | 使用通用任务槽（ag-mem-19）的遗忘阈值作为默认值 | 分槽配置补全 |
| 条目I值异常（<0或>1） | 保守处理：标记为遗忘候选（假设I值异常意味着数据损坏） | I值被修正 |
| 判定过程中条目被其他模块删除 | 跳过该条目，不报错（使用快照数据判定） | — |
| 遗忘候选清单过大（>1000条） | 分批发送至 ag-mem-42，每批最多500条 | — |
| 紧急熔断 | 中断判定，未完成的条目保留在源层级 | 紧急解除后恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| F-01 | L5核心层条目永不参与遗忘判定，编译期硬编码，任何扫描请求均不得包含L5条目 |
| F-02 | L4长期层默认受遗忘保护，仅在明确请求“强制扫描L4”时才参与判定 |
| F-03 | 遗忘判定必须基于条目的实际I值与分槽专属阈值，不得使用全局默认阈值一刀切 |
| F-04 | 用户收藏或人工锁定的条目必须跳过遗忘判定，保留至人工解除标记 |
| F-05 | 遗忘判定仅输出候选清单，不得直接执行任何删除或修改操作 |
