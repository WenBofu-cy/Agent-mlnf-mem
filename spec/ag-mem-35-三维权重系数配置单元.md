# ag-mem-35-三维权重系数配置单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-35 |
| 模块名称 | 三维权重系数配置单元 |
| 所属分区 | 三、漏斗二：任务经验漏斗 / 三维重要度计算引擎 |
| 核心职责 | 统一存储与管理漏斗二中三维重要度计算公式 I = I₀ + α·S + β·V + γ·C 的权重系数α、β、γ，以及各场景分槽专属的晋升阈值与遗忘策略参数。提供全局默认配置与分槽级别的差异化配置，支持运行时查询与受控更新。确保各计算模块（ag-mem-31 S值、ag-mem-32 V值、ag-mem-33 C值、ag-mem-34 I₀值、ag-mem-36 聚合计算）使用统一且一致的参数源。不参与任何计算或决策，仅提供参数数据的存取与校验服务 |
| 依赖模块 | 无（作为底层配置服务，不依赖其他模块） |
| 被依赖模块 | ag-mem-15~19（各场景分槽，查询本槽专属晋升/遗忘阈值）、ag-mem-31（S值计算，查询S值权重系数α）、ag-mem-32（V值计算，查询V值权重系数β与分槽调整系数）、ag-mem-33（C值统计，查询C值权重系数γ与分槽饱和阈值）、ag-mem-34（I₀赋值，查询I₀赋值规则配置）、ag-mem-36（重要度聚合，查询全局权重系数）、ag-mem-38（晋升判定，查询各层晋升阈值）、ag-mem-40（遗忘判定，查询各层遗忘阈值）、ag-mem-21（L1衰减评估，查询L1晋升/遗忘阈值） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常服务 | `NORMAL_SERVICE` | 所有参数配置已成功加载并通过校验，可正常响应查询 | 系统初始化完成，参数加载成功 |
| 参数缺失降级 | `DEGRADED_SERVICE` | 部分参数加载失败或缺失，使用编译期内置默认值响应查询 | 参数文件校验失败或部分字段缺失 |
| 参数更新中 | `UPDATING` | 正在进行参数更新，暂不响应查询 | 收到参数更新指令 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |

## 核心处理逻辑

```
FUNCTION weight_config_manager_main_loop():
    STATE_NORMAL = NORMAL_SERVICE
    STATE_DEGRADED = DEGRADED_SERVICE
    STATE_UPDATING = UPDATING
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_UPDATING
    加载默认参数配置（编译期内置默认值）
    标记所有参数为“默认值”

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_NORMAL

        // 第2步：系统初始化时加载参数
        IF internal_state == STATE_UPDATING:
            参数文件 = 加载外部参数配置文件()
            IF 参数文件.校验通过:
                FOR EACH 参数 IN 参数文件:
                    IF 参数值在有效范围内:
                        更新参数存储表(参数名, 参数值)
                        标记参数为“已加载”
                    ELSE:
                        保持默认值
                        记录告警日志("参数值越界: " + 参数名)
                SET internal_state = STATE_NORMAL
            ELSE:
                SET internal_state = STATE_DEGRADED
                记录告警日志("参数文件校验失败，全部使用默认值")
                CONTINUE

        // 第3步：响应参数查询请求
        IF 收到参数查询请求:
            IF internal_state == STATE_UPDATING:
                向请求来源返回查询失败("参数更新中，暂不可查询")
                CONTINUE

            CASE 请求.查询参数类型 OF:
                "全局权重":
                    结果 = {α: 参数表["alpha"], β: 参数表["beta"], γ: 参数表["gamma"]}
                "晋升阈值":
                    结果 = 构建晋升阈值表(请求.分槽编号)
                "遗忘阈值":
                    结果 = 构建遗忘阈值表(请求.分槽编号)
                "分槽配置":
                    结果 = 构建分槽完整配置(请求.分槽编号)
                "全部":
                    结果 = 构建全量参数快照()

            向请求来源返回参数查询回执(结果, 各参数是否为默认值, 最近更新时间)

        // 第4步：处理参数更新指令
        IF 收到参数更新指令:
            // 4a. 校验授权令牌
            IF NOT 验证授权令牌(指令.授权令牌):
                向请求方返回参数更新确认回执(失败, "授权令牌无效")
                CONTINUE

            SET internal_state = STATE_UPDATING

            // 4b. 备份当前参数
            旧参数快照 = 当前参数表.clone()

            // 4c. 逐项校验并更新
            更新成功 = True
            FOR EACH 更新项 IN 指令.新参数值:
                // 校验参数名存在
                IF 更新项.参数名 NOT IN 参数表:
                    记录错误("未知参数: " + 更新项.参数名)
                    更新成功 = False
                    BREAK

                // 校验取值范围
                IF NOT 校验取值范围(更新项.参数名, 更新项.新值):
                    记录错误("参数值越界: " + 更新项.参数名)
                    更新成功 = False
                    BREAK

                // 特殊校验：α+β+γ=1.0
                IF 更新项.参数名 IN ["alpha", "beta", "gamma"]:
                    临时α = IF "alpha" IN 更新项 THEN 更新项["alpha"] ELSE 参数表["alpha"]
                    临时β = IF "beta" IN 更新项 THEN 更新项["beta"] ELSE 参数表["beta"]
                    临时γ = IF "gamma" IN 更新项 THEN 更新项["gamma"] ELSE 参数表["gamma"]
                    IF ABS(临时α + 临时β + 临时γ - 1.0) > 0.01:
                        记录错误("权重系数之和必须等于1.0")
                        更新成功 = False
                        BREAK

            // 4d. 应用更新或回滚
            IF 更新成功:
                应用新参数值()
                标记参数为非默认值
                记录更新时间 = NOW()
                向请求方返回参数更新确认回执(成功, 旧值, 新值)
                SET internal_state = STATE_NORMAL
            ELSE:
                恢复参数快照(旧参数快照)
                向请求方返回参数更新确认回执(失败, "校验失败，已回滚")
                SET internal_state = STATE_NORMAL

        // 第5步：周期性校验状态上报
        IF 初始化完成 OR 参数更新完成:
            统计参数加载情况(总数, 已加载数, 默认值替代数)
            向 ag-mem-03 上报参数校验状态

        SLEEP 100ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 外部参数配置文件缺失或损坏 | 全部参数使用编译期内置默认值，标记DEGRADED_SERVICE | 重新加载有效配置文件 |
| 参数更新时α+β+γ≠1.0 | 拒绝更新，返回错误提示，保持旧参数不变 | 提供满足约束的新参数值 |
| 单个参数值超出有效范围 | 拒绝该参数的更新，其他参数正常更新（部分成功） | 提供合法取值 |
| 参数更新过程中系统断电 | 上电后回滚至更新前旧参数（原子更新） | 重新尝试更新 |
| 查询不存在的参数名 | 返回错误码“参数不存在”，记录日志 | — |
| 紧急熔断 | 暂停服务，保持当前参数不变 | 紧急解除后恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| W-01 | 权重系数α+β+γ=1.0为强制约束，编译期与运行期双重校验，任何更新不得打破此约束 |
| W-02 | 参数更新必须持有有效授权令牌，防止未授权修改导致记忆系统行为异常 |
| W-03 | 参数更新前必须完整备份当前参数，更新失败时原子回滚 |
| W-04 | 所有参数必须定义合法取值范围，超出范围的值在加载与更新时均被拒绝 |
| W-05 | 参数配置文件缺失时使用编译期内置默认值，确保系统在任何情况下均可正常运行 |
