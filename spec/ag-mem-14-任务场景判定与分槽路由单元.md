# ag-mem-14-任务场景判定与分槽路由单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-14 |
| 模块名称 | 任务场景判定与分槽路由单元 |
| 所属分区 | 三、漏斗二：任务经验漏斗 / 场景分槽管理 |
| 核心职责 | 接收 ag-mem-03（漏斗二专属调度单元）下发的场景判定请求，基于任务描述、用户历史行为特征与当前上下文，判定当前任务所属的场景类别（对话交互、工具调用、信息检索、创作生成、通用任务），并输出目标分槽编号。支持多场景匹配时的置信度排序与关联标记。为漏斗二的经验路由提供唯一的场景分类依据。不参与认知决策，仅执行场景判定与分槽推荐 |
| 依赖模块 | ag-mem-03（漏斗二专属调度单元，下发场景判定请求）、ag-mem-10（偏好累积统计单元，可选地查询用户历史偏好辅助判定） |
| 被依赖模块 | ag-mem-03（接收场景判定结果用于分槽路由）、ag-mem-15~19（各场景分槽，接收带场景标签的经验条目） |

## 场景判定规则库

### 一、场景类别与特征关键词/模式

| 场景类别 | 目标分槽编号 | 判定关键词 | 判定行为模式 | 默认置信度 |
|----------|:---:|------|------|:---:|
| 对话交互 | ag-mem-15 | 聊天、问候、闲聊、角色扮演、日常对话 | 无工具调用、多轮问答 | 0.75 |
| 工具调用 | ag-mem-16 | 执行、调用、运行、查询API、操作文件 | 包含明确的工具名称或API调用意图 | 0.85 |
| 信息检索 | ag-mem-17 | 搜索、查找、什么是、如何、最新 | 包含搜索意图词、期望返回信息列表 | 0.80 |
| 创作生成 | ag-mem-18 | 写、生成、创作、画、翻译、总结 | 包含内容生成类动词 | 0.80 |
| 通用任务 | ag-mem-19 | — | 无法匹配以上任何类别，或混合意图 | 0.50 |

### 二、多场景匹配规则

- 当一条任务同时匹配多个场景的关键词时，按关键词匹配数量加权排序
- 若两个场景置信度差值 < 0.1，则同时返回两个场景作为“主场景”和“次选场景”，标记为关联经验
- 若所有场景置信度均 < 0.3，判定为“通用任务”，置信度 0.3

### 三、用户偏好辅助修正

当请求中标记“需要用户偏好辅助”且用户历史偏好可用时：
- 若用户在过去30天内对某场景的交互频次占比 > 50%，将该场景置信度上调 0.1
- 若用户历史中从未出现过某场景，将该场景置信度下调 0.05


## 核心处理逻辑

```
FUNCTION scene_judgment_main_loop():
    STATE_IDLE = IDLE
    STATE_FEATURE = FEATURE_EXTRACT
    STATE_MATCH = RULE_MATCH
    STATE_OUTPUT = RESULT_OUTPUT
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_IDLE
    初始化场景判定计数器（各场景命中次数）
    加载场景规则库

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_IDLE

        // 第2步：接收场景判定请求
        IF 收到 ag-mem-03 场景判定请求:
            SET internal_state = STATE_FEATURE
            请求ID = 请求.请求ID
            任务描述 = 请求.任务描述文本
            上下文 = 请求.上下文特征
            用户ID = 请求.用户ID

            // 2a. 特征提取
            关键词列表 = 分词提取关键词(任务描述)
            意图动词 = 识别意图动词(任务描述)
            是否包含工具名称 = 检测工具名称(任务描述)

            // 2b. 可选：查询用户历史偏好
            用户偏好修正 = False
            IF 请求.需要用户偏好辅助:
                偏好摘要 = 向 ag-mem-10 查询用户偏好摘要(用户ID)
                IF 偏好摘要 != None:
                    用户偏好修正 = True

            // 第3步：规则匹配
            SET internal_state = STATE_MATCH
            候选场景列表 = []

            FOR EACH 场景规则 IN 场景规则库:
                匹配得分 = 0
                匹配依据 = []

                // 关键词匹配
                FOR EACH 关键词 IN 场景规则.判定关键词:
                    IF 关键词 IN 任务描述:
                        匹配得分 += 1
                        匹配依据.append("关键词匹配: " + 关键词)

                // 意图动词匹配
                IF 场景规则.意图动词 与 意图动词 匹配:
                    匹配得分 += 2
                    匹配依据.append("意图动词匹配")

                // 工具名称检测
                IF 场景规则 == "工具调用" AND 包含工具名称:
                    匹配得分 += 3
                    匹配依据.append("检测到工具名称")

                IF 匹配得分 > 0:
                    基础置信度 = 场景规则.默认置信度 + 匹配得分 × 0.05
                    IF 基础置信度 > 1.0:
                        基础置信度 = 1.0

                    候选场景列表.append({
                        场景类别: 场景规则.场景类别,
                        目标分槽: 场景规则.目标分槽编号,
                        置信度: 基础置信度,
                        匹配依据: 匹配依据
                    })

            // 3a. 用户偏好修正
            IF 用户偏好修正:
                FOR EACH 候选场景 IN 候选场景列表:
                    场景占比 = 偏好摘要.场景分布.获取(候选场景.场景类别, 0)
                    IF 场景占比 > 0.5:
                        候选场景.置信度 += 0.1
                        候选场景.匹配依据.append("用户高频场景+0.1")
                    ELSE IF 场景占比 == 0:
                        候选场景.置信度 -= 0.05
                        候选场景.匹配依据.append("用户历史无此场景-0.05")

            // 3b. 按置信度排序
            候选场景列表 = 按置信度降序排列(候选场景列表)

            // 第4步：输出结果
            SET internal_state = STATE_OUTPUT

            IF 候选场景列表为空 OR 候选场景列表[0].置信度 < 0.3:
                // 无法判定，使用通用任务
                判定结果 = {
                    请求ID: 请求ID,
                    主场景类别: "通用任务",
                    置信度: 0.30,
                    建议目标分槽编号: "ag-mem-19",
                    次选场景列表: [],
                    判定依据摘要: "无法匹配任何场景规则"
                }
            ELSE:
                主场景 = 候选场景列表[0]
                次选场景列表 = []

                // 检查是否有次选场景（置信度接近）
                IF 候选场景列表.size > 1 AND (主场景.置信度 - 候选场景列表[1].置信度) < 0.1:
                    次选场景列表 = 候选场景列表[1: min(3, 候选场景列表.size)]

                判定结果 = {
                    请求ID: 请求ID,
                    主场景类别: 主场景.场景类别,
                    置信度: 主场景.置信度,
                    建议目标分槽编号: 主场景.目标分槽,
                    次选场景列表: 次选场景列表,
                    判定依据摘要: 主场景.匹配依据
                }

            // 4a. 返回结果
            向 ag-mem-03 发送场景判定结果(判定结果)

            // 4b. 更新统计计数器
            更新场景命中计数(判定结果.主场景类别)

            SET internal_state = STATE_IDLE

        // 第5步：周期性状态上报
        IF 距上次状态上报 >= 30次判定:
            向 ag-mem-03 上报判定状态(最近判定耗时均值, 各场景命中率)
            重置计数器

        SLEEP 10ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 任务描述为空或仅包含无意义字符 | 返回通用任务 + 置信度 0.1，标记“无效输入” | 收到有效任务描述 |
| 场景规则库加载失败 | 使用编译期内置的简化规则库，标记降级 | 规则库文件恢复 |
| 用户偏好查询超时 | 跳过偏好修正步骤，仅使用规则匹配结果 | 查询恢复正常 |
| 所有场景置信度均 < 0.3 | 返回通用任务 + 置信度 0.3，不强制匹配 | — |
| 紧急熔断 | 暂停服务，未完成的判定请求丢弃 | 紧急解除后恢复 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| 内部调度总线 | 读 | 场景判定请求 | 只读 | ag-mem-03 发送 |
| 内部调度总线 | 读 | 用户历史偏好摘要（可选） | 只读 | ag-mem-10 返回 |
| 内部调度总线 | 写 | 场景判定结果 | 专属写入 | 向 ag-mem-03 返回 |
| 内部调度总线 | 写 | 判定状态上报 | 周期性写入 | 向 ag-mem-03 发送 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 场景判定仅基于任务描述的元数据特征（关键词、意图动词），不得解析用户原始输入中的敏感内容 |
| S-02 | 用户偏好数据仅用于置信度修正，不得作为场景判定的唯一依据 |
| S-03 | 当所有场景置信度均低于阈值时，必须返回通用任务，不得随机选择场景 |
| S-04 | 本模块不持久化任何用户数据，判定过程完全基于内存 |
