## V1.1 核心规则（动态子漏斗 + 哈希索引双体系）
### 一、双大类分区（永久固定）
所有动态子漏斗只归属两大类别，彻底废弃固定场景：
- `profile`：用户画像类（偏好、习惯、人设、长期特征）
- `experience`：任务经验类（执行记录、工具操作、任务流程、解题经验）

### 二、动态子漏斗三大决策逻辑
1. **高匹配复用**
   已有子漏斗相似度得分 ≥ 0.65 → 直接复用该 funnel_id，复用存量哈希索引分区
2. **低匹配新建（配额未满）**
   最高相似度 ＜ 0.65，且当前大类数量 **未达上限** → 自动新建子漏斗，同步创建独立空哈希索引桶
3. **低匹配归并（配额已满）**
   最高相似度 ＜ 0.65，且 **已达配额上限** → 归入最相似存量子漏斗，新哈希标签合并入目标漏斗索引库

### 三、配额上限规则（全局统一，由 ag-mem-01 管控）
- 画像子漏斗最大数量：**8**
- 经验子漏斗最大数量：**32**
> V1.1 宏观自收敛底层约束：有限数量动态分化，自动合并防膨胀

### 四、相似度加权规则（叠加哈希索引预筛选加速）
1. 哈希标签精确命中加分：单标签完全匹配 +0.2（索引预筛选优先执行，大幅减少全量文本比对）
2. 任务领域关键词匹配：+0~0.6
3. 任务意图模式匹配：+0~0.25
4. 用户历史领域偏好加成：最高 +0.15
- 最终得分区间：`0.0 ~ 1.0`

### 五、哈希索引配套规则（V1.1新增，完全融入路由流程）
1. **索引隔离机制**：每个子漏斗拥有独立哈希索引桶 `funnel_index_bucket`，不同漏斗索引物理隔离，检索仅遍历单桶，解决全库遍历卡顿
2. **hash_tag 生成规则**
   - 自动提取：领域名词、工具名称、任务类型、用户专属偏好关键词
   - 一条记忆生成多条短哈希标签，存入对应漏斗索引字典
3. **索引容量约束**：单漏斗哈希索引条目上限可配置，达到阈值自动触发旧低复用标签清理（配合遗忘机制）
4. **检索加速逻辑**：记忆查询时，先通过本模块输出的 `funnel_id + hash_tag_list` 直接定位单漏斗索引桶，跳过全记忆库扫描
5. **新建漏斗自动初始化空索引**，归并漏斗自动合并双方哈希标签集合，去重存储

## 核心处理逻辑（完整 V1.1 新版伪代码，集成哈希索引全流程）
```
FUNCTION dynamic_route_main_loop():
    // 状态定义
    STATE_IDLE = IDLE
    STATE_EXTRACT = DOMAIN_TAG_EXTRACT
    STATE_MATCH = SIM_INDEX_MATCH
    STATE_DECIDE = FUNNEL_INDEX_DECIDE
    STATE_OUTPUT = RESULT_OUTPUT
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_IDLE
    // 加载全局相似度权重、哈希索引生成规则
    加载全局哈希索引配置
    初始化路由&索引统计计数器

    WHILE 系统运行中:

        // 1. 全局熔断管控
        IF 收到 ag-mem-01 熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令:
            SET internal_state = STATE_IDLE

        // 2. 接收路由请求
        IF 收到动态路由请求:
            SET internal_state = STATE_EXTRACT
            请求ID = 请求.请求ID
            任务文本 = 请求.任务描述文本
            大类标识 = 请求.大类(profile/experience)
            用户ID = 请求.用户ID

            // 2a. 领域特征 + 哈希检索标签双提取（V1.1新增索引生成）
            领域标签集合 = 提取领域特征(任务文本)
            任务意图模式 = 识别任务意图(任务文本)
            // 自动生成本条记忆专属哈希检索标签列表
            hash_tag_list = 生成哈希索引标签(领域标签集合, 任务意图模式)

            // 2b. 获取全局漏斗配额、存量漏斗列表、各漏斗哈希索引桶
            存量漏斗列表 = ag-mem-01.get_funnel_list_with_index(大类标识)
            配额上限 = ag-mem-01.get_max_quota(大类标识)
            当前数量 = len(存量漏斗列表)

            // 2c. 可选：用户偏好相似度&历史索引匹配修正
            偏好权重 = 0.0
            IF 请求允许偏好辅助:
                偏好摘要 = ag-mem-10.get_domain_preference(用户ID)
                偏好权重 = 计算领域&索引标签重合度(hash_tag_list, 偏好摘要.历史哈希标签)

            // 3. 相似度匹配 + 哈希索引预筛选加速（V1.1核心优化）
            SET internal_state = SIM_INDEX_MATCH
            匹配候选列表 = []

            FOR EACH funnel IN 存量漏斗列表:
                // 第一步：哈希索引快速预匹配，标签命中直接加分，减少全文比对
                index_hit_count = 统计交集标签数量(hash_tag_list, funnel.index_tags)
                index_score = index_hit_count * 0.2

                // 第二步：基础文本语义相似度
                base_score = 计算文本相似度(领域标签集合, funnel.domain_tags)
                intent_score = 意图模式匹配分(任务意图模式, funnel.intent_pattern)
                final_score = base_score + intent_score + index_score + 偏好权重

                // 分值封顶
                IF final_score > 1.0: final_score = 1.0

                匹配候选列表.append({
                    funnel_id: funnel.id,
                    funnel_name: funnel.name,
                    index_bucket: funnel.index_bucket,
                    existing_index_tags: funnel.index_tags,
                    score: final_score
                })

            // 按得分倒序排序候选漏斗
            匹配候选列表 = 排序降序(匹配候选列表)

            // 4. V1.1 核心三分支决策 + 哈希索引分配
            SET internal_state = FUNNEL_INDEX_DECIDE

            best_match = 匹配候选列表[0] 若存在 else None
            target_hash_bucket = ""
            merged_tags = hash_tag_list

            IF best_match 存在 AND best_match.score >= 0.65:
                // 分支1：高匹配 → 复用存量漏斗+原有索引桶
                target_funnel_id = best_match.funnel_id
                target_name = best_match.funnel_name
                target_hash_bucket = best_match.index_bucket
                is_new = False
                reason = "领域&哈希标签相似度达标，复用存量子漏斗及专属索引桶"

            ELSE:
                // 无高匹配漏斗
                IF 当前数量 < 配额上限:
                    // 分支2：配额未满 → 新建动态子漏斗 + 全新独立哈希索引桶
                    target_funnel_id = ag-mem-01.create_new_funnel_with_index(大类标识, 领域标签集合, hash_tag_list)
                    target_name = 自动生成领域名称(领域标签集合)
                    target_hash_bucket = 自动生成独立索引桶ID(target_funnel_id)
                    is_new = True
                    reason = "无高匹配漏斗且配额空闲，新建子漏斗并初始化独立哈希索引分区"
                ELSE:
                    // 分支3：配额已满 → 强制归并最优相似漏斗，合并哈希标签
                    target_funnel_id = best_match.funnel_id
                    target_name = best_match.funnel_name
                    target_hash_bucket = best_match.index_bucket
                    // 合并新旧哈希标签，自动去重
                    merged_tags = 去重合并标签(hash_tag_list, best_match.existing_index_tags)
                    is_new = False
                    reason = "配额已满，归并至最优相似存量漏斗，新哈希标签合并入目标索引桶"

            // 5. 输出完整路由+索引结果
            SET internal_state = RESULT_OUTPUT

            路由索引结果 = {
                request_id: 请求ID,
                funnel_category: 大类标识,
                funnel_id: target_funnel_id,
                funnel_name: target_name,
                match_score: best_match.score if best_match else 0.0,
                is_new_created: is_new,
                route_reason: reason,
                hash_tag_list: merged_tags,
                funnel_index_bucket: target_hash_bucket
            }

            // 下发至写入/存储模块，用于分层存储与索引写入
            推送路由索引结果至 ag-mem-03
            // 更新路由、索引统计计数器
            更新路由索引统计计数器(is_new, target_hash_bucket)

            SET internal_state = STATE_IDLE

        // 周期性状态上报（包含哈希索引全局占用数据）
        IF 判定次数达标:
            上报路由&索引统计数据至 ag-mem-01

        SLEEP 10ms
```

## 约束与异常处理（V1.1 新版，新增索引异常分支）
| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 领域特征/哈希标签提取为空 | 归入通用经验漏斗，不新建、不归并，仅写入通用索引桶 | 下次有效任务输入 |
| 全局配额/索引规则读取失败 | 临时冻结新建漏斗逻辑，仅允许存量漏斗复用，索引只读不新增 | 总控漏斗配置服务恢复 |
| 用户偏好查询超时 | 不叠加偏好&历史索引权重，仅依赖原始标签+语义相似度 | 偏好模块恢复正常 |
| 新建漏斗哈希索引桶生成失败 | 降级复用相似度最高存量漏斗，写入其索引分区 | 索引ID生成器恢复 |
| 单漏斗哈希索引达到容量上限 | 自动触发该漏斗低复用标签清理，再写入新标签 | 索引清理完成 |
| 紧急熔断 | 停止一切新建/归并/索引写入操作，仅支持只读路由查询 | 熔断解除 |
