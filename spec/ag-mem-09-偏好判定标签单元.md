# ag-mem-09-偏好判定标签单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-09 |
| 模块名称 | 偏好判定标签单元 |
| 所属分区 | 二、漏斗一：用户画像漏斗 |
| 核心职责 | 接收 ag-mem-07 输出的结构化行为观测条目，将其与当前活跃画像槽中的历史偏好基线进行比对。根据用户交互的显式反馈信号、行为频率及模式一致性，将每条行为判定为“显式偏好”、“隐式倾向”或“偶发行为”三类标签之一。将带标签的行为条目输出至 ag-mem-10 供累积统计使用。不参与任何认知决策，仅负责行为标签的客观判定与附加 |
| 依赖模块 | ag-mem-07（用户行为观测记录单元，提供待判定的行为条目）、ag-mem-10（偏好累积统计单元，提供历史偏好统计摘要用于比对）、ag-mem-02（漏斗一专属调度单元，提供当前活跃槽位号）、ag-mem-06（画像槽数据隔离管控单元，获取读取令牌） |
| 被依赖模块 | ag-mem-10（偏好累积统计单元，消费带标签的行为条目）、ag-mem-11（个性化建议生成单元，可参考标签等级调整建议强度） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 无待判定条目，等待输入 | 系统初始化完成，无待处理行为 |
| 判定进行中 | `JUDGING` | 正在获取历史基线并判定标签 | 收到 ag-mem-07 的行为观测条目 |
| 数据不足 | `INSUFFICIENT_BASELINE` | 历史偏好基线不足，无法精确判定，使用保守标签 | 新用户或画像槽数据稀疏 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 结构化行为观测条目 | Struct（条目ID + 行为类型 + 行为参数 + 场景标签 + 时间戳） | ag-mem-07 用户行为观测记录单元 | 每次批量写入完成后推送 | **高** |
| 当前活跃槽位绑定信息 | Struct（会话ID + 活跃槽位编号 + 用户ID） | ag-mem-02 漏斗一专属调度单元 | 槽位激活或切换时 | **高** |
| 历史偏好统计摘要 | Struct（槽位编号 + 各行为维度统计 + 显式/隐式/偶发分布 + 最新偏好向量） | ag-mem-10 偏好累积统计单元 | 每次判定请求时查询 | **高** |
| 读取许可令牌 | Struct（令牌ID + 授权槽位编号 + 有效期） | ag-mem-06 画像槽数据隔离管控单元 | 查询历史基线前申请 | **高** |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-mem-01 总控漏斗F0 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 带标签的行为条目 | Struct（原始条目 + 偏好标签 + 判定置信度 + 判定依据摘要） | ag-mem-10 偏好累积统计单元 | 判定完成时 | **高** |

## 偏好判定规则

### 一、三类标签定义

| 标签类别 | 标识 | 定义 | 典型触发条件 |
|----------|:---:|------|------|
| 显式偏好 | `EXPLICIT` | 用户通过明确操作表达的偏好 | 点赞、收藏、正向反馈、明确要求记住、设置中的偏好开关 |
| 隐式倾向 | `IMPLICIT` | 通过行为频率与模式推断的倾向 | 高频重复同类操作、长时间查看、复制/分享、持续选择同一选项 |
| 偶发行为 | `OCCASIONAL` | 一次性或低频行为，无法判定为偏好 | 首次操作、与历史模式明显不符、随机浏览、误触 |

### 二、判定规则优先级矩阵

| 行为类型 | 判定规则 | 默认标签 | 判定置信度 |
|----------|------|:---:|:---:|
| `FEEDBACK_LIKE` | 直接判定为显式偏好 | EXPLICIT | 0.95 |
| `FEEDBACK_DISLIKE` | 直接判定为显式偏好（负向） | EXPLICIT | 0.95 |
| `FEEDBACK_SKIP` | 若同类行为历史中连续跳过≥3次，判定为隐式负向倾向 | IMPLICIT | 0.60 |
| `TOOL_INVOKE` | 若该工具在近7日内调用≥5次，判定为隐式倾向 | IMPLICIT | 0.75 |
| `RESULT_COPY` | 复制行为直接判定为隐式倾向 | IMPLICIT | 0.70 |
| `RESULT_SHARE` | 分享行为直接判定为隐式倾向 | IMPLICIT | 0.80 |
| `RESULT_VIEW` | 若查看时长超过历史同类平均时长的150%，判定为隐式倾向 | IMPLICIT | 0.65 |
| `RETRY_ACTION` | 重试≥2次判定为隐式倾向（用户执着于此操作） | IMPLICIT | 0.70 |
| `TEXT_INPUT` | 若输入内容与历史偏好关键词匹配，判定为隐式倾向 | IMPLICIT | 0.55 |
| 其他 | 与历史基线无显著相关性，判定为偶发行为 | OCCASIONAL | 0.50 |
| 首次出现的任何行为 | 无历史基线可对比，判定为偶发行为 | OCCASIONAL | 0.40 |

### 三、判定置信度说明

- **≥ 0.80**：高置信度，用户意图明确，可直接用于偏好更新
- **0.60 – 0.79**：中等置信度，需累积多条同类标签后方可升级为偏好
- **< 0.60**：低置信度，仅用于统计参考，不触发偏好变更


## 核心处理逻辑

```
FUNCTION preference_labeling_main_loop():
    STATE_IDLE = IDLE
    STATE_JUDGE = JUDGING
    STATE_INSUFF = INSUFFICIENT_BASELINE
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_IDLE
    初始化判定计数器（各类标签累计数）

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_IDLE

        // 第2步：更新活跃槽位信息
        IF 收到 ag-mem-02 活跃槽位绑定信息:
            当前活跃槽位 = 绑定信息.活跃槽位编号
            CONTINUE

        // 第3步：接收行为观测条目
        IF 收到 ag-mem-07 行为观测条目列表:
            SET internal_state = STATE_JUDGE

            // 3a. 获取历史偏好基线
            读取令牌 = 向 ag-mem-06 申请读取令牌(当前活跃槽位)
            IF 读取令牌 == None:
                // 无法获取令牌，使用保守标签
                SET internal_state = STATE_INSUFF
                FOR EACH 条目 IN 行为观测条目列表:
                    保守标签条目 = 附加标签(条目, OCCASIONAL, 0.30, "无法获取画像槽读取权限")
                    向 ag-mem-10 发送带标签的行为条目(保守标签条目)
                SET internal_state = STATE_IDLE
                CONTINUE

            历史统计 = 向 ag-mem-10 查询历史偏好统计摘要(当前活跃槽位)

            // 3b. 逐条判定
            FOR EACH 条目 IN 行为观测条目列表:
                标签 = 判定标签(条目, 历史统计)
                带标签条目 = 附加标签(条目, 标签.类别, 标签.置信度, 标签.依据)
                向 ag-mem-10 发送带标签的行为条目(带标签条目)
                更新判定计数器(标签.类别)

            SET internal_state = STATE_IDLE

        // 第4步：周期性状态上报
        IF 距上次状态上报 >= 30秒:
            向 ag-mem-02 上报判定状态(
                当前状态=internal_state,
                已判定条目总数=判定计数器.总计,
                标签分布=判定计数器.分布比例,
                基线可用率=IF 历史统计 != None THEN 1.0 ELSE 0.0
            )

        SLEEP 10ms


FUNCTION 判定标签(条目, 历史统计):
    // 规则1：显式偏好——直接反馈信号
    IF 条目.行为类型 IN ["FEEDBACK_LIKE", "FEEDBACK_DISLIKE"]:
        RETURN {类别: EXPLICIT, 置信度: 0.95, 依据: "用户主动给出反馈"}

    // 规则2：显式偏好——用户明确要求记住
    IF 条目.行为参数 包含 "save_preference" 标记:
        RETURN {类别: EXPLICIT, 置信度: 0.90, 依据: "用户明确要求记住偏好"}

    // 规则3：隐式倾向——高频工具调用
    IF 条目.行为类型 == "TOOL_INVOKE":
        该工具调用次数 = 历史统计.工具调用频次.获取(条目.行为参数.工具名称, 0)
        IF 该工具调用次数 >= 5:
            RETURN {类别: IMPLICIT, 置信度: 0.75, 依据: "近7日该工具调用≥5次"}

    // 规则4：隐式倾向——复制/分享
    IF 条目.行为类型 == "RESULT_COPY":
        RETURN {类别: IMPLICIT, 置信度: 0.70, 依据: "用户复制了结果内容"}
    IF 条目.行为类型 == "RESULT_SHARE":
        RETURN {类别: IMPLICIT, 置信度: 0.80, 依据: "用户分享了结果"}

    // 规则5：隐式倾向——长时间查看
    IF 条目.行为类型 == "RESULT_VIEW":
        历史平均查看时长 = 历史统计.查看时长均值.获取(条目.场景标签.任务类型, 0)
        IF 条目.行为参数.查看时长 > 历史平均查看时长 × 1.5:
            RETURN {类别: IMPLICIT, 置信度: 0.65, 依据: "查看时长超过历史均值150%"}

    // 规则6：隐式倾向——连续跳过
    IF 条目.行为类型 == "FEEDBACK_SKIP":
        连续跳过次数 = 历史统计.连续跳过计数.获取(条目.场景标签.任务类型, 0)
        IF 连续跳过次数 >= 3:
            RETURN {类别: IMPLICIT, 置信度: 0.60, 依据: "连续跳过同类内容≥3次"}

    // 规则7：隐式倾向——重复重试
    IF 条目.行为类型 == "RETRY_ACTION":
        IF 条目.行为参数.重试次数 >= 2:
            RETURN {类别: IMPLICIT, 置信度: 0.70, 依据: "用户重复尝试同一操作"}

    // 规则8：隐式倾向——输入关键词匹配历史偏好
    IF 条目.行为类型 == "TEXT_INPUT" AND 历史统计 != None:
        历史偏好关键词 = 历史统计.偏好关键词集合
        IF 条目.行为参数.输入内容 包含 历史偏好关键词中的任意词:
            RETURN {类别: IMPLICIT, 置信度: 0.55, 依据: "输入内容与历史偏好关键词匹配"}

    // 规则9：首次出现——偶发行为
    IF 历史统计 == None OR 该行为类型在历史统计中无记录:
        RETURN {类别: OCCASIONAL, 置信度: 0.40, 依据: "首次出现的用户行为"}

    // 默认：偶发行为
    RETURN {类别: OCCASIONAL, 置信度: 0.50, 依据: "与历史基线无显著相关性"}
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 无法获取画像槽读取权限（令牌申请失败） | 所有条目使用 OCCASIONAL 标签 + 低置信度0.3，标记“权限不足” | 令牌申请成功 |
| 历史偏好基线为空（新用户首次使用） | 所有条目使用 OCCASIONAL 标签 + 置信度0.4，标记“数据不足” | 积累足够行为数据 |
| 行为观测条目包含未知行为类型 | 判定为 OCCASIONAL + 置信度0.3，记录未知类型日志 | 行为类型被注册 |
| 判定置信度低于0.5的条目占比过高（>70%） | 上报告警，提示画像基线可能异常 | 基线数据正常化 |
| 紧急熔断 | 暂停所有判定，未处理的条目保留在队列中 | 紧急解除后恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 判定过程中获取的历史偏好统计摘要为聚合数据，不得包含其他用户的任何信息 |
| S-02 | 输入内容的关键词匹配仅基于已脱敏的偏好关键词集合，不得存储或分析原始输入文本 |
| S-03 | 低置信度标签（<0.5）不得触发任何自动化偏好更新，仅作为统计参考 |
| S-04 | 本模块仅附加标签，不修改原始行为观测条目的任何字段 |
