# ag-mem-10-偏好累积统计单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-10 |
| 模块名称 | 偏好累积统计单元 |
| 所属分区 | 二、漏斗一：用户画像漏斗 |
| 核心职责 | 接收 ag-mem-09 输出的带标签行为条目，将其按照行为维度、场景标签与时间窗口进行累积统计。维护当前活跃画像槽内的多维偏好统计表，包括各行为维度的频次分布、显式/隐式/偶发标签比例、偏好强度趋势等。为偏好判定单元（ag-mem-09）提供历史基线查询，为个性化建议生成单元（ag-mem-11）提供量化偏好数据。不参与任何认知决策，仅负责偏好数据的定量统计与汇总 |
| 依赖模块 | ag-mem-09（偏好判定标签单元，提供带标签的行为条目）、ag-mem-02（漏斗一专属调度单元，提供当前活跃槽位号）、ag-mem-06（画像槽数据隔离管控单元，获取读写令牌） |
| 被依赖模块 | ag-mem-09（偏好判定标签单元，查询历史偏好统计摘要用于判定）、ag-mem-11（个性化建议生成单元，获取量化偏好数据） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 无待统计条目，等待输入 | 系统初始化完成，无待处理条目 |
| 统计更新中 | `STAT_UPDATING` | 正在将新行为条目计入统计表 | 收到 ag-mem-09 的带标签行为条目 |
| 查询响应中 | `QUERY_RESPONDING` | 正在处理历史偏好统计摘要查询 | 收到 ag-mem-09 或 ag-mem-11 的统计查询请求 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 带标签的行为条目 | Struct（原始行为观测条目 + 偏好标签 + 判定置信度） | ag-mem-09 偏好判定标签单元 | 每次判定完成后 | **高** |
| 统计查询请求 | Struct（查询类型：偏好摘要/维度详情/趋势 + 查询参数：时间窗口、行为维度、标签类别） | ag-mem-09（用于判定基线）、ag-mem-11（用于生成建议） | 按需查询 | **高** |
| 当前活跃槽位绑定信息 | Struct（会话ID + 活跃槽位编号 + 用户ID） | ag-mem-02 漏斗一专属调度单元 | 槽位激活或切换时 | **高** |
| 读写许可令牌 | Struct（令牌ID + 授权槽位编号 + 操作类型 + 有效期） | ag-mem-06 画像槽数据隔离管控单元 | 每次数据操作前申请 | **高** |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-mem-01 总控漏斗F0 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 历史偏好统计摘要 | Struct（槽位编号 + 各行为维度频次 + 显式/隐式/偶发分布 + 偏好强度向量 + 时间窗口） | ag-mem-09（用于判定基线）、ag-mem-11（用于生成建议） | 收到统计查询请求时 | **高** |
| 统计更新确认 | Struct（更新条目数 + 各维度增量 + 当前统计项总数） | ag-mem-09（确认写入完成） | 统计更新完成时 | 普通 |
| 统计异常告警 | Struct（告警类型 + 异常维度 + 当前值 + 历史均值 + 偏差幅度） | ag-mem-02 漏斗一专属调度单元 | 检测到统计异常波动时 | 普通 |
| 统计状态上报 | Struct（当前状态 + 累计统计条目数 + 活跃维度数 + 存储使用量） | ag-mem-02 漏斗一专属调度单元 | 周期性（每60秒）或状态变更时 | 普通 |

### 二、时间窗口定义

| 窗口类型 | 范围 | 用途 |
|----------|------|------|
| 实时窗口 | 当前会话 | 会话内偏好变化检测 |
| 短期窗口 | 最近7日 | 近期偏好趋势判定 |
| 中期窗口 | 最近30日 | 偏好稳定性评估 |
| 长期窗口 | 全部历史 | 整体偏好画像基线 |


## 核心处理逻辑

```
FUNCTION preference_statistics_main_loop():
    STATE_IDLE = IDLE
    STATE_UPDATE = STAT_UPDATING
    STATE_QUERY = QUERY_RESPONDING
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_IDLE
    初始化统计缓存（当前活跃槽位的统计摘要内存镜像）

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_IDLE

        // 第2步：更新活跃槽位
        IF 收到 ag-mem-02 活跃槽位绑定信息:
            当前活跃槽位 = 绑定信息.活跃槽位编号
            // 加载该槽位的统计摘要到内存缓存
            加载槽位统计缓存(当前活跃槽位)
            CONTINUE

        // 第3步：接收带标签的行为条目并更新统计
        IF 收到 ag-mem-09 带标签的行为条目列表:
            SET internal_state = STATE_UPDATE

            写入令牌 = 向 ag-mem-06 申请写入令牌(当前活跃槽位)
            IF 写入令牌 == None:
                暂存待更新条目，等待重试
                CONTINUE

            更新计数 = 0
            FOR EACH 条目 IN 行为条目列表:
                // 3a. 按行为维度更新
                行为维度 = 条目.行为类型
                更新维度统计(行为维度, 条目.偏好标签, 条目.判定置信度)

                // 3b. 按场景标签更新
                场景类别 = 条目.场景标签.场景类别
                更新场景统计(场景类别, 条目.偏好标签)

                // 3c. 更新标签类别总分布
                更新标签分布(条目.偏好标签)

                // 3d. 更新偏好关键词集合（从TEXT_INPUT中提取）
                IF 条目.行为类型 == "TEXT_INPUT":
                    提取关键词并更新偏好关键词集合(条目.行为参数.关键词)

                // 3e. 检测统计异常波动
                IF 检测异常波动(行为维度, 条目):
                    向 ag-mem-02 发送统计异常告警

                更新计数++

            // 3f. 重新计算偏好强度向量
            重新计算偏好强度向量()

            // 3g. 更新内存缓存
            更新槽位统计缓存(当前活跃槽位)

            // 3h. 持久化写入画像槽
            将统计摘要持久化写入画像槽(当前活跃槽位)

            向 ag-mem-09 返回统计更新确认(更新计数, 各维度增量)
            SET internal_state = STATE_IDLE

        // 第4步：响应统计查询请求
        IF 收到统计查询请求:
            SET internal_state = STATE_QUERY

            读取令牌 = 向 ag-mem-06 申请读取令牌(当前活跃槽位)
            IF 读取令牌 == None:
                向请求来源返回错误("无法获取读取权限")
                SET internal_state = STATE_IDLE
                CONTINUE

            // 4a. 根据查询类型构建统计摘要
            CASE 查询请求.查询类型 OF:
                "偏好摘要":
                    结果 = 构建偏好摘要(查询请求.时间窗口)
                "维度详情":
                    结果 = 构建维度详情(查询请求.行为维度, 查询请求.时间窗口)
                "趋势":
                    结果 = 构建趋势数据(查询请求.行为维度, 查询请求.标签类别)

            向请求来源返回历史偏好统计摘要(结果)
            SET internal_state = STATE_IDLE

        // 第5步：周期性状态上报
        IF 距上次状态上报 >= 60秒:
            向 ag-mem-02 上报统计状态(
                当前状态=internal_state,
                累计统计条目数=统计缓存.总条目数,
                活跃维度数=统计缓存.活跃维度数量,
                存储使用量=统计缓存.存储估算
            )

        SLEEP 10ms


FUNCTION 检测异常波动(行为维度, 新条目):
    IF 行为维度 NOT IN 统计缓存.按行为维度:
        RETURN False  // 新维度，无历史基线

    历史统计 = 统计缓存.按行为维度[行为维度]
    近7日均值 = 历史统计.最近7日次数 / 7

    IF 近7日均值 > 0 AND 新条目.置信度 > 0.7:
        当前频率 = 统计缓存.当前小时该维度次数
        IF 当前频率 > 近7日均值 × 3:
            RETURN True  // 异常高频

    RETURN False
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 写入令牌申请失败 | 暂存待更新条目在内存队列，定时重试 | 令牌申请成功 |
| 统计缓存与画像槽数据不一致 | 以画像槽持久化数据为准，重新加载缓存 | 缓存刷新完成 |
| 查询请求的时间窗口超出数据范围 | 返回可用范围内的数据，标记“数据不完整” | 数据积累到覆盖查询范围 |
| 统计异常波动告警过于频繁 | 同一维度告警冷却期60秒，期间不重复发送 | 冷却期结束 |
| 紧急熔断 | 暂停统计更新，保留内存缓存中的未持久化数据 | 紧急解除后恢复并刷新缓存 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 统计摘要在响应查询前必须通过 ag-mem-06 隔离校验，确保仅返回当前活跃槽位的数据 |
| S-02 | 偏好关键词集合仅提取自用户主动输入，不得包含系统生成内容的分析结果 |
| S-03 | 统计异常告警仅基于当前槽位的历史数据对比，不得跨用户进行异常检测 |
| S-04 | 统计数据的持久化写入必须完整，写入中断时保留未损坏的前一版本作为回滚点 |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-M10-01 | `IDLE`，收到带标签条目 | 3条带标签行为条目（TOOL_INVOKE×2, RESULT_COPY×1） | 统计更新成功，返回更新确认，各维度计数增加 |
| TC-M10-02 | `IDLE`，收到偏好摘要查询 | 查询请求（类型=偏好摘要，时间窗口=最近7日） | 返回7日内各维度频次与偏好强度向量 |
| TC-M10-03 | `IDLE`，检测到异常高频行为 | 某维度当前小时频率超过近7日均值3倍 | 发送统计异常告警至ag-mem-02 |
| TC-M10-04 | `IDLE`，查询超出数据范围 | 查询请求（时间窗口=最近30日，但仅积累7日数据） | 返回7日数据，标记“数据不完整” |
| TC-M10-05 | `SYSTEM_PAUSED`，收到统计查询 | 统计查询请求 | 拒绝处理，保持暂停状态 |
