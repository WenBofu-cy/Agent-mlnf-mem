# ag-mem-26-L4长期层存储单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-26 |
| 模块名称 | L4长期层存储单元 |
| 所属分区 | 三、漏斗二：任务经验漏斗 / 五层存储 |
| 核心职责 | 作为漏斗二五层记忆存储架构的第四层，专门存储从L3晋升而来的跨场景可泛化复用的高阶任务技能经验。L4是经验从“稳定策略”进入“长期固化”的关键层，占漏斗二总容量的4.5%。对写入的经验数据进行去个性化处理，通过ag-mem-27提取通用规则。为ag-mem-27提供待抽象提炼的同类经验条目集。L4层条目默认受遗忘保护，仅当I值降至遗忘阈值以下且复用频次验证不足时才被遗忘。晋升至L5核心层的经验需通过更高门槛。不参与晋升判定或遗忘决策，仅执行经验的接收、存储、泛化管理与基础管理 |
| 依赖模块 | ag-mem-24（L3中期层存储单元，发送晋升条目列表）、ag-mem-27（L4长期层经验抽象提炼单元，执行通用规则提取）、ag-mem-28（L5核心层存储单元，接收晋升条目）、ag-mem-40（遗忘阈值判定单元，执行遗忘扫描）、ag-mem-48（全局容量配额管控单元，查询L4当前容量） |
| 被依赖模块 | ag-mem-24（返回写入确认）、ag-mem-27（接收抽象提炼触发指令）、ag-mem-15~19（各场景分槽，通过查询接口检索L4经验） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常服务 | `NORMAL` | L4存储正常，接收晋升条目 | 系统初始化完成，存储分区就绪 |
| 容量预警 | `CAPACITY_WARNING` | L4存储使用率超过80%，触发温和清理 | L4使用率 ≥ 80% |
| 容量紧急 | `CAPACITY_CRITICAL` | L4存储使用率超过95%，暂停写入并强制清理 | L4使用率 ≥ 95% |
| 抽象提炼中 | `ABSTRACTING` | 正在对L4中同类经验执行通用规则提取 | 定时触发或收到维护指令 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |

## L4存储配置

| 配置项 | 默认值 | 说明 |
|--------|:---:|------|
| L4占漏斗二总容量比例 | **4.5%** | L4是较小容量存储层，承载高质量泛化经验 |
| L4最大条目数 | 1000条 | 硬上限，达到后强制触发清理 |
| L4条目最大留存时长 | **无硬性限制**（仅受遗忘策略约束） | L4层经验默认长期保留 |
| L4单条目最大大小 | 25KB | 允许更丰富的经验上下文与泛化规则数据 |
| 写入超时 | 300ms | 单次写入操作的最大等待时间 |
| 容量预警阈值 | 80% | 触发温和清理 |
| 容量紧急阈值 | 95% | 触发强制清理并暂停写入 |
| 抽象提炼触发间隔 | 每新增20条同类经验 或 每72小时 | 触发通用规则提取 |
| 遗忘保护标记 | **强保护** | L4遗忘阈值显著低于L3，仅在I值极低且复用不足时遗忘 |


## 去个性化处理规则

### 一、去个性化目的

L4层经验将被用于跨用户、跨场景的通用技能泛化。在写入L4时，必须执行去个性化处理，确保经验中不包含任何可关联到特定用户的个人信息。

### 二、去个性化字段清单

| 字段 | 处理方式 |
|------|------|
| 用户ID | **删除**，替换为“匿名用户”标记 |
| 会话ID | **删除** |
| 设备指纹 | **删除** |
| 地理位置信息 | **删除** |
| 用户原始输入文本 | **删除**，仅保留任务特征向量与意图标签 |
| 用户个性化偏好参数 | **删除**，仅保留通用工具调用序列 |
| 时间戳 | 保留（仅用于时序分析） |
| 任务特征向量 | 保留 |
| 工具调用序列 | 保留 |
| 结果标签 | 保留 |
| 重要度I值 | 保留（重新计算，去除V值权重影响） |
| 来源分槽编号 | 保留 |

### 三、去个性化后重要度重算

```
L4_I值 = I₀ + α·S + γ·C
// 注意：去除了V值（用户价值）维度，仅保留安全显著性与复用频次
// α和γ使用各分槽的专属权重系数
```


## 核心处理逻辑

```
FUNCTION l4_storage_main_loop():
    STATE_NORMAL = NORMAL
    STATE_WARN = CAPACITY_WARNING
    STATE_CRITICAL = CAPACITY_CRITICAL
    STATE_ABSTRACT = ABSTRACTING
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_NORMAL
    初始化L4存储分区（内存索引 + 持久化存储）
    初始化条目计数器 = 0
    初始化同类经验累计计数器 = {}  // 分槽编号 → 自上次抽象提炼后的新增条目数

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_NORMAL

        // 第2步：接收L3晋升条目列表
        IF 收到 ag-mem-24 L4晋升条目列表:
            IF internal_state == STATE_PAUSED:
                向 ag-mem-24 返回L4写入确认回执(0, 0, 0, "系统熔断中")
                CONTINUE

            // 2a. 校验：L4不接受警示条目
            FOR EACH 晋升条目 IN 晋升条目列表:
                IF 晋升条目.警示标签 == "CAUTION":
                    向 ag-mem-24 返回错误("L4不接受警示条目，条目ID=" + 晋升条目.条目ID)
                    从晋升条目列表中移除该条目

            IF 晋升条目列表为空:
                CONTINUE

            // 2b. 查询当前L4容量
            容量回执 = 向 ag-mem-48 查询L4当前容量()
            L4使用率 = 容量回执.当前使用率

            // 2c. 容量状态更新
            IF L4使用率 >= 95%:
                SET internal_state = STATE_CRITICAL
            ELSE IF L4使用率 >= 80%:
                SET internal_state = STATE_WARN

            // 2d. 容量紧急处理
            IF internal_state == STATE_CRITICAL:
                IF 条目计数器 >= L4最大条目数:
                    向 ag-mem-40 发送遗忘扫描触发指令(L4中重要度最低的15%条目, "容量紧急")
                    等待清理结果
                    IF 清理后使用率仍 >= 95%:
                        向 ag-mem-24 返回L4写入确认回执(接收数, 0, 0, "L4容量已满")
                        CONTINUE

            // 2e. 逐条写入L4（含去个性化处理）
            成功写入数 = 0
            去个性化处理数 = 0
            FOR EACH 晋升条目 IN 晋升条目列表:
                // 校验条目大小
                IF 晋升条目.经验数据.大小 > 25KB:
                    CONTINUE

                // 去个性化处理
                去个性化经验 = 执行去个性化处理(晋升条目.经验数据)
                去个性化处理数++

                // 重新计算重要度（去除V值影响）
                L4_I值 = 晋升条目.I₀ + α × 晋升条目.S值 + γ × 晋升条目.C值

                // 写入L4存储
                存储位置 = L4存储分区.追加写入(
                    条目ID=晋升条目.条目ID,
                    经验数据=去个性化经验,
                    重要度I值=L4_I值,
                    来源分槽=晋升条目.来源分槽编号,
                    警示标签="NORMAL",  // L4不接受警示条目，确保为NORMAL
                    写入时间=NOW()
                )
                IF 存储位置 != None:
                    成功写入数++
                    条目计数器++

                    // 更新同类经验累计计数器
                    分槽编号 = 晋升条目.来源分槽编号
                    IF 分槽编号 NOT IN 同类经验累计计数器:
                        同类经验累计计数器[分槽编号] = 0
                    同类经验累计计数器[分槽编号]++

            // 2f. 返回确认
            向 ag-mem-24 返回L4写入确认回执(
                接收条目数=晋升条目列表.size,
                成功写入数=成功写入数,
                去个性化处理数=去个性化处理数,
                L4当前使用率=L4使用率
            )

            // 2g. 检查是否需要触发抽象提炼
            FOR EACH 分槽编号, 累计数 IN 同类经验累计计数器:
                IF 累计数 >= 20:
                    向 ag-mem-27 发送抽象提炼触发指令(
                        提炼范围=该分槽的同类经验,
                        来源槽位编号="ag-mem-26",
                        同类经验分组=该分槽近20条新条目
                    )
                    同类经验累计计数器[分槽编号] = 0

        // 第3步：响应经验查询请求
        IF 收到来自 ag-mem-15~19 的L4经验查询请求:
            查询条件 = 请求.查询条件
            来源分槽 = 请求.来源分槽编号

            // 3a. 检索匹配条目
            匹配条目 = L4存储.查询(查询条件, 来源分槽, 请求.最大返回条数)
            
            // 3b. 标记是否已抽象提炼
            FOR EACH 条目 IN 匹配条目:
                条目.has_abstracted = 条目.已关联通用规则

            // 3c. 返回结果
            向来源分槽返回L4查询结果列表(匹配条目, 层级="L4")

        // 第4步：定时抽象提炼触发（每72小时）
        IF 距上次定时抽象提炼 >= 72小时:
            SET internal_state = STATE_ABSTRACT
            // 对L4中所有未提炼的条目进行分组提炼
            FOR EACH 分槽编号 IN L4存储.获取所有分槽():
                该分槽未提炼条目 = L4存储.筛选(分槽编号, 已抽象提炼=False)
                IF 该分槽未提炼条目.size >= 5:
                    向 ag-mem-27 发送抽象提炼触发指令(
                        提炼范围=该分槽未提炼条目,
                        来源槽位编号="ag-mem-26",
                        同类经验分组=该分槽未提炼条目
                    )
            SET internal_state = STATE_NORMAL
            重置定时抽象提炼计时器

        // 第5步：定时遗忘扫描（每24小时，使用强保护阈值）
        IF 距上次遗忘扫描 >= 24小时 AND internal_state IN [NORMAL, WARN]:
            向 ag-mem-40 发送遗忘扫描触发指令(
                遗忘范围=L4全量条目,
                遗忘阈值=各分槽L4专属遗忘阈值,
                遗忘保护标记="强保护",
                来源槽位编号="ag-mem-26"
            )
            重置遗忘扫描计时器

        // 第6步：接收抽象提炼完成回执
        IF 收到 ag-mem-27 抽象提炼完成回执:
            // 更新L4中被提炼条目的状态标记
            FOR EACH 提炼规则 IN 回执.提炼规则列表:
                来源条目ID列表 = 提炼规则.来源条目ID列表
                FOR EACH 条目ID IN 来源条目ID列表:
                    L4存储.更新抽象标记(条目ID, 已提炼=True, 关联规则ID=提炼规则.规则ID)

        // 第7步：周期性状态上报
        IF 距上次状态上报 >= 120秒:
            统计已抽象提炼条目数
            向 ag-mem-48 和 ag-mem-03 上报L4状态(
                当前状态=internal_state,
                条目总数=条目计数器,
                使用率=L4使用率,
                已抽象提炼条目数=统计已提炼数
            )

        SLEEP 10ms


FUNCTION 执行去个性化处理(原始经验):
    去个性化 = 原始经验.clone()
    去个性化.用户ID = "匿名用户"
    去个性化.会话ID = None
    去个性化.设备指纹 = None
    去个性化.地理位置信息 = None
    去个性化.用户原始输入文本 = None
    去个性化.用户个性化偏好参数 = None
    // 保留：任务特征向量、工具调用序列、结果标签、时间戳、来源分槽编号
    RETURN 去个性化
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 晋升条目包含警示标签 | 拒绝该条目写入L4，返回错误提示 | 警示标签降级为NORMAL后重新晋升 |
| L4存储写入异常 | 返回部分成功写入数，上报告警 | 存储恢复正常 |
| 条目大小超过25KB上限 | 跳过该条目，记录跳过日志 | — |
| L4使用率达95%且遗忘清理后仍无法释放空间 | 拒绝新写入，上报告警 | 清理成功或人工扩容 |
| 去个性化处理异常（数据格式不兼容） | 使用保守去个性化（删除所有非必要字段），标记处理异常 | 数据格式修复 |
| 抽象提炼期间收到写入请求 | 写入请求排队等待，提炼完成后处理 | 提炼完成 |
| 紧急熔断 | 暂停所有操作，保持L4数据完整性 | 紧急解除后恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 所有写入L4的经验必须经过去个性化处理，禁止保留任何可关联到特定用户的个人信息 |
| S-02 | L4不接受警示条目标签为CAUTION的经验，确保长期层不包含失败策略 |
| S-03 | L4层经验受强遗忘保护，遗忘阈值显著低于其他层级，仅在I值极低且复用频次验证不足时才允许遗忘 |
| S-04 | 去个性化后的重要度重算必须去除V值（用户价值）维度，仅基于安全显著性与复用频次 |
| S-05 | L4层经验在晋升L5时必须额外通过安全底线校验（由ag-mem-43执行） |
