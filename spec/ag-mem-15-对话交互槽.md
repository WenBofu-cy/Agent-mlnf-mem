# ag-mem-15-对话交互槽 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-15 |
| 模块名称 | 对话交互槽 |
| 所属分区 | 三、漏斗二：任务经验漏斗 / 场景分槽管理 |
| 核心职责 | 作为漏斗二中专门承载“对话交互”类任务经验的场景分槽。接收 ag-mem-03 路由的对话交互场景经验条目，管理该场景下的五层记忆存储（L1-L5）。在本槽内，用户价值（V值）权重自动上调20%，以强化用户个性化对话策略的沉淀。同时管理该槽专属的晋升阈值与遗忘策略参数。不参与认知决策，仅执行对话交互类经验的存储、检索与生命周期管理 |
| 依赖模块 | ag-mem-03（漏斗二专属调度单元，下发经验路由）、ag-mem-20~30（五层存储单元，实际存储操作）、ag-mem-35（三维权重系数配置单元，读取并应用本槽专属权重） |
| 被依赖模块 | ag-mem-03（返回查询/写入结果）、ag-mem-25（相似经验归并单元，可扫描本槽内的重复经验） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 无操作请求，等待指令 | 系统初始化完成，槽位已激活 |
| 写入处理 | `WRITING` | 正在将新经验条目写入五层存储 | 收到 ag-mem-03 的经验写入请求 |
| 查询处理 | `QUERYING` | 正在检索匹配的历史对话经验 | 收到 ag-mem-03 的经验查询请求 |
| 维护整理 | `MAINTENANCE` | 正在执行槽内经验归并或遗忘扫描 | 收到维护指令或定时触发 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 经验写入请求 | Struct（经验条目数据 + 用户ID + 重要度预估值 + 场景标签确认） | ag-mem-03 漏斗二专属调度单元（通过内部调度总线） | 对话交互类任务执行完成，产生新经验时 | **高** |
| 经验查询请求 | Struct（查询条件 + 用户ID + 最大返回条数 + 时间窗口） | ag-mem-03 漏斗二专属调度单元（通过内部调度总线） | ECC 需要检索历史对话经验时 | **高** |
| 维护扫描指令 | Struct（扫描类型：归并/遗忘 + 扫描深度） | ag-mem-03 漏斗二专属调度单元 | 定时触发或容量告警时 | 普通 |
| 专属权重配置参数 | Struct（V值权重系数 + 各层晋升阈值 + 遗忘阈值） | ag-mem-35 三维权重系数配置单元 | 槽位激活或配置更新时 | 普通 |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-mem-01 总控漏斗F0 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 写入确认回执 | Struct（条目ID + 分配层级 + 预估晋升时间 + 写入耗时） | ag-mem-03 漏斗二专属调度单元 | 写入完成时 | **高** |
| 查询结果列表 | Struct（匹配经验列表 + 各条目层级 + 重要度 + 查询耗时） | ag-mem-03 漏斗二专属调度单元 | 查询完成时 | **高** |
| 维护完成回执 | Struct（归并条目数 + 遗忘条目数 + 释放空间量） | ag-mem-03 漏斗二专属调度单元 | 维护扫描完成时 | 普通 |
| 槽位状态上报 | Struct（当前条目总数 + 各层级分布 + 存储使用量 + 最近活跃时间） | ag-mem-03 漏斗二专属调度单元 | 周期性（每60秒）或状态变更时 | 普通 |


## 专属参数配置

| 参数名称 | 默认值 | 说明 |
|----------|:---:|------|
| V值权重系数 | **1.2**（标准为1.0，上调20%） | 使对话交互经验更重视用户价值信号 |
| L1→L2晋升阈值 | I ≥ 0.35 | 略低于标准0.40，促进对话经验更快晋升 |
| L2→L3晋升阈值 | I ≥ 0.55 | 标准 |
| L3→L4晋升阈值 | I ≥ 0.75 | 标准 |
| L4→L5晋升阈值 | I ≥ 0.90 | 标准 |
| L1遗忘阈值 | I < 0.08 | 略低于标准0.10，保留更多临时对话 |
| L2遗忘阈值 | I < 0.20 | 标准 |
| L3遗忘阈值 | I < 0.30 | 标准 |


## 核心处理逻辑

```
FUNCTION dialogue_slot_main_loop():
    STATE_IDLE = IDLE
    STATE_WRITE = WRITING
    STATE_QUERY = QUERYING
    STATE_MAINT = MAINTENANCE
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_IDLE
    初始化槽位编号 = "ag-mem-15"
    加载专属权重配置（V值系数=1.2）

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_IDLE

        // 第2步：接收经验写入请求
        IF 收到 ag-mem-03 经验写入请求:
            SET internal_state = STATE_WRITE
            经验条目 = 请求.经验条目数据

            // 2a. 应用专属权重：V值上调20%
            IF 经验条目.v_value != None:
                经验条目.v_value = 经验条目.v_value × 1.2
                IF 经验条目.v_value > 1.0:
                    经验条目.v_value = 1.0

            // 2b. 重新计算重要度（使用上调后的V值）
            调整后重要度 = 计算重要度(经验条目.i0, 经验条目.s_value, 经验条目.v_value, 经验条目.c_value)

            // 2c. 写入L1临时层
            写入结果 = 向 ag-mem-20 发起L1写入(经验条目, 调整后重要度)

            // 2d. 返回确认
            向 ag-mem-03 返回写入确认回执(
                条目ID=写入结果.条目ID,
                分配层级="L1",
                预估晋升时间=计算预估晋升时间(调整后重要度, 当前时间)
            )
            SET internal_state = STATE_IDLE

        // 第3步：接收经验查询请求
        IF 收到 ag-mem-03 经验查询请求:
            SET internal_state = STATE_QUERY
            查询条件 = 请求.查询条件

            // 3a. 从五层存储中检索匹配经验
            结果集 = []
            FOR EACH 层级 IN [L5, L4, L3, L2, L1]:  // 从核心层到临时层检索
                层级结果 = 向对应存储单元发起查询(层级, 查询条件, 请求.最大返回条数)
                结果集.合并(层级结果)
                IF 结果集.size >= 请求.最大返回条数:
                    BREAK

            // 3b. 按重要度排序
            结果集 = 按重要度降序排列(结果集)
            结果集 = 截断到最大返回条数(结果集, 请求.最大返回条数)

            // 3c. 返回结果
            向 ag-mem-03 返回查询结果列表(结果集, 各条目层级, 重要度)
            SET internal_state = STATE_IDLE

        // 第4步：执行维护扫描
        IF 收到维护扫描指令:
            SET internal_state = STATE_MAINT

            // 4a. 归并相似经验
            归并结果 = 向 ag-mem-25 发起槽内归并扫描(槽位编号=ag-mem-15)
            
            // 4b. 遗忘低重要度条目
            遗忘结果 = 向 ag-mem-40 发起槽内遗忘扫描(槽位编号=ag-mem-15, 专属遗忘阈值)

            // 4c. 返回维护结果
            向 ag-mem-03 返回维护完成回执(
                归并条目数=归并结果.归并数,
                遗忘条目数=遗忘结果.遗忘数,
                释放空间量=遗忘结果.释放空间
            )
            SET internal_state = STATE_IDLE

        // 第5步：周期性状态上报
        IF 距上次状态上报 >= 60秒:
            统计各层级条目数、存储使用量
            向 ag-mem-03 上报槽位状态

        SLEEP 20ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 写入L1失败（存储满） | 触发L1清理（遗忘低重要度条目），重试写入 | 清理完成且有足够空间 |
| 查询结果为空 | 返回空列表 + 标记“无匹配经验” | — |
| V值上调后溢出（>1.0） | 截断为1.0 | — |
| 维护扫描期间收到写入请求 | 写入请求排队等待，维护完成后处理 | 维护完成 |
| 紧急熔断 | 暂停所有操作，保持槽位状态 | 紧急解除后恢复 |

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|

| 内部调度总线 | 读 | 维护扫描指令 | 只读 | ag-mem-03 发送 |

| 内部调度总线 | 写 | 查询结果列表 | 专属写入 | 向 ag-mem-03 返回 |
| 内部调度总线 | 写 | 维护完成回执 | 专属写入 | 向 ag-mem-03 返回 |
| 内部调度总线 | 写 | 槽位状态上报 | 周期性写入 | 向 ag-mem-03 发送 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 本槽位仅接受场景标签确认为“对话交互”的经验条目，其他场景条目将被拒绝 |
| S-02 | V值权重上调仅在本槽位内生效，不得影响其他分槽的重要度计算 |
| S-03 | 槽内经验的跨槽查询必须通过 ag-mem-03 统一路由，不得直接响应外部模块的查询 |
| S-04 | 维护扫描期间的遗忘操作必须遵循本槽专属的遗忘阈值，不得使用全局默认值 |

| 用例编号 | 前置条件 | 输入 | 预期输出 |

| TC-M15-06 | `SYSTEM_PAUSED`，收到查询请求 | 查询请求 | 拒绝处理，保持暂停状态 |
