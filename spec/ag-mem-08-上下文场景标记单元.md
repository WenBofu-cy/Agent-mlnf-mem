# ag-mem-08-上下文场景标记单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-08 |
| 模块名称 | 上下文场景标记单元 |
| 所属分区 | 二、漏斗一：用户画像漏斗 |
| 核心职责 | 接收 ag-mem-07（用户行为观测记录单元）的场景查询请求，基于当前会话的最近行为序列、任务特征及时间窗口，动态判定当前用户所处的场景类别与交互阶段，输出标准化的上下文场景标签。为行为观测条目提供场景语义锚点，使同一用户在不同场景下的行为能够被准确标记和区分。不参与任何认知决策，仅提供场景判定与标签输出 |
| 依赖模块 | ag-mem-02（漏斗一专属调度单元，提供当前活跃会话ID）、ag-mem-07（用户行为观测记录单元，可选地传入最近行为序列用于判定） |
| 被依赖模块 | ag-mem-07（用户行为观测记录单元，获取场景标签后写入行为条目）、ag-mem-09（偏好判定标签单元，可参考场景标签辅助判定） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 无查询请求，等待指令 | 系统初始化完成，无待处理查询 |
| 场景分析 | `ANALYZING` | 正在基于当前上下文判定场景类别 | 收到 ag-mem-07 的场景查询请求 |
| 判定完成 | `JUDGED` | 场景判定完成，已输出标签 | 场景标签已返回 ag-mem-07 |
| 数据不足 | `INSUFFICIENT_DATA` | 可用于判定的行为数据不足，使用默认标签 | 行为序列为空或过短 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 场景查询请求 | Struct（会话ID + 请求来源模块 + 可选：最近行为序列列表） | ag-mem-07 用户行为观测记录单元 | 每次构建行为观测条目时 | **高** |
| 当前活跃会话绑定信息 | Struct（会话ID + 活跃槽位编号 + 用户ID） | ag-mem-02 漏斗一专属调度单元 | 槽位激活时，供场景分析参考 | 普通 |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-mem-01 总控漏斗F0 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 上下文场景标签 | Struct（场景类别 + 任务类型 + 交互阶段 + 置信度 + 判定依据） | ag-mem-07 用户行为观测记录单元 | 场景判定完成时 | **高** |
| 数据不足默认标签 | Struct（场景类别=通用任务 + 置信度=0.3 + 判定依据=数据不足） | ag-mem-07 用户行为观测记录单元 | 行为数据不足以支持判定时 | **高** |
| 场景切换事件通知 | Struct（前场景 + 新场景 + 切换时间戳） | ag-mem-09 偏好判定标签单元（可选） | 检测到场景发生明显切换时 | 普通 |


## 场景判定规则

### 一、场景类别定义（与漏斗二场景分槽对齐）

| 场景类别 | 判定关键词/特征 | 典型行为序列示例 |
|----------|------|------|
| 对话交互 | 连续多轮问答、闲聊、角色扮演 | TEXT_INPUT → RESULT_VIEW → TEXT_INPUT → RESULT_VIEW |
| 工具调用 | 调用外部API、执行代码、文件操作 | TEXT_INPUT → TOOL_INVOKE → RESULT_VIEW |
| 信息检索 | 搜索、查询、知识获取 | TEXT_INPUT → RESULT_VIEW（含引用链接）→ RESULT_COPY |
| 创作生成 | 文本写作、代码生成、图像描述 | TEXT_INPUT（含"写""生成""创作"）→ RESULT_VIEW → RESULT_COPY/SHARE |
| 通用任务 | 混合操作、无法明确归类的行为 | 行为序列不匹配以上任何模式 |

### 二、交互阶段定义

| 交互阶段 | 判定条件 |
|----------|------|
| 任务开始 | 当前为会话首条消息，或前一条为 SESSION_START，或检测到全新任务意图 |
| 执行中 | 已发出任务请求但尚未收到最终结果，或正在浏览/处理中间结果 |
| 结果评估 | 收到任务结果后的浏览、复制、分享、点赞、踩等行为 |
| 任务结束 | 会话结束，或连续沉默超过阈值时间 |

### 三、判定规则优先级

1. 优先根据最近一条关键行为类型确定场景（如 TOOL_INVOKE → 工具调用）
2. 若最近一条不足以确定，回溯最近3条行为序列进行模式匹配
3. 若仍无法确定，根据当前会话的整体任务意图判定（由 ECC 传入的意图标签辅助）
4. 完全无可用数据时，返回通用任务 + 低置信度


## 核心处理逻辑

```
FUNCTION scene_marker_main_loop():
    STATE_IDLE = IDLE
    STATE_ANALYZE = ANALYZING
    STATE_JUDGED = JUDGED
    STATE_INSUFF = INSUFFICIENT_DATA
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_IDLE
    初始化场景切换检测器

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_IDLE

        // 第2步：接收场景查询请求
        IF 收到 ag-mem-07 场景查询请求:
            SET internal_state = STATE_ANALYZE

            行为序列 = 请求.最近行为序列
            IF 行为序列 == None OR 行为序列为空:
                // 尝试从当前会话上下文获取
                行为序列 = 获取当前会话最近行为序列(请求.会话ID, 最近5条)

            IF 行为序列 == None OR 行为序列长度 == 0:
                SET internal_state = STATE_INSUFF
                向 ag-mem-07 返回数据不足默认标签(
                    场景类别="通用任务",
                    置信度=0.3,
                    判定依据="无可用的行为数据"
                )
                SET internal_state = STATE_IDLE
                CONTINUE

            // 第3步：场景判定
            最近行为 = 行为序列[0]
            判定结果 = 基于规则判定场景(行为序列)

            // 3a. 检测场景切换
            IF 上一次场景标签 != None AND 上一次场景标签.场景类别 != 判定结果.场景类别:
                向 ag-mem-09 发送场景切换事件通知(上一次场景标签, 判定结果, NOW())

            上一次场景标签 = 判定结果

            // 第4步：输出标签
            SET internal_state = STATE_JUDGED
            向 ag-mem-07 返回上下文场景标签(判定结果)
            SET internal_state = STATE_IDLE

        SLEEP 10ms


FUNCTION 基于规则判定场景(行为序列):
    // 提取最近一条关键行为
    最近关键行为 = 从行为序列中找到第一个非 SESSION_START/SESSION_END 的行为
    IF 最近关键行为 == None:
        RETURN {场景类别: "通用任务", 置信度: 0.3, 判定依据: "无关键行为"}

    // 规则1：工具调用场景
    IF 最近关键行为.类型 == "TOOL_INVOKE":
        RETURN {场景类别: "工具调用", 置信度: 0.90, 判定依据: "检测到工具调用行为"}

    // 规则2：搜索类场景（查询后快速查看结果并复制）
    IF 最近关键行为.类型 == "RESULT_COPY" AND 行为序列中存在("TEXT_INPUT" 包含搜索关键词):
        RETURN {场景类别: "信息检索", 置信度: 0.75, 判定依据: "搜索-查看-复制模式"}

    // 规则3：创作类场景（输入包含创作关键词）
    IF 最近关键行为.类型 == "RESULT_VIEW" AND 行为序列中存在("TEXT_INPUT" 包含创作关键词):
        RETURN {场景类别: "创作生成", 置信度: 0.80, 判定依据: "输入包含创作意图关键词"}

    // 规则4：连续多轮问答 → 对话交互
    IF 行为序列中连续出现 TEXT_INPUT → RESULT_VIEW 模式超过2轮:
        RETURN {场景类别: "对话交互", 置信度: 0.70, 判定依据: "多轮问答模式匹配"}

    // 默认：通用任务
    RETURN {场景类别: "通用任务", 置信度: 0.50, 判定依据: "无匹配特征，使用默认场景"}
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 行为序列为空 | 返回通用任务标签 + 低置信度0.3 | 积累足够行为数据后正常判定 |
| 行为序列仅包含系统事件（SESSION_START/END） | 返回通用任务标签 + 置信度0.3 | 用户产生有效交互行为 |
| 场景切换检测到频繁切换（短时间内切换超过3次） | 锁定当前场景标签，延迟上报切换事件 | 行为模式稳定后恢复 |
| 紧急熔断 | 暂停服务，保持最后场景标签 | 紧急解除后恢复 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| 内部调度总线 | 读 | 场景查询请求 | 只读 | ag-mem-07 发送 |
| 内部调度总线 | 读 | 当前活跃会话绑定信息 | 只读 | ag-mem-02 发送 |
| 内部调度总线 | 写 | 上下文场景标签 | 专属写入 | 向 ag-mem-07 返回 |
| 内部调度总线 | 写 | 场景切换事件通知 | 事件触发写入 | 向 ag-mem-09 发送（可选） |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 场景判定仅基于行为元数据（类型、顺序），不得访问用户原始输入内容 |
| S-02 | 本模块不持久化任何用户数据，判定过程完全基于内存中的行为序列快照 |
| S-03 | 数据不足时不得强行猜测场景，必须标记低置信度并明确告知下游模块 |
