# ag-mem-42-冗余记忆删除与归档单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-42 |
| 模块名称 | 冗余记忆删除与归档单元 |
| 所属分区 | 三、漏斗二：任务经验漏斗 / 晋升与遗忘执行机制 |
| 核心职责 | 作为漏斗二中唯一有权物理清除数据的模块，接收来自 ag-mem-40（遗忘阈值判定单元）和 ag-mem-25（L3归并单元）的条目清除指令，对指定的经验条目执行安全删除或冷归档。根据条目的当前层级与建议处理方式，L1/L2层级条目直接安全擦除，L3及以上层级条目压缩归档至冷存储以保留可追溯性。在完成删除或归档后，向 ag-mem-48（全局容量配额管控单元）退还已释放的存储配额，并向 ag-mem-51（记忆变更日志追溯单元）记录完整的删除/归档事件。不参与遗忘判定或存储管理决策，仅执行已批准的条目清理操作 |
| 依赖模块 | ag-mem-40（遗忘阈值判定单元，提供遗忘候选清单）、ag-mem-25（L3归并单元，提供归并淘汰的原始条目清单）、ag-mem-48（全局容量配额管控单元，接收退还的存储配额）、ag-mem-49（存储压缩与冷归档单元，执行L3及以上条目的压缩归档） |
| 被依赖模块 | ag-mem-40（返回删除/归档完成回执）、ag-mem-25（返回归并清理完成回执）、ag-mem-20~26（各层级存储单元，接收物理删除指令）、ag-mem-51（记忆变更日志追溯单元，记录删除/归档事件） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 无清理任务，等待指令 | 系统初始化完成，无待处理请求 |
| 删除执行中 | `DELETING` | 正在对L1/L2层级条目执行直接安全擦除 | 收到遗忘候选清单或归并淘汰清单 |
| 归档执行中 | `ARCHIVING` | 正在对L3/L4层级条目执行压缩归档 | 收到遗忘候选清单（含L3/L4条目） |
| 配额退还中 | `QUOTA_RETURN` | 正在向 ag-mem-48 退还已释放的存储配额 | 删除或归档操作完成 |
| 操作失败 | `OPERATION_FAILED` | 删除或归档过程中发生不可恢复的异常 | 存储写入失败或配额退还被拒 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |

## 处理策略

### 一、按层级与建议处理方式的执行策略

| 条目层级 | 建议处理方式 | 实际操作 | 说明 |
|:---:|------|:---:|------|
| L1 | 删除 | **直接安全擦除** | L1经验价值最低，无需归档 |
| L2 | 删除 | **直接安全擦除** | L2经验价值中等，直接删除 |
| L3 | 删除 | **直接安全擦除** | 当ag-mem-40明确建议删除时 |
| L3 | 归档 | **压缩归档至冷存储** | L3经验有一定价值，归档可追溯 |
| L4 | 归档 | **压缩归档至冷存储** | L4经验价值高，归档保留可追溯性 |
| L5 | — | **禁止操作** | L5为永久记忆，编译期拒绝任何删除/归档请求 |

### 二、安全擦除标准

| 擦除方式 | 适用层级 | 说明 |
|----------|:---:|------|
| 直接标记删除 | L1 | 标记存储空间为可复用，不覆写 |
| 单次覆写擦除 | L2 | 覆写随机数据后标记为可复用 |
| 归档后擦除 | L3/L4 | 先完整归档至冷存储，确认归档成功后从源层级删除 |

### 三、分批处理规则

| 规则项 | 默认值 | 说明 |
|--------|:---:|------|
| 每批最大处理条目数 | **200条** | 避免单次操作占用过多IO资源 |
| 批次间隔时间 | **100ms** | 给下游模块喘息时间 |
| 单条目处理超时 | **5秒** | 超过此时限标记为失败，继续处理下一条 |


## 核心处理逻辑

```
FUNCTION redundant_memory_pruner_main_loop():
    STATE_IDLE = IDLE
    STATE_DELETE = DELETING
    STATE_ARCHIVE = ARCHIVING
    STATE_QUOTA = QUOTA_RETURN
    STATE_FAIL = OPERATION_FAILED
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_IDLE
    初始化处理队列 = []
    初始化处理统计计数器

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_IDLE

        // 第2步：接收遗忘候选清单
        IF 收到 ag-mem-40 遗忘候选清单:
            IF internal_state != STATE_IDLE:
                将清单追加到处理队列
                CONTINUE

            SET internal_state = STATE_DELETE
            处理条目列表(遗忘候选清单, "遗忘判定", ag-mem-40)

        // 第3步：接收归并淘汰条目清单
        IF 收到 ag-mem-25 归并淘汰条目清单:
            IF internal_state != STATE_IDLE:
                将清单追加到处理队列
                CONTINUE

            SET internal_state = STATE_DELETE
            处理条目列表(归并淘汰条目清单, "归并淘汰", ag-mem-25)


FUNCTION 处理条目列表(条目列表, 处理原因, 请求来源):
    成功删除数 = 0
    成功归档数 = 0
    失败条目列表 = []
    总释放空间 = 0

    // 按层级分类：L1/L2走删除，L3/L4走归档
    FOR EACH 条目 IN 条目列表:
        // L5保护检查
        IF 条目.来源层级 == "L5":
            失败条目列表.append({条目ID: 条目.条目ID, 失败原因: "L5为永久记忆，禁止删除"})
            CONTINUE

        // 确定处理方式
        IF 条目.来源层级 IN ["L1", "L2"] OR 条目.建议处理方式 == "删除":
            处理方式 = "删除"
        ELSE IF 条目.来源层级 IN ["L3", "L4"] AND 条目.建议处理方式 == "归档":
            处理方式 = "归档"
        ELSE:
            处理方式 = "删除"  // 默认

        CASE 处理方式 OF:
            "删除":
                // 发送源层级删除指令
                向 条目.来源层级存储单元 发送删除指令(
                    条目ID=条目.条目ID,
                    删除原因=处理原因 + ": " + 条目.遗忘原因,
                    是否归档=False
                )
                等待删除确认超时=5秒
                IF 删除确认.删除状态 == "成功":
                    成功删除数++
                    总释放空间 += 删除确认.释放空间量
                    // 记录删除事件日志
                    向 ag-mem-51 发送删除事件日志(
                        事件类型="删除",
                        条目ID=条目.条目ID,
                        来源层级=条目.来源层级,
                        处理原因=处理原因,
                        I值=条目.当前I值
                    )
                ELSE:
                    失败条目列表.append({条目ID: 条目.条目ID, 失败原因: "删除失败"})

            "归档":
                // 先执行归档写入
                向 ag-mem-49 发送归档写入指令(
                    条目ID=条目.条目ID,
                    经验数据=条目.经验数据,
                    元数据=条目.元数据,
                    归档原因=处理原因,
                    来源层级=条目.来源层级,
                    来源分槽=条目.来源分槽
                )
                等待归档确认超时=5秒
                IF 归档确认.归档状态 == "成功":
                    成功归档数++
                    归档位置 = 归档确认.归档存储位置

                    // 归档成功后从源层级删除
                    向 条目.来源层级存储单元 发送删除指令(
                        条目ID=条目.条目ID,
                        删除原因="已归档至" + 归档位置,
                        是否归档=True,
                        归档存储位置=归档位置
                    )
                    // 等待删除确认
                    等待删除确认超时=5秒
                    IF 删除确认.删除状态 == "成功":
                        总释放空间 += 删除确认.释放空间量
                    ELSE:
                        // 归档成功但删除失败，条目同时存在于源层级和冷存储
                        成功归档数--  // 归档仍算成功，但标记异常
                        失败条目列表.append({条目ID: 条目.条目ID, 失败原因: "归档成功但源层级删除失败"})

                    // 记录归档事件日志
                    向 ag-mem-51 发送归档事件日志(
                        事件类型="归档",
                        条目ID=条目.条目ID,
                        来源层级=条目.来源层级,
                        处理原因=处理原因,
                        归档位置=归档位置
                    )
                ELSE:
                    失败条目列表.append({条目ID: 条目.条目ID, 失败原因: "归档写入失败"})

    // 全部处理完成后，退还存储配额
    IF 总释放空间 > 0:
        SET internal_state = STATE_QUOTA
        向 ag-mem-48 发送配额退还请求(总释放空间)

    // 构建完成回执
    向 请求来源 返回删除/归档完成回执(
        处理条目总数=条目列表.size,
        成功删除数=成功删除数,
        成功归档数=成功归档数,
        失败条目列表=失败条目列表,
        总释放空间量=总释放空间,
        处理耗时=NOW() - 处理开始时间
    )

    // 检查处理队列
    IF 处理队列 非空:
        下一批 = 处理队列.pop(0)
        处理条目列表(下一批.条目列表, 下一批.处理原因, 下一批.请求来源)
    ELSE:
        SET internal_state = STATE_IDLE
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 清单中包含L5层级条目 | 跳过该条目，记录告警，不执行任何操作 | — |
| 删除指令发送后存储单元无响应（超时5秒） | 标记该条目失败，继续处理下一条 | 存储恢复正常 |
| 归档写入失败 | 标记该条目失败，保留在源层级（不删除），等待下次处理 | ag-mem-49恢复正常 |
| 归档成功但源层级删除失败 | 条目在源层级和冷存储同时存在，记录异常日志 | 下次维护周期由源层级清理 |
| 配额退还失败 | 记录告警，标记“配额未释放”，不影响删除/归档完成回执 | ag-mem-48恢复后手动触发退还 |
| 紧急熔断 | 完成当前正在处理的条目（安全优先），然后暂停接收新清单 | 紧急解除后恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| D-01 | L5核心层条目编译期拒绝任何删除或归档请求，确保永久记忆不被任何自动化模块清除 |
| D-02 | L3/L4层级条目必须先完整归档至冷存储，确认归档成功后，方可从源层级删除 |
| D-03 | 本模块为漏斗二中唯一有权物理清除数据的模块，其他模块不得绕过本模块直接删除经验条目 |
| D-04 | 所有删除和归档操作必须记录完整的不可变事件日志（条目ID、层级、原因、时间戳） |
| D-05 | 配额退还必须在删除或归档操作确认完成后执行，不得在操作未完成时提前退还 |
