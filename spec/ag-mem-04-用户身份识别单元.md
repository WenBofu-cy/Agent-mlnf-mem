# ag-mem-04-用户身份识别单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-04 |
| 模块名称 | 用户身份识别单元 |
| 所属分区 | 二、漏斗一：用户画像漏斗 |
| 核心职责 | 融合设备指纹、用户登录凭证、声纹特征等多模态信号，在每次用户会话开始时确认用户身份。将识别结果（已知用户ID + 置信度 + 是否新用户标记）上报至 ag-mem-02，由调度单元决定匹配已有画像槽或创建新槽。支持长期用户、临时用户和访客三种身份模式的识别与区分。不参与任何认知决策，仅提供身份识别结果 |
| 依赖模块 | 设备指纹采集模块（操作系统/浏览器提供）、用户登录系统（外部认证服务）、声纹识别引擎（可选，本地运行） |
| 被依赖模块 | ag-mem-02（漏斗一专属调度单元，接收身份识别结果用于槽位匹配）、ag-mem-07（用户行为观测记录单元，接收用户ID用于行为关联） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 等待会话 | `WAITING_SESSION` | 无活跃会话，等待用户接入 | 系统初始化完成，无用户交互 |
| 识别进行中 | `IDENTIFYING` | 正在采集多模态信号并执行身份识别 | 检测到新用户会话或收到身份查询请求 |
| 识别完成 | `IDENTIFIED` | 身份识别完成，已输出结果 | 识别流程结束，结果已上报 |
| 信号不足 | `INSUFFICIENT_SIGNAL` | 可用信号不足以完成高置信度识别，降级为临时用户或访客 | 多模态信号均不可用或置信度过低 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 会话启动信号 | Struct（会话ID + 接入方式 + 时间戳） | 系统会话管理器 | 用户打开应用或发起对话时 | **高** |
| 设备指纹数据 | Struct（设备ID + 浏览器指纹 + 操作系统 + 屏幕分辨率 + 时区） | 客户端采集（通过 API 传入） | 会话启动时自动采集 | **高** |
| 用户登录凭证 | Struct（用户ID + 认证令牌 + 有效期） | 外部认证服务（OAuth/JWT） | 用户主动登录时 | **高** |
| 声纹特征向量（可选） | Struct（特征向量 + 采集质量 + 环境噪声等级） | 本地声纹识别引擎 | 用户语音输入时 | 普通 |
| 身份查询请求 | Struct（请求来源模块 + 会话ID） | ag-mem-02 漏斗一专属调度单元 | 调度单元需要确认当前用户身份时 | **高** |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-mem-01 总控漏斗F0 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 用户身份识别结果 | Struct（用户ID + 识别方式 + 置信度 + 是否新用户 + 推荐槽位类型） | ag-mem-02 漏斗一专属调度单元 | 识别完成时 | **高** |
| 识别状态上报 | Struct（当前状态 + 可用信号列表 + 识别耗时 + 匹配度） | ag-mem-02 漏斗一专属调度单元 | 周期性或状态变更时 | 普通 |
| 新用户通知 | Struct（临时用户ID + 建议创建长期槽的触发条件） | ag-mem-02 漏斗一专属调度单元 | 检测到全新用户时 | 普通 |


## 身份识别策略

### 一、多模态信号优先级与置信度

| 信号类型 | 优先级 | 单独置信度 | 说明 |
|----------|:---:|:---:|------|
| 用户登录凭证 | **最高** | 0.98 | 已通过外部认证服务验证的登录令牌 |
| 设备指纹匹配 | **高** | 0.85 | 与已有画像槽关联的设备ID匹配 |
| 声纹特征匹配 | **中** | 0.75 | 与已有用户声纹模板匹配 |
| 仅设备指纹（无历史） | **低** | 0.50 | 设备未关联任何已知用户 |
| 无任何可用信号 | — | 0.0 | 标记为访客 |

### 二、综合置信度计算

```
综合置信度 = 取最高优先级可用信号的置信度

IF 登录凭证有效:
    用户ID = 登录凭证.用户ID
    置信度 = 0.98
    识别方式 = "登录凭证"
ELSE IF 设备指纹匹配已有用户:
    用户ID = 匹配到的用户ID
    置信度 = 0.85
    识别方式 = "设备指纹"
ELSE IF 声纹匹配已有用户:
    用户ID = 匹配到的用户ID
    置信度 = 0.75
    识别方式 = "声纹特征"
ELSE IF 设备指纹有效但无关联用户:
    用户ID = 生成临时ID
    置信度 = 0.50
    识别方式 = "设备指纹（新设备）"
    标记为潜在新用户
ELSE:
    用户ID = "GUEST_" + 会话ID
    置信度 = 0.0
    识别方式 = "无可用信号"
    标记为访客
```

### 三、用户类型判定

| 判定条件 | 用户类型 | 推荐槽位类型 |
|----------|:---:|:---:|
| 置信度 ≥ 0.85 且匹配已有长期槽 | 已知用户 | 长期槽 |
| 置信度 ≥ 0.50 且标记为潜在新用户 | 新用户 | 临时槽（可升级） |
| 置信度 = 0.0 | 访客 | 访客槽 |


## 核心处理逻辑

```
FUNCTION user_identity_main_loop():
    STATE_WAIT = WAITING_SESSION
    STATE_IDENTIFY = IDENTIFYING
    STATE_DONE = IDENTIFIED
    STATE_LOW = INSUFFICIENT_SIGNAL
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_WAIT
    初始化已知用户数据库连接
    初始化设备-用户关联表

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_WAIT

        // 第2步：检测新会话
        IF 收到会话启动信号:
            SET internal_state = STATE_IDENTIFY
            会话ID = 信号.会话ID
            清空上一轮识别缓存

            // 第3步：采集多模态信号
            登录凭证 = 获取用户登录凭证(会话ID)
            设备指纹 = 获取设备指纹数据(会话ID)
            声纹特征 = 获取声纹特征向量(会话ID)  // 可选，等待首次语音输入

            // 第4步：分级识别
            IF 登录凭证 != None AND 登录凭证.有效:
                // 最高优先级：登录凭证
                用户ID = 登录凭证.用户ID
                置信度 = 0.98
                识别方式 = "登录凭证"
                推荐槽位 = "长期槽"
                是否新用户 = False

            ELSE IF 设备指纹 != None:
                关联用户 = 设备-用户关联表.查询(设备指纹.设备ID)
                IF 关联用户 != None:
                    用户ID = 关联用户.用户ID
                    置信度 = 0.85
                    识别方式 = "设备指纹"
                    推荐槽位 = "长期槽"
                    是否新用户 = False
                ELSE:
                    // 设备未关联任何用户
                    用户ID = "TEMP_" + 生成唯一ID()
                    置信度 = 0.50
                    识别方式 = "设备指纹（新设备）"
                    推荐槽位 = "临时槽"
                    是否新用户 = True
                    设备-用户关联表.暂存(设备指纹.设备ID, 用户ID)

            ELSE IF 声纹特征 != None AND 声纹特征.质量 > 0.6:
                匹配用户 = 声纹数据库.匹配(声纹特征.特征向量)
                IF 匹配用户 != None AND 匹配用户.相似度 > 阈值:
                    用户ID = 匹配用户.用户ID
                    置信度 = 0.75
                    识别方式 = "声纹特征"
                    推荐槽位 = "长期槽"
                    是否新用户 = False
                ELSE:
                    用户ID = "TEMP_" + 生成唯一ID()
                    置信度 = 0.40
                    识别方式 = "声纹（无匹配）"
                    推荐槽位 = "临时槽"
                    是否新用户 = True

            ELSE:
                // 无任何可用信号
                SET internal_state = STATE_LOW
                用户ID = "GUEST_" + 会话ID
                置信度 = 0.0
                识别方式 = "无可用信号"
                推荐槽位 = "访客槽"
                是否新用户 = True

            // 第5步：输出识别结果
            SET internal_state = STATE_DONE
            识别结果 = 构建识别结果(用户ID, 识别方式, 置信度, 是否新用户, 推荐槽位)
            向 ag-mem-02 发送用户身份识别结果
            SET internal_state = STATE_WAIT

        // 第6步：响应外部身份查询
        IF 收到 ag-mem-02 身份查询请求:
            IF 最近一次识别结果仍有效（距上次识别 < 会话超时）:
                向 ag-mem-02 返回缓存的识别结果
            ELSE:
                向 ag-mem-02 返回错误(原因="身份已过期，请重新识别")

        SLEEP 50ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 声纹采集质量过低（噪声>阈值） | 跳过声纹识别，降级使用设备指纹 | 环境噪声降低，声纹质量恢复 |
| 设备指纹被篡改或不可靠 | 标记置信度下调至0.4，优先等待用户登录 | 用户完成登录认证 |
| 外部认证服务不可用 | 超时后降级为设备指纹识别，标记“认证服务不可用” | 认证服务恢复正常 |
| 同一设备关联多个用户 | 返回最近活跃的用户ID，提示可能存在身份混淆 | 用户主动登录确认身份 |
| 紧急熔断 | 暂停所有识别流程，保持最后识别结果 | 紧急解除后恢复 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |

| 内部调度总线 | 写 | 识别状态上报 | 周期性写入 | 向 ag-mem-02 发送 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 声纹特征向量仅用于本地匹配，原始音频数据不存储、不上传 |
| S-02 | 设备指纹数据仅用于用户关联，不共享给漏斗二或任何外部模块 |
| S-03 | 访客模式的用户ID不关联任何持久化数据，会话结束后自动失效 |
| S-04 | 身份识别结果仅向 ag-mem-02 输出，不得被其他模块直接访问 |
| S-05 | 识别置信度低于0.5时，必须明确标记为低置信度，下游模块应限制敏感操作 |

| TC-M04-04 | `WAITING_SESSION`，声纹匹配成功 | 声纹特征匹配用户U003，相似度>阈值 | 置信度0.75，用户ID=U003，推荐长期槽 |
