## 一、仓库定位

本目录包含 Agent-mlnf-mem 全部 51 个模块的接口规格文档，每个模块独立成档，严格遵循统一的十章标准模板。

Agent-mlnf-mem 采用双漏斗架构——漏斗一负责用户画像，漏斗二负责任务自成长经验。双漏斗物理隔离，严格遵循 MLNF-Mem 五层单向晋升与三维重要度驱动核心理论。漏斗外挂扩展区独立运行，不参与记忆沉淀、筛选、晋升与遗忘机制。

本套51模块MLNF架构继承EM-Core-HR人形机器人、EM-Core-AD自动驾驶同源底层设计思路，针对AI Agent端侧轻量化、低存储需求完成定制裁剪。

## 三、接口规格文档标准结构

每个模块的接口规格文档包含以下十章：

1. **基本信息**：模块编号、名称、分区、职责、依赖/被依赖模块
2. **内部状态定义**：状态机及触发条件
3. **输入数据**：数据类型、来源模块、触发条件、优先级
4. **输出数据**：数据类型、目标模块、输出条件、优先级
5. **核心处理逻辑**：伪代码描述
6. **约束与异常处理**：异常场景及恢复条件
7. **总线契约**：总线操作、数据内容、权限
8. **安全边界**：安全规则逐条列出
9. **接口校验用例**：覆盖正常/边界/异常场景
10. **质量自检清单**
