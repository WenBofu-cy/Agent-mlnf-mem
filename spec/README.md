## 一、仓库定位
本仓库为 EM-Core 通用智能系统中 MLNF-Mem 记忆中枢的 AI Agent 专项实现，承载 60 个记忆模块。

采用双漏斗统一架构——漏斗一用户画像（懂你），漏斗二 Agent 自成长经验（练己）。双漏斗底层机制完全相同（总控漏斗 F₀ 统一调度、动态子漏斗集群水平分类、五层单向晋升垂直深浅、三维重要度驱动、宏观自收敛），仅分类维度不同：漏斗一按用户领域分类，漏斗二按任务类型分类。

完整继承 HR 人形机器人 60 模块 MLNF-Mem 全部核心机制。漏斗数据采用双写加校验和，启动自动完整性校验，损坏自动从备份分区恢复。三个外挂扩展模块已全部融入 ECC 对应模块，记忆中枢回归纯粹的记忆管理。

### 三中枢标准化并发数据流时序
1. 用户输入经由 ECC-12 发起 LLM 调用，同步通过 MemoryBus 向 MLNF 01 总控漏斗下发当前会话长效记忆只读锁；
2. ECC 并行检索 MLNF 画像、历史经验，生成带依赖标记 DAG 任务树，经 CerebellumBus 下发 MCC 执行多会话隔离/同会话并行任务；
3. MCC 逐条回流各并行子任务执行观测数据，MLNF 仅临时缓存行为记录，不触发记忆晋升、归档操作；
4. 当前会话全部串行、并行子任务执行完毕，ECC 统一下发会话完成回执至 MLNF；01 总控漏斗接收后解除该会话只读锁；任意并行子任务报错、会话中途终止，锁持续持有，缓存观测数据不进入长效记忆晋升链路；
5. 系统低负载时段，ECC 批量审核会话草稿并推送合规经验至 MLNF，解锁状态下数据才可执行五层晋升、宏观自收敛归档；
6. 多会话并发访问记忆采用哈希分片隔离+会话只读锁双重隔离，单会话内全部并行任务共享同一把会话锁，互不干扰其他会话数据读写。

## 二、核心架构

### 2.1 双漏斗统一架构

| 维度 | 漏斗一：用户画像 | 漏斗二：Agent自成长 |
|------|:---:|:---:|
| 总控调度 | F₀ 统一规则约束 | F₀ 统一规则约束 |
| 水平分类 | 动态子漏斗（按用户领域） | 动态子漏斗（按任务类型） |
| 分类维度 | 工作/创作/社交/生活/学习/通用 | 文件管理/代码开发/信息检索/沟通协作/数据分析/自动化/通用 |
| 垂直深浅 | 五层单向晋升 L1→L5 | 五层单向晋升 L1→L5 |
| 宏观自收敛 | 子漏斗达 Nmax 后强制归并 | 子漏斗达 Nmax 后强制归并 |
| 闲置处理 | 超90天无新观测自动休眠 | 超30天无新经验自动回收 |
| 哈希索引 | 场景特征 → 子漏斗定位 | 场景特征 → 子漏斗定位 |
| 容量占比 | 20% | 80% |
| 并发适配 | 多会话画像检索分片隔离，并行写入带缓冲队列 | 批量并行执行观测数据分片批量入库，避免IO峰值阻塞 |

### 2.2 漏斗一：用户画像漏斗

```
漏斗一：用户画像漏斗
├── 总控漏斗 F₀ 统一调度
├── 动态子漏斗集群（按用户领域，Nmax=8）
│   ├── u₁ 工作场景·专业领域 → L1→L2→L3→L4→L5
│   ├── u₂ 创作场景·表达领域 → L1→L2→L3→L4→L5
│   ├── u₃ 社交场景·沟通领域 → L1→L2→L3→L4→L5
│   ├── u₄ 生活场景·日常领域 → L1→L2→L3→L4→L5
│   ├── u₅ 学习场景·求知领域 → L1→L2→L3→L4→L5
│   └── u₆ 通用场景·跨领域特征 → L1→L2→L3→L4→L5
├── 宏观自收敛（子漏斗达 Nmax 后归并）
└── 闲置休眠（超90天无新观测自动冻结，再次激活需用户确认）
```

漏斗一核心机制：
- 不参与时间衰减遗忘，偏好不因时间流逝而失效
- 新习惯连续3次确认后覆盖旧习惯
- 多条具体行为经宏观自收敛抽象为高阶标签
- 长期未激活的子漏斗冻结但不删除
- 并发多会话行为观测数据分片缓冲写入，防止多任务同时写入造成磁盘IO卡顿

### 2.3 漏斗二：Agent 自成长漏斗

```
漏斗二：Agent自成长漏斗
├── 总控漏斗 F₀ 统一调度
├── 动态子漏斗集群（按任务类型，Nmax=12）
│   ├── f₁ 文件管理场景 → L1→L2→L3→L4→L5
│   ├── f₂ 信息检索场景 → L1→L2→L3→L4→L5
│   ├── f₃ 代码开发场景 → L1→L2→L3→L4→L5
│   ├── f₄ 沟通协作场景 → L1→L2→L3→L4→L5
│   ├── f₅ 数据分析场景 → L1→L2→L3→L4→L5
│   ├── f₆ 跨应用自动化场景 → L1→L2→L3→L4→L5
│   └── f₇ 通用策略场景 → L1→L2→L3→L4→L5
├── 宏观自收敛（子漏斗达 Nmax 后归并）
└── 闲置回收（超30天无新经验自动回收）
```

### 2.4 五层单向记忆晋升通路

```
L1 临时层 → L2 近期层 → L3 中期层 → L4 长期层 → L5 核心层
（本次会话）  （近7日）    （近30日）   （泛化技能）  （永久锁定）
```

晋升条件：留存时长达标 + 重要度达标。仅可单向晋升，不可回退。
并发约束：会话未完成全部MCC并行/串行子任务、会话只读锁未释放时，临时层L1数据禁止向上层级晋升归档。

### 2.5 三维重要度公式

```
I = I₀ + α·S + β·V + γ·C
```

| 驱动信号 | 定义 | 取值范围 |
|----------|------|:------:|
| S | 安全显著性 | [0,1] |
| V | 用户满意度 | [0,1] |
| C | 复用频次 | [0,1] |

默认权重：α=0.5、β=0.2、γ=0.3，可配置。
并发场景适配：同会话多条并行子任务分别独立计算S/V/C指标，会话完成后聚合生成整体I值用于晋升判定。

### 2.6 宏观自收敛机制

| 阶段 | 说明 |
|:---:|------|
| 初期扩张 | 新场景触发子漏斗持续新建 |
| 约束触发 | 子漏斗达 Nmax，停止新建 |
| 归并收敛 | 新经验强制匹配已有子漏斗，相似漏斗合并 |
| 全局统合 | F₀ 统一规则约束，行为风格趋于稳定 |
| 稳态达成 | 从多分支分散自然回归整体行为一致 |
并发适配：低负载后台批量执行多会话收敛合并计算，高负载同步接收ECC降级信号，降低收敛并行算力占用。

### 2.7 核心特性

| 特性 | 说明 |
|------|------|
| 双漏斗统一架构 | 相同底层机制，不同分类维度 |
| 宏观自收敛 | 子漏斗达上限后自动合并复用，行为风格趋于稳定 |
| 轻量化终身记忆 | PC 端 1GB，手机端 200MB，核心技能常驻离线可用 |
| 经验驱动人格 | 无预设人格，通过记忆沉淀自主学习 |
| 用户画像不过期 | 覆盖式更新替代时间衰减遗忘 |
| 记忆可管控 | 每条记忆带唯一ID与时间戳，支持查看、编辑、删除、修正，L4/L5层可导出/导入文件 |
| 零成本复制迁移 | 记忆包支持文件导出/分发，一台Agent学会一键复制到另一台Agent |
| 数据崩溃恢复 | 双写+校验和写入，启动自动完整性校验，损坏从备份分区恢复 |
| 多会话并发隔离 | 会话只读锁+哈希分片缓存双重隔离，并行任务数据互不污染，未完结会话数据禁止长效归档 |

| 50 | Resource_Balance_Unit | 冷热记忆负载均衡；接收ECC并发降级指令，降低后台批量任务并行度 |

### 分区九：哈希索引·检索·调度·加速（51–56）

| 编号 | 模块名称 | 核心职责 |
|:---:|----------|----------|
| 51 | Mem_Lookup_Cache_Unit | 内存级查找缓存，启动时建立漏斗名→漏斗对象哈希映射，热门经验条目 LRU 缓存；多会话并发检索分片隔离缓存，互不抢占内存 |
| 52 | Mem_Retrieve_Unit | 条件化记忆检索查询，含模糊指令语义匹配；前台多会话并行检索高优先级调度 |
| 53 | Hot_Cache_Unit | 高频记忆热点缓存加速；单会话多条并行任务共享同一会话热点缓存 |
| 54 | Mem_Relation_Index | 跨漏斗记忆关联索引；多会话关联查询分片隔离 |
| 55 | Mem_StateMachine_Unit | 记忆全生命周期状态机；标记会话锁持有状态，管控并发读写权限 |
| 56 | Batch_Task_Scheduler | 记忆后台批处理调度，含疑问缓存暂存子功能；区分前台实时查询、后台晋升/收敛/清理，同步ECC降级信号调低后台并发数量，支撑多会话分片并行批处理 |

### 分区十：跨设备加密同步单元（57–58）

| 编号 | 模块名称 | 核心职责 |
|:---:|----------|----------|

| 58 | Profile_Sync_Unit | 各领域画像摘要与 L4/L5 泛化技能的加密同步，冲突检测与合并，同步内容仅含脱敏数据；批量多会话同步分片传输 |

### 分区十一：ID索引与变更日志 + 导入导出及完整性校验（59–60）

|:---:|----------|----------|
| 59 | Mem_ID_Log_Unit | 全局记忆唯一ID索引管理，全链路变更日志写入，不可变审计记录；每条并发会话、并行子任务操作单独标记session_id留痕审计 |
| 60 | Mem_Import_Export_Unit | L4/L5经验导入导出执行（权限校验由ECC-05完成），脱敏处理；双写+校验和写入，启动自动完整性校验，损坏从备份分区恢复；支持完整记忆包一键导出/导入，导出文件为 AES-256-GCM 加密压缩包，一台Agent学会的技能可一键复制到另一台Agent，零成本迁移；多会话并发导出/导入加互斥锁防止文件损坏 |

| 规则编号 | 内容 |
|:---:|------|
| 1 | 记忆系统内部 01–60 模块编号连续完整、永久固定，版本内禁止新增、删减、重排 |

| 3 | 双漏斗物理存储分区隔离，编译期强制实施 |
| 4 | 漏斗一不参与时间衰减遗忘，仅执行覆盖式更新、抽象提炼、闲置休眠 |

| 6 | 跨系统通信统一走 MemoryBus 全局记忆总线中转交互 |
| 7 | 记忆访问权限判定归 ECC-05 伦理仲裁，记忆操作路由归 ECC-12 资源调度 |

| 9 | 漏斗数据写入采用双写+校验和机制，启动自动完整性校验，损坏自动从备份分区恢复 |
| 10 | L4/L5 记忆支持加密导出/导入，完整记忆包一键迁移，零成本跨设备复制 |
| 11 | 多会话并发访问记忆时采用分片哈希隔离+会话只读锁双重隔离，单会话并行任务共享同一会话锁；当前会话未完成全部MCC并行/串行子任务、锁未释放前，禁止长效记忆写入与层级晋升操作 |
