# ag-mem-07-用户行为观测记录单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-07 |
| 模块名称 | 用户行为观测记录单元 |
| 所属分区 | 二、漏斗一：用户画像漏斗 |
| 核心职责 | 持续观测并结构化记录用户在交互过程中的操作行为、偏好选择与反馈信号。将原始交互事件转化为标准化的行为观测条目，包含行为类型、上下文场景、时间戳与关联会话ID。通过 ag-mem-06 隔离管控校验后写入当前活跃画像槽。为偏好判定（ag-mem-09）和偏好统计（ag-mem-10）提供原始数据来源。不参与任何行为判定或认知决策，仅负责行为数据的采集与结构化存储 |
| 依赖模块 | ag-mem-02（漏斗一专属调度单元，提供当前活跃槽位号）、ag-mem-06（画像槽数据隔离管控单元，获取写入令牌）、ag-mem-08（上下文场景标记单元，为行为条目打场景标签） |
| 被依赖模块 | ag-mem-09（偏好判定标签单元，消费行为观测条目）、ag-mem-10（偏好累积统计单元，消费行为观测条目） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 等待会话 | `WAITING_SESSION` | 无活跃会话，等待用户交互 | 系统初始化完成，无活跃槽位绑定 |
| 正常记录 | `NORMAL_RECORDING` | 活跃会话中，正常记录用户行为 | 活跃槽位已绑定，用户产生交互行为 |
| 批量暂存 | `BATCH_BUFFERING` | 行为条目暂存于内存缓冲区，等待批量写入 | 单次会话中行为频率过高，达到缓冲区阈值 |
| 写入暂停 | `WRITE_PAUSED` | 画像槽写入权限被暂停，暂存行为但无法落盘 | ag-mem-06 令牌过期或写入被拒 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 用户交互事件 | Struct（会话ID + 事件类型 + 事件参数 + 时间戳） | 系统交互层（对话界面、工具调用接口、用户反馈控件） | 用户每次点击、输入、选择、确认、取消等操作时 | **高** |
| 当前活跃槽位绑定信息 | Struct（会话ID + 活跃槽位编号 + 用户ID） | ag-mem-02 漏斗一专属调度单元 | 槽位激活或切换时 | **高** |
| 上下文场景标签 | Struct（场景类别 + 任务类型 + 交互阶段 + 时间窗口） | ag-mem-08 上下文场景标记单元 | 每次行为观测记录时查询 | 普通 |
| 写入许可令牌 | Struct（令牌ID + 授权槽位编号 + 有效期） | ag-mem-06 画像槽数据隔离管控单元 | 校验通过后签发 | **高** |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-mem-01 总控漏斗F0 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 结构化行为观测条目 | Struct（条目ID + 会话ID + 用户ID + 槽位编号 + 行为类型 + 行为参数 + 场景标签 + 时间戳） | ag-mem-09 偏好判定标签单元、ag-mem-10 偏好累积统计单元 | 行为条目写入画像槽后 | **高** |
| 批量写入请求 | Struct（目标槽位编号 + 条目列表 + 写入令牌） | ag-mem-06 画像槽数据隔离管控单元 | 缓冲区达到阈值或会话结束时 | **高** |
| 行为记录状态上报 | Struct（当前状态 + 已记录条目数 + 缓冲区使用率 + 写入成功率） | ag-mem-02 漏斗一专属调度单元 | 周期性（每10秒）或状态变更时 | 普通 |


## 用户交互行为分类

### 一、行为类型定义

| 行为类型 | 说明 | 记录参数示例 |
|----------|------|------|
| `TEXT_INPUT` | 用户文本输入 | 输入长度、语言、输入速度 |
| `VOICE_INPUT` | 用户语音输入 | 语音时长、语速、音量 |
| `BUTTON_CLICK` | 按钮点击 | 按钮ID、按钮类型、点击耗时 |
| `MENU_SELECT` | 菜单选择 | 菜单项ID、层级路径 |
| `TOOL_INVOKE` | 工具调用请求 | 工具名称、参数复杂度 |
| `RESULT_VIEW` | 结果查看 | 查看时长、滚动深度 |
| `RESULT_COPY` | 结果复制 | 复制内容长度 |
| `RESULT_SHARE` | 结果分享 | 分享目标平台 |
| `FEEDBACK_LIKE` | 点赞/正向反馈 | 反馈对象（消息ID/工具ID） |
| `FEEDBACK_DISLIKE` | 踩/负向反馈 | 反馈对象、是否附带原因 |
| `FEEDBACK_SKIP` | 跳过/忽略 | 跳过的内容类型 |
| `SESSION_START` | 会话开始 | 接入方式、设备类型 |
| `SESSION_END` | 会话结束 | 会话时长、结束原因 |
| `ERROR_ENCOUNTER` | 遇到错误 | 错误类型、错误码 |
| `RETRY_ACTION` | 重试操作 | 重试次数、原始操作类型 |

## 核心处理逻辑

```
FUNCTION behavior_observation_main_loop():
    STATE_WAIT = WAITING_SESSION
    STATE_NORMAL = NORMAL_RECORDING
    STATE_BUFFER = BATCH_BUFFERING
    STATE_PAUSED_WRITE = WRITE_PAUSED
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_WAIT
    初始化行为条目缓冲区 = []
    设置缓冲区阈值 = 20 条
    设置批量写入间隔 = 5 秒
    活跃槽位编号 = None
    当前会话ID = None

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            IF 缓冲区非空:
                尝试强制写入缓冲区内容
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_WAIT

        // 第2步：更新活跃槽位绑定
        IF 收到 ag-mem-02 活跃槽位绑定信息:
            活跃槽位编号 = 绑定信息.活跃槽位编号
            当前用户ID = 绑定信息.用户ID
            当前会话ID = 绑定信息.会话ID
            IF internal_state == STATE_WAIT:
                SET internal_state = STATE_NORMAL
            CONTINUE

        // 第3步：接收用户交互事件
        IF 收到用户交互事件:
            IF internal_state IN [STATE_WAIT, STATE_PAUSED]:
                // 无活跃槽位，无法记录，丢弃事件
                CONTINUE

            // 3a. 构建行为观测条目
            条目 = 构建观测条目(
                会话ID=当前会话ID,
                用户ID=当前用户ID,
                槽位编号=活跃槽位编号,
                行为类型=事件.事件类型,
                行为参数=事件.事件参数,
                时间戳=NOW()
            )

            // 3b. 查询上下文场景标签
            场景标签 = 向 ag-mem-08 查询当前上下文场景()
            IF 场景标签 != None:
                条目.场景标签 = 场景标签

            // 3c. 尝试关联前置行为
            IF 事件.事件类型 IN [RESULT_VIEW, FEEDBACK_LIKE, FEEDBACK_DISLIKE, RETRY_ACTION]:
                最近工具调用 = 查找缓冲区中最近一次 TOOL_INVOKE 类型的条目
                IF 最近工具调用 != None:
                    条目.关联条目ID = 最近工具调用.条目ID

            // 3d. 加入缓冲区
            缓冲区.append(条目)

            // 3e. 检查是否需要批量写入
            IF 缓冲区.size >= 缓冲区阈值:
                SET internal_state = STATE_BUFFER
                批量写入结果 = 执行批量写入(缓冲区, 活跃槽位编号)
                IF 批量写入结果.成功:
                    清空缓冲区
                    向 ag-mem-09 推送新写入的行为条目列表
                    SET internal_state = STATE_NORMAL
                ELSE:
                    SET internal_state = STATE_PAUSED_WRITE
                    保留缓冲区内容，等待恢复

        // 第4步：定时批量写入
        IF 距上次批量写入 >= 5秒 AND 缓冲区非空 AND internal_state == STATE_NORMAL:
            SET internal_state = STATE_BUFFER
            批量写入结果 = 执行批量写入(缓冲区, 活跃槽位编号)
            IF 批量写入结果.成功:
                清空缓冲区
                向 ag-mem-09 推送新写入的行为条目列表
                SET internal_state = STATE_NORMAL
            ELSE:
                SET internal_state = STATE_PAUSED_WRITE

        // 第5步：写入暂停恢复检测
        IF internal_state == STATE_PAUSED_WRITE:
            // 尝试重新获取写入令牌并重试
            重试结果 = 执行批量写入(缓冲区, 活跃槽位编号)
            IF 重试结果.成功:
                清空缓冲区
                SET internal_state = STATE_NORMAL

        // 第6步：会话结束强制写入
        IF 收到会话结束信号 AND 缓冲区非空:
            执行批量写入(缓冲区, 活跃槽位编号)
            清空缓冲区
            SET internal_state = STATE_WAIT

        // 第7步：周期性状态上报
        IF 距上次状态上报 >= 10秒:
            向 ag-mem-02 上报行为记录状态(当前状态, 已记录条目总数, 缓冲区使用率, 写入成功率)

        SLEEP 20ms


FUNCTION 执行批量写入(缓冲区, 目标槽位编号):
    // 获取写入令牌
    令牌 = 向 ag-mem-06 申请写入令牌(目标槽位编号)
    IF 令牌 == None:
        RETURN {成功: False, 原因: "无法获取写入令牌"}

    // 批量写入画像槽
    FOR EACH 条目 IN 缓冲区:
        写入结果 = 画像槽存储.追加写入(目标槽位编号, 条目)
        IF 写入结果.失败:
            RETURN {成功: False, 原因: "存储写入失败"}

    RETURN {成功: True, 写入数量: 缓冲区.size}
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 无活跃槽位时收到交互事件 | 丢弃事件，记录丢弃计数 | 槽位激活后正常记录 |
| 写入令牌过期 | 缓冲区保留数据，定时重试获取新令牌 | 令牌刷新成功 |
| 缓冲区溢出（超过最大容量200条） | 丢弃最旧的非关键条目（优先保留FEEDBACK类和ERROR类） | 缓冲区写入成功 |
| 会话异常中断（无结束信号） | 超时检测（30分钟无新事件）后强制写入缓冲区 | 超时触发或新会话开始 |
| 紧急熔断 | 强制写入缓冲区后暂停服务 | 紧急解除后恢复 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| 系统交互层API | 读 | 用户交互事件 | 只读 | 各交互控件推送 |
| 内部调度总线 | 读 | 当前活跃槽位绑定信息 | 只读 | ag-mem-02 发送 |
| 内部调度总线 | 读 | 上下文场景标签 | 只读 | ag-mem-08 返回 |
| 内部调度总线 | 读 | 写入许可令牌 | 只读 | ag-mem-06 签发 |
| 内部调度总线 | 写 | 结构化行为观测条目 | 广播 | 向 ag-mem-09、ag-mem-10 推送 |
| 内部调度总线 | 写 | 批量写入请求 | 专属写入 | 向 ag-mem-06 发送 |
| 内部调度总线 | 写 | 行为记录状态上报 | 周期性写入 | 向 ag-mem-02 发送 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 所有行为观测条目在写入画像槽前必须通过 ag-mem-06 隔离管控校验，禁止绕过隔离机制直接写入 |
| S-02 | 行为条目中不得包含用户的原始输入内容（如完整文本、语音录音），仅保留行为元数据（类型、时长、参数摘要） |
| S-03 | 缓冲区中的数据在未写入画像槽前仅存于内存，进程重启后自动丢弃（非持久化），会话中断不产生残留数据 |
| S-04 | 写入失败时缓冲区数据保留重试，重试超过3次仍未成功则丢弃并上报告警 |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-M07-01 | `NORMAL_RECORDING`，用户输入文本 | 交互事件（类型=TEXT_INPUT，参数=长度50） | 构建观测条目，加入缓冲区 |
| TC-M07-02 | `NORMAL_RECORDING`，缓冲区达到20条 | 第20个交互事件 | 触发批量写入，清空缓冲区，推送条目至ag-mem-09 |
| TC-M07-03 | `NORMAL_RECORDING`，用户点赞 | 交互事件（类型=FEEDBACK_LIKE） | 关联最近的TOOL_INVOKE条目，写入缓冲区 |
| TC-M07-04 | `WAITING_SESSION`，收到交互事件 | 交互事件 | 丢弃事件，记录丢弃计数 |
| TC-M07-05 | `WRITE_PAUSED`，写入令牌过期 | 定时重试触发 | 重新申请令牌，若成功则写入缓冲区 |
| TC-M07-06 | `NORMAL_RECORDING`，会话结束 | 会话结束信号 + 缓冲区有5条未写入 | 强制写入缓冲区，状态切换至WAITING_SESSION |
