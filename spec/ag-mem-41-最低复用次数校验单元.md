# ag-mem-41-最低复用次数校验单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-41 |
| 模块名称 | 最低复用次数校验单元 |
| 所属分区 | 三、漏斗二：任务经验漏斗 / 晋升与遗忘执行机制 |
| 核心职责 | 接收 ag-mem-40（遗忘阈值判定单元）在判定过程中对接近遗忘阈值边界的条目发起的复用次数保护校验请求。校验该条目在同类任务场景下的历史复用次数是否满足最低保护条件：若复用次数高于最低保护阈值，即使I值低于遗忘阈值也暂时保留（给予一次保护周期）；若复用次数不足且I值确实偏低，则确认遗忘判定有效。通过引入复用次数保护机制，避免那些I值因自然衰减而暂时低于阈值但仍具有较高实用价值的经验被过早误删。不参与遗忘判定决策，仅提供复用次数的客观校验与保护建议 |
| 依赖模块 | ag-mem-33（C值统计单元，提供条目的历史复用次数数据）、ag-mem-40（遗忘阈值判定单元，发起保护校验请求）、ag-mem-35（三维权重系数配置单元，获取各分槽的最低保护次数阈值） |
| 被依赖模块 | ag-mem-40（返回校验回执与保护建议） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 无校验请求，等待输入 | 系统初始化完成，无待处理请求 |
| 查询复用数据 | `QUERYING_USAGE` | 正在向 ag-mem-33 查询条目的历史复用次数 | 收到 ag-mem-40 的保护校验请求 |
| 结果输出 | `OUTPUTTING` | 校验完成，正在返回保护建议 | 复用数据查询完成 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |

## 复用次数保护规则

### 一、最低保护次数阈值

| 层级 | 最低保护次数 | 说明 |
|:---:|:---:|------|
| L1（临时层） | **1次** | L1条目留存时间短，复用1次即可获得保护 |
| L2（近期层） | **3次** | L2条目需要一定的复用验证 |
| L3（中期层） | **5次** | L3条目需要较充分的复用验证 |
| L4（长期层） | **8次** | L4条目需要充分的复用验证才能获得遗忘保护 |

### 二、各分槽最低保护次数调整

| 分槽编号 | 调整系数 | 调整后L1 | 调整后L2 | 调整后L3 | 调整后L4 |
|----------|:---:|:---:|:---:|:---:|:---:|
| ag-mem-15（对话交互） | ×1.0 | 1 | 3 | 5 | 8 |
| ag-mem-16（工具调用） | ×1.2 | 2 | 4 | 6 | 10 |
| ag-mem-17（信息检索） | ×1.0 | 1 | 3 | 5 | 8 |
| ag-mem-18（创作生成） | ×1.0 | 1 | 3 | 5 | 8 |
| ag-mem-19（通用任务） | ×0.8 | 1 | 2 | 4 | 6 |

### 三、保护判定规则

```
受保护条件（满足任一即可）:
1. 当前复用次数 ≥ 最低保护次数 → 受保护，建议保留
2. 近30日内复用次数 ≥ 1 且 当前I值 ≥ 遗忘阈值 × 0.7 → 受保护，给予30天保护期
3. 最近复用时间在7日内 → 受保护，给予7天保护期

不受保护条件:
- 当前复用次数 < 最低保护次数 且 不满足上述条件2或条件3 → 不受保护，确认遗忘判定
```


## 核心处理逻辑

```
FUNCTION min_usage_validator_main_loop():
    STATE_IDLE = IDLE
    STATE_QUERY = QUERYING_USAGE
    STATE_OUTPUT = OUTPUTTING
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_IDLE
    加载各分槽最低复用次数保护阈值配置（从 ag-mem-35 获取）
    初始化校验统计计数器

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_IDLE

        // 第2步：接收复用次数保护校验请求
        IF 收到 ag-mem-40 复用次数保护校验请求:
            SET internal_state = STATE_QUERY
            条目ID = 请求.条目ID
            当前I值 = 请求.当前I值
            当前层级 = 请求.当前层级
            来源分槽 = 请求.来源分槽编号
            遗忘阈值 = 请求.遗忘阈值

            // 2a. 查询该条目的历史复用次数
            向 ag-mem-33 发送条目复用次数查询请求(条目ID)
            // 等待查询回执

        // 第3步：处理复用次数查询回执
        IF 收到 ag-mem-33 条目复用次数查询回执:
            SET internal_state = STATE_OUTPUT
            总复用次数 = 回执.总成功复用次数
            近30日复用次数 = 回执.近30日复用次数
            最近复用时间 = 回执.最近复用时间

            // 3a. 获取该分槽与层级的最低保护次数
            分槽调整系数 = 获取分槽保护次数调整系数(来源分槽)
            基础保护次数 = 获取层级基础保护次数(当前层级)
            最低保护次数 = ROUND(基础保护次数 × 分槽调整系数)

            // 3b. 执行保护判定
            受保护 = False
            保护原因 = ""
            保护有效期 = 0

            // 条件1: 复用次数 ≥ 最低保护次数
            IF 总复用次数 >= 最低保护次数:
                受保护 = True
                保护原因 = "复用次数满足最低保护条件"
                保护有效期 = 30天

            // 条件2: 近30日有复用且I值不低于遗忘阈值×0.7
            ELSE IF 近30日复用次数 >= 1 AND 当前I值 >= 遗忘阈值 × 0.7:
                受保护 = True
                保护原因 = "近30日有复用且I值不低于遗忘阈值的70%"
                保护有效期 = 30天

            // 条件3: 最近7日内被复用
            ELSE IF (NOW() - 最近复用时间) <= 7天:
                受保护 = True
                保护原因 = "最近7日内被复用"
                保护有效期 = 7天

            // 3c. 构建校验回执
            IF 受保护:
                保护建议 = "建议暂时保留，给予" + 保护有效期 + "天保护期"
            ELSE:
                保护建议 = "复用次数不足，建议确认遗忘判定"

            校验回执 = {
                条目ID: 条目ID,
                是否受保护: 受保护,
                当前复用次数: 总复用次数,
                最低保护阈值: 最低保护次数,
                保护建议: 保护建议,
                保护有效期: 保护有效期
            }

            // 3d. 发送回执
            向 ag-mem-40 返回复用次数校验回执(校验回执)

            // 3e. 更新统计
            更新校验统计(受保护, 总复用次数)
            SET internal_state = STATE_IDLE

        // 第4步：周期性状态上报
        IF 距上次状态上报 >= 180秒:
            向 ag-mem-03 上报校验状态(
                当前状态=internal_state,
                最近校验次数=校验统计.总次数,
                受保护比例=校验统计.受保护比例,
                校验耗时均值=校验统计.耗时均值
            )
            重置统计计数器

        SLEEP 10ms


FUNCTION 获取层级基础保护次数(层级):
    基础表 = {
        "L1": 1,
        "L2": 3,
        "L3": 5,
        "L4": 8
    }
    RETURN 基础表.GET(层级, 3)


FUNCTION 获取分槽保护次数调整系数(分槽编号):
    系数表 = {
        "ag-mem-15": 1.0,
        "ag-mem-16": 1.2,
        "ag-mem-17": 1.0,
        "ag-mem-18": 1.0,
        "ag-mem-19": 0.8
    }
    RETURN 系数表.GET(分槽编号, 1.0)
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| ag-mem-33 查询复用次数超时或返回空 | 保守处理：视为不受保护，确认遗忘判定 | 查询恢复正常 |
| 条目在C值存储表中不存在（新条目） | 复用次数=0，不受保护 | 条目积累复用次数 |
| 分槽编号在配置表中不存在 | 使用默认系数1.0，基础保护次数按层级取默认值 | 分槽配置补全 |
| 同一短时间内重复请求校验同一条目 | 使用上次校验结果缓存（60秒内有效），避免频繁查询 | — |
| 紧急熔断 | 暂停校验，未完成的请求丢弃 | 紧急解除后恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| U-01 | 复用次数校验仅读取条目的复用频次统计数据，不得修改任何经验数据 |
| U-02 | 保护有效期到期后，该条目在下次遗忘判定时需重新校验，不得自动续期 |
| U-03 | 工具调用槽（ag-mem-16）的最低保护次数上调20%，确保高安全场景的经验更不易被误删 |
| U-04 | 校验结果缓存仅用于避免短时间内重复查询，有效期60秒，超时自动失效 |
