# ag-mem-41-最低复用次数校验单元 接口规格
## 基本信息
| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-41 |
| 模块名称 | 最低复用次数校验单元 |
| 所属分区 | 三、漏斗二：任务经验漏斗 / 晋升与遗忘执行机制 |
| 核心职责 | 作为遗忘判定辅助校验模块，接收ag-mem-40下发的边界条目，读取条目工具/任务复用统计数据，基于全局/分槽最低复用保护阈值判定条目是否满足复用保护条件；向ag-mem-40返回校验回执；接收ag-mem-40推送的受保护条目通知，留存复用保护记录；不参与遗忘判定、不删除/修改经验条目，仅提供复用维度的辅助校验能力，为V1.1长效记忆提供复用频次保护机制。 |
| 依赖模块 | ag-mem-20~26（分层存储单元，读取条目复用次数元数据）、ag-mem-35（三维权重系数配置单元，获取分槽专属最低复用阈值）、ag-mem-01（总控漏斗F0，全局熔断调度） |
| 被依赖模块 | ag-mem-40（遗忘阈值判定单元，接收复用校验回执、推送保护条目通知）、ag-mem-03（漏斗二调度单元，周期上报复用校验统计）、ag-mem-51（记忆变更日志追溯单元，记录复用保护校验事件） |

## 内部状态定义
| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 无校验任务，等待ag-mem-40下发校验请求 | 系统初始化完成，无待处理请求 |
| 数据读取 | `META_FETCH` | 向分层存储拉取条目复用次数元数据 | 收到复用校验请求 |
| 阈值比对 | `THRESHOLD_JUDGE` | 读取分槽复用阈值，完成保护判定 | 条目复用数据拉取完成 |
| 回执输出 | `RESPONSE_OUT` | 生成校验回执推送至ag-mem-40 | 复用次数比对完成 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断，停止所有校验任务 | 收到全局熔断指令 |

## 复用校验核心规则
### 一、分槽复用阈值标准
由ag-mem-35统一下发，无专属配置分槽自动使用全局通用阈值：
| 层级 | 全局默认最低复用保护次数 | 说明 |
|:---:|:---:|------|
| L1 | 2 | 短期临时记忆，复用门槛最低 |
| L2 | 3 | 近期经验基础保护线 |
| L3 | 5 | 中期长效经验，复用门槛提高 |
| L4 | 8 | 长期核心经验，复用保护门槛最高 |

### 二、复用保护判定逻辑
1. 总复用次数 = 任务复用次数 + 工具序列复用次数
2. 判定公式：
`if 总复用次数 ≥ 分槽对应层级最低复用阈值 → 条目受复用保护`
3. 保护联动规则（对接ag-mem-40）：
条目满足复用保护，且I值 > 层级遗忘阈值×0.8，直接跳过遗忘清理；
仅复用达标但I值过低，仍会进入遗忘候选清单。

### 三、特殊豁免规则
1. L5条目无需进入本模块校验，由ag-mem-40直接拦截遗忘；
2. 人工锁定、用户收藏条目无需复用校验，直接判定受保护；
3. 24小时内有访问记录条目，跳过复用校验。

## 核心处理逻辑
```
FUNCTION reuse_protect_judge_main_loop():
    STATE_IDLE = IDLE
    STATE_FETCH_META = META_FETCH
    STATE_THRESHOLD_JUDGE = THRESHOLD_JUDGE
    STATE_RSP_OUT = RESPONSE_OUT
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_IDLE
    加载全分槽复用阈值配置（从ag-mem-35同步）
    初始化校验统计指标
    初始化保护记录缓存 = {}

    WHILE 系统运行中:
        // 1. 全局熔断最高优先级处理
        IF 收到紧急熔断指令:
            SET internal_state = SYSTEM_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == SYSTEM_PAUSED:
            SET internal_state = IDLE

        // 2. 接收ag-mem-40下发复用校验请求
        IF 收到复用保护校验请求:
            校验条目ID列表 = 请求.条目ID列表
            来源分槽 = 请求.来源分槽编号
            目标层级 = 请求.目标层级
            SET internal_state = META_FETCH

            // 拉取条目复用元数据
            复用元数据列表 = 批量查询ag-mem-20~26获取条目复用次数
            IF 复用元数据列表为空:
                向ag-mem-40返回空校验回执
                SET internal_state = IDLE
                CONTINUE

            // 获取当前分槽对应层级最低复用阈值
            分槽阈值配置 = 全分槽阈值配置表.get(来源分槽, 全局通用阈值)
            CASE 目标层级 OF:
                "L1": min_reuse = 分槽阈值配置.L1_min_reuse
                "L2": min_reuse = 分槽阈值配置.L2_min_reuse
                "L3": min_reuse = 分槽阈值配置.L3_min_reuse
                "L4": min_reuse = 分槽阈值配置.L4_min_reuse

            SET internal_state = THRESHOLD_JUDGE
            回执结果列表 = []
            受保护条目计数 = 0

            FOR EACH 条目元数据 IN 复用元数据列表:
                条目ID = 条目元数据.条目ID
                总复用 = 条目元数据.任务复用次数 + 条目元数据.工具序列复用次数
                受保护标记 = False
                保护描述 = ""

                IF 总复用 >= min_reuse:
                    受保护标记 = True
                    保护描述 = f"总复用次数{总复用}达到层级最低保护阈值{min_reuse}"
                    受保护条目计数++
                    // 存入本地保护缓存
                    保护记录缓存[条目ID] = {
                        "reuse_total":总复用,
                        "threshold":min_reuse,
                        "record_ts":NOW()
                    }

                // 组装单条校验结果
                回执结果列表.append({
                    "item_id":条目ID,
                    "is_protected":受保护标记,
                    "protect_reason":保护描述,
                    "total_reuse":总复用,
                    "slot_min_threshold":min_reuse
                })

            // 输出校验回执至ag-mem-40
            SET internal_state = RESPONSE_OUT
            向ag-mem-40发送最低复用次数保护回执(回执结果列表)

            // 存在受保护条目则输出审计日志至ag-mem-51
            IF 受保护条目计数 > 0:
                日志摘要 = f"本次校验共{受保护条目计数}条条目满足复用保护条件"
                向ag-mem-51推送复用保护事件日志(日志摘要, 来源分槽, NOW())

            // 更新统计指标
            更新校验统计(总校验数=复用元数据列表.size, 保护数=受保护条目计数)
            SET internal_state = IDLE

        // 3. 接收ag-mem-40推送的遗忘保护条目通知，留存记录
        IF 收到遗忘保护条目通知:
            条目ID = 通知.条目ID
            保护记录缓存[条目ID] = {
                "protect_source":"遗忘复用校验",
                "item_I":通知.I值,
                "layer_thresh":通知.遗忘阈值,
                "record_ts":NOW()
            }

        // 4. 每180秒周期上报统计状态
        IF 距上次上报 >= 180秒:
            状态上报结构体 = 组装复用校验统计上报(内部状态、累计校验总数、累计保护条目数、缓存保护记录数量)
            向ag-mem-03发送复用校验周期状态上报
            重置上报计时器

        SLEEP 10ms
```

## 约束与异常处理
| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 批量校验条目无复用元数据 | 返回空回执，不产生告警日志 | 分层存储同步条目复用统计数据 |
| 当前分槽无配置复用阈值 | 自动加载全局默认层级复用阈值兜底 | ag-mem-35补充分槽专属阈值 |
| 条目复用次数为0 | 直接判定不满足复用保护，回执标记is_protected=false | 条目后续产生工具/任务复用记录 |
| 校验过程中条目被并发删除 | 基于快照元数据完成校验，不中断流程 | 无，快照隔离并发变更 |
| 全局紧急熔断 | 终止当前校验任务，清空临时缓存，拒绝新请求 | 下发RESUME恢复指令 |

## 安全边界
| 规则编号 | 内容 |
|:---:|------|
| R-01 | 本模块仅读取条目复用统计元数据，禁止修改、新增、删除任何经验条目数据 |
| R-02 | L5层级条目不进入复用校验流程，上游ag-mem-40直接拦截 |
| R-03 | 复用保护判定阈值统一由ag-mem-35管控，本模块不可自定义修改阈值 |
| R-04 | 所有复用校验、保护记录操作必须留存审计日志推送至ag-mem-51，不可丢弃 |
| R-05 | 仅响应ag-mem-40发起的校验请求，拒绝其他模块直接调用复用校验接口 |
