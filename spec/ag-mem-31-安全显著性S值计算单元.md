# ag-mem-31-安全显著性S值计算单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-31 |
| 模块名称 | 安全显著性S值计算单元 |
| 所属分区 | 三、漏斗二：任务经验漏斗 / 三维重要度计算引擎 |
| 核心职责 | 从任务经验条目中提取与安全相关的物理信号与事件特征，计算安全显著性分值S（0.0–1.0）。S值反映该任务经验对系统安全的重要性：涉及敏感操作、权限提升、高错误代价、高风险工具调用等事件将获得较高S值；常规安全操作获得较低S值。当S值 ≥ 0.9 时，触发S值直达L5写入机制（由ag-mem-16工具调用槽执行）。S值作为三维重要度I值的关键组成部分，直接影响经验条目的留存优先级与晋升速度。不参与认知决策，仅执行S值的客观量化计算 |
| 依赖模块 | ag-mem-15~19（各场景分槽，在写入新经验时附带原始任务数据供S值计算）、ag-mem-34（基础重要度I₀赋值单元，可参考S值辅助确定I₀） |
| 被依赖模块 | ag-mem-36（综合重要度I值聚合计算单元，消费S值）、ag-mem-16（工具调用槽，接收S≥0.9的L5直达触发信号）、ag-mem-35（三维权重系数配置单元，提供S值权重系数α） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 无计算请求，等待输入 | 系统初始化完成，无待处理经验 |
| 计算进行中 | `CALCULATING` | 正在基于经验数据计算S值 | 收到各场景分槽的S值计算请求 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| S值计算请求 | Struct（条目ID + 任务类型 + 工具调用列表 + 操作参数 + 结果标签 + 错误码 + 执行上下文） | ag-mem-15~19 各场景分槽（通过内部调度总线） | 每次新经验写入L1前 | **高** |
| S值计算规则配置 | Struct（各类安全信号的权重系数 + 阈值表 + 敏感操作清单） | ag-mem-35 三维权重系数配置单元 | 系统初始化加载，配置更新时 | 普通 |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-mem-01 总控漏斗F0 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| S值计算结果 | Struct（条目ID + S值 + 触发的主要安全信号列表 + L5直达资格标记） | 请求来源分槽（ag-mem-15~19）、ag-mem-36（重要度聚合） | 计算完成时 | **高** |
| L5直达触发信号 | Struct（条目ID + S值 + 经验数据摘要 + 直达原因） | ag-mem-16 工具调用槽（当S≥0.9且来源为工具调用槽时） | S值 ≥ 0.9 且任务类型为工具调用 | **高** |
| 计算状态上报 | Struct（当前状态 + 最近S值分布 + 高S值事件计数） | ag-mem-03 漏斗二专属调度单元 | 周期性（每60秒）或状态变更时 | 普通 |


## S值计算规则

### 一、安全信号分类与权重

| 信号类别 | 信号示例 | 基准分值 | 权重系数 |
|----------|------|:---:|:---:|
| 敏感操作 | 删除文件、修改系统配置、权限提升、访问敏感数据 | 0.80 | 0.40 |
| 高风险工具 | 执行shell命令、修改数据库、调用外部支付API | 0.70 | 0.30 |
| 错误与异常 | 任务执行失败、工具返回错误码、超时中断 | 0.60 | 0.15 |
| 用户隐私相关 | 访问通讯录、读取位置、获取浏览历史 | 0.90 | 0.10 |
| 结果验证 | 结果校验未通过、偏差超阈值 | 0.50 | 0.05 |

### 二、S值计算公式

```
S值 = Σ (信号类别基准分 × 是否触发该信号 × 信号强度归一化) × 权重系数

S值 = MIN(计算值, 1.0)   // 上限截断

其中:
- 是否触发该信号: 0 或 1（或根据严重程度取0.0–1.0之间的值）
- 信号强度归一化: 对于分级信号（如错误严重等级），将原始等级映射到[0,1]区间
```

### 三、快速判定规则（无需完整计算）

| 条件 | S值直接赋值 | 说明 |
|------|:---:|------|
| 任务结果标签 = "严重失败" 且涉及敏感操作 | 0.95 | 最高安全级别 |
| 任务结果标签 = "成功" 且无任何安全信号触发 | 0.05 | 最低安全级别（常规安全操作） |
| 任务涉及用户隐私数据访问（无论成功与否） | ≥ 0.70 | 隐私相关操作强制高S值 |
| 任务涉及系统级敏感操作（删除、权限修改） | ≥ 0.80 | 系统级操作强制高S值 |

### 四、L5直达资格判定

| 条件 | L5直达资格 |
|------|:---:|
| S值 ≥ 0.9 | **是** — 发送L5直达触发信号至ag-mem-16 |
| S值 < 0.9 | 否 — 正常写入L1，按常规晋升流程 |


## 核心处理逻辑

```
FUNCTION s_value_calculator_main_loop():
    STATE_IDLE = IDLE
    STATE_CALC = CALCULATING
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_IDLE
    加载S值计算规则配置（信号权重、敏感操作清单、阈值表）

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_IDLE

        // 第2步：接收S值计算请求
        IF 收到来自 ag-mem-15~19 的S值计算请求:
            SET internal_state = STATE_CALC
            条目ID = 请求.条目ID
            任务类型 = 请求.任务类型
            工具调用列表 = 请求.工具调用列表
            操作参数 = 请求.操作参数
            结果标签 = 请求.结果标签
            错误码 = 请求.错误码
            执行上下文 = 请求.执行上下文

            // 2a. 快速判定：严重失败 + 敏感操作 → 直接赋值
            IF 结果标签 == "严重失败":
                IF 检测敏感操作(工具调用列表, 操作参数):
                    S值 = 0.95
                    触发信号列表 = ["严重失败+敏感操作"]
                    跳转到输出步骤

            // 2b. 快速判定：完全成功 + 无安全信号 → 直接赋值
            IF 结果标签 == "成功":
                安全信号触发列表 = 检测所有安全信号(工具调用列表, 操作参数, 错误码)
                IF 安全信号触发列表为空:
                    S值 = 0.05
                    触发信号列表 = ["无安全信号"]
                    跳转到输出步骤

            // 2c. 标准S值计算流程
            S值 = 0.0
            触发信号列表 = []

            // 信号1：敏感操作检测（权重0.40）
            IF 检测敏感操作(工具调用列表, 操作参数):
                敏感操作等级 = 评估敏感操作严重度(工具调用列表, 操作参数)
                敏感操作得分 = 0.80 × 敏感操作等级
                S值 += 敏感操作得分 × 0.40
                触发信号列表.append("敏感操作:" + 列出具体操作)

            // 信号2：高风险工具检测（权重0.30）
            IF 检测高风险工具(工具调用列表):
                高风险工具等级 = 评估高风险工具等级(工具调用列表)
                高风险得分 = 0.70 × 高风险工具等级
                S值 += 高风险得分 × 0.30
                触发信号列表.append("高风险工具:" + 列出具体工具)

            // 信号3：错误与异常检测（权重0.15）
            IF 错误码 != None AND 错误码 != 0:
                错误严重度 = 映射错误码到严重度(错误码)
                错误得分 = 0.60 × 错误严重度
                S值 += 错误得分 × 0.15
                触发信号列表.append("错误异常:" + 错误码)

            // 信号4：用户隐私检测（权重0.10）
            IF 检测用户隐私访问(工具调用列表, 操作参数):
                隐私得分 = 0.90 × 1.0  // 隐私访问一旦触发即为最高
                S值 += 隐私得分 × 0.10
                触发信号列表.append("用户隐私访问")

            // 信号5：结果验证异常（权重0.05）
            IF 结果标签 IN ["部分成功", "结果异常"]:
                结果偏差得分 = 0.50 × 0.7
                S值 += 结果偏差得分 × 0.05
                触发信号列表.append("结果验证异常")

            // 2d. 边界处理
            IF S值 > 1.0:
                S值 = 1.0
            IF S值 < 0.0:
                S值 = 0.0

            // 输出步骤：
            // 3a. 判定L5直达资格
            L5直达资格 = S值 >= 0.9

            // 3b. 构建S值计算结果
            计算结果 = {
                条目ID: 条目ID,
                S值: ROUND(S值, 2),
                触发的主要安全信号列表: 触发信号列表,
                L5直达资格标记: L5直达资格
            }

            // 3c. 发送至请求来源分槽和重要度聚合单元
            向请求来源分槽返回S值计算结果
            向 ag-mem-36 发送S值计算结果

            // 3d. 若满足L5直达条件，发送触发信号至工具调用槽
            IF L5直达资格 AND 任务类型 == "工具调用":
                向 ag-mem-16 发送L5直达触发信号(
                    条目ID=条目ID,
                    S值=S值,
                    经验数据摘要=提取经验摘要(请求),
                    直达原因="S值≥0.9，触发L5直达"
                )

            // 3e. 更新统计计数器
            IF S值 >= 0.7:
                高S值事件计数++

            SET internal_state = STATE_IDLE

        // 第4步：周期性状态上报
        IF 距上次状态上报 >= 60秒:
            向 ag-mem-03 上报计算状态(
                当前状态=internal_state,
                最近S值分布=统计分布,
                高S值事件计数=高S值事件计数
            )
            重置统计计数器

        SLEEP 10ms


FUNCTION 检测敏感操作(工具调用列表, 操作参数):
    敏感操作清单 = 加载敏感操作清单()
    FOR EACH 工具调用 IN 工具调用列表:
        IF 工具调用.工具名称 IN 敏感操作清单:
            RETURN True
        IF 操作参数 包含 敏感关键词:
            RETURN True
    RETURN False


FUNCTION 检测高风险工具(工具调用列表):
    高风险工具清单 = ["shell_exec", "db_write", "payment_api", "system_config", "sudo"]
    FOR EACH 工具调用 IN 工具调用列表:
        IF 工具调用.工具名称 IN 高风险工具清单:
            RETURN True
    RETURN False


FUNCTION 检测用户隐私访问(工具调用列表, 操作参数):
    隐私相关操作 = ["read_contacts", "get_location", "browser_history", "read_messages"]
    FOR EACH 工具调用 IN 工具调用列表:
        IF 工具调用.工具名称 IN 隐私相关操作:
            RETURN True
    RETURN False
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| S值计算规则配置加载失败 | 使用编译期内置默认权重与阈值，标记降级 | 规则配置恢复 |
| 计算请求缺少必要字段（工具调用列表为空） | 默认返回S值=0.10（低安全信号），标记“数据不完整” | 字段补充完整 |
| 多个安全信号同时触发 | 按权重加和，上限截断为1.0 | — |
| S值计算中检测到无法归类的安全信号 | 标记为“未知安全信号”，赋予默认基准分0.30 | 信号类型被注册到规则库 |
| 紧急熔断 | 暂停计算，未完成的请求丢弃 | 紧急解除后恢复 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|

| 内部调度总线 | 写 | 计算状态上报 | 周期性写入 | 向 ag-mem-03 发送 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | S值计算仅基于任务元数据（工具名称、操作类型、错误码），不得解析用户原始输入内容 |
| S-02 | 敏感操作清单与高风险工具清单为只读配置，运行时不得被任何模块修改 |
| S-03 | L5直达资格判定仅基于S值阈值（≥0.9），不得通过其他路径绕过 |
| S-04 | S值计算结果必须可追溯，每个S值均附带触发的主要安全信号列表 |
