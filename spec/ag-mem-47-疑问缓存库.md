# ag-mem-47-疑问缓存库 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-47 |
| 模块名称 | 疑问缓存库 |
| 所属分区 | 四、漏斗外挂扩展区（物理隔离） |
| 核心职责 | 作为漏斗外挂扩展区的临时疑问存储库，暂存 ECC 认知大脑在推理过程中产生的推理断点、未确认认知项、低置信度场景数据。当 ECC 模块（如 ag-ecc-08 元认知模块）判定当前场景的置信度低于阈值或存在无法即时解决的逻辑矛盾时，将相关数据写入疑问缓存库，供系统离线复盘、根因分析、策略修正与模型迭代使用。本库完全独立于双漏斗记忆系统运行，不参与记忆沉淀、筛选、晋升与遗忘机制。仅提供暂存与查询服务，缓存条目在系统确认问题已解决或超过保留期限后自动清除 |
| 依赖模块 | 无（作为独立缓存服务，不依赖记忆系统内部模块） |
| 被依赖模块 | ag-ecc-08（元认知模块，写入低置信度推理断点）、ag-ecc-03（因果推理模块，写入未解决的逻辑矛盾）、ag-mem-01（总控漏斗 F₀，可转发查询请求以支持离线复盘） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常服务 | `NORMAL_SERVICE` | 缓存库正常运行，接收疑问写入与查询 | 系统初始化完成 |
| 容量预警 | `CAPACITY_WARNING` | 疑问条目数量超过最大容量的80% | 缓存条目数 ≥ 最大容量的80% |
| 容量已满 | `CAPACITY_FULL` | 缓存条目数量达到硬上限 | 缓存条目数 = 最大容量 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 疑问提交请求 | Struct（提交来源模块 + 疑问类型：推理断点/逻辑矛盾/低置信度场景 + 相关条目ID列表 + 场景描述 + 当前置信度 + 建议关注优先级） | ag-ecc-08（元认知）、ag-ecc-03（因果推理） | ECC 模块遇到无法即时解决的问题时 | **高** |
| 疑问查询请求 | Struct（请求来源模块 + 查询条件：时间范围/疑问类型/优先级/处理状态 + 最大返回条数） | ag-mem-01（转发离线复盘请求）、系统管理接口 | 离线复盘或系统调试时 | 普通 |
| 疑问状态更新指令 | Struct（疑问条目ID + 新处理状态 + 解决方案摘要 + 操作来源） | 系统管理接口或自动化分析流程 | 问题被确认解决时 | 普通 |
| 定时清理触发信号 | 内部定时器信号 | 本模块内部 | 每24小时触发一次过期条目清理 | 普通 |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-mem-01 总控漏斗 F₀ | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 疑问写入确认回执 | Struct（疑问条目ID + 写入状态 + 当前缓存使用率 + 建议清理提示） | 提交来源模块 | 疑问写入完成时 | **高** |
| 疑问查询结果列表 | Struct（匹配疑问条目列表 + 各条目详情 + 查询耗时 + 总匹配数） | 请求来源模块 | 查询完成时 | **高** |
| 疑问状态更新确认 | Struct（条目ID + 旧状态 + 新状态 + 更新时间戳） | 状态更新发起方 | 状态更新完成时 | 普通 |
| 缓存状态上报 | Struct（当前状态 + 总条目数 + 各类型分布 + 使用率 + 最早/最新条目时间） | ag-mem-01（周期性）、ag-mem-03（调度单元） | 初始化完成或状态变更时 | 普通 |


## 疑问条目结构

| 字段名称 | 数据类型 | 说明 |
|----------|----------|------|
| 疑问条目ID | String | 唯一标识，格式：Q-{时间戳}-{UUID} |
| 提交来源模块 | String | 写入该疑问的 ECC 模块编号 |
| 疑问类型 | Enum | 推理断点 / 逻辑矛盾 / 低置信度场景 |
| 相关条目ID列表 | List | 关联的记忆条目或决策记录 ID |
| 场景描述 | Struct | 包含场景快照哈希、关键参数、上下文摘要 |
| 当前置信度 | Float（0.0–1.0） | 写入时的系统置信度 |
| 优先级 | Enum | 高 / 中 / 低 |
| 处理状态 | Enum | 待分析 / 分析中 / 已解决 / 已忽略 / 已过期 |
| 写入时间戳 | Float | UTC 时间戳 |
| 最后更新时间戳 | Float | UTC 时间戳 |
| 解决方案摘要 | String | 问题确认解决后填写的方案描述 |


## 核心处理逻辑

```
FUNCTION question_cache_main_loop():
    STATE_NORMAL = NORMAL_SERVICE
    STATE_WARN = CAPACITY_WARNING
    STATE_FULL = CAPACITY_FULL
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_NORMAL
    初始化疑问条目存储（内存列表 + 持久化备份）
    设置最大容量 = 5000 条
    设置默认保留天数 = 90 天

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_NORMAL

        // 第2步：接收疑问提交请求
        IF 收到疑问提交请求:
            IF internal_state == STATE_FULL:
                向提交来源返回疑问写入确认回执(条目ID=Null, 写入状态="拒绝", 原因="缓存已满")
                CONTINUE

            // 2a. 生成疑问条目
            疑问条目ID = "Q-" + NOW() + "-" + 生成UUID()
            新条目 = 构建疑问条目(疑问条目ID, 请求)

            // 2b. 写入存储
            疑问存储.追加写入(新条目)

            // 2c. 更新容量状态
            当前条目数 = 疑问存储.统计条目数()
            IF 当前条目数 >= 最大容量:
                SET internal_state = STATE_FULL
            ELSE IF 当前条目数 >= 最大容量 × 0.8:
                SET internal_state = STATE_WARN

            // 2d. 返回确认
            向提交来源返回疑问写入确认回执(
                疑问条目ID=疑问条目ID,
                写入状态="成功",
                当前缓存使用率=当前条目数 / 最大容量,
                建议清理提示=IF internal_state == STATE_WARN THEN "缓存使用率已达80%，建议启动复盘流程清理已解决条目" ELSE ""
            )

        // 第3步：接收疑问查询请求
        IF 收到疑问查询请求:
            查询结果 = 疑问存储.按条件查询(请求.查询条件)
            查询结果 = 截断到最大返回条数(查询结果, 请求.最大返回条数)
            向请求来源返回疑问查询结果列表(查询结果)

        // 第4步：接收疑问状态更新指令
        IF 收到疑问状态更新指令:
            IF 指令.疑问条目ID IN 疑问存储:
                更新疑问条目状态(指令.疑问条目ID, 指令.新处理状态, 指令.解决方案摘要)
                向状态更新发起方返回疑问状态更新确认(指令.疑问条目ID, 旧状态, 新状态)
            ELSE:
                向状态更新发起方返回更新失败("疑问条目不存在")

        // 第5步：定时清理过期条目（每24小时）
        IF 收到定时清理触发信号:
            过期条目列表 = 疑问存储.筛选(写入时间戳 < NOW() - 90天 AND 处理状态 IN ["已解决", "已忽略"])
            FOR EACH 过期条目 IN 过期条目列表:
                疑问存储.标记为已过期(过期条目.疑问条目ID)

            // 更新容量状态
            当前条目数 = 疑问存储.统计条目数(排除已过期)
            IF 当前条目数 < 最大容量 × 0.8:
                SET internal_state = STATE_NORMAL

        // 第6步：周期性状态上报
        IF 距上次状态上报 >= 120秒:
            统计各类型、各状态条目分布
            向 ag-mem-01 和 ag-mem-03 上报缓存状态(当前状态, 统计信息)

        SLEEP 50ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 疑问条目数量达到硬上限（5000条） | 拒绝新写入，返回“缓存已满”通知 | 过期条目清理或人工删除释放空间 |
| 提交请求缺少必要字段（疑问类型为空） | 拒绝写入，返回“参数不完整”错误 | 补充完整参数后重试 |
| 查询请求的时间范围超出存储范围 | 返回可用范围内的结果，标记“数据可能不完整” | — |
| 定时清理过程中系统资源紧张 | 暂停清理，等待下一周期 | 资源恢复正常 |
| 疑问条目存储持久化异常 | 使用内存缓存继续服务，上报告警 | 存储恢复正常 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| 内部调度总线 / MemoryBus | 读 | 疑问提交请求 | 只读 | ag-ecc-08/03 发送 |
| 内部调度总线 / MemoryBus | 读 | 疑问查询请求 | 只读 | ag-mem-01（转发）、管理接口 |
| 内部调度总线 / MemoryBus | 读 | 疑问状态更新指令 | 只读 | 管理接口、自动化分析流程 |
| 内部调度总线 / MemoryBus | 写 | 疑问写入确认回执 | 专属写入 | 向提交来源返回 |
| 内部调度总线 / MemoryBus | 写 | 疑问查询结果列表 | 专属写入 | 向请求来源返回 |
| 内部调度总线 / MemoryBus | 写 | 疑问状态更新确认 | 专属写入 | 向更新发起方返回 |
| 内部调度总线 | 写 | 缓存状态上报 | 周期性写入 | 向 ag-mem-01/03 发送 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| C-01 | 疑问缓存条目在系统确认问题解决或超过保留期限后自动清除，不得无限期保留 |
| C-02 | 疑问条目中的场景描述不得包含用户的原始输入文本或个人身份信息，仅保留场景特征哈希与结构化的上下文参数 |
| C-03 | 本库完全独立于漏斗记忆系统，不参与记忆沉淀、晋升、遗忘任何环节 |
| C-04 | 疑问缓存数据仅用于离线复盘与系统调试，不得作为实时决策依据 |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-M47-01 | `NORMAL_SERVICE`，提交低置信度场景疑问 | 提交请求（疑问类型=低置信度场景，置信度=0.35） | 写入成功，返回疑问条目ID |
| TC-M47-02 | `NORMAL_SERVICE`，查询待分析疑问 | 查询请求（处理状态=待分析，最大返回=50） | 返回匹配的疑问条目列表 |
| TC-M47-03 | `NORMAL_SERVICE`，更新疑问处理状态 | 状态更新指令（新状态=已解决，解决方案摘要="已确认原因"） | 更新成功，返回确认 |
| TC-M47-04 | `CAPACITY_FULL`，缓存已满时提交疑问 | 提交请求 + 当前条目数=5000 | 拒绝写入，返回“缓存已满” |

| TC-M47-06 | `SYSTEM_PAUSED`，收到提交请求 | 提交请求 | 拒绝处理，保持暂停状态 |
