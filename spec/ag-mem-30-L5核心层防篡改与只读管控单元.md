# ag-mem-30-L5核心层防篡改与只读管控单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-30 |
| 模块名称 | L5核心层防篡改与只读管控单元 |
| 所属分区 | 三、漏斗二：任务经验漏斗 / 五层存储 |
| 核心职责 | 作为L5核心层数据的唯一对外访问接口，管控所有外部模块对L5层经验的读写权限。对外部模块仅提供经过令牌验证的只读查询服务，任何写入请求必须携带由ag-mem-29签发的有效写入安全令牌。管理查询令牌的签发、验证与吊销，确保L5数据不被未授权模块访问或修改。不参与经验内容的管理决策，仅执行访问权限的强制管控 |
| 依赖模块 | ag-mem-28（L5核心层存储单元，接收查询令牌验证请求）、ag-mem-29（L5核心层安全规则硬锁定单元，提供写入安全令牌的验证基准） |
| 被依赖模块 | ag-mem-15~19（各场景分槽，通过本模块授权的查询接口检索L5经验）、ag-mem-28（接收查询令牌验证回执） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常管控 | `NORMAL_GATE` | 访问管控正常运行，签发查询令牌并验证写入令牌 | 系统初始化完成 |
| 写入令牌校验中 | `WRITE_TOKEN_CHECK` | 正在验证携带的写入安全令牌是否与ag-mem-29签发的活跃令牌一致 | 收到外部模块的L5写入请求 |
| 查询令牌签发中 | `QUERY_TOKEN_ISSUE` | 正在为合法查询请求签发短期查询令牌 | 收到各分槽的L5查询请求 |
| 违规阻断 | `VIOLATION_BLOCK` | 检测到未授权访问或令牌伪造，已拦截并告警 | 令牌验证失败或未携带令牌的请求 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| L5查询请求 | Struct（查询条件 + 请求来源分槽编号 + 请求模块ID + 最大返回条数） | ag-mem-15~19 各场景分槽（通过内部调度总线） | 各分槽需要检索L5核心经验时 | **高** |
| L5写入请求 | Struct（条目数据 + 写入来源 + 携带的写入安全令牌） | 外部模块（通过ag-mem-28转发或直接请求） | 任何模块尝试写入L5时 | **最高** |
| 活跃写入令牌信息 | Struct（令牌ID + 授权写入条目数上限 + 有效期 + 签名 + 当前状态） | ag-mem-29 L5核心层安全规则硬锁定单元 | L5写入令牌签发或吊销时同步 | **最高** |
| 查询令牌吊销请求 | Struct（令牌ID + 吊销原因） | 会话管理器或ag-mem-28 | 会话结束或令牌泄露时 | **高** |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-mem-01 总控漏斗F0 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 查询令牌签发回执 | Struct（令牌ID + 授权查询槽位编号 + 有效期 + 签名） | 请求来源分槽（ag-mem-15~19） | 查询请求通过验证时 | **高** |
| 查询令牌验证回执 | Struct（令牌有效 + 授权槽位编号 + 授权操作类型 + 剩余有效期） | ag-mem-28 L5核心层存储单元 | L5存储单元验证查询令牌时 | **高** |
| 写入令牌验证回执 | Struct（令牌有效 + 授权写入条目数上限 + 剩余有效期 + 写入来源确认） | ag-mem-28 L5核心层存储单元 | 写入令牌验证通过时 | **最高** |
| 访问拒绝通知 | Struct（拒绝原因 + 请求详情 + 违规严重等级 + 时间戳） | 请求来源模块 + ag-mem-29（写入违规时） + ag-mem-51（日志） | 令牌验证失败或未授权访问时 | **高** |
| 违规告警 | Struct（违规类型 + 来源模块 + 请求内容摘要 + 时间戳） | ag-mem-29 L5锁定单元、ag-mem-51 日志追溯单元 | 检测到未授权访问或令牌伪造时 | **高** |
| 令牌吊销确认 | Struct（令牌ID + 吊销状态 + 已失效时间） | 请求吊销方（会话管理器或ag-mem-28） | 令牌吊销完成时 | 普通 |
| 管控状态上报 | Struct（当前状态 + 今日签发令牌数 + 今日拒绝次数 + 活跃令牌数 + 违规次数） | ag-mem-03 漏斗二专属调度单元 | 周期性（每120秒）或状态变更时 | 普通 |


## 访问权限矩阵

| 请求来源模块 | 允许的操作 | 所需令牌类型 | 令牌有效期 | 附加条件 |
|------|:---:|:---:|:---:|------|
| ag-mem-15~19（各场景分槽） | 只读查询 | 查询令牌 | 300秒 | 仅返回本分槽来源的条目 |
| ag-mem-28（L5存储单元） | 查询令牌验证 | — | — | 仅接受来自L5存储单元的验证请求 |
| ag-mem-16（S值直达写入） | 写入 | 写入安全令牌（ag-mem-29签发） | 30秒 | 需验证令牌签名 |
| ag-mem-27（L4推送写入） | 写入 | 写入安全令牌（ag-mem-29签发） | 30秒 | 需验证令牌签名 |
| 人工操作接口 | 写入/删除 | 写入安全令牌（ag-mem-29签发） | 30秒 | 需双重确认 |
| 任何其他模块 | 禁止 | — | — | 编译期直接拒绝 |


## 令牌管理规则

| 规则项 | 默认值 | 说明 |
|--------|:---:|------|
| 查询令牌有效期 | **300秒** | 超过有效期自动失效 |
| 写入令牌有效期 | **30秒** | 与ag-mem-29签发的令牌有效期一致 |
| 最大活跃查询令牌数 | **20个** | 超过上限则优先吊销最旧的令牌 |
| 令牌签名算法 | HMAC-SHA256 | 防止令牌伪造，密钥由本模块与ag-mem-29共享 |
| 令牌吊销延迟 | **即时生效** | 吊销后立即拒绝携带该令牌的请求 |


## 核心处理逻辑

```
FUNCTION l5_access_control_main_loop():
    STATE_NORMAL = NORMAL_GATE
    STATE_WRITE_CHECK = WRITE_TOKEN_CHECK
    STATE_QUERY_ISSUE = QUERY_TOKEN_ISSUE
    STATE_BLOCK = VIOLATION_BLOCK
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_NORMAL
    初始化查询令牌签发记录表 = {}
    初始化写入令牌缓存（从ag-mem-29同步）
    初始化违规计数器 = 0

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            吊销所有活跃查询令牌
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_NORMAL

        // 第2步：同步活跃写入令牌信息
        IF 收到 ag-mem-29 活跃写入令牌信息:
            更新写入令牌缓存 = 信息
            CONTINUE

        // 第3步：接收L5查询请求（签发查询令牌）
        IF 收到 L5查询请求:
            // 3a. 校验请求来源是否在访问权限矩阵中
            请求来源 = 请求.来源模块ID
            IF 请求来源 NOT IN [ag-mem-15~19]:
                SET internal_state = STATE_BLOCK
                向请求来源返回访问拒绝通知("未授权模块", 请求详情, "严重")
                向 ag-mem-51 发送违规告警("未授权L5查询", 请求来源)
                违规计数器++
                SET internal_state = STATE_NORMAL
                CONTINUE

            // 3b. 检查活跃查询令牌数是否超限
            IF 查询令牌签发记录表.size >= 20:
                // 吊销最旧的查询令牌
                最旧令牌 = 按签发时间升序排列(查询令牌签发记录表)[0]
                吊销查询令牌(最旧令牌.令牌ID, "令牌数超限")

            // 3c. 签发查询令牌
            SET internal_state = STATE_QUERY_ISSUE
            令牌ID = "L5-QTOKEN-" + 生成UUID()
            查询令牌 = {
                令牌ID: 令牌ID,
                授权查询槽位编号: 请求.来源分槽编号,
                授权操作类型: "只读查询",
                有效期: 300秒,
                签发时间: NOW(),
                签名: HMAC签名(令牌ID + 请求.来源分槽编号 + NOW())
            }
            查询令牌签发记录表[令牌ID] = 查询令牌

            // 3d. 返回令牌
            向请求来源返回查询令牌签发回执(查询令牌)
            SET internal_state = STATE_NORMAL

        // 第4步：接收ag-mem-28的查询令牌验证请求
        IF 收到来自 ag-mem-28 的查询令牌验证请求:
            令牌ID = 请求.令牌ID
            IF 令牌ID IN 查询令牌签发记录表:
                令牌 = 查询令牌签发记录表[令牌ID]
                // 检查是否过期
                IF NOW() - 令牌.签发时间 > 300:
                    从查询令牌签发记录表中移除令牌ID
                    向 ag-mem-28 返回查询令牌验证回执(令牌有效=False, 原因="令牌已过期")
                ELSE:
                    向 ag-mem-28 返回查询令牌验证回执(
                        令牌有效=True,
                        授权槽位编号=令牌.授权查询槽位编号,
                        授权操作类型=令牌.授权操作类型,
                        剩余有效期=300 - (NOW() - 令牌.签发时间)
                    )
            ELSE:
                向 ag-mem-28 返回查询令牌验证回执(令牌有效=False, 原因="令牌不存在或已被吊销")

        // 第5步：接收L5写入请求（验证写入安全令牌）
        IF 收到 L5写入请求:
            SET internal_state = STATE_WRITE_CHECK
            携带令牌ID = 请求.携带的写入安全令牌

            // 5a. 检查是否有活跃写入令牌
            IF 写入令牌缓存 == None OR 写入令牌缓存.当前状态 != "有效":
                SET internal_state = STATE_BLOCK
                向请求来源返回访问拒绝通知("无活跃写入令牌", 请求详情, "严重")
                向 ag-mem-29 发送违规告警("无令牌写入尝试", 请求来源)
                违规计数器++
                SET internal_state = STATE_NORMAL
                CONTINUE

            // 5b. 验证令牌ID匹配
            IF 携带令牌ID != 写入令牌缓存.令牌ID:
                SET internal_state = STATE_BLOCK
                向请求来源返回访问拒绝通知("写入令牌不匹配", 请求详情, "严重")
                向 ag-mem-29 发送违规告警("令牌伪造或过期", 请求来源)
                违规计数器++
                SET internal_state = STATE_NORMAL
                CONTINUE

            // 5c. 验证令牌签名
            期望签名 = HMAC签名(携带令牌ID + 写入令牌缓存.授权写入条目数上限 + 写入令牌缓存.有效期)
            IF 携带令牌的签名 != 期望签名:
                SET internal_state = STATE_BLOCK
                向请求来源返回访问拒绝通知("令牌签名校验失败", 请求详情, "严重")
                向 ag-mem-29 发送违规告警("令牌签名校验失败", 请求来源)
                违规计数器++
                SET internal_state = STATE_NORMAL
                CONTINUE

            // 5d. 检查令牌是否过期
            IF NOW() - 写入令牌缓存.签发时间 > 30:
                向请求来源返回访问拒绝通知("写入令牌已过期", 请求详情, "一般")
                SET internal_state = STATE_NORMAL
                CONTINUE

            // 5e. 验证通过
            向 ag-mem-28 返回写入令牌验证回执(
                令牌有效=True,
                授权写入条目数上限=写入令牌缓存.授权写入条目数上限,
                剩余有效期=30 - (NOW() - 写入令牌缓存.签发时间),
                写入来源确认=写入令牌缓存.写入来源
            )
            SET internal_state = STATE_NORMAL

        // 第6步：处理查询令牌吊销请求
        IF 收到查询令牌吊销请求:
            吊销查询令牌(请求.令牌ID, 请求.吊销原因)
            向请求方返回令牌吊销确认(请求.令牌ID, "已吊销")

        // 第7步：定时清理过期查询令牌（每60秒）
        IF 距上次清理 >= 60秒:
            FOR EACH 令牌ID, 令牌 IN 查询令牌签发记录表:
                IF NOW() - 令牌.签发时间 > 300:
                    从查询令牌签发记录表中移除令牌ID
            重置清理计时器

        // 第8步：周期性状态上报
        IF 距上次状态上报 >= 120秒:
            向 ag-mem-03 上报管控状态(
                当前状态=internal_state,
                今日签发令牌数=统计签发计数,
                今日拒绝次数=统计拒绝计数,
                活跃令牌数=查询令牌签发记录表.size,
                违规次数=违规计数器
            )

        SLEEP 10ms


FUNCTION 吊销查询令牌(令牌ID, 吊销原因):
    IF 令牌ID IN 查询令牌签发记录表:
        从查询令牌签发记录表中移除令牌ID
        记录吊销日志(令牌ID, 吊销原因)
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 查询令牌签发记录表满（达20个上限） | 自动吊销最旧的查询令牌，为新令牌腾出空间 | — |
| 写入令牌缓存未同步（系统启动初期） | 拒绝所有写入请求，标记“等待写入令牌同步” | ag-mem-29完成令牌信息同步 |
| 查询令牌被伪造（签名不匹配） | 拒绝查询，触发违规告警 | 使用合法令牌重新请求 |
| 同一来源模块短时间内多次违规 | 累计违规次数，超过阈值后锁定该模块的L5访问权限600秒 | 锁定时间到期且人工确认 |
| 紧急熔断 | 立即吊销所有活跃查询令牌，拒绝所有请求 | 紧急解除后恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | L5层数据的所有访问必须经过本模块的令牌验证，禁止任何模块绕过本模块直接访问L5存储 |
| S-02 | 查询令牌仅授权只读操作，任何携带查询令牌的写入请求一律拒绝 |
| S-03 | 写入令牌必须与ag-mem-29签发的活跃令牌完全一致（ID+签名双重校验），任何不匹配视为令牌伪造 |
| S-04 | 令牌签名采用HMAC-SHA256，密钥仅本模块与ag-mem-29共享，外部模块无法伪造 |
| S-05 | 违规访问尝试必须实时阻断并完整记录告警日志，不可篡改 |
| S-06 | 查询令牌过期后自动失效，不得通过任何方式延长有效期 |
