# ag-mem-12-临时画像槽自动清除单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-12 |
| 模块名称 | 临时画像槽自动清除单元 |
| 所属分区 | 二、漏斗一：用户画像漏斗 |
| 核心职责 | 管理漏斗一中临时槽和访客槽的生命周期终止处理。在临时槽超过7天有效期或访客槽会话结束时，接收 ag-mem-02 下发的清除指令，对目标画像槽执行安全擦除（覆写后删除）并释放存储配额。确保被清除的用户数据不可恢复，满足隐私合规要求。不参与槽位创建或内容管理决策，仅执行安全清除操作 |
| 依赖模块 | ag-mem-02（漏斗一专属调度单元，下发清除指令）、ag-mem-06（画像槽数据隔离管控单元，吊销已清除槽位的所有访问令牌）、ag-mem-48（全局容量配额管控单元，退还已释放的存储配额） |
| 被依赖模块 | ag-mem-02（接收清除完成回执）、ag-mem-51（记忆变更日志追溯单元，记录清除事件） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 无清除任务，等待指令 | 系统初始化完成，无待处理请求 |
| 清除准备 | `PREPARING` | 正在吊销目标槽位的访问令牌并锁定存储分区 | 收到 ag-mem-02 清除指令 |
| 安全擦除 | `ERASING` | 正在对槽位存储执行覆写擦除 | 令牌吊销完成，存储分区锁定 |
| 清除完成 | `COMPLETED` | 清除操作完成，已释放存储配额 | 擦除完成且配额退还 |
| 清除失败 | `FAILED` | 擦除过程中发生异常 | 存储操作异常或权限不足 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |

## 安全擦除策略

### 一、擦除方法

| 槽位类型 | 擦除方法 | 覆写次数 | 说明 |
|----------|:---:|:---:|------|
| 临时槽 | DoD 5220.22-M 标准单次覆写 | 1次 | 全分区覆写随机数据后删除 |
| 访客槽 | 直接删除 + 元数据清除 | 0次 | 访客数据不保留，直接释放空间 |
| 长期槽（手动删除时） | DoD 5220.22-M 标准三次覆写 | 3次 | 确保不可恢复 |

### 二、清除流程时序

```
收到清除指令 → 吊销所有令牌 → 锁定存储分区 → 执行安全擦除 → 退还配额 → 记录清除日志
```

每步超时时间：30秒，超时则进入失败处理流程。


## 核心处理逻辑

```
FUNCTION slot_cleanup_main_loop():
    STATE_IDLE = IDLE
    STATE_PREPARE = PREPARING
    STATE_ERASE = ERASING
    STATE_DONE = COMPLETED
    STATE_FAIL = FAILED
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_IDLE
    初始化清除任务队列

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            // 当前正在进行的清除任务需要回滚吗？不，清除是单向操作，继续完成
            等待当前清除任务完成（如有）
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_IDLE

        // 第2步：接收清除指令
        IF 收到 ag-mem-02 槽位清除指令:
            IF internal_state == STATE_IDLE:
                SET internal_state = STATE_PREPARE
                当前槽位编号 = 指令.目标槽位编号
                当前清除原因 = 指令.清除原因
                当前槽位类型 = 指令.槽位类型
                擦除方法 = 根据槽位类型选择擦除策略(当前槽位类型)

                // 2a. 吊销所有访问令牌
                向 ag-mem-06 发送令牌吊销请求(当前槽位编号, "槽位清除")

            ELSE:
                // 非空闲状态，加入任务队列
                清除任务队列.append(指令)
                CONTINUE

        // 第3步：等待令牌吊销确认
        IF internal_state == STATE_PREPARE AND 收到 ag-mem-06 令牌吊销确认:
            IF 确认.槽位编号 == 当前槽位编号:
                // 3a. 锁定存储分区
                锁定结果 = 漏斗一存储管理器.锁定分区(当前槽位编号)
                IF 锁定结果.成功:
                    SET internal_state = STATE_ERASE
                    开始擦除计时器 = NOW()
                    执行安全擦除(当前槽位编号, 擦除方法)
                ELSE:
                    SET internal_state = STATE_FAIL
                    向 ag-mem-02 发送清除失败通知(当前槽位编号, "存储分区锁定失败")
                    处理队列中下一个任务()

        // 第4步：等待擦除完成
        IF internal_state == STATE_ERASE:
            // 擦除是异步操作，轮询检查完成状态
            擦除状态 = 漏斗一存储管理器.查询擦除状态(当前槽位编号)
            IF 擦除状态.完成:
                擦除耗时 = NOW() - 开始擦除计时器
                SET internal_state = STATE_DONE

                // 4a. 退还存储配额
                释放空间量 = 擦除状态.释放空间量
                向 ag-mem-48 发送配额退还请求(当前槽位编号, 释放空间量)

            ELSE IF 擦除状态.失败:
                SET internal_state = STATE_FAIL
                向 ag-mem-02 发送清除失败通知(当前槽位编号, "安全擦除异常: " + 擦除状态.错误信息)
                处理队列中下一个任务()

            ELSE IF (NOW() - 开始擦除计时器) > 30秒:
                SET internal_state = STATE_FAIL
                向 ag-mem-02 发送清除失败通知(当前槽位编号, "安全擦除超时")
                处理队列中下一个任务()

        // 第5步：等待配额退还确认，完成清除流程
        IF internal_state == STATE_DONE AND 收到 ag-mem-48 配额退还确认:
            IF 确认.槽位编号 == 当前槽位编号:
                // 5a. 记录清除事件
                向 ag-mem-51 发送清除事件日志(
                    事件类型="画像槽清除",
                    槽位编号=当前槽位编号,
                    槽位类型=当前槽位类型,
                    清除原因=当前清除原因,
                    释放空间=释放空间量,
                    擦除耗时=擦除耗时
                )

                // 5b. 返回清除完成回执
                向 ag-mem-02 发送清除完成回执(
                    槽位编号=当前槽位编号,
                    清除结果="成功",
                    释放空间量=释放空间量,
                    擦除耗时=擦除耗时,
                    是否可恢复=False
                )

                // 5c. 处理队列中下一个任务
                处理队列中下一个任务()

        SLEEP 50ms


FUNCTION 处理队列中下一个任务():
    IF 清除任务队列 非空:
        下一个指令 = 清除任务队列.pop(0)
        // 重新执行步骤2
        当前槽位编号 = 下一个指令.目标槽位编号
        当前清除原因 = 下一个指令.清除原因
        SET internal_state = STATE_PREPARE
        向 ag-mem-06 发送令牌吊销请求(当前槽位编号, "槽位清除")
    ELSE:
        SET internal_state = STATE_IDLE
        清除当前任务上下文
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 令牌吊销超时（ag-mem-06 无响应） | 等待30秒后强制进入擦除阶段，标记“令牌未完全吊销” | ag-mem-06 恢复正常 |
| 存储分区锁定失败（已被其他进程锁定） | 返回清除失败，等待下次触发 | 分区锁释放 |
| 安全擦除过程中断电/系统崩溃 | 重启后检测到未完成清除的槽位，重新执行擦除 | 系统重启且分区可访问 |
| 配额退还失败 | 记录告警，标记“配额未释放”，不影响清除完成回执 | ag-mem-48 恢复后手动触发退还 |
| 紧急熔断 | 完成当前正在进行的擦除操作（安全优先），然后暂停接收新指令 | 紧急解除后恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 临时槽和访客槽的清除必须执行安全擦除（临时槽覆写1次，访客槽直接删除），不可仅标记删除 |
| S-02 | 清除操作开始前必须先吊销目标槽位的所有访问令牌，防止擦除过程中数据被读取 |
| S-03 | 安全擦除过程中不得被任何读取或写入请求中断 |
| S-04 | 清除操作的每个步骤（吊销→锁定→擦除→退还）必须完整记录日志，不可篡改 |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-M12-01 | `IDLE`，收到临时槽过期清除指令 | 清除指令（槽位编号=SLOT-TEMP-1，原因=过期，策略=单次覆写） | 吊销令牌→锁定分区→覆写擦除→退还配额→返回完成回执 |
| TC-M12-02 | `IDLE`，收到访客槽会话结束清除指令 | 清除指令（槽位编号=SLOT-GUEST-1，原因=会话结束） | 直接删除→退还配额→返回完成回执 |
| TC-M12-03 | `IDLE`，收到长期槽手动删除指令 | 清除指令（槽位编号=SLOT-LONG-2，原因=用户手动删除） | 三次覆写擦除→退还配额→返回完成回执 |
| TC-M12-04 | `PREPARING`，令牌吊销超时30秒 | 令牌吊销请求无响应 | 强制进入擦除阶段，标记“令牌未完全吊销” |
| TC-M12-05 | `ERASING`，擦除过程中存储异常 | 存储操作返回错误 | 进入 FAILED，返回清除失败通知 |
| TC-M12-06 | `IDLE`，队列中有多个待清除任务 | 连续收到3个清除指令 | 依次处理，每个任务独立完成后再处理下一个 |
