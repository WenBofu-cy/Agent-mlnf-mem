# ag-mem-23-L2近期层热度统计单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-23 |
| 模块名称 | L2近期层热度统计单元 |
| 所属分区 | 三、漏斗二：任务经验漏斗 / 五层存储 |
| 核心职责 | 统计L2近期层中各经验条目被查询命中的频率。接收 ag-mem-22（L2存储单元）在每次完成经验查询后推送的命中条目列表，更新对应条目的命中计数。为 ag-mem-33（复用频次C值统计单元）提供L2层的命中频次数据，作为C值计算的原始输入；为 ag-mem-38（晋升双条件判定单元）提供热度参考权重。不参与晋升决策，仅执行命中事件的记录与统计 |
| 依赖模块 | ag-mem-22（L2近期层存储单元，推送查询命中条目列表）、ag-mem-33（复用频次C值统计单元，消费L2命中频次数据）、ag-mem-38（晋升双条件判定单元，查询热度参考数据） |
| 被依赖模块 | ag-mem-33（获取L2命中频次）、ag-mem-38（获取热度参考权重） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 无统计任务，等待事件 | 系统初始化完成 |
| 统计更新中 | `UPDATING` | 正在更新命中计数 | 收到 ag-mem-22 的命中条目列表 |
| 查询响应中 | `RESPONDING` | 正在处理热度数据查询请求 | 收到 ag-mem-33 或 ag-mem-38 的查询请求 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| L2查询命中条目列表 | List（条目ID + 命中时间戳 + 查询来源分槽编号） | ag-mem-22 L2近期层存储单元（通过内部调度总线） | L2每次完成经验查询后 | **高** |
| L2新条目通知 | Struct（新增条目ID列表 + 来源分槽编号 + 写入时间戳） | ag-mem-22 L2近期层存储单元 | L2接收新晋升条目时 | 普通 |
| 热度数据查询请求 | Struct（条目ID列表 + 查询类型：单条/批量 + 统计时间窗口） | ag-mem-33（C值统计）或 ag-mem-38（晋升判定） | 需要热度数据时 | **高** |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-mem-01 总控漏斗F0 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| L2命中频次数据 | Struct（条目ID + 总命中次数 + 近7日命中次数 + 近24小时命中次数 + 最近命中时间） | ag-mem-33（C值统计）或 ag-mem-38（晋升判定） | 收到查询请求时 | **高** |
| 热度统计状态上报 | Struct（当前状态 + 统计条目总数 + 今日总命中次数 + 平均命中率） | ag-mem-03 漏斗二专属调度单元 | 周期性（每60秒）或状态变更时 | 普通 |


## 统计数据结构

```
L2热度统计表 = {
    条目ID: {
        总命中次数: 整数,
        近7日命中次数: 整数,
        近24小时命中次数: 整数,
        最近命中时间: 时间戳,
        首次命中时间: 时间戳,
        命中来源分槽分布: {分槽编号: 命中次数}
    }
}
```


## 核心处理逻辑

```
FUNCTION l2_heat_statistics_main_loop():
    STATE_IDLE = IDLE
    STATE_UPDATE = UPDATING
    STATE_RESPOND = RESPONDING
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_IDLE
    初始化L2热度统计表 = {}

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_IDLE

        // 第2步：接收L2新条目通知（初始化热度计数）
        IF 收到 ag-mem-22 L2新条目通知:
            FOR EACH 条目ID IN 通知.新增条目ID列表:
                IF 条目ID NOT IN L2热度统计表:
                    L2热度统计表[条目ID] = {
                        总命中次数: 0,
                        近7日命中次数: 0,
                        近24小时命中次数: 0,
                        最近命中时间: None,
                        首次命中时间: NOW(),
                        命中来源分槽分布: {}
                    }
            CONTINUE

        // 第3步：接收L2查询命中条目列表（更新命中计数）
        IF 收到 ag-mem-22 L2查询命中条目列表:
            SET internal_state = STATE_UPDATE
            当前时间 = NOW()

            FOR EACH 命中条目 IN 查询命中条目列表:
                条目ID = 命中条目.条目ID
                查询来源分槽 = 命中条目.查询来源分槽编号

                IF 条目ID NOT IN L2热度统计表:
                    // 如果条目不在统计表中（可能是旧数据），初始化
                    L2热度统计表[条目ID] = {
                        总命中次数: 0,
                        近7日命中次数: 0,
                        近24小时命中次数: 0,
                        最近命中时间: None,
                        首次命中时间: 当前时间,
                        命中来源分槽分布: {}
                    }

                统计记录 = L2热度统计表[条目ID]
                统计记录.总命中次数++
                统计记录.近7日命中次数++
                统计记录.近24小时命中次数++
                统计记录.最近命中时间 = 当前时间

                // 更新分槽分布
                IF 查询来源分槽 NOT IN 统计记录.命中来源分槽分布:
                    统计记录.命中来源分槽分布[查询来源分槽] = 0
                统计记录.命中来源分槽分布[查询来源分槽]++

            SET internal_state = STATE_IDLE

        // 第4步：响应热度数据查询请求
        IF 收到热度数据查询请求:
            SET internal_state = STATE_RESPOND
            查询结果 = {}

            FOR EACH 条目ID IN 请求.条目ID列表:
                IF 条目ID IN L2热度统计表:
                    统计 = L2热度统计表[条目ID]
                    查询结果[条目ID] = {
                        总命中次数: 统计.总命中次数,
                        近7日命中次数: 统计.近7日命中次数,
                        近24小时命中次数: 统计.近24小时命中次数,
                        最近命中时间: 统计.最近命中时间
                    }
                ELSE:
                    查询结果[条目ID] = {
                        总命中次数: 0,
                        近7日命中次数: 0,
                        近24小时命中次数: 0,
                        最近命中时间: None
                    }

            向请求来源返回L2命中频次数据(查询结果)
            SET internal_state = STATE_IDLE

        // 第5步：定时衰减旧计数（每小时）
        IF 距上次衰减 >= 1小时:
            // 衰减逻辑：超过7天的命中不计入"近7日命中次数"，超过24小时不计入"近24小时命中次数"
            // 此处简化处理：保留计数器不清零，因为实际统计时我们无法追溯每条命中的精确时间
            // 真正的滑动窗口统计需要记录每次命中的时间戳列表，复杂度较高
            // 对于L2层级，采用近似方案：仅在新命中时递增，依赖ag-mem-33的归一化处理
            重置衰减计时器

        // 第6步：清理已删除条目的统计数据（每24小时）
        IF 距上次清理 >= 24小时:
            // 向ag-mem-22查询当前L2中存在的条目ID列表
            有效条目列表 = 向 ag-mem-22 查询当前L2条目ID列表()
            有效条目集合 = SET(有效条目列表)
            FOR EACH 条目ID IN L2热度统计表:
                IF 条目ID NOT IN 有效条目集合:
                    从L2热度统计表中移除条目ID
            重置清理计时器

        // 第7步：周期性状态上报
        IF 距上次状态上报 >= 60秒:
            统计总数 = L2热度统计表.size
            今日总命中 = SUM(各条目近24小时命中次数)
            向 ag-mem-03 上报热度统计状态(统计总数, 今日总命中)

        SLEEP 20ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 查询命中列表包含不存在的条目ID | 自动初始化该条目的统计记录，不报错 | — |
| 热度统计表占用内存过大 | 触发LRU淘汰：移除最近30天无任何命中的条目统计 | 内存使用率回落 |
| 查询请求的条目ID列表为空 | 返回空结果 | — |
| ag-mem-22 查询有效条目列表超时 | 跳过本次清理，保留现有统计数据 | 下次清理周期正常 |
| 紧急熔断 | 暂停所有操作，保持统计数据 | 紧急解除后恢复 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| 内部调度总线 | 读 | L2查询命中条目列表 | 只读 | ag-mem-22 发送 |
| 内部调度总线 | 读 | L2新条目通知 | 只读 | ag-mem-22 发送 |
| 内部调度总线 | 读 | 热度数据查询请求 | 只读 | ag-mem-33/38 发送 |
| 内部调度总线 | 写 | L2命中频次数据 | 专属写入 | 向请求来源返回 |
| 内部调度总线 | 写 | 热度统计状态上报 | 周期性写入 | 向 ag-mem-03 发送 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | L2热度统计数据仅反映查询命中频率，不包含任何用户个人信息或经验内容 |
| S-02 | 热度统计表仅存于内存，不持久化，系统重启后数据从零开始积累 |
| S-03 | 本模块不主动查询L2存储内容，仅被动接收ag-mem-22推送的命中事件 |
| S-04 | 热度数据的跨槽查询禁止：某个分槽查询时仅返回与该分槽相关的命中分布 |

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-M23-01 | `IDLE`，收到命中条目列表 | 命中条目列表（3条，来自ag-mem-15的查询） | 更新3条记录，命中计数+1 |

| TC-M23-06 | `SYSTEM_PAUSED`，收到查询请求 | 热度查询请求 | 拒绝处理，保持暂停状态 |
