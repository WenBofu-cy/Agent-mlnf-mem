# ag-mem-36-综合重要度I值聚合计算单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-36 |
| 模块名称 | 综合重要度I值聚合计算单元 |
| 所属分区 | 三、漏斗二：任务经验漏斗 / 三维重要度计算引擎 |
| 核心职责 | 作为三维重要度计算引擎的聚合中枢，接收来自 ag-mem-31（S值）、ag-mem-32（V值）、ag-mem-33（C值）、ag-mem-34（I₀值）分别计算的四个维度的分值，从 ag-mem-35 获取全局权重系数α、β、γ，执行聚合公式 I = I₀ + α·S + β·V + γ·C，输出最终的综合重要度评分I值。当新经验写入时，负责协调四个计算模块依次完成各自维度的计算，并在收集全部四个分值后执行聚合。同时响应来自 ag-mem-37（重要度增量定时刷新单元）的全量重算请求，对已有经验重新聚合I值。不参与各维度分值的计算逻辑，仅执行聚合加权与边界裁剪 |
| 依赖模块 | ag-mem-31（S值计算结果）、ag-mem-32（V值计算结果）、ag-mem-33（C值更新确认）、ag-mem-34（I₀赋值结果）、ag-mem-35（三维权重系数配置，获取α/β/γ及各分槽专属调整系数） |
| 被依赖模块 | ag-mem-15~19（各场景分槽，接收聚合后的I值用于写入L1）、ag-mem-37（重要度增量定时刷新单元，接收全量重算结果）、ag-mem-38（晋升双条件判定单元，消费I值）、ag-mem-40（遗忘阈值判定单元，消费I值） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 无聚合任务，等待各维度分值输入 | 系统初始化完成，无待处理经验 |
| 分值收集中 | `COLLECTING` | 已收到部分维度的分值，等待其余维度完成计算 | 新经验写入触发聚合流程，但四个维度尚未全部返回 |
| 聚合计算中 | `AGGREGATING` | 四个维度分值已全部收集，正在执行聚合计算 | 全部四个维度的分值均已到达 |
| 全量重算中 | `FULL_RECALC` | 正在对漏斗二中已有经验进行全量I值重算 | 收到 ag-mem-37 的全量刷新指令 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |

## I值聚合公式

### 一、标准聚合公式

```
I = I₀ + α·S + β·V + γ·C

其中:
- I₀: 基础重要度（0.05–1.00），由 ag-mem-34 赋值
- S:  安全显著性（0.00–1.00），由 ag-mem-31 计算
- V:  用户价值（0.00–1.00），由 ag-mem-32 计算，已包含分槽专属调整
- C:  复用频次（0.00–1.00），由 ag-mem-33 统计，已包含分槽饱和阈值调整
- α:  安全显著性权重（默认0.40），由 ag-mem-35 提供
- β:  用户价值权重（默认0.30），由 ag-mem-35 提供
- γ:  复用频次权重（默认0.30），由 ag-mem-35 提供

约束: α + β + γ = 1.0（由 ag-mem-35 保证）
```

### 二、I值边界裁剪

```
IF I > 1.0:
    I = 1.0
IF I < 0.05:
    I = 0.05
```

### 三、I值分级

| I值区间 | 级别 | 说明 |
|:---:|:---:|------|
| 0.90–1.00 | 极高 | 核心经验，快速晋升至L4/L5 |
| 0.70–0.89 | 高 | 重要经验，晋升优先级高 |
| 0.40–0.69 | 中 | 普通经验，正常晋升节奏 |
| 0.15–0.39 | 低 | 低价值经验，容易触发遗忘 |
| 0.05–0.14 | 极低 | 即将被遗忘的经验 |


## 核心处理逻辑

```
FUNCTION i_value_aggregator_main_loop():
    STATE_IDLE = IDLE
    STATE_COLLECT = COLLECTING
    STATE_AGGREGATE = AGGREGATING
    STATE_FULL = FULL_RECALC
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_IDLE
    初始化分值收集表 = {}  // 条目ID → {I₀, S, V, C, 来源分槽, 收集状态}
    初始化聚合统计计数器
    加载权重系数配置（从 ag-mem-35 获取 α, β, γ 及分槽调整表）

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            // 清空未完成的收集表
            清空分值收集表
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_IDLE

        // 第2步：接收新经验聚合触发（启动四维度计算协调）
        IF 收到来自 ag-mem-15~19 的新经验聚合触发:
            条目ID = 触发.条目ID
            来源分槽 = 触发.来源分槽编号

            // 2a. 初始化分值收集记录
            分值收集表[条目ID] = {
                I₀: None, S: None, V: None, C: None,
                来源分槽: 来源分槽,
                收集状态: {"I₀": False, "S": False, "V": False, "C": False},
                触发时间: NOW()
            }
            SET internal_state = STATE_COLLECT

            // 2b. 向四个计算模块依次发起计算请求
            // 注意：I₀和S可并行，V需要用户行为数据（可能有延迟），C初始为0（新经验无复用记录）
            向 ag-mem-34 发送I₀赋值请求(条目ID, 触发.经验元数据)
            向 ag-mem-31 发送S值计算请求(条目ID, 触发.经验元数据)
            向 ag-mem-32 发送V值计算请求(条目ID, 触发.经验元数据)
            // C值：新经验初始C=0，由 ag-mem-33 自动初始化
            分值收集表[条目ID].C = 0.0
            分值收集表[条目ID].收集状态["C"] = True

        // 第3步：收集各维度分值
        IF 收到 ag-mem-34 I₀赋值结果:
            条目ID = 结果.条目ID
            IF 条目ID IN 分值收集表:
                分值收集表[条目ID].I₀ = 结果.I₀值
                分值收集表[条目ID].收集状态["I₀"] = True
                检查是否全部收集完成(条目ID)

        IF 收到 ag-mem-31 S值计算结果:
            条目ID = 结果.条目ID
            IF 条目ID IN 分值收集表:
                分值收集表[条目ID].S = 结果.S值
                分值收集表[条目ID].收集状态["S"] = True
                检查是否全部收集完成(条目ID)

        IF 收到 ag-mem-32 V值计算结果:
            条目ID = 结果.条目ID
            IF 条目ID IN 分值收集表:
                分值收集表[条目ID].V = 结果.V值
                分值收集表[条目ID].收集状态["V"] = True
                检查是否全部收集完成(条目ID)

        // 第4步：全部收集完成，执行聚合计算
        IF 检查全部收集完成(条目ID):
            SET internal_state = STATE_AGGREGATE
            记录 = 分值收集表[条目ID]

            // 4a. 获取该分槽的专属权重调整系数
            分槽配置 = 权重系数配置表[记录.来源分槽]
            IF 分槽配置 == None:
                分槽配置 = {α调整: 1.0, β调整: 1.0, γ调整: 1.0}

            // 4b. 应用分槽专属调整后的权重
            有效α = α × 分槽配置.α调整
            有效β = β × 分槽配置.β调整
            有效γ = γ × 分槽配置.γ调整

            // 4c. 执行聚合公式
            I值 = 记录.I₀ + 有效α × 记录.S + 有效β × 记录.V + 有效γ × 记录.C

            // 4d. 边界裁剪
            IF I值 > 1.0:
                I值 = 1.0
            IF I值 < 0.05:
                I值 = 0.05

            // 4e. 构建聚合结果
            聚合结果 = {
                条目ID: 条目ID,
                I值: ROUND(I值, 3),
                I₀值: 记录.I₀,
                S值: 记录.S,
                V值: 记录.V,
                C值: 记录.C,
                各维度加权贡献: {
                    I₀贡献: ROUND(记录.I₀, 3),
                    S贡献: ROUND(有效α × 记录.S, 3),
                    V贡献: ROUND(有效β × 记录.V, 3),
                    C贡献: ROUND(有效γ × 记录.C, 3)
                },
                聚合时间戳: NOW()
            }

            // 4f. 发送至请求来源分槽
            向 记录.来源分槽 发送综合重要度I值(聚合结果)

            // 4g. 清理收集表
            从分值收集表中移除条目ID
            更新聚合统计计数器
            SET internal_state = STATE_IDLE

        // 第5步：超时检测
        FOR EACH 条目ID, 记录 IN 分值收集表:
            IF NOW() - 记录.触发时间 > 2秒:
                未完成维度 = []
                FOR EACH 维度, 状态 IN 记录.收集状态:
                    IF NOT 状态:
                        未完成维度.append(维度)

                // 使用默认值补齐缺失维度
                IF "I₀" IN 未完成维度: 记录.I₀ = 0.30
                IF "S" IN 未完成维度: 记录.S = 0.10
                IF "V" IN 未完成维度: 记录.V = 0.20

                // 标记所有维度为完成，强制执行聚合
                FOR EACH 维度 IN 未完成维度:
                    记录.收集状态[维度] = True

                向 ag-mem-03 发送聚合超时告警(条目ID, 未完成维度, 2秒)

                // 强制执行聚合
                检查是否全部收集完成(条目ID)

        // 第6步：全量重算
        IF 收到 ag-mem-37 全量刷新指令:
            SET internal_state = STATE_FULL

            // 6a. 获取需要重算的条目列表
            全量条目列表 = 向 ag-mem-03 查询漏斗二全量经验条目列表()
            重算结果列表 = []

            FOR EACH 条目 IN 全量条目列表:
                // 查询各维度的当前值
                当前I₀ = 条目.I₀值
                当前S = 条目.S值
                当前V = 条目.V值
                当前C = 条目.C值

                // 重新聚合
                有效α = α × 获取分槽α调整(条目.来源分槽)
                有效β = β × 获取分槽β调整(条目.来源分槽)
                有效γ = γ × 获取分槽γ调整(条目.来源分槽)

                新I值 = 当前I₀ + 有效α × 当前S + 有效β × 当前V + 有效γ × 当前C
                新I值 = CLAMP(新I值, 0.05, 1.0)

                重算结果列表.append({
                    条目ID: 条目.条目ID,
                    新I值: ROUND(新I值, 3)
                })

            // 6b. 发送全量重算结果
            向 ag-mem-37 返回全量重算完成回执(重算结果列表, 重算耗时)
            SET internal_state = STATE_IDLE

        // 第7步：周期性状态上报
        IF 距上次状态上报 >= 120秒:
            向 ag-mem-03 上报聚合状态(
                当前状态=internal_state,
                最近聚合次数=聚合统计.总次数,
                平均I值=聚合统计.I值均值,
                聚合耗时均值=聚合统计.耗时均值
            )
            重置统计计数器

        SLEEP 10ms


FUNCTION 检查是否全部收集完成(条目ID):
    IF 条目ID NOT IN 分值收集表:
        RETURN False
    记录 = 分值收集表[条目ID]
    RETURN 记录.收集状态["I₀"] AND 记录.收集状态["S"] AND 记录.收集状态["V"] AND 记录.收集状态["C"]
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 某维度分值计算超时（>2秒） | 使用默认值补齐缺失维度（I₀=0.30, S=0.10, V=0.20），发送超时告警 | 对应计算模块恢复正常 |
| 权重系数α+β+γ≠1.0（配置异常） | 拒绝聚合，上报告警，等待配置修复 | ag-mem-35 修复配置 |
| 分槽配置缺失（未知分槽编号） | 使用全局默认权重系数（α=0.40, β=0.30, γ=0.30） | 分槽配置补全 |
| 全量重算时条目列表查询失败 | 返回部分重算结果，标记“重算未完成” | 下次刷新周期继续 |
| 聚合结果I值超出[0.05, 1.0] | 强制裁剪至边界，记录异常日志 | — |
| 同一经验条目重复触发聚合 | 以最新触发的为准，前一次触发未完成则覆盖 | — |
| 紧急熔断 | 清空分值收集表，暂停所有聚合操作 | 紧急解除后恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| A-01 | 聚合计算必须严格遵循 I = I₀ + α·S + β·V + γ·C 公式，不得增加额外维度或修改公式结构 |
| A-02 | 权重系数α+β+γ=1.0为硬约束，聚合前必须校验，不满足时拒绝执行并上报告警 |
| A-03 | 超时补齐使用的默认值（I₀=0.30, S=0.10, V=0.20）为保守值，偏向低估而非高估 |
| A-04 | I值边界[0.05, 1.0]为硬约束，不得输出超出边界的重要度值 |
| A-05 | 全量重算时不得修改条目的各维度原始分值（I₀/S/V/C），仅重新执行聚合计算 |
