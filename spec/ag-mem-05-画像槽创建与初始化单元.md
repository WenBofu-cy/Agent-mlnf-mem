# ag-mem-05-画像槽创建与初始化单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-05 |
| 模块名称 | 画像槽创建与初始化单元 |
| 所属分区 | 二、漏斗一：用户画像漏斗 |
| 核心职责 | 接收 ag-mem-02 下发的新槽创建指令，在漏斗一存储分区中为新用户分配独立的画像槽存储空间。初始化槽内数据结构（偏好维度统计基线、行为累计计数器、活跃时间戳等），配置槽位类型（长期/临时/访客）对应的生命周期参数。创建完成后返回槽位编号与存储分区指针至 ag-mem-02。不参与任何认知决策，仅执行画像槽的物理创建与初始化 |
| 依赖模块 | ag-mem-02（漏斗一专属调度单元，下发创建指令）、ag-mem-06（画像槽数据隔离管控单元，为新槽配置隔离策略）、ag-mem-48（全局容量配额管控单元，申请存储配额） |
| 被依赖模块 | ag-mem-02（接收创建完成回执）、ag-mem-07（用户行为观测记录单元，获取新槽位号用于行为写入） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 无创建任务，等待指令 | 系统初始化完成，无待处理请求 |
| 配额申请 | `QUOTA_REQUEST` | 正在向 ag-mem-48 申请存储配额 | 收到新槽创建指令 |
| 存储分配 | `STORAGE_ALLOC` | 正在物理分配存储分区并初始化数据结构 | 配额申请通过 |
| 创建完成 | `CREATED` | 新槽创建成功，等待返回确认 | 存储分配与初始化完成 |
| 创建失败 | `CREATE_FAILED` | 配额不足或存储分配失败 | 配额申请被拒或存储写入异常 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 新槽创建指令 | Struct（用户ID + 槽位类型 + 初始存储配额 + 关联会话ID） | ag-mem-02 漏斗一专属调度单元（通过内部调度总线） | 调度单元判定需要为新用户创建画像槽时 | **高** |
| 存储配额审批结果 | Struct（是否批准 + 分配配额 + 剩余可用空间） | ag-mem-48 全局容量配额管控单元 | 配额申请后返回 | **高** |
| 隔离策略配置参数 | Struct（槽位编号 + 访问控制列表 + 加密密钥） | ag-mem-06 画像槽数据隔离管控单元 | 新槽创建时 | **高** |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-mem-01 总控漏斗F0 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 槽位创建完成回执 | Struct（槽位编号 + 存储分区指针 + 槽位类型 + 初始化状态 + 创建耗时） | ag-mem-02 漏斗一专属调度单元 | 创建成功时 | **高** |
| 创建失败通知 | Struct（失败原因 + 用户ID + 建议操作） | ag-mem-02 漏斗一专属调度单元 | 创建失败时 | **高** |
| 隔离策略申请 | Struct（槽位编号 + 槽位类型 + 用户ID） | ag-mem-06 画像槽数据隔离管控单元 | 存储分配成功后 | **高** |
| 创建事件日志 | Struct（事件类型 + 用户ID + 槽位编号 + 时间戳） | ag-mem-51 记忆变更日志追溯单元 | 每次创建完成或失败时 | 普通 |


## 画像槽初始化参数

### 一、不同槽位类型的默认配置

| 参数 | 长期槽 | 临时槽 | 访客槽 |
|------|:---:|:---:|:---:|
| 初始存储配额 | 5MB | 2MB | 500KB |
| 最大存储配额 | 20MB | 5MB | 1MB |
| 生命周期 | 永久（手动删除） | 7天 | 当前会话 |
| 偏好维度数量 | 20个 | 10个 | 5个 |
| 行为统计精度 | 精确（按日/周/月） | 简化（按周） | 不统计 |
| 是否启用加密 | ✅ | ✅ | ✅ |
| 是否参与长期分析 | ✅ | ❌ | ❌ |

### 二、初始化数据结构清单

| 数据结构 | 初始值 | 说明 |
|----------|------|------|
| 用户ID | 指令传入 | 唯一用户标识 |
| 槽位编号 | 自动生成 | 格式：SLOT-{类型}-{序号} |
| 创建时间戳 | 当前UTC时间 | 用于生命周期计算 |
| 最后活跃时间 | 当前UTC时间 | 用于未活跃检测 |
| 偏好维度统计表 | 空字典 | 维度名→{显式偏好计数, 隐式倾向计数, 偶发行为计数} |
| 行为累计计数器 | 全部置零 | 按行为类型分类计数 |
| 会话历史列表 | 空列表 | 记录每次会话的时间与摘要 |
| 显式反馈记录 | 空列表 | 用户明确点赞/踩/收藏的操作 |
| 加密密钥 | 随机生成 | 槽级独立密钥 |
| 访问控制令牌 | 新生成 | 仅返回给 ag-mem-02 |


## 核心处理逻辑

```
FUNCTION slot_creation_main_loop():
    STATE_IDLE = IDLE
    STATE_QUOTA = QUOTA_REQUEST
    STATE_ALLOC = STORAGE_ALLOC
    STATE_DONE = CREATED
    STATE_FAIL = CREATE_FAILED
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_IDLE
    初始化槽位计数器（按类型分别计数）

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_IDLE

        // 第2步：接收新槽创建指令
        IF 收到 ag-mem-02 新槽创建指令:
            SET internal_state = STATE_QUOTA
            用户ID = 指令.用户ID
            槽位类型 = 指令.槽位类型
            请求配额 = 指令.初始存储配额

            // 2a. 校验槽位类型合法性
            IF 槽位类型 NOT IN ["长期槽", "临时槽", "访客槽"]:
                SET internal_state = STATE_FAIL
                向 ag-mem-02 返回创建失败通知("非法槽位类型")
                SET internal_state = STATE_IDLE
                CONTINUE

            // 2b. 获取槽位类型默认配置
            默认配置 = 获取槽位默认配置(槽位类型)
            IF 请求配额 == 0:
                请求配额 = 默认配置.初始存储配额
            IF 请求配额 > 默认配置.最大存储配额:
                请求配额 = 默认配置.最大存储配额

            // 2c. 申请存储配额
            向 ag-mem-48 发送配额申请(漏斗一, 请求配额)
            // 等待审批结果

        // 第3步：处理配额审批结果
        IF 收到 ag-mem-48 配额审批结果:
            IF NOT 审批结果.是否批准:
                SET internal_state = STATE_FAIL
                向 ag-mem-02 返回创建失败通知("存储配额不足")
                SET internal_state = STATE_IDLE
                CONTINUE

            // 配额批准，进入存储分配阶段
            SET internal_state = STATE_ALLOC
            分配配额 = 审批结果.分配配额

            // 3a. 生成槽位编号
            槽位编号 = 生成槽位编号(槽位类型)

            // 3b. 物理分配存储分区
            存储指针 = 漏斗一存储管理器.分配分区(槽位编号, 分配配额)
            IF 存储指针 == None:
                SET internal_state = STATE_FAIL
                向 ag-mem-02 返回创建失败通知("存储分配失败")
                向 ag-mem-48 退还已申请配额(分配配额)
                SET internal_state = STATE_IDLE
                CONTINUE

            // 3c. 初始化数据结构
            初始化偏好维度统计表(槽位编号, 默认配置.偏好维度数量)
            初始化行为累计计数器(槽位编号)
            初始化会话历史列表(槽位编号)
            生成槽级加密密钥
            设置生命周期参数(默认配置.生命周期)

            // 3d. 申请隔离策略
            向 ag-mem-06 发送隔离策略申请(槽位编号, 槽位类型, 用户ID)
            // 等待隔离策略返回

        // 第4步：接收隔离策略配置
        IF 收到 ag-mem-06 隔离策略配置参数:
            将访问控制列表与加密密钥绑定到槽位
            设置槽位访问权限（仅允许 ag-mem-02 和 ag-mem-07 写入，ag-mem-01 可读）

            // 第5步：创建完成
            SET internal_state = STATE_DONE
            递增槽位计数器(槽位类型)
            创建回执 = 构建完成回执(槽位编号, 存储指针, 槽位类型, "初始化成功")
            向 ag-mem-02 发送槽位创建完成回执

            // 记录创建事件
            向 ag-mem-51 发送创建事件日志(用户ID, 槽位编号, 槽位类型)

            SET internal_state = STATE_IDLE

        SLEEP 20ms


FUNCTION 生成槽位编号(槽位类型):
    序号 = 槽位计数器(槽位类型) + 1
    SWITCH 槽位类型:
        CASE "长期槽":
            RETURN "SLOT-LONG-" + 序号
        CASE "临时槽":
            RETURN "SLOT-TEMP-" + 序号
        CASE "访客槽":
            RETURN "SLOT-GUEST-" + 序号
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 存储配额不足 | 拒绝创建，返回失败通知，提示清理旧数据或扩容 | 存储空间释放 |
| 存储分区分配失败（写入错误） | 退还已申请配额，返回失败通知 | 存储介质恢复正常 |
| 隔离策略申请超时 | 使用默认隔离策略（仅允许创建者访问），标记“隔离策略默认” | ag-mem-06 恢复正常 |
| 同一用户ID重复创建 | 返回已存在的槽位编号（幂等操作），不重复分配存储 | — |
| 槽位计数器溢出 | 上报告警，暂停创建同类型槽位 | 管理员介入处理 |
| 紧急熔断 | 中断当前创建流程，已分配的存储回滚（释放配额） | 紧急解除后恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 每个画像槽独立分配加密密钥，禁止跨槽复用密钥 |
| S-02 | 槽位创建时必须同步申请隔离策略，禁止在隔离策略生效前接收任何数据写入 |
| S-03 | 创建失败时必须完整回滚已分配的存储配额，不得残留无效分区占用空间 |
| S-04 | 访客槽的生命周期参数硬编码为“当前会话”，不得被任何模块修改为永久保留 |
| S-05 | 槽位编号生成后不可更改，该编号作为全系统唯一标识，用于所有后续操作关联 |
