# ag-mem-24-L3中期层存储单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-24 |
| 模块名称 | L3中期层存储单元 |
| 所属分区 | 三、漏斗二：任务经验漏斗 / 五层存储 |
| 核心职责 | 作为漏斗二五层记忆存储架构的第三层，专门存储近30日内从L2晋升而来的稳定有效的任务策略经验。L3是经验从“近期有效”进入“中期稳定”的关键层，占漏斗二总容量的10%。对写入的失败经验强制标记警示标签并锁定在L3层，禁止其晋升至L4，但仍保留供查询参考。当同一场景连续三次无警示安全通过后，警示标签自动降级为普通经验。管理L3层容量与条目生命周期，定期触发相似经验归并，为L4长期层输送泛化能力。不参与晋升判定或遗忘决策，仅执行经验的接收、存储、警示标签管理与基础管理 |
| 依赖模块 | ag-mem-22（L2近期层存储单元，发送晋升条目列表）、ag-mem-25（L3中期层相似经验归并单元，执行归并操作）、ag-mem-26（L4长期层存储单元，接收晋升条目）、ag-mem-40（遗忘阈值判定单元，执行遗忘扫描）、ag-mem-48（全局容量配额管控单元，查询L3当前容量） |
| 被依赖模块 | ag-mem-22（返回写入确认）、ag-mem-25（接收归并触发指令）、ag-mem-15~19（各场景分槽，通过查询接口检索L3经验） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常服务 | `NORMAL` | L3存储正常，接收晋升条目 | 系统初始化完成，存储分区就绪 |
| 容量预警 | `CAPACITY_WARNING` | L3存储使用率超过80%，触发温和清理 | L3使用率 ≥ 80% |
| 容量紧急 | `CAPACITY_CRITICAL` | L3存储使用率超过95%，暂停写入并强制清理 | L3使用率 ≥ 95% |
| 维护整理 | `MAINTENANCE` | 正在执行归并或遗忘扫描 | 定时触发或收到维护指令 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |

## L3存储配置

| 配置项 | 默认值 | 说明 |
|--------|:---:|------|
| L3占漏斗二总容量比例 | **10%** | L3是中等容量存储层，承载稳定策略经验 |
| L3最大条目数 | 2000条 | 硬上限，达到后强制触发清理 |
| L3条目最大留存时长 | **30天**（720小时） | 超过此时长的条目必须晋升至L4或清除 |
| L3单条目最大大小 | 20KB | 允许更丰富的经验上下文数据 |
| 写入超时 | 300ms | 单次写入操作的最大等待时间 |
| 容量预警阈值 | 80% | 触发温和清理 |
| 容量紧急阈值 | 95% | 触发强制清理并暂停写入 |
| 警示标签降级条件 | 同一场景连续3次无警示安全通过 | 警示经验自动降级为普通经验 |


## 失败经验警示标签管理

### 一、警示标签规则

| 条件 | 处理方式 |
|------|------|
| 条目结果标签为“失败”或“策略失误” | 自动标记警示标签 `CAUTION`，锁定在L3层，禁止晋升至L4 |
| 警示条目仍可被L3查询接口检索 | 返回时标记 `caution: true`，供ECC决策时参考（不作为自动执行依据） |
| 同一场景（相同分槽+相似任务特征）连续3次无警示安全通过 | `CAUTION` 标签自动降级为普通标签 `NORMAL`，解除L4晋升限制 |
| 警示条目在L3留存超过30天且从未触发降级 | 保留警示标签但允许清除（按遗忘策略处理），不强制晋升 |

### 二、警示标签状态机

```
NORMAL ──（遇到失败经验）──→ CAUTION
CAUTION ──（连续3次同场景安全通过）──→ NORMAL
CAUTION ──（留存超30天且触发遗忘）──→ 清除
```


## 核心处理逻辑

```
FUNCTION l3_storage_main_loop():
    STATE_NORMAL = NORMAL
    STATE_WARN = CAPACITY_WARNING
    STATE_CRITICAL = CAPACITY_CRITICAL
    STATE_MAINT = MAINTENANCE
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_NORMAL
    初始化L3存储分区（内存索引 + 持久化存储）
    初始化条目计数器 = 0
    初始化警示条目跟踪表 = {}  // 条目ID → {场景签名, 连续安全次数}

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_NORMAL

        // 第2步：接收L2晋升条目列表
        IF 收到 ag-mem-22 L3晋升条目列表:
            IF internal_state == STATE_PAUSED:
                向 ag-mem-22 返回L3写入确认回执(0, 0, 0, "系统熔断中")
                CONTINUE

            // 2a. 查询当前L3容量
            容量回执 = 向 ag-mem-48 查询L3当前容量()
            L3使用率 = 容量回执.当前使用率

            // 2b. 容量状态更新
            IF L3使用率 >= 95%:
                SET internal_state = STATE_CRITICAL
            ELSE IF L3使用率 >= 80%:
                SET internal_state = STATE_WARN

            // 2c. 容量紧急处理
            IF internal_state == STATE_CRITICAL:
                IF 条目计数器 >= L3最大条目数:
                    向 ag-mem-40 发送遗忘扫描触发指令(L3中重要度最低的20%条目, "容量紧急")
                    等待清理结果
                    IF 清理后使用率仍 >= 95%:
                        向 ag-mem-22 返回L3写入确认回执(接收数, 0, 0, "L3容量已满")
                        CONTINUE

            // 2d. 逐条写入L3
            成功写入数 = 0
            警示标记数 = 0
            FOR EACH 晋升条目 IN 晋升条目列表:
                // 校验条目大小
                IF 晋升条目.经验数据.大小 > 20KB:
                    CONTINUE

                // 失败经验警示标签判定
                警示标签 = "NORMAL"
                IF 晋升条目.结果标签 IN ["失败", "策略失误"]:
                    警示标签 = "CAUTION"
                    警示标记数++
                    // 初始化警示跟踪
                    场景签名 = 生成场景签名(晋升条目.来源分槽编号, 晋升条目.经验数据.任务特征)
                    警示条目跟踪表[晋升条目.条目ID] = {
                        场景签名: 场景签名,
                        连续安全次数: 0
                    }

                // 写入L3存储
                存储位置 = L3存储分区.追加写入(
                    条目ID=晋升条目.条目ID,
                    经验数据=晋升条目.经验数据,
                    重要度I值=晋升条目.重要度I值,
                    来源分槽=晋升条目.来源分槽编号,
                    警示标签=警示标签,
                    写入时间=NOW()
                )
                IF 存储位置 != None:
                    成功写入数++
                    条目计数器++

            // 2e. 返回确认
            向 ag-mem-22 返回L3写入确认回执(
                接收条目数=晋升条目列表.size,
                成功写入数=成功写入数,
                警示标记数=警示标记数,
                L3当前使用率=L3使用率
            )

        // 第3步：响应经验查询请求
        IF 收到来自 ag-mem-15~19 的L3经验查询请求:
            查询条件 = 请求.查询条件
            来源分槽 = 请求.来源分槽编号
            包含警示 = 请求.是否包含警示条目

            // 3a. 检索匹配条目
            匹配条目 = L3存储.查询(查询条件, 来源分槽, 请求.最大返回条数)
            
            // 3b. 标记警示条目
            FOR EACH 条目 IN 匹配条目:
                IF 条目.警示标签 == "CAUTION":
                    条目.caution_flag = True
                    IF NOT 包含警示:
                        从匹配条目中移除该条目  // 如果请求不包含警示，则过滤掉
                ELSE:
                    条目.caution_flag = False

            // 3c. 返回结果
            向来源分槽返回L3查询结果列表(匹配条目, 层级="L3")

        // 第4步：定时归并相似经验（每12小时）
        IF 距上次归并 >= 12小时 AND internal_state IN [NORMAL, WARN]:
            SET internal_state = STATE_MAINT
            向 ag-mem-25 发送归并触发指令(L3全量条目, "定时归并", "ag-mem-24")
            // 等待归并完成
            SET internal_state = STATE_NORMAL
            重置归并计时器

        // 第5步：定时检查超期条目（每小时）
        IF 距上次超期检查 >= 1小时:
            // 清除在L3留存超过30天的条目
            超期条目 = L3存储.筛选(留存时长 > 30天)
            IF 超期条目 非空:
                FOR EACH 条目 IN 超期条目:
                    IF 条目.警示标签 == "CAUTION":
                        // 警示条目超期：按遗忘策略处理，不强制晋升
                        向 ag-mem-40 发送遗忘评估请求(条目)
                    ELSE IF 条目.重要度I值 >= L3→L4晋升阈值:
                        向 ag-mem-26 发送晋升条目(条目)
                    ELSE:
                        向 ag-mem-40 发送遗忘评估请求(条目)
                条目计数器 -= 超期条目.size
            重置超期检查计时器

        // 第6步：警示标签降级检测（每次同场景安全通过时触发）
        IF 收到来自 ag-mem-15~19 的“场景安全通过”通知:
            场景签名 = 通知.场景签名
            // 遍历警示跟踪表，找到匹配场景的警示条目
            FOR EACH 条目ID, 跟踪信息 IN 警示条目跟踪表:
                IF 跟踪信息.场景签名 == 场景签名:
                    跟踪信息.连续安全次数++
                    IF 跟踪信息.连续安全次数 >= 3:
                        // 降级为普通经验
                        L3存储.更新警示标签(条目ID, "NORMAL")
                        从警示条目跟踪表中移除条目ID
                        记录日志("警示标签降级", 条目ID)

        // 第7步：周期性状态上报
        IF 距上次状态上报 >= 60秒:
            统计警示条目数 = 警示条目跟踪表.size
            向 ag-mem-48 和 ag-mem-03 上报L3状态(
                当前状态=internal_state,
                条目总数=条目计数器,
                使用率=L3使用率,
                警示条目数=统计警示条目数
            )

        SLEEP 10ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| L3存储写入异常 | 返回部分成功写入数，上报告警 | 存储恢复正常 |
| 条目大小超过20KB上限 | 跳过该条目，记录跳过日志 | — |
| L3使用率达95%且遗忘清理后仍无法释放空间 | 拒绝新写入，上报告警 | 清理成功或人工扩容 |
| 警示条目在L3留存超过30天 | 不强制晋升，按遗忘策略评估清除 | — |
| 归并期间收到写入请求 | 写入请求排队等待，归并完成后处理 | 归并完成 |
| 紧急熔断 | 暂停所有操作，保持L3数据完整性 | 紧急解除后恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 失败经验必须强制标记警示标签并锁定在L3层，禁止通过任何路径晋升至L4 |
| S-02 | 警示条目在查询时明确标注 `caution: true`，ECC认知大脑应避免将其作为自动决策的唯一依据 |
| S-03 | 警示标签降级必须严格满足“连续3次同场景安全通过”条件，不得手动或通过其他模块强制降级 |
| S-04 | L3层条目在留存超过30天后必须处理（晋升L4或清除），不得无限期滞留 |
