# ag-mem-33-复用频次C值统计单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-33 |
| 模块名称 | 复用频次C值统计单元 |
| 所属分区 | 三、漏斗二：任务经验漏斗 / 三维重要度计算引擎 |
| 核心职责 | 统计漏斗二中各经验条目在同类任务场景下被成功调用并执行的次数，将复用频次归一化为C值（0.0–1.0）。C值反映该经验的实用性与可复用程度：被频繁成功复用的经验获得高C值，加速晋升；从未被调用或调用失败的条目获得低C值，逐渐被遗忘。C值作为三维重要度I值的关键组成部分，是驱动经验从“偶发行为”进化为“稳定技能”的核心动力。通过ag-mem-37的重要度增量定时刷新机制，定期全量更新所有条目的C值。不参与认知决策，仅执行复用频次的客观统计与归一化 |
| 依赖模块 | ag-mem-23（L2近期层热度统计单元，提供L2层条目的命中频次数据）、ag-mem-15~19（各场景分槽，在经验被成功调用后推送复用事件）、ag-mem-36（综合重要度I值聚合计算单元，可提供上次评估的C值用于增量更新） |
| 被依赖模块 | ag-mem-36（综合重要度I值聚合计算单元，消费C值）、ag-mem-35（三维权重系数配置单元，提供C值权重系数γ）、ag-mem-37（重要度增量定时刷新单元，触发C值全量更新） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 无统计任务，等待事件或刷新指令 | 系统初始化完成，无待处理请求 |
| 增量更新中 | `INCREMENTAL_UPDATING` | 正在基于新收到的复用事件更新单条或多条经验的C值 | 收到各场景分槽的复用事件通知 |
| 全量刷新中 | `FULL_REFRESHING` | 正在执行全量经验的C值重新计算 | 收到ag-mem-37的定时刷新指令 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |

## C值计算规则

### 一、C值计算基本公式

```
C值 = 成功调用次数归一化值 × 时间衰减因子

其中:
- 成功调用次数归一化值 = MIN(成功调用总次数 / C值饱和阈值, 1.0)
- C值饱和阈值 = 10次（即成功复用10次后C值达到1.0）
- 时间衰减因子 = e^(-λ × 距上次成功调用的天数)
- λ = 0.01（衰减系数，约100天衰减至0.37）
```

### 二、调用结果处理

| 调用结果 | 对C值的影响 | 说明 |
|----------|:---:|------|
| 成功 | 成功调用计数器+1，时间衰减重置（距上次成功调用天数=0） | C值上升 |
| 失败 | 成功调用计数器不变，但距上次成功调用天数继续累积 | C值自然衰减 |
| 未被调用 | 成功调用计数器不变，时间衰减持续 | C值逐渐降低 |

### 三、C值与经验层级的关系

| 经验层级 | C值计算特点 | 说明 |
|----------|------|------|
| L1（临时层） | 初始C值=0.0，仅在L1滞留超过24小时后开始统计 | 避免短期爆发性调用虚增C值 |
| L2（近期层） | 直接使用L2命中频次数据计算，响应及时 | L2是高频经验聚集层 |
| L3（中期层） | 使用L3存储的累计调用次数，结合时间衰减 | 稳定策略层 |
| L4（长期层） | 使用L4存储的累计调用次数，衰减系数降低（λ=0.005） | 长期经验衰减更慢 |
| L5（核心层） | C值固定为1.0，不参与衰减 | 核心经验永久保持最高C值 |

### 四、各场景分槽的C值饱和阈值

| 来源分槽 | C值饱和阈值 | 说明 |
|----------|:---:|------|
| ag-mem-15（对话交互） | 8次 | 对话经验复用门槛略低 |
| ag-mem-16（工具调用） | 10次 | 标准阈值 |
| ag-mem-17（信息检索） | 12次 | 检索经验需要更高频次验证 |
| ag-mem-18（创作生成） | 10次 | 标准阈值 |
| ag-mem-19（通用任务） | 15次 | 通用经验需要更充分的复用验证 |


## 核心处理逻辑

```
FUNCTION c_value_statistics_main_loop():
    STATE_IDLE = IDLE
    STATE_INCREMENTAL = INCREMENTAL_UPDATING
    STATE_FULL = FULL_REFRESHING
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_IDLE
    初始化C值存储表 = {}  // 条目ID → {成功调用总数, 距上次成功调用天数, 当前C值, 来源分槽编号, 当前层级}
    加载C值计算规则配置

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_IDLE

        // 第2步：接收复用事件通知（增量更新）
        IF 收到来自 ag-mem-15~19 的复用事件通知:
            SET internal_state = STATE_INCREMENTAL
            条目ID = 通知.条目ID
            调用结果 = 通知.调用结果
            来源分槽 = 通知.调用场景分槽编号

            // 2a. 查询当前C值记录
            IF 条目ID NOT IN C值存储表:
                // 新条目，初始化
                分槽饱和阈值 = 获取分槽C值饱和阈值(来源分槽)
                C值存储表[条目ID] = {
                    成功调用总数: 0,
                    距上次成功调用天数: 0,
                    当前C值: 0.0,
                    来源分槽编号: 来源分槽,
                    当前层级: "L1",
                    饱和阈值: 分槽饱和阈值
                }

            记录 = C值存储表[条目ID]

            // 2b. 根据调用结果更新计数
            IF 调用结果 == "成功":
                记录.成功调用总数++
                记录.距上次成功调用天数 = 0
            ELSE:
                // 失败不增加计数，时间衰减继续
                记录.距上次成功调用天数 += 自上次更新以来的实际天数

            // 2c. 重新计算C值
            新C值 = 计算C值(记录.成功调用总数, 记录.距上次成功调用天数, 记录.饱和阈值, 记录.当前层级)
            记录.当前C值 = 新C值

            // 2d. 发送更新确认
            向 ag-mem-36 发送C值更新确认(
                更新条目数=1,
                各条目新C值={条目ID: 新C值}
            )

            SET internal_state = STATE_IDLE

        // 第3步：接收全量刷新指令
        IF 收到 ag-mem-37 C值全量刷新指令:
            SET internal_state = STATE_FULL

            // 3a. 获取所有需要刷新的条目列表
            全量条目列表 = 向 ag-mem-03 查询漏斗二全量经验条目列表()
            IF 全量条目列表为空:
                SET internal_state = STATE_IDLE
                CONTINUE

            // 3b. 批量获取L2命中频次数据（用于L2条目的C值计算）
            L2命中数据 = 向 ag-mem-23 查询全量L2命中频次()

            // 3c. 逐条重新计算C值
            更新计数 = 0
            更新结果 = {}
            FOR EACH 条目 IN 全量条目列表:
                条目ID = 条目.条目ID
                当前层级 = 条目.当前层级
                来源分槽 = 条目.来源分槽编号

                // 获取该条目的当前记录或初始化
                IF 条目ID NOT IN C值存储表:
                    分槽饱和阈值 = 获取分槽C值饱和阈值(来源分槽)
                    C值存储表[条目ID] = {
                        成功调用总数: 条目.累计成功调用次数 IF 条目有记录 ELSE 0,
                        距上次成功调用天数: 计算距上次成功调用天数(条目),
                        当前C值: 条目.当前C值 IF 条目有记录 ELSE 0.0,
                        来源分槽编号: 来源分槽,
                        当前层级: 当前层级,
                        饱和阈值: 分槽饱和阈值
                    }

                记录 = C值存储表[条目ID]

                // L2层条目额外使用热度统计数据进行修正
                IF 当前层级 == "L2" AND 条目ID IN L2命中数据:
                    热度数据 = L2命中数据[条目ID]
                    // 热度数据补充到成功调用计数中
                    记录.成功调用总数 = MAX(记录.成功调用总数, 热度数据.近7日命中次数)

                // L5层条目C值固定为1.0
                IF 当前层级 == "L5":
                    记录.当前C值 = 1.0
                    记录.成功调用总数 = 10  // 保持饱和状态
                    记录.距上次成功调用天数 = 0
                ELSE:
                    // 更新时间衰减天数
                    记录.距上次成功调用天数 += 自上次全量刷新以来的天数
                    // 重新计算C值
                    新C值 = 计算C值(记录.成功调用总数, 记录.距上次成功调用天数, 记录.饱和阈值, 当前层级)
                    记录.当前C值 = 新C值

                // 检测异常波动
                IF ABS(新C值 - 条目.上次C值) > 0.5 AND 条目.上次C值 > 0:
                    向 ag-mem-03 发送C值异常告警(条目ID, "C值异常波动", 新C值, "变化幅度>0.5")

                更新结果[条目ID] = 记录.当前C值
                更新计数++

            // 3d. 发送全量刷新确认
            向 ag-mem-36 发送C值更新确认(
                更新条目数=更新计数,
                各条目新C值=更新结果,
                更新耗时=NOW() - 刷新开始时间
            )
            向 ag-mem-37 发送刷新完成确认

            SET internal_state = STATE_IDLE

        // 第4步：周期性状态上报
        IF 距上次状态上报 >= 120秒:
            全量条目数 = C值存储表.size
            平均C值 = MEAN(所有记录的当前C值) IF 全量条目数 > 0 ELSE 0.0
            向 ag-mem-03 上报计算状态(
                当前状态=internal_state,
                全量条目数=全量条目数,
                平均C值=ROUND(平均C值, 2),
                最近一次刷新时间=最近刷新时间
            )

        SLEEP 10ms


FUNCTION 计算C值(成功调用总数, 距上次成功调用天数, 饱和阈值, 当前层级):
    // 归一化调用次数
    IF 成功调用总数 >= 饱和阈值:
        调用次数归一化 = 1.0
    ELSE:
        调用次数归一化 = 成功调用总数 / 饱和阈值

    // 计算时间衰减因子
    IF 当前层级 == "L4":
        衰减系数 = 0.005
    ELSE:
        衰减系数 = 0.01

    时间衰减因子 = EXP(-衰减系数 × 距上次成功调用天数)

    // C值 = 调用次数归一化 × 时间衰减
    C值 = 调用次数归一化 × 时间衰减因子

    RETURN ROUND(C值, 2)
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| C值存储表中条目ID不存在（首次写入） | 自动初始化记录，成功调用数=0，C值=0.0 | — |
| 全量刷新时条目列表查询超时 | 分批查询，每批最多500条，返回部分结果并标记“刷新未完成” | 下次刷新周期继续 |
| C值计算结果超出[0.0, 1.0] | 强制裁剪至[0.0, 1.0]，记录异常日志 | — |
| L2命中频次数据不可用 | 跳过L2热度修正，仅使用累计调用次数计算 | 数据恢复后下次刷新时补充 |
| 同一经验条目短时间内重复上报复用事件 | 去重：同一经验ID+同一调用场景在300秒内的重复事件合并为一次 | — |
| 紧急熔断 | 暂停所有统计操作，保留当前C值存储表 | 紧急解除后恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| C-01 | C值计算仅基于调用频次与时间衰减，不得访问或修改经验条目的内容数据 |
| C-02 | L5核心层条目的C值固定为1.0，编译期硬编码，不得通过任何接口修改 |
| C-03 | C值异常波动告警阈值（单次变化>0.5）为保护机制，不得用于自动化决策 |
| C-04 | 复用事件的去重窗口（300秒）为可配置参数，但不得设为0（禁用去重会导致统计失真） |
