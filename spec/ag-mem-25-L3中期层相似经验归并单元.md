# ag-mem-25-L3中期层相似经验归并单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-25 |
| 模块名称 | L3中期层相似经验归并单元 |
| 所属分区 | 三、漏斗二：任务经验漏斗 / 五层存储 |
| 核心职责 | 定期扫描L3中期层中来自同一场景分槽的经验条目，检测并合并场景特征高度相似的重复经验。归并操作将多条相似经验整合为一条更精炼的通用经验条目，原条目的重要度I值取归并集中最大值，复用频次C值累加，释放冗余条目的存储空间。通过归并减少L3层经验碎片化，提升检索效率，为L4长期层提供更高质量的经验候选。不参与晋升决策或遗忘判定，仅执行相似检测与归并整合 |
| 依赖模块 | ag-mem-24（L3中期层存储单元，提供待归并条目列表与归并触发指令）、ag-mem-48（全局容量配额管控单元，确认归并后释放的存储空间） |
| 被依赖模块 | ag-mem-24（返回归并完成回执与释放空间量）、ag-mem-42（冗余记忆删除与归档单元，接收归并后被淘汰的原始条目用于清除或归档） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 无归并任务，等待指令 | 系统初始化完成，无待处理请求 |
| 相似检测 | `SIMILARITY_CHECK` | 正在遍历L3条目进行特征提取与相似度计算 | 收到 ag-mem-24 的归并触发指令 |
| 归并执行 | `MERGING` | 正在将检测到的相似条目组合并为新条目 | 相似检测完成，存在满足归并阈值的条目组 |
| 结果输出 | `OUTPUTTING` | 归并完成，正在向目标模块输出结果 | 归并执行完成 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 归并触发指令 | Struct（归并范围 + 归并策略 + 触发原因：定时/容量预警 + 来源槽位编号） | ag-mem-24 L3中期层存储单元（通过内部调度总线） | 定时（每12小时）或容量预警时 | **高** |

| L3存储容量确认 | Struct（当前使用率 + 可用空间） | ag-mem-48 全局容量配额管控单元 | 归并执行前查询 | 普通 |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-mem-01 总控漏斗F0 | 模式切换或紧急事件时 | **紧急** |

|--------|----------|----------|----------|:---:|
| 归并完成回执 | Struct（扫描条目总数 + 检测到的相似组数 + 归并后新条目数 + 清除原始条目数 + 释放空间量 + 归并耗时） | ag-mem-24 L3中期层存储单元 | 归并操作完成时 | **高** |
| 归并后新条目列表 | List（新条目ID + 归并来源条目ID列表 + 整合后经验数据 + 整合后I值 + 累加后C值 + 来源分槽编号） | ag-mem-24（写入新条目）、ag-mem-42（归档被淘汰的原始条目） | 归并执行完成时 | **高** |
| 清除条目列表 | List（被归并淘汰的原始条目ID列表 + 清除原因） | ag-mem-42 冗余记忆删除与归档单元 | 归并执行完成时 | **高** |

## 相似度判定规则

### 一、相似度计算维度

| 维度 | 权重 | 说明 |
|------|:---:|------|
| 场景类别匹配 | **必须完全一致** | 不同场景分槽的经验不参与跨槽归并 |
| 任务特征向量余弦相似度 | 0.40 | 基于任务描述提取的关键词与意图向量 |
| 工具调用序列编辑距离 | 0.30 | 工具名称序列的Levenshtein距离归一化 |
| 经验结果标签一致性 | 0.20 | 成功/失败/部分成功等标签是否一致 |
| 时间窗口接近度 | 0.10 | 写入时间相近的经验更可能属于同一模式（30天内） |

### 二、归并阈值

| 阈值名称 | 默认值 | 说明 |
|----------|:---:|------|
| 综合相似度归并阈值 | **≥ 0.80** | 综合相似度达到此值即触发归并 |
| 单组最大归并条目数 | **5条** | 一组归并最多合并5条相似经验 |
| 最小归并条目数 | **2条** | 至少2条相似经验才触发归并 |
| 警示条目归并限制 | **禁止归并** | 标记为CAUTION的警示条目不参与归并，保留原始数据供分析 |


## 归并策略

### 一、数据整合规则

| 字段 | 整合方式 |
|------|------|
| 经验数据 | 保留归并组中I值最高的条目作为主条目，补充其他条目的差异化特征作为附加标签 |
| 重要度I值 | 取归并组中最大值 |
| 复用频次C值 | 归并组所有条目C值累加（上限1.0） |
| 安全显著性S值 | 取归并组中最大值 |
| 用户价值V值 | 取归并组中最大值 |
| 来源分槽编号 | 保持不变 |
| 写入时间戳 | 使用当前归并时间 |
| 归并来源记录 | 新增字段记录所有被归并的原始条目ID列表 |

### 二、归并后处理

1. 新条目写入L3，替代归并组中的所有原始条目
2. 原始条目从L3索引中移除
3. 原始条目数据发送至 ag-mem-42 进行归档（L3级归档，可追溯）或直接删除
4. 释放的存储空间量上报 ag-mem-48


## 核心处理逻辑

```
FUNCTION l3_similarity_merge_main_loop():
    STATE_IDLE = IDLE
    STATE_CHECK = SIMILARITY_CHECK
    STATE_MERGE = MERGING
    STATE_OUTPUT = OUTPUTTING
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_IDLE
    初始化归并统计计数器

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_IDLE

        // 第2步：接收归并触发指令
        IF 收到 ag-mem-24 归并触发指令:
            SET internal_state = STATE_CHECK
            待归并条目列表 = 指令.待归并条目列表
            触发原因 = 指令.触发原因

            IF 待归并条目列表.size < 2:
                // 条目数不足，无需归并
                向 ag-mem-24 返回归并完成回执(0, 0, 0, 0, 0, "条目数不足")
                SET internal_state = STATE_IDLE
                CONTINUE

            // 2a. 按来源分槽分组
            分槽分组 = 按来源分槽编号分组(待归并条目列表)

            // 2b. 在各分槽内部进行相似度检测
            所有相似组列表 = []
            FOR EACH 分槽, 条目列表 IN 分槽分组:
                相似组列表 = 检测分槽内相似组(条目列表)
                所有相似组列表.合并(相似组列表)

            IF 所有相似组列表为空:
                向 ag-mem-24 返回归并完成回执(待归并条目列表.size, 0, 0, 0, 0, "未检测到相似经验")
                SET internal_state = STATE_IDLE
                CONTINUE

            // 第3步：执行归并
            SET internal_state = STATE_MERGE
            归并后新条目列表 = []
            清除条目列表 = []
            总释放空间 = 0

            FOR EACH 相似组 IN 所有相似组列表:
                // 3a. 检查是否有警示条目
                IF 相似组中存在警示标签为CAUTION的条目:
                    跳过该组，不参与归并
                    CONTINUE

                // 3b. 选择主条目（I值最高）
                主条目 = 按I值降序排列(相似组)[0]
                辅助条目列表 = 相似组 - 主条目

                // 3c. 整合经验数据
                整合经验 = 主条目.经验数据.clone()
                FOR EACH 辅助条目 IN 辅助条目列表:
                    整合经验.附加标签.合并(辅助条目.经验数据.差异化特征)

                // 3d. 整合数值字段
                整合I值 = 相似组中最大I值
                整合C值 = MIN(相似组所有C值之和, 1.0)
                整合S值 = 相似组中最大S值
                整合V值 = 相似组中最大V值

                // 3e. 生成新条目
                新条目 = {
                    条目ID: "L3-MERGED-" + 生成UUID(),
                    来源分槽编号: 主条目.来源分槽编号,
                    经验数据: 整合经验,
                    I值: 整合I值,
                    C值: 整合C值,
                    S值: 整合S值,
                    V值: 整合V值,
                    警示标签: "NORMAL",
                    归并来源: 相似组中所有条目的ID列表,
                    写入时间戳: NOW()
                }
                归并后新条目列表.append(新条目)

                // 3f. 记录待清除的原始条目
                清除条目列表.合并(相似组中所有条目ID)

                // 3g. 估算释放空间
                释放空间 = SUM(相似组中所有条目.存储大小) - 新条目.预估存储大小
                总释放空间 += 释放空间

            // 第4步：输出结果
            SET internal_state = STATE_OUTPUT

            // 4a. 发送新条目至L3
            IF 归并后新条目列表 非空:
                向 ag-mem-24 发送归并后新条目列表

            // 4b. 发送清除列表
            IF 清除条目列表 非空:
                向 ag-mem-42 发送清除条目列表(清除条目列表, "L3归并淘汰")

            // 4c. 返回完成回执
            向 ag-mem-24 返回归并完成回执(
                扫描条目总数=待归并条目列表.size,
                检测到的相似组数=所有相似组列表.size,
                归并后新条目数=归并后新条目列表.size,
                清除原始条目数=清除条目列表.size,
                释放空间量=总释放空间,
                归并耗时=NOW() - 归并开始时间
            )

            // 4d. 上报状态
            向 ag-mem-03 上报归并状态(归并统计)
            更新归并统计计数器

            SET internal_state = STATE_IDLE

        SLEEP 50ms


FUNCTION 检测分槽内相似组(条目列表):
    相似组列表 = []
    已分组标记 = SET()

    FOR i, 条目A IN 条目列表:
        IF 条目A.条目ID IN 已分组标记:
            CONTINUE
        当前组 = [条目A]
        已分组标记.add(条目A.条目ID)

        FOR j, 条目B IN 条目列表:
            IF j <= i OR 条目B.条目ID IN 已分组标记:
                CONTINUE
            IF 条目B.警示标签 == "CAUTION":
                CONTINUE

            相似度 = 计算综合相似度(条目A, 条目B)
            IF 相似度 >= 0.80:
                当前组.append(条目B)
                已分组标记.add(条目B.条目ID)
                IF 当前组.size >= 5:
                    BREAK

        IF 当前组.size >= 2:
            相似组列表.append(当前组)

    RETURN 相似组列表


FUNCTION 计算综合相似度(条目A, 条目B):
    // 维度1：任务特征向量余弦相似度（权重0.40）
    余弦相似度 = 计算余弦相似度(条目A.任务特征向量, 条目B.任务特征向量)

    // 维度2：工具调用序列编辑距离（权重0.30）
    编辑距离 = Levenshtein距离(条目A.工具调用序列, 条目B.工具调用序列)
    最大长度 = MAX(条目A.工具调用序列长度, 条目B.工具调用序列长度)
    序列相似度 = IF 最大长度 > 0 THEN 1 - 编辑距离/最大长度 ELSE 1.0

    // 维度3：结果标签一致性（权重0.20）
    标签相似度 = IF 条目A.结果标签 == 条目B.结果标签 THEN 1.0 ELSE 0.5

    // 维度4：时间窗口接近度（权重0.10）
    时间差天数 = ABS(条目A.写入时间戳 - 条目B.写入时间戳) / 86400
    时间相似度 = MAX(0, 1 - 时间差天数 / 30)

    综合相似度 = 0.40 × 余弦相似度 + 0.30 × 序列相似度 + 0.20 × 标签相似度 + 0.10 × 时间相似度
    RETURN 综合相似度
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 待归并条目列表为空或仅1条 | 直接返回完成回执（全0），不报错 | — |
| 归并组中包含警示条目 | 跳过该组，警示条目不参与归并 | 警示标签降级为NORMAL后参与下次归并 |
| 归并后新条目写入L3失败 | 回滚：保留原始条目不变，本次归并组全部跳过 | L3存储恢复正常 |
| 相似度检测耗时过长（>30秒） | 中断检测，返回已完成的部分结果，标记“检测未完成” | 下次归并周期继续 |
| 归并过程中系统资源紧张 | 降低归并处理速率，每次仅处理前10组，剩余组下周期继续 | 资源恢复正常 |
| 紧急熔断 | 中断当前归并，未完成的归并组保留原始条目不变 | 紧急解除后恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 警示条目标记为CAUTION的经验禁止参与归并，确保失败经验的原始数据完整可追溯 |
| S-02 | 归并操作仅修改L3层经验的组织形式，不得改变经验的核心语义与来源归属 |
| S-03 | 归并后的新条目必须完整记录所有被归并的原始条目ID列表，支持追溯 |
| S-04 | 不同场景分槽的经验禁止跨槽归并，即使相似度极高也必须保持分槽隔离 |
| S-05 | 归并过程中不得修改任何条目的警示标签状态 |
