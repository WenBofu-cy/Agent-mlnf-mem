# ag-mem-21-L1临时层时序衰减单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-21 |
| 模块名称 | L1临时层时序衰减单元 |
| 所属分区 | 三、漏斗二：任务经验漏斗 / 五层存储 |
| 核心职责 | 接收 ag-mem-20（L1临时层存储单元）发起的衰减评估请求，对L1中留存超过24小时的条目进行时序衰减评估。基于条目的留存时长、重要度（I值）及来源场景分槽的专属晋升策略，判定每个条目的处理去向：晋升至L2近期层（满足晋升条件）、继续保留在L1（未达晋升条件但仍有价值）、或直接清除（重要度过低）。不参与条目内容修改或认知决策，仅执行基于时间与重要度的确定性筛选 |
| 依赖模块 | ag-mem-20（L1临时层存储单元，发送待评估条目列表）、ag-mem-22（L2近期层存储单元，接收晋升条目）、ag-mem-42（冗余记忆删除与归档单元，接收清除条目） |
| 被依赖模块 | ag-mem-20（返回衰减评估结果与清理确认） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 无评估任务，等待指令 | 系统初始化完成，无待处理请求 |
| 评估进行中 | `EVALUATING` | 正在逐条评估待处理条目的衰减状态 | 收到 ag-mem-20 的衰减评估请求 |
| 结果输出 | `OUTPUTTING` | 评估完成，正在向目标模块输出结果 | 所有条目评估完成 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|

| L1条目详细信息 | Struct（条目ID + 重要度I值 + 写入时间戳 + 来源分槽编号 + 经验类型标签） | ag-mem-20 L1临时层存储单元 | 随衰减评估请求附带 | **高** |
| 各场景分槽晋升阈值配置 | Struct（分槽编号 + L1→L2晋升I值阈值 + L1→L2晋升留存时长阈值） | ag-mem-35 三维权重系数配置单元 | 系统初始化加载，配置更新时 | 普通 |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-mem-01 总控漏斗F0 | 模式切换或紧急事件时 | **紧急** |

|--------|----------|----------|----------|:---:|
| 晋升条目列表 | List（条目ID + 目标层级L2 + 晋升原因 + 当前I值 + 留存时长） | ag-mem-22 L2近期层存储单元 | 评估完成后，包含所有满足晋升条件的条目 | **高** |
| 清除条目列表 | List（条目ID + 清除原因 + 当前I值 + 留存时长） | ag-mem-42 冗余记忆删除与归档单元 | 评估完成后，包含所有应清除的条目 | **高** |

| 衰减评估完成回执 | Struct（评估条目总数 + 晋升数 + 清除数 + 保留数 + 评估耗时） | ag-mem-20 L1临时层存储单元 | 评估完成时 | **高** |


## 衰减判定规则

### 一、二维判定矩阵

| 留存时长 | I值条件 | 处理方式 | 说明 |
|----------|------|:---:|------|
| < 24小时 | 任意 | **继续保留在L1** | 未达到晋升评估的最小留存时长，暂不评估 |
| ≥ 24小时 | I ≥ 晋升阈值 | **晋升至L2** | 满足晋升双条件（时长+重要度） |
| ≥ 24小时 | 遗忘阈值 ≤ I < 晋升阈值 | **继续保留在L1** | 有一定价值但不足以晋升，等待下次评估 |
| ≥ 24小时 | I < 遗忘阈值 | **直接清除** | 重要度过低，不再保留 |
| ≥ 72小时 | I < 晋升阈值 | **直接清除** | 在L1滞留超过72小时仍未晋升，即使高于遗忘阈值也清除 |

### 二、各场景分槽的晋升阈值（由 ag-mem-35 配置）

| 来源分槽 | L1→L2晋升I值阈值 | L1遗忘I值阈值 |
|----------|:---:|:---:|
| ag-mem-15（对话交互） | 0.35 | 0.08 |
| ag-mem-16（工具调用） | 0.40 | 0.10 |
| ag-mem-17（信息检索） | 0.38 | 0.08 |
| ag-mem-18（创作生成） | 0.38 | 0.10 |
| ag-mem-19（通用任务） | 0.42 | 0.06 |

### 三、容量紧急时的策略调整

当触发原因为“容量紧急”时，规则收紧：
- 最小留存时长从24小时临时下调为6小时
- 遗忘阈值临时上调20%（更易清除低价值条目）
- 晋升阈值不变


## 核心处理逻辑

```
FUNCTION l1_decay_assessment_main_loop():
    STATE_IDLE = IDLE
    STATE_EVAL = EVALUATING
    STATE_OUTPUT = OUTPUTTING
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_IDLE
    加载各分槽晋升阈值配置

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_IDLE

        // 第2步：接收衰减评估请求
        IF 收到 ag-mem-20 L1衰减评估请求:
            SET internal_state = STATE_EVAL
            待评估条目列表 = 请求.待评估条目列表
            触发原因 = 请求.触发原因
            晋升列表 = []
            清除列表 = []
            保留列表 = []
            当前时间 = NOW()

            // 2a. 判断是否容量紧急
            是否容量紧急 = 触发原因 == "容量紧急"

            // 2b. 逐条评估
            FOR EACH 条目 IN 待评估条目列表:
                留存时长 = 当前时间 - 条目.写入时间戳
                来源分槽 = 条目.来源分槽编号
                I值 = 条目.重要度I值

                // 获取该分槽的专属阈值
                分槽配置 = 晋升阈值配置表[来源分槽]
                晋升阈值 = 分槽配置.L1晋升I值阈值
                遗忘阈值 = 分槽配置.L1遗忘I值阈值

                // 容量紧急时调整阈值
                IF 是否容量紧急:
                    最小留存 = 6小时
                    遗忘阈值 = 遗忘阈值 × 1.2
                ELSE:
                    最小留存 = 24小时

                // 判定处理方式
                IF 留存时长 >= 72小时 AND I值 < 晋升阈值:
                    // 滞留超过72小时仍未达到晋升条件，直接清除
                    条目.处理方式 = "清除"
                    条目.清除原因 = "L1滞留超72小时未晋升"
                    清除列表.append(条目)

                ELSE IF 留存时长 >= 最小留存:
                    IF I值 >= 晋升阈值:
                        // 满足晋升条件
                        条目.处理方式 = "晋升L2"
                        条目.晋升原因 = "满足留存时长+重要度双条件"
                        晋升列表.append(条目)
                    ELSE IF I值 < 遗忘阈值:
                        // 低于遗忘阈值，清除
                        条目.处理方式 = "清除"
                        条目.清除原因 = "I值低于遗忘阈值"
                        清除列表.append(条目)
                    ELSE:
                        // 有一定价值，继续保留
                        条目.处理方式 = "保留"
                        条目.评估时间戳 = 当前时间
                        保留列表.append(条目)
                ELSE:
                    // 未达最小留存时长，继续保留
                    条目.处理方式 = "保留"
                    条目.评估时间戳 = 当前时间
                    保留列表.append(条目)

            // 第3步：输出结果
            SET internal_state = STATE_OUTPUT

            // 3a. 发送晋升条目至L2
            IF 晋升列表 非空:
                向 ag-mem-22 发送晋升条目列表(晋升列表)

            // 3b. 发送清除条目至冗余删除单元
            IF 清除列表 非空:
                向 ag-mem-42 发送清除条目列表(清除列表)

            // 3c. 告知L1保留条目
            向 ag-mem-20 发送保留条目确认(保留列表.size, 保留列表)

            // 3d. 发送完成回执
            向 ag-mem-20 发送衰减评估完成回执(
                评估条目总数=待评估条目列表.size,
                晋升数=晋升列表.size,
                清除数=清除列表.size,
                保留数=保留列表.size,
                评估耗时=NOW() - 评估开始时间
            )

            SET internal_state = STATE_IDLE

        SLEEP 10ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 条目缺少来源分槽编号（无法确定阈值） | 使用通用任务槽（ag-mem-19）的阈值作为默认值 | 分槽信息补全 |
| 条目重要度I值缺失或异常（<0或>1） | 保守处理：保留在L1，标记异常，下次评估再判定 | I值被修正 |
| 晋升写入L2失败（L2返回拒绝） | 条目退回保留列表，下次评估时重新判定 | L2恢复正常 |
| 待评估条目列表为空 | 直接返回完成回执（全0），不做任何操作 | — |
| 紧急熔断 | 暂停评估，未完成的条目列表保留在内存中，恢复后继续 | 紧急解除后恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 衰减评估仅读取条目元数据（时间戳、I值、分槽编号），不得访问或修改条目的经验内容 |
| S-02 | 晋升至L2的条目必须完整保留其原始来源分槽编号，用于L2层的后续管理 |
| S-03 | 容量紧急时的阈值调整仅在本次评估中生效，不得持久化修改各分槽的默认阈值配置 |
| S-04 | 清除条目必须通过 ag-mem-42 执行安全删除，本模块不得直接操作存储删除 |
