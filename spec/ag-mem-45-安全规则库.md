# ag-mem-45-安全规则库 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-45 |
| 模块名称 | 安全规则库 |
| 所属分区 | 四、漏斗外挂扩展区（物理隔离） |
| 核心职责 | 作为漏斗外挂扩展区的安全策略知识库，存储并管理工具调用安全边界、权限分级规则、敏感操作白名单/黑名单、合规校验基准等结构化安全规则。为 ag-mem-43（失败经验安全仲裁三道校验单元）提供第一道安全合规校验的依据，为 ag-ecc-04（安全仲裁模块）提供全局安全规则查询，为 ag-mem-29（L5 核心层安全规则硬锁定单元）提供 L5 安全合规校验基准。本库完全独立于双漏斗记忆系统运行，不参与记忆沉淀、筛选、晋升与遗忘机制。安全规则在系统部署前预置，支持定期离线更新，是 EM-Core Agent 安全体系的底层规则源头。仅提供只读查询服务，不参与任何认知决策 |
| 依赖模块 | 无（作为独立安全规则服务，不依赖记忆系统内部模块） |
| 被依赖模块 | ag-mem-43（失败经验安全仲裁，获取安全合规校验结果）、ag-mem-29（L5 锁定单元，获取 L5 安全合规校验基准）、ag-ecc-04（安全仲裁模块，获取全局安全规则）、ag-mem-01（总控漏斗 F₀，可转发安全规则查询请求） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常服务 | `NORMAL_SERVICE` | 安全规则库加载完成，正常响应查询 | 系统初始化完成，规则索引构建成功 |
| 规则加载中 | `LOADING` | 正在加载或更新安全规则数据 | 系统启动时或收到规则更新指令 |
| 规则库降级 | `DEGRADED` | 部分规则数据损坏或不可用，使用内置最小安全规则集 | 规则库完整性校验失败 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |

## 安全规则分类与结构

### 一、规则分类

| 分类ID | 分类名称 | 内容说明 | 典型规则数 |
|:---:|------|------|:---:|
| WHITELIST | 工具白名单 | 允许 Agent 自主调用的工具清单及其参数约束 | 50-200 |
| BLACKLIST | 工具黑名单 | 禁止 Agent 以任何方式调用的工具或操作 | 10-30 |
| SENSITIVE | 敏感操作清单 | 需要用户确认或二次验证的操作清单 | 20-50 |
| PERMISSION | 权限分级规则 | 各工具的权限等级、所需令牌级别、可访问数据范围 | 30-60 |
| COMPLIANCE | 合规校验规则 | 工具调用序列的合规模式匹配规则（如：禁止在读取敏感数据后调用外部上传API） | 15-40 |
| SAFETY_LEVEL | 安全等级定义 | 各操作的安全等级（低/中/高/严重）与对应的系统响应策略 | 10-20 |

### 二、安全规则条目结构

```
安全规则条目 = {
    规则ID: "SEC-{分类}-{UUID}",
    分类: "WHITELIST | BLACKLIST | SENSITIVE | PERMISSION | COMPLIANCE | SAFETY_LEVEL",
    规则名称: 规则描述名称,
    适用工具/操作: [工具名称列表],
    规则条件: 触发该规则的条件表达式,
    规则动作: "允许 | 拒绝 | 确认 | 二次验证 | 告警 | 阻断",
    严重等级: "低 | 中 | 高 | 严重",
    优先级: 1-100（数字越大优先级越高）,
    是否可覆盖: True/False（是否允许 ECC-05 在极端伦理困境下临时豁免）,
    豁免条件: 可豁免时的严格条件描述,
    版本: 规则版本号,
    更新时间: 最后更新时间戳,
    来源: "预置 | 更新 | 人工录入"
}
```


## 核心处理逻辑

```
FUNCTION safety_rule_engine_main_loop():
    STATE_LOADING = LOADING
    STATE_NORMAL = NORMAL_SERVICE
    STATE_DEGRADED = DEGRADED
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_LOADING
    初始化规则索引（按分类、工具名称、操作类型、严重等级构建多级索引）
    初始化查询缓存（LRU，最大200条，因为安全规则查询重复率高）

    // 系统启动时加载安全规则库
    加载结果 = 加载安全规则数据()
    IF 加载结果.完整性校验通过:
        SET internal_state = STATE_NORMAL
        构建全量规则索引()
    ELSE:
        SET internal_state = STATE_DEGRADED
        加载内置最小安全规则集()  // 确保基本安全底线可用
        构建部分规则索引()

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_NORMAL

        // 第2步：接收安全规则查询请求
        IF 收到安全规则查询请求:
            IF internal_state NOT IN [STATE_NORMAL, STATE_DEGRADED]:
                向请求来源返回查询失败("安全规则库当前不可用")
                CONTINUE

            请求ID = 请求.请求ID
            查询类型 = 请求.查询类型
            查询参数 = 请求.查询参数

            // 2a. 检查查询缓存（仅对相同参数的查询生效）
            缓存键 = 生成缓存键(查询类型, 查询参数)
            IF 查询缓存.存在(缓存键) AND 查询类型 != "合规校验":
                // 合规校验不缓存，每次实时判定
                向请求来源返回安全规则查询结果(查询缓存[缓存键], 来源="缓存")
                CONTINUE

            // 2b. 按查询类型分发处理
            CASE 查询类型 OF:
                "合规校验":
                    // 这是最核心的查询类型，用于 ag-mem-43 的安全仲裁
                    工具名称 = 查询参数.工具名称
                    操作类型 = 查询参数.操作类型
                    参数列表 = 查询参数.参数列表

                    // 第一步：检查黑名单
                    黑名单匹配 = 规则索引.按分类("BLACKLIST").匹配(工具名称, 操作类型)
                    IF 黑名单匹配 非空:
                        判定结果 = 构建合规校验回执(
                            是否合规=False,
                            违规规则列表=黑名单匹配,
                            严重等级=黑名单匹配.最高严重等级,
                            建议措施="禁止执行，工具或操作在黑名单中"
                        )
                        向请求来源返回安全合规校验回执(判定结果)
                        CONTINUE

                    // 第二步：检查白名单
                    白名单匹配 = 规则索引.按分类("WHITELIST").匹配(工具名称, 操作类型)
                    是否在白名单 = 白名单匹配 非空

                    // 第三步：检查敏感操作清单
                    敏感操作匹配 = 规则索引.按分类("SENSITIVE").匹配(工具名称, 操作类型)
                    是否需要确认 = 敏感操作匹配 非空

                    // 第四步：检查权限分级
                    权限规则 = 规则索引.按分类("PERMISSION").匹配(工具名称)
                    所需权限等级 = 权限规则.所需令牌级别 IF 权限规则 非空 ELSE "低"

                    // 第五步：检查合规模式（序列模式匹配）
                    合规模式匹配 = 规则索引.按分类("COMPLIANCE").序列匹配(查询参数.工具调用序列历史)

                    // 综合判定
                    IF NOT 是否在白名单 AND 敏感操作匹配 非空:
                        判定结果 = 构建合规校验回执(是否合规=False, 违规规则列表=["不在白名单的敏感操作"], 严重等级="严重")
                    ELSE IF 合规模式匹配 非空:
                        判定结果 = 构建合规校验回执(是否合规=False, 违规规则列表=合规模式匹配, 严重等级=合规模式匹配.最高严重等级)
                    ELSE IF 是否需要确认 AND 所需权限等级 == "高":
                        判定结果 = 构建合规校验回执(是否合规=True, 附加条件="需要用户确认", 严重等级="中")
                    ELSE:
                        判定结果 = 构建合规校验回执(是否合规=True, 严重等级=权限规则.安全等级 IF 权限规则 非空 ELSE "低")

                    向请求来源返回安全合规校验回执(判定结果)

                "权限分级":
                    匹配规则 = 规则索引.按分类("PERMISSION").模糊匹配(查询参数.工具名称)
                    向请求来源返回安全规则查询结果(匹配规则)

                "敏感操作判定":
                    匹配规则 = 规则索引.按分类("SENSITIVE").精确匹配(查询参数.工具名称, 查询参数.操作类型)
                    向请求来源返回安全规则查询结果(匹配规则)

                "安全等级查询":
                    匹配规则 = 规则索引.按分类("SAFETY_LEVEL").精确匹配(查询参数.操作类型)
                    向请求来源返回安全规则查询结果(匹配规则)

                "全量规则":
                    IF 请求来源模块 IN [ag-ecc-04, ag-mem-29]:
                        向请求来源返回安全规则查询结果(全量规则快照)
                    ELSE:
                        向请求来源返回查询失败("全量规则查询权限不足")

            // 写入缓存（合规校验除外）
            IF 查询类型 != "合规校验":
                查询缓存[缓存键] = 查询结果

        // 第3步：接收安全规则更新指令
        IF 收到安全规则更新指令:
            // 3a. 安全规则更新必须经过严格的多重校验
            IF NOT 验证更新授权令牌(指令.授权令牌):
                向更新发起方返回规则更新确认回执(失败, "授权令牌无效")
                CONTINUE

            IF NOT 校验签名(指令.校验签名):
                向更新发起方返回规则更新确认回执(失败, "签名校验失败")
                CONTINUE

            // 3b. 备份当前规则库
            旧规则库快照 = 当前规则数据.clone()
            旧版本号 = 当前版本号

            // 3c. 在沙箱中验证新规则的逻辑一致性
            沙箱验证结果 = 在隔离环境验证规则(指令.更新数据包)
            IF NOT 沙箱验证结果.通过:
                向更新发起方返回规则更新确认回执(失败, "规则逻辑一致性验证失败: " + 沙箱验证结果.错误信息)
                CONTINUE

            // 3d. 执行更新
            SET internal_state = STATE_LOADING
            更新结果 = 执行规则更新(指令.更新数据包, 指令.更新范围)
            IF 更新结果.成功:
                重新构建规则索引()
                清空查询缓存()
                更新版本号 = 指令.版本号
                SET internal_state = STATE_NORMAL
                记录安全事件("安全规则库更新", 旧版本号, 新版本号)
            ELSE:
                恢复旧规则库快照
                恢复旧版本号
                SET internal_state = STATE_NORMAL
                向更新发起方返回规则更新确认回执(失败, 更新结果.错误信息)
                CONTINUE

            向更新发起方返回规则更新确认回执(成功, 更新统计, 新版本号)

        // 第4步：接收完整性校验指令
        IF 收到完整性校验指令:
            校验结果 = 执行规则库完整性校验()
            向 ag-mem-01 返回安全规则完整性报告(校验结果)
            IF 校验结果.损坏规则数 > 0:
                SET internal_state = STATE_DEGRADED
                加载内置最小安全规则集作为补充
            ELSE IF internal_state == STATE_DEGRADED:
                SET internal_state = STATE_NORMAL

        // 第5步：周期性状态上报
        IF 距上次状态上报 >= 60秒:
            向 ag-mem-01 和 ag-mem-03 上报安全规则状态(当前状态, 规则统计, 版本号)

        SLEEP 10ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 安全规则数据文件缺失或损坏 | 进入 DEGRADED 状态，加载内置最小安全规则集（仅包含黑名单和基础敏感操作），上报告警 | 重新加载有效规则数据文件 |
| 合规校验请求缺少必要参数（工具名称/操作类型为空） | 保守处理，视为不合规，返回“信息不足，默认拒绝” | 请求补充完整参数 |
| 规则更新签名校验失败 | 拒绝更新，记录安全事件，触发告警 | 使用正确签名重试 |
| 规则更新沙箱验证失败（新规则存在逻辑冲突） | 拒绝更新，保留旧版本，返回详细错误信息 | 修正规则冲突后重试 |
| 规则更新过程中系统断电 | 重启后自动回滚至更新前旧版本 | 重新尝试更新 |
| 全量规则查询请求来自未授权模块 | 拒绝查询，记录违规日志 | — |
| 紧急熔断 | 暂停服务，保持当前规则索引与版本不变 | 紧急解除后恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| SR-01 | 安全规则库为只读数据，运行时不得被任何控制模块或自动化流程修改 |
| SR-02 | 规则更新必须经过授权令牌验证、签名校验与沙箱逻辑一致性验证三道关卡 |
| SR-03 | 规则库降级时强制加载内置最小安全规则集，确保安全底线在任何情况下均可用 |
| SR-04 | 合规校验请求信息不足时，默认返回“不合规”，遵循安全保守原则 |
| SR-05 | 全量规则查询仅向 ag-ecc-04（安全仲裁模块）和 ag-mem-29（L5 锁定单元）开放，其他模块无权获取 |
| SR-06 | 所有规则更新与拒绝事件必须记录完整的安全事件日志，不可篡改 |
| SR-07 | 本库完全独立于漏斗记忆系统，不参与记忆沉淀、晋升、遗忘任何环节 |
