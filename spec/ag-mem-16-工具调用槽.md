# ag-mem-16-工具调用槽 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-16 |
| 模块名称 | 工具调用槽 |
| 所属分区 | 三、漏斗二：任务经验漏斗 / 场景分槽管理 |
| 核心职责 | 作为漏斗二中专门承载“工具调用”类任务经验的场景分槽。接收 ag-mem-03 路由的工具调用场景经验条目，管理该场景下的五层记忆存储（L1-L5）。在本槽内，安全显著性（S值）权重自动上调20%，以强化安全相关操作经验的优先留存。同时管理该槽专属的晋升阈值与遗忘策略参数。不参与认知决策，仅执行工具调用类经验的存储、检索与生命周期管理 |
| 依赖模块 | ag-mem-03（漏斗二专属调度单元，下发经验路由）、ag-mem-20~30（五层存储单元，实际存储操作）、ag-mem-35（三维权重系数配置单元，读取并应用本槽专属权重） |
| 被依赖模块 | ag-mem-03（返回查询/写入结果）、ag-mem-25（相似经验归并单元，可扫描本槽内的重复经验）、ag-mem-43（失败经验安全仲裁三道校验单元，工具调用失败经验需过安全校验） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 无操作请求，等待指令 | 系统初始化完成，槽位已激活 |
| 写入处理 | `WRITING` | 正在将新经验条目写入五层存储 | 收到 ag-mem-03 的经验写入请求 |
| 查询处理 | `QUERYING` | 正在检索匹配的历史工具调用经验 | 收到 ag-mem-03 的经验查询请求 |
| 维护整理 | `MAINTENANCE` | 正在执行槽内经验归并或遗忘扫描 | 收到维护指令或定时触发 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 经验写入请求 | Struct（经验条目数据 + 用户ID + 重要度预估值 + 场景标签确认） | ag-mem-03 漏斗二专属调度单元（通过内部调度总线） | 工具调用类任务执行完成，产生新经验时 | **高** |
| 经验查询请求 | Struct（查询条件 + 用户ID + 最大返回条数 + 时间窗口） | ag-mem-03 漏斗二专属调度单元（通过内部调度总线） | ECC 需要检索历史工具调用经验时 | **高** |
| 维护扫描指令 | Struct（扫描类型：归并/遗忘 + 扫描深度） | ag-mem-03 漏斗二专属调度单元 | 定时触发或容量告警时 | 普通 |
| 专属权重配置参数 | Struct（S值权重系数 + 各层晋升阈值 + 遗忘阈值） | ag-mem-35 三维权重系数配置单元 | 槽位激活或配置更新时 | 普通 |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-mem-01 总控漏斗F0 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 写入确认回执 | Struct（条目ID + 分配层级 + 预估晋升时间 + 写入耗时 + 安全校验标记） | ag-mem-03 漏斗二专属调度单元 | 写入完成时 | **高** |
| 查询结果列表 | Struct（匹配经验列表 + 各条目层级 + 重要度 + 查询耗时） | ag-mem-03 漏斗二专属调度单元 | 查询完成时 | **高** |
| 维护完成回执 | Struct（归并条目数 + 遗忘条目数 + 释放空间量） | ag-mem-03 漏斗二专属调度单元 | 维护扫描完成时 | 普通 |
| 槽位状态上报 | Struct（当前条目总数 + 各层级分布 + 存储使用量 + 最近活跃时间） | ag-mem-03 漏斗二专属调度单元 | 周期性（每60秒）或状态变更时 | 普通 |


## 专属参数配置

| 参数名称 | 默认值 | 说明 |
|----------|:---:|------|
| S值权重系数 | **1.2**（标准为1.0，上调20%） | 使工具调用经验更重视安全显著性信号 |
| L1→L2晋升阈值 | I ≥ 0.40 | 标准 |
| L2→L3晋升阈值 | I ≥ 0.60 | 标准 |
| L3→L4晋升阈值 | I ≥ 0.80 | 标准 |
| L4→L5晋升阈值 | I ≥ 0.90 | 标准，但S≥0.9可直接触发L5直达写入 |
| L1遗忘阈值 | I < 0.10 | 标准 |
| L2遗忘阈值 | I < 0.25 | 略高于标准0.20，优先清理低安全性的工具经验 |
| L3遗忘阈值 | I < 0.35 | 略高于标准0.30 |


## 核心处理逻辑

```
FUNCTION tool_call_slot_main_loop():
    STATE_IDLE = IDLE
    STATE_WRITE = WRITING
    STATE_QUERY = QUERYING
    STATE_MAINT = MAINTENANCE
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_IDLE
    初始化槽位编号 = "ag-mem-16"
    加载专属权重配置（S值系数=1.2）

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_IDLE

        // 第2步：接收经验写入请求
        IF 收到 ag-mem-03 经验写入请求:
            // 校验场景标签
            IF 请求.场景标签 != "工具调用":
                向 ag-mem-03 返回错误("场景标签不匹配，本槽仅接受工具调用类经验")
                CONTINUE

            SET internal_state = STATE_WRITE
            经验条目 = 请求.经验条目数据

            // 2a. 检查是否为失败经验
            IF 经验条目.结果标签 == "失败" OR 经验条目.结果标签 == "策略失误":
                // 失败经验在写入L1前，先标记为待安全校验
                经验条目.待安全校验 = True
                // 初始层级锁定为L1，不得直接进入更高层级

            // 2b. 应用专属权重：S值上调20%
            IF 经验条目.s_value != None:
                经验条目.s_value = 经验条目.s_value × 1.2
                IF 经验条目.s_value > 1.0:
                    经验条目.s_value = 1.0

            // 2c. S值直达L5判定
            IF 经验条目.s_value >= 0.9 AND 经验条目.结果标签 == "成功":
                写入层级 = "L5"
                向 ag-mem-28 发起L5直达写入(经验条目)
                记录安全事件("高安全显著性工具调用经验直达L5")
            ELSE:
                写入层级 = "L1"
                向 ag-mem-20 发起L1写入(经验条目)

            // 2d. 返回确认
            向 ag-mem-03 返回写入确认回执(
                条目ID=经验条目.条目ID,
                分配层级=写入层级,
                安全校验标记=经验条目.待安全校验
            )
            SET internal_state = STATE_IDLE

        // 第3步：接收经验查询请求
        IF 收到 ag-mem-03 经验查询请求:
            SET internal_state = STATE_QUERY

            // 3a. 从五层存储中检索匹配经验（安全相关经验优先返回）
            结果集 = []
            FOR EACH 层级 IN [L5, L4, L3, L2, L1]:
                层级结果 = 向对应存储单元发起查询(层级, 查询条件)
                // 对L5和L4结果标记“安全优先”
                结果集.合并(层级结果)

            // 3b. 排序：安全显著性高的条目优先
            结果集 = 按重要度降序 + S值降序排列(结果集)

            // 3c. 返回结果
            向 ag-mem-03 返回查询结果列表(结果集)
            SET internal_state = STATE_IDLE

        // 第4步：执行维护扫描
        IF 收到维护扫描指令:
            SET internal_state = STATE_MAINT
            归并结果 = 向 ag-mem-25 发起槽内归并扫描(槽位编号=ag-mem-16)
            遗忘结果 = 向 ag-mem-40 发起槽内遗忘扫描(槽位编号=ag-mem-16, 专属遗忘阈值)
            向 ag-mem-03 返回维护完成回执(归并结果, 遗忘结果)
            SET internal_state = STATE_IDLE

        // 第5步：周期性状态上报
        IF 距上次状态上报 >= 60秒:
            向 ag-mem-03 上报槽位状态

        SLEEP 20ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 写入请求场景标签不匹配 | 拒绝写入，返回错误提示 | — |
| 失败经验写入 | 标记待安全校验，锁定在L1层级，后续晋升须通过ag-mem-43三道校验 | 安全校验通过 |
| S值直达L5时L5存储满 | 优先清理L5中重要度最低的非安全条目，再写入 | 清理完成 |
| 工具调用经验中检测到敏感操作（如删除文件、修改权限） | 自动提升S值到≥0.8，确保高留存优先级 | — |
| 紧急熔断 | 暂停所有操作 | 紧急解除后恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 本槽位仅接受场景标签确认为“工具调用”的经验条目 |
| S-02 | S值权重上调仅在本槽位内生效，不得影响其他分槽的重要度计算 |
| S-03 | 失败经验必须标记待安全校验，在未通过ag-mem-43校验前不得晋升至L2及以上 |
| S-04 | 检测到敏感操作（删除、权限修改等）时，必须自动提升S值确保高留存优先级 |
| S-05 | 槽内经验的跨槽查询必须通过 ag-mem-03 统一路由 |
