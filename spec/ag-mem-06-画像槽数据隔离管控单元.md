# ag-mem-06-画像槽数据隔离管控单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-06 |
| 模块名称 | 画像槽数据隔离管控单元 |
| 所属分区 | 二、漏斗一：用户画像漏斗 |
| 核心职责 | 在编译期与运行时强制实施漏斗一内部各画像槽之间的物理存储隔离。拦截并校验所有跨槽数据访问请求，确保任意时刻任意模块只能访问当前活跃会话绑定的唯一画像槽。管理槽级访问令牌的签发、验证与吊销。任何试图跨用户读取或混合画像数据的操作均被实时阻断并记录告警。不参与任何认知决策，仅执行数据隔离的强制管控 |
| 依赖模块 | ag-mem-02（漏斗一专属调度单元，提供当前活跃槽位号）、ag-mem-05（画像槽创建与初始化单元，新建槽时申请隔离策略） |
| 被依赖模块 | ag-mem-07（用户行为观测记录单元，需通过本模块校验写入权限）、ag-mem-09（偏好判定标签单元，需通过本模块校验读取权限）、ag-mem-51（记忆变更日志追溯单元，接收隔离违规告警） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常管控 | `NORMAL_GATE` | 隔离策略正常运行，放行合规请求 | 系统初始化完成，活跃槽位已绑定 |
| 违规阻断 | `VIOLATION_BLOCK` | 检测到跨槽访问违规，已拦截并告警 | 请求访问的槽位号与当前活跃槽位号不匹配 |
| 无活跃槽 | `NO_ACTIVE_SLOT` | 当前无活跃画像槽，拒绝所有数据访问 | 会话未开始或槽位已释放 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 数据访问请求 | Struct（请求来源模块 + 请求操作类型：读/写/修改/删除 + 目标槽位编号 + 请求数据字段列表） | ag-mem-07/09/11 等漏斗一内部模块 | 任何需要对画像槽执行数据操作时 | **高** |
| 当前活跃槽位绑定信息 | Struct（会话ID + 活跃用户ID + 活跃槽位编号 + 绑定时间戳） | ag-mem-02 漏斗一专属调度单元 | 槽位激活或切换时 | **高** |
| 隔离策略配置参数 | Struct（槽位编号 + 访问控制列表 + 读写权限矩阵 + 加密密钥引用） | ag-mem-05 画像槽创建与初始化单元 | 新槽创建时，或策略更新时 | **高** |
| 令牌吊销请求 | Struct（槽位编号 + 吊销原因 + 立即生效标记） | ag-mem-02 漏斗一专属调度单元 | 槽位释放或用户登出时 | **高** |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-mem-01 总控漏斗F0 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 访问许可令牌 | Struct（令牌ID + 授权槽位编号 + 授权操作类型 + 有效期 + 签名） | 请求来源模块（ag-mem-07/09/11等） | 校验通过时 | **高** |
| 访问拒绝回执 | Struct（拒绝原因 + 请求详情 + 违规严重等级 + 时间戳） | 请求来源模块 + ag-mem-51（日志） | 校验不通过时 | **高** |
| 隔离违规告警 | Struct（违规类型 + 来源模块 + 目标槽位 + 当前活跃槽位 + 请求内容摘要） | ag-mem-02（调度单元）、ag-mem-51（日志追溯） | 检测到跨槽访问时 | **高** |
| 令牌吊销确认 | Struct（槽位编号 + 吊销状态 + 已失效令牌数量） | ag-mem-02 漏斗一专属调度单元 | 令牌吊销完成时 | **高** |
| 隔离状态上报 | Struct（活跃槽位数 + 今日违规次数 + 已签发令牌数 + 已吊销令牌数） | ag-mem-01 总控漏斗F0 | 周期性（每10秒）或状态变更时 | 普通 |


## 隔离管控规则

### 一、访问权限矩阵

| 请求来源模块 | 允许的操作 | 允许的目标槽位 | 附加条件 |
|------|:---:|:---:|------|
| ag-mem-07（行为观测） | 写入 | 仅当前活跃槽位 | 需持有有效写入令牌 |
| ag-mem-09（偏好判定） | 读写 | 仅当前活跃槽位 | 需持有有效读取令牌 |
| ag-mem-10（偏好统计） | 读取 | 仅当前活跃槽位 | 需持有有效读取令牌 |
| ag-mem-11（个性化建议） | 读取 | 仅当前活跃槽位 | 需持有有效读取令牌 |
| ag-mem-01（总控F0） | 读取 | 仅当前活跃槽位 | 仅响应用户画像查询请求 |
| 任何其他模块 | 禁止 | — | 编译期直接拒绝 |

### 二、违规判定规则

| 违规类型 | 判定条件 | 严重等级 |
|----------|------|:---:|
| 跨槽访问 | 请求目标槽位编号 ≠ 当前活跃槽位编号 | 严重 |
| 无活跃槽访问 | 当前无绑定活跃槽位，但收到数据访问请求 | 一般 |
| 令牌过期访问 | 令牌有效期已过，仍携带过期令牌请求 | 一般 |
| 未授权模块访问 | 请求来源模块不在访问权限矩阵中 | 严重 |
| 操作类型越权 | 请求的操作类型超出该模块的授权范围 | 严重 |


## 核心处理逻辑

```
FUNCTION slot_isolation_gate_main_loop():
    STATE_NORMAL = NORMAL_GATE
    STATE_BLOCK = VIOLATION_BLOCK
    STATE_NO_SLOT = NO_ACTIVE_SLOT
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_NO_SLOT
    初始化活跃槽位 = None
    初始化令牌签发记录表 = {}
    初始化违规计数器 = 0

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            吊销所有活跃令牌
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_NO_SLOT

        // 第2步：更新活跃槽位绑定
        IF 收到 ag-mem-02 活跃槽位绑定信息:
            活跃槽位 = 绑定信息.活跃槽位编号
            活跃用户 = 绑定信息.活跃用户ID
            SET internal_state = STATE_NORMAL
            CONTINUE

        // 第3步：处理令牌吊销
        IF 收到令牌吊销请求:
            FOR EACH 令牌 IN 令牌签发记录表:
                IF 令牌.槽位编号 == 吊销请求.槽位编号:
                    标记令牌为已吊销
                    令牌失效计数++
            向 ag-mem-02 返回令牌吊销确认(吊销请求.槽位编号, 已失效令牌数量)
            IF 吊销请求.槽位编号 == 活跃槽位:
                活跃槽位 = None
                SET internal_state = STATE_NO_SLOT
            CONTINUE

        // 第4步：接收数据访问请求
        IF 收到数据访问请求:
            // 4a. 检查是否有活跃槽位
            IF internal_state == STATE_NO_SLOT OR 活跃槽位 == None:
                SET internal_state = STATE_NO_SLOT
                向请求来源返回访问拒绝回执("无活跃画像槽", 请求详情, "一般")
                CONTINUE

            // 4b. 检查请求来源模块是否在权限矩阵中
            来源模块 = 请求.来源模块
            IF 来源模块 NOT IN 访问权限矩阵:
                SET internal_state = STATE_BLOCK
                向 ag-mem-02 发送隔离违规告警("未授权模块访问", 来源模块, 请求.目标槽位)
                向 ag-mem-51 发送隔离违规告警
                向请求来源返回访问拒绝回执("未授权模块", 请求详情, "严重")
                CONTINUE

            // 4c. 检查目标槽位是否为当前活跃槽位
            IF 请求.目标槽位编号 != 活跃槽位:
                SET internal_state = STATE_BLOCK
                向 ag-mem-02 发送隔离违规告警("跨槽访问", 来源模块, 请求.目标槽位, 活跃槽位)
                向 ag-mem-51 发送隔离违规告警
                违规计数器++
                向请求来源返回访问拒绝回执("跨槽访问被禁止", 请求详情, "严重")
                CONTINUE

            // 4d. 检查操作类型是否在授权范围内
            授权操作 = 访问权限矩阵[来源模块].允许的操作
            IF 请求.操作类型 NOT IN 授权操作:
                SET internal_state = STATE_BLOCK
                向请求来源返回访问拒绝回执("操作类型越权", 请求详情, "严重")
                CONTINUE

            // 4e. 校验通过，签发访问令牌
            令牌 = 生成访问令牌(来源模块, 活跃槽位, 请求.操作类型, 有效期=300秒)
            令牌签发记录表.添加(令牌)
            向请求来源返回访问许可令牌

            IF internal_state == STATE_BLOCK:
                SET internal_state = STATE_NORMAL

        // 第5步：令牌过期清理
        IF 距上次令牌清理 >= 60秒:
            FOR EACH 令牌 IN 令牌签发记录表:
                IF 令牌.已过期 OR 令牌.已吊销:
                    移除令牌
            更新令牌签发记录表

        // 第6步：周期性状态上报
        IF 距上次状态上报 >= 10秒:
            向 ag-mem-01 上报隔离状态(活跃槽位, 今日违规次数, 已签发令牌数, 已吊销令牌数)

        SLEEP 10ms


FUNCTION 生成访问令牌(来源模块, 槽位编号, 操作类型, 有效期):
    令牌ID = "TOKEN-" + 生成UUID()
    签名 = HMAC签名(令牌ID + 来源模块 + 槽位编号 + 操作类型 + 当前时间戳)
    RETURN 访问许可令牌(令牌ID, 槽位编号, 操作类型, 有效期, 签名)
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 活跃槽位未绑定时收到数据请求 | 拒绝所有请求，返回“无活跃画像槽” | ag-mem-02 下发新的槽位绑定信息 |
| 同一模块短时间内多次违规 | 累计违规次数，超过阈值后锁定该模块的访问权限300秒 | 锁定时间到期且人工确认 |
| 令牌签名校验失败 | 视为伪造令牌，触发严重告警，立即吊销该令牌 | 请求来源重新申请合法令牌 |
| 隔离策略配置参数丢失 | 使用编译期默认隔离策略（仅允许当前槽位读写），标记降级 | 策略参数恢复 |
| 紧急熔断 | 立即吊销所有活跃令牌，清空槽位绑定，拒绝所有请求 | 紧急解除后恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 编译期硬编码访问权限矩阵，运行时禁止任何模块修改权限配置 |
| S-02 | 所有跨槽访问请求必须被实时阻断，不得以“仅读取不修改”为理由放行 |
| S-03 | 隔离违规事件必须完整记录（来源模块、目标槽位、活跃槽位、请求内容摘要），不可篡改 |
| S-04 | 访问令牌有效期不得超过300秒，过期令牌自动失效 |
| S-05 | 本模块为漏斗一数据隔离的唯一强制执行点，任何模块不得绕过本模块直接访问画像槽存储 |
