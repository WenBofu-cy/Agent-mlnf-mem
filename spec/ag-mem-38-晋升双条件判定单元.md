# ag-mem-38-晋升双条件判定单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-38 |
| 模块名称 | 晋升双条件判定单元 |
| 所属分区 | 三、漏斗二：任务经验漏斗 / 晋升与遗忘执行机制 |
| 核心职责 | 接收来自 ag-mem-21（L1衰减评估）、ag-mem-22（L2超期处理）、ag-mem-24（L3超期处理）以及 ag-mem-37（重要度增量定时刷新后触发的晋升复检）的晋升候选条目，逐一校验其是否同时满足留存时长与综合重要度I值的双条件阈值。满足条件的条目整理为“晋升候选清单”，发送至 ag-mem-39（层级单向搬运写入单元）执行物理晋升；不满足条件的条目返回来源层级继续保留或进入遗忘评估。同时校验条目是否带有警示标签（CAUTION）——警示条目默认禁止晋升至 L4，需先通过 ag-mem-43（失败经验安全仲裁）解除警示后方可参与晋升判定。不参与搬运执行或内容修改，仅负责晋升条件的客观校验与清单生成 |
| 依赖模块 | ag-mem-21（L1衰减评估，推送L1晋升候选）、ag-mem-22（L2存储，推送L2超期晋升候选）、ag-mem-24（L3存储，推送L3超期晋升候选）、ag-mem-26（L4存储，推送L4超期晋升候选）、ag-mem-35（三维权重系数配置单元，获取各层级晋升阈值）、ag-mem-37（定时刷新后触发的晋升复检） |
| 被依赖模块 | ag-mem-39（层级单向搬运写入单元，接收晋升候选清单）、ag-mem-21/22/24/26（返回判定结果） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 无判定任务，等待晋升候选推送 | 系统初始化完成，无待处理条目 |
| 判定进行中 | `JUDGING` | 正在逐条校验晋升候选条目的双条件 | 收到晋升候选条目列表 |
| 结果输出 | `OUTPUTTING` | 判定完成，正在向 ag-mem-39 发送晋升清单 | 全部条目判定完成 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |

## 晋升双条件定义

### 一、各层级晋升条件

| 晋升路径 | 留存时长达标条件 | 重要度I值达标条件 | 其他条件 |
|----------|:---:|:---:|------|
| L1→L2 | ≥ 24小时 | I ≥ 分槽L1晋升阈值 | 无警示标签 |
| L2→L3 | ≥ 7天（168小时） | I ≥ 分槽L2晋升阈值 | 无警示标签 |
| L3→L4 | ≥ 30天（720小时） | I ≥ 分槽L3晋升阈值 | 警示标签必须为 NORMAL；若为 CAUTION，需先通过 ag-mem-43 安全仲裁 |
| L4→L5 | ≥ 90天（2160小时） | I ≥ 分槽L4晋升阈值 + S值 ≥ 0.9 或 规则置信度 ≥ 0.85 | 必须通过 ag-mem-29 的安全令牌校验 |

### 二、各场景分槽的晋升阈值（从 ag-mem-35 获取）

| 来源分槽 | L1→L2 I值阈值 | L2→L3 I值阈值 | L3→L4 I值阈值 | L4→L5 I值阈值 |
|----------|:---:|:---:|:---:|:---:|
| ag-mem-15（对话交互） | 0.35 | 0.55 | 0.75 | 0.90 |
| ag-mem-16（工具调用） | 0.40 | 0.60 | 0.80 | 0.90 |
| ag-mem-17（信息检索） | 0.38 | 0.58 | 0.78 | 0.88 |
| ag-mem-18（创作生成） | 0.38 | 0.58 | 0.78 | 0.90 |
| ag-mem-19（通用任务） | 0.42 | 0.62 | 0.82 | 0.92 |

### 三、警示标签拦截规则

| 警示标签 | 允许晋升的目标层级 | 拦截时的处理 |
|----------|:---:|------|
| NORMAL | L1→L2, L2→L3, L3→L4, L4→L5 | 正常参与晋升判定 |
| CAUTION | 仅 L1→L2, L2→L3 | 当目标层级为 L4 时，自动拦截并通知 ag-mem-43 执行安全仲裁 |
| PERMANENT_CAUTION | 仅 L1→L2 | 永久锁定，禁止晋升至 L3 及以上，仅人工可解除 |


## 核心处理逻辑

```
FUNCTION promotion_judge_main_loop():
    STATE_IDLE = IDLE
    STATE_JUDGE = JUDGING
    STATE_OUTPUT = OUTPUTTING
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_IDLE
    加载各层级晋升阈值配置（从 ag-mem-35 获取）
    初始化晋升统计计数器

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_IDLE

        // 第2步：接收晋升候选列表
        IF 收到晋升候选列表:
            SET internal_state = STATE_JUDGE
            源层级 = 列表.源层级
            目标层级 = 根据源层级确定目标层级(源层级)
            晋升候选清单 = []
            拒绝条目列表 = []

            FOR EACH 条目 IN 列表.条目列表:
                // 2a. 获取该条目的分槽专属晋升阈值
                分槽配置 = 晋升阈值配置表[条目.来源分槽编号]
                IF 分槽配置 == None:
                    分槽配置 = 晋升阈值配置表["通用任务槽"]

                // 根据晋升路径选择对应阈值
                CASE (源层级, 目标层级) OF:
                    (L1, L2): 晋升I值阈值 = 分槽配置.L1晋升I值阈值
                    (L2, L3): 晋升I值阈值 = 分槽配置.L2晋升I值阈值
                    (L3, L4): 晋升I值阈值 = 分槽配置.L3晋升I值阈值
                    (L4, L5): 晋升I值阈值 = 分槽配置.L4晋升I值阈值

                // 2b. 校验警示标签
                IF 条目.警示标签 == "PERMANENT_CAUTION":
                    拒绝条目列表.append({条目ID: 条目.条目ID, 拒绝原因: "永久警示标签，禁止晋升"})
                    CONTINUE

                IF 条目.警示标签 == "CAUTION" AND 目标层级 == "L4":
                    // 拦截并通知安全仲裁
                    向 ag-mem-24 发送警示条目拦截通知(条目.条目ID, 源层级, "需先通过安全仲裁")
                    向 ag-mem-43 发送失败经验仲裁请求(条目)
                    拒绝条目列表.append({条目ID: 条目.条目ID, 拒绝原因: "警示标签需先通过安全仲裁"})
                    CONTINUE

                // 2c. 校验留存时长
                最小留存时长 = 获取层级最小留存时长(源层级, 目标层级)
                IF 条目.留存时长 < 最小留存时长:
                    拒绝条目列表.append({条目ID: 条目.条目ID, 拒绝原因: "留存时长不足"})
                    CONTINUE

                // 2d. 校验重要度I值
                IF 条目.I值 < 晋升I值阈值:
                    拒绝条目列表.append({条目ID: 条目.条目ID, 拒绝原因: "I值不满足晋升阈值"})
                    CONTINUE

                // 2e. L4→L5 额外校验
                IF 源层级 == "L4" AND 目标层级 == "L5":
                    IF 条目.S值 < 0.9 AND 条目.规则置信度 < 0.85:
                        拒绝条目列表.append({条目ID: 条目.条目ID, 拒绝原因: "不满足L5特殊晋升条件"})
                        CONTINUE

                // 2f. 通过所有校验，加入晋升候选清单
                晋升候选清单.append({
                    条目ID: 条目.条目ID,
                    I值: 条目.I值,
                    留存时长: 条目.留存时长,
                    晋升原因: "满足双条件: 留存" + 条目.留存时长 + " + I值" + 条目.I值 + " ≥ " + 晋升I值阈值,
                    来源分槽编号: 条目.来源分槽编号
                })

            // 第3步：输出结果
            SET internal_state = STATE_OUTPUT

            IF 晋升候选清单 非空:
                向 ag-mem-39 发送晋升候选清单(源层级, 目标层级, 晋升候选清单)

            向 请求来源模块 返回判定完成回执(
                源层级=源层级,
                判定条目总数=列表.条目列表.size,
                晋升候选数=晋升候选清单.size,
                拒绝数=拒绝条目列表.size,
                拒绝原因分布=统计拒绝原因分布(拒绝条目列表)
            )

            更新晋升统计(源层级, 目标层级, 晋升候选清单.size, 拒绝条目列表.size)
            SET internal_state = STATE_IDLE

        // 第4步：周期性状态上报
        IF 距上次状态上报 >= 120秒:
            向 ag-mem-03 上报判定状态(晋升统计)
            重置统计计数器

        SLEEP 10ms


FUNCTION 获取层级最小留存时长(源层级, 目标层级):
    时长表 = {
        ("L1", "L2"): 24小时,
        ("L2", "L3"): 168小时,
        ("L3", "L4"): 720小时,
        ("L4", "L5"): 2160小时
    }
    RETURN 时长表.GET((源层级, 目标层级), 24小时)
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 条目缺少来源分槽编号（无法确定阈值） | 使用通用任务槽（ag-mem-19）的阈值作为默认值 | 分槽信息补全 |
| 条目I值异常（<0.05或>1.0） | 视为不满足晋升条件，拒绝晋升，记录异常日志 | I值被修正 |
| 警示标签CAUTION且目标为L4 | 自动拦截，向ag-mem-43发起安全仲裁，不等待结果 | 安全仲裁通过后重新参与下次判定 |
| 同一批次中包含重复条目ID | 仅判定一次，以首次出现的为准，后续重复跳过 | — |
| 紧急熔断 | 暂停判定，未完成的条目列表保留，恢复后继续 | 紧急解除后恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| P-01 | 警示标签为 PERMANENT_CAUTION 的条目编译期禁止任何层级的晋升，仅人工可解除 |
| P-02 | 警示标签为 CAUTION 的条目禁止直接晋升至 L4，必须先通过 ag-mem-43 三道安全仲裁 |
| P-03 | 晋升双条件（留存时长 + I值）必须同时满足，任一条不满足即拒绝晋升 |
| P-04 | 晋升阈值必须使用各分槽专属配置，不得使用全局默认值一刀切 |
