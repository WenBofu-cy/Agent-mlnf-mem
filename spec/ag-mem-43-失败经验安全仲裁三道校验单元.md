## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-43 |
| 模块名称 | 失败经验安全仲裁三道校验单元 |
| 所属分区 | 三、漏斗二：任务经验漏斗 / 晋升与遗忘执行机制 |
| 核心职责 | 对结果标签为“失败”或“策略失误”且当前位于 L3 中期层的经验条目，在其晋升至 L4 长期层之前，强制执行三道递进式安全仲裁校验。第一道：安全规则合规校验——向 ag-mem-45（安全规则库）发起请求，验证该经验中涉及的工具调用与操作序列是否违反安全边界；第二道：任务逻辑一致性校验——检查经验中的任务目标、工具选择与执行结果之间是否存在不可调和的逻辑冲突；第三道：模拟复现校验——在隔离的代码执行沙箱中尝试复现工具调用序列的关键步骤，验证其是否仍存在已知的失败模式。仅当三道校验全部通过时，该失败经验方可获得晋升资格，其警示标签从“CAUTION”降级为“NORMAL”；任一校验不通过，经验将被锁定在当前层级并标记为“永久警示”，仅可人工干预解除。不参与晋升决策或数据修改，仅执行安全仲裁判定 |
| 依赖模块 | ag-mem-24（L3中期层存储单元，提供待晋升的失败经验条目与元数据）、ag-mem-45（安全规则库，提供安全规则与合规校验基准）、ag-mem-07（代码执行沙箱，提供隔离的模拟复现环境）、ag-mem-35（三维权重系数配置单元，获取校验超时等配置参数） |
| 被依赖模块 | ag-mem-24（返回安全仲裁结果，通过则允许晋升）、ag-mem-39（层级单向搬运写入单元，接收通过仲裁的条目执行 L3→L4 搬运）、ag-mem-51（记忆变更日志追溯单元，记录仲裁事件） |

## 三道校验详细规则

### 一、第一道：安全规则合规校验

*   **执行方**：向 ag-mem-45（安全规则库）发起同步校验请求。
*   **校验内容**：提取经验条目中涉及的全部工具名称、操作类型（读取/写入/删除/执行）及关键参数。ag-mem-45 在其规则库中逐一比对，检查是否存在违规项。
*   **判定标准**：
    *   **通过**：所有工具调用与操作均在安全规则白名单内，且参数组合未触发任何高风险规则。
    *   **不通过**：存在任一工具调用被安全规则库标记为“高风险且禁止自动执行”，或操作序列与参数组合命中了拒绝规则。
*   **结果处理**：若不通过，直接终止后续校验，返回“永久警示”。

### 二、第二道：任务逻辑一致性校验

*   **执行方**：本模块内置的轻量级逻辑一致性引擎。
*   **校验内容**：分析经验中的任务目标、所选的工具序列及最终的失败结果，检查是否存在明显的逻辑错误。
*   **常见不通过模式**：
    *   工具选择与任务目标完全无关（如：目标为“查询天气”，却调用了“文件删除”工具）。
    *   工具调用顺序存在循环依赖或逻辑死锁。
    *   执行结果的错误码表明为不可恢复的配置错误（如：API 密钥无效），而非运行时偶发错误。
*   **判定标准**：
    *   **通过**：任务目标、工具序列与失败结果之间存在合理的因果链路，失败原因可归因于外部环境或偶发异常。
    *   **不通过**：存在明显的逻辑错误，表明该策略本身不成立，不应被复用或固化。
*   **结果处理**：若不通过，直接终止后续校验，返回“永久警示”。

### 三、第三道：模拟复现校验

*   **执行方**：向 ag-mem-07（代码执行沙箱）发起异步的模拟复现请求。
*   **校验内容**：将经验条目的原始工具调用序列在沙箱中进行隔离重放。沙箱会尝试模拟相同的输入参数，并观察是否仍会产生相同的失败错误码。
*   **判定标准**：
    *   **通过**：沙箱中模拟执行成功（说明原失败为临时环境问题），或虽然失败但产生了新的、不同的错误（说明环境已变化，原策略可能重新有效）。
    *   **不通过**：沙箱中精确复现了完全相同的失败错误码（说明该策略在当前环境下稳定失败，不应晋升）。
*   **结果处理**：若通过，则整体仲裁通过；若不通过，返回“永久警示”。


## 核心处理逻辑

```
FUNCTION failure_arbitration_main_loop():
    STATE_IDLE = IDLE
    STATE_CHECK1 = CHECK_ONE
    STATE_CHECK2 = CHECK_TWO
    STATE_CHECK3 = CHECK_THREE
    STATE_DONE = ARBITRATION_DONE
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_IDLE
    加载校验超时配置

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_IDLE

        // 第2步：接收失败经验仲裁请求
        IF 收到 ag-mem-24 的失败经验仲裁请求:
            IF internal_state != STATE_IDLE:
                将请求加入待处理队列
                CONTINUE

            当前条目ID = 请求.条目ID
            当前经验数据 = 请求.经验数据
            仲裁日志 = 初始化仲裁日志(当前条目ID)

            // --- 第一道校验：安全规则合规 ---
            SET internal_state = STATE_CHECK1
            向 ag-mem-45 发送安全合规校验请求(当前经验数据.工具调用序列)
            等待回执，超时则按“不通过”处理

            IF 回执.是否合规 == False:
                仲裁日志.第一道结果 = "不通过: " + 回执.违规规则列表
                生成永久警示标记()
                CONTINUE  // 跳转至输出结果

            仲裁日志.第一道结果 = "通过"

            // --- 第二道校验：任务逻辑一致性 ---
            SET internal_state = STATE_CHECK2
            逻辑校验结果 = 执行本地逻辑一致性检查(当前经验数据)
            
            IF 逻辑校验结果.是否一致 == False:
                仲裁日志.第二道结果 = "不通过: " + 逻辑校验结果.冲突描述
                生成永久警示标记()
                CONTINUE

            仲裁日志.第二道结果 = "通过"

            // --- 第三道校验：模拟复现 ---
            SET internal_state = STATE_CHECK3
            向 ag-mem-07 发送模拟复现请求(当前经验数据.工具调用序列)
            等待回执，超时则按“通过”处理（超时意味着环境可能已变化，不能证明策略稳定失败）

            IF 回执.是否复现失败 == True:
                仲裁日志.第三道结果 = "不通过: 稳定复现相同错误"
                生成永久警示标记()
                CONTINUE

            仲裁日志.第三道结果 = "通过"

            // --- 全部通过 ---
            仲裁日志.最终结果 = "通过"
            向 ag-mem-39 发送晋升放行通知(当前条目ID, 新警示标签=NORMAL)

        // 第3步：输出结果
        IF 仲裁日志.最终结果 != None:
            SET internal_state = STATE_DONE
            向 ag-mem-24 返回安全仲裁结果(仲裁日志)
            向 ag-mem-51 发送仲裁事件日志(仲裁日志)
            SET internal_state = STATE_IDLE
            // 处理队列中的下一个请求

        SLEEP 10ms


FUNCTION 生成永久警示标记():
    仲裁日志.最终结果 = "拒绝"
    向 ag-mem-24 发送永久警示标记通知(当前条目ID, 新标签=PERMANENT_CAUTION, 拒绝原因摘要)
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 第一道校验超时（ag-mem-45 无响应） | 保守处理，视为“不通过”，标记永久警示 | ag-mem-45 恢复正常 |
| 第二道校验发现不可判定的模糊逻辑 | 保守处理，视为“不通过”，记录详细日志供人工分析 | 人工介入判定 |
| 第三道校验超时（ag-mem-07 沙箱无响应） | 保守处理，视为“通过”（超时通常意味着环境改变，不能判定为稳定失败） | ag-mem-07 恢复正常 |
| 同一失败条目重复申请仲裁（已标记永久警示后） | 直接返回上次仲裁结果，除非人工解除了永久警示标记 | 人工解除标记 |
| 经验数据格式不兼容导致第二道校验失败 | 视为“不通过”，记录异常日志 | 数据格式修复 |
| 紧急熔断 | 中断当前仲裁流程，已部分完成的校验结果丢弃 | 紧急解除后恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| R-01 | 任何一道校验不通过，该失败经验必须被永久锁定在 L3 层，不得晋升至 L4 或 L5 |
| R-02 | 永久警示标记仅可通过人工干预解除，不得被任何自动化模块（包括重要度刷新）修改 |
| R-03 | 模拟复现校验必须在与主系统完全隔离的沙箱环境中执行，防止失败策略影响在线服务 |
| R-04 | 安全仲裁的每一步结果都必须完整记录日志，不可篡改，供事后责任追溯 |
| R-05 | 仲裁请求在系统繁忙时排队处理，但单条目总仲裁时间不得超过配置的最大超时 |
