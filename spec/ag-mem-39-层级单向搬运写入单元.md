# ag-mem-39-层级单向搬运写入单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-39 |
| 模块名称 | 层级单向搬运写入单元 |
| 所属分区 | 三、漏斗二：任务经验漏斗 / 晋升与遗忘执行机制 |
| 核心职责 | 接收 ag-mem-38（晋升双条件判定单元）下发的晋升候选清单，执行经验条目从当前层级向上一层的物理搬运。确保搬运过程严格单向——经验只能从低层级向高层级晋升（L1→L2→L3→L4→L5），禁止任何形式的回退或越级跳转。在搬运完成后，通知源层级存储单元删除已搬运的原始条目（或标记为已晋升），并向目标层级确认写入成功。当搬运失败（如目标层级存储满或写入异常）时，保留原始条目不变并回滚搬运操作。不参与晋升条件判定，仅执行已批准晋升条目的物理搬运 |
| 依赖模块 | ag-mem-38（晋升双条件判定单元，下发晋升候选清单）、ag-mem-20~28（各层级存储单元，执行实际的写入与删除操作）、ag-mem-48（全局容量配额管控单元，确认目标层级容量） |
| 被依赖模块 | ag-mem-38（返回搬运完成回执）、ag-mem-51（记忆变更日志追溯单元，记录晋升事件） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 无搬运任务，等待晋升候选清单 | 系统初始化完成，无待处理任务 |
| 容量校验中 | `CAPACITY_CHECK` | 正在向 ag-mem-48 查询目标层级当前容量是否足以容纳晋升条目 | 收到晋升候选清单 |
| 搬运执行中 | `TRANSFERRING` | 正在逐条执行经验数据从源层级到目标层级的物理搬运 | 容量校验通过 |
| 回滚中 | `ROLLING_BACK` | 搬运过程中发生异常，正在回滚已部分完成的操作 | 搬运写入失败或目标层级返回异常 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 晋升候选清单 | Struct（源层级 + 目标层级 + 条目列表：条目ID + 经验数据 + 重要度I值 + 来源分槽编号 + 晋升原因） | ag-mem-38 晋升双条件判定单元（通过内部调度总线） | 晋升判定完成，存在满足条件的条目时 | **高** |
| 目标层级容量确认回执 | Struct（当前使用率 + 可用空间 + 能否容纳晋升条目数 + 建议分批数） | ag-mem-48 全局容量配额管控单元 | 容量查询时返回 | **高** |
| 目标层级写入确认 | Struct（条目ID + 写入状态 + 分配存储位置） | ag-mem-22/24/26/28 各层级存储单元 | 每条晋升条目写入目标层级完成后 | **高** |
| 源层级删除确认 | Struct（条目ID + 删除状态 + 释放空间量） | ag-mem-20/22/24/26 各层级存储单元 | 每条晋升条目从源层级删除完成后 | **高** |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-mem-01 总控漏斗F0 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 搬运完成回执 | Struct（源层级 + 目标层级 + 成功搬运条目数 + 失败条目列表 + 失败原因 + 搬运耗时 + 各条目新存储位置） | ag-mem-38 晋升双条件判定单元 | 搬运全部完成（含部分失败）时 | **高** |
| 目标层级写入指令 | Struct（条目ID + 经验数据 + 重要度I值 + 来源分槽编号 + 晋升来源层级） | ag-mem-22/24/26/28 各层级存储单元 | 容量校验通过后，逐条发送 | **高** |
| 源层级删除指令 | Struct（条目ID + 删除原因="晋升至{目标层级}" + 是否归档） | ag-mem-20/22/24/26 各层级存储单元 | 目标层级写入成功后发送 | **高** |
| 搬运异常告警 | Struct（异常类型 + 失败条目ID + 源层级 + 目标层级 + 异常详情） | ag-mem-03（调度单元）、ag-mem-51（日志） | 搬运失败或回滚时 | **高** |

## 层级搬运规则

### 一、合法晋升路径

| 源层级 | 合法目标层级 | 禁止的路径 |
|:---:|:---:|------|
| L1 | L2 | 禁止直接晋升至L3/L4/L5 |
| L2 | L3 | 禁止回退至L1、禁止直接晋升至L4/L5 |
| L3 | L4 | 禁止回退至L1/L2、禁止直接晋升至L5 |
| L4 | L5 | 禁止回退至L1/L2/L3 |
| L5 | — | L5为最高层级，不可再晋升 |

### 二、搬运策略

| 规则项 | 说明 |
|------|------|
| 逐条搬运 | 每条条目独立搬运，一条失败不影响其他条目的搬运 |
| 先写后删 | 先将经验数据完整写入目标层级，确认成功后，再从源层级删除 |
| 原子性保证 | 若目标层级写入成功但源层级删除失败，条目在两级同时存在（由后续归并单元清理） |
| 禁止回退 | 编译期禁止任何从高层级向低层级的搬运请求，发现即拒绝 |
| 禁止越级 | 编译期禁止任何跳过中间层级的搬运请求，发现即拒绝 |
| L5搬运特殊要求 | 晋升至L5需额外通过 ag-mem-29 的安全令牌校验 |


## 核心处理逻辑

```
FUNCTION layer_transfer_main_loop():
    STATE_IDLE = IDLE
    STATE_CHECK = CAPACITY_CHECK
    STATE_TRANSFER = TRANSFERRING
    STATE_ROLLBACK = ROLLING_BACK
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_IDLE
    初始化合法晋升路径映射表 = {L1→L2, L2→L3, L3→L4, L4→L5}

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_IDLE

        // 第2步：接收晋升候选清单
        IF 收到 ag-mem-38 晋升候选清单:
            源层级 = 清单.源层级
            目标层级 = 清单.目标层级
            晋升条目列表 = 清单.条目列表

            // 2a. 校验晋升路径合法性
            IF (源层级, 目标层级) NOT IN 合法晋升路径映射表:
                向 ag-mem-38 返回搬运完成回执(
                    成功搬运条目数=0,
                    失败条目列表=晋升条目列表中所有条目ID,
                    失败原因="非法晋升路径: " + 源层级 + " → " + 目标层级
                )
                CONTINUE

            IF 晋升条目列表为空:
                向 ag-mem-38 返回搬运完成回执(成功搬运条目数=0)
                CONTINUE

            // 2b. 容量校验
            SET internal_state = STATE_CHECK
            向 ag-mem-48 发送目标层级容量查询请求(目标层级, 晋升条目列表.size)
            // 等待容量确认回执

        // 第3步：处理容量校验结果
        IF internal_state == STATE_CHECK AND 收到容量确认回执:
            IF NOT 容量确认回执.能否容纳:
                向 ag-mem-38 返回搬运完成回执(
                    成功搬运条目数=0,
                    失败条目列表=晋升条目列表中所有条目ID,
                    失败原因="目标层级容量不足"
                )
                SET internal_state = STATE_IDLE
                CONTINUE

            // 容量充足，开始搬运
            SET internal_state = STATE_TRANSFER
            成功计数 = 0
            失败条目列表 = []
            已写入条目列表 = []  // 用于回滚追踪

            // 逐条搬运
            FOR EACH 条目 IN 晋升条目列表:
                // 3a. 先写入目标层级
                向 目标层级存储单元 发送写入指令(
                    条目ID=条目.条目ID,
                    经验数据=条目.经验数据,
                    重要度I值=条目.重要度I值,
                    来源分槽编号=条目.来源分槽编号,
                    晋升来源层级=源层级
                )
                // 等待目标层级写入确认

        // 第4步：处理目标层级写入确认
        IF internal_state == STATE_TRANSFER AND 收到目标层级写入确认:
            IF 确认.写入状态 == "成功":
                // 写入成功，记录已写入（用于回滚）
                已写入条目列表.append(确认.条目ID)

                // 发送源层级删除指令
                向 源层级存储单元 发送删除指令(
                    条目ID=确认.条目ID,
                    删除原因="晋升至" + 目标层级,
                    是否归档=(源层级 == "L3" AND 目标层级 == "L4")  // L3→L4归档保留
                )
                // 等待源层级删除确认
            ELSE:
                // 写入失败
                失败条目列表.append({条目ID: 确认.条目ID, 失败原因: "目标层级写入失败"})

                // 检查是否还有待处理的写入确认
                IF 已写入条目列表.size + 失败条目列表.size >= 晋升条目列表.size:
                    进入完成处理阶段

        // 第5步：处理源层级删除确认
        IF internal_state == STATE_TRANSFER AND 收到源层级删除确认:
            IF 确认.删除状态 == "成功":
                成功计数++
                // 记录晋升事件日志
                向 ag-mem-51 发送晋升事件日志(确认.条目ID, 源层级, 目标层级)
            ELSE:
                // 删除失败但写入已成功，条目在两级同时存在
                成功计数++  // 仍计为成功，重复条目由归并单元后续处理
                向 ag-mem-51 发送异常日志("晋升删除源条目失败", 确认.条目ID)

            // 检查是否全部处理完成
            IF 成功计数 + 失败条目列表.size >= 晋升条目列表.size:
                进入完成处理阶段

        // 第6步：搬运过程中异常回滚
        IF internal_state == STATE_TRANSFER:
            IF 收到目标层级写入异常 OR 目标层级返回"存储故障":
                SET internal_state = STATE_ROLLBACK

                // 回滚已写入的条目：从目标层级删除
                FOR EACH 已写入条目ID IN 已写入条目列表:
                    向 目标层级存储单元 发送删除指令(
                        条目ID=已写入条目ID,
                        删除原因="搬运回滚",
                        是否归档=False
                    )

                // 未完成的条目全部标记为失败
                FOR EACH 条目 IN 晋升条目列表:
                    IF 条目.条目ID NOT IN 已写入条目列表 AND 条目.条目ID NOT IN 失败条目列表:
                        失败条目列表.append({条目ID: 条目.条目ID, 失败原因: "搬运异常回滚"})

                向 ag-mem-03 发送搬运异常告警("目标层级写入异常，已回滚")

                // 进入完成处理阶段
                SET internal_state = STATE_IDLE
                发送完成回执

        SLEEP 10ms


FUNCTION 进入完成处理阶段():
    // 构建搬运完成回执
    向 ag-mem-38 返回搬运完成回执(
        源层级=源层级,
        目标层级=目标层级,
        成功搬运条目数=成功计数,
        失败条目列表=失败条目列表,
        失败原因=IF 失败条目列表为空 THEN "全部成功" ELSE "部分失败",
        搬运耗时=NOW() - 搬运开始时间,
        各条目新存储位置=已写入条目列表
    )
    SET internal_state = STATE_IDLE
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 非法晋升路径（越级或回退） | 编译期直接拒绝，返回错误 | — |
| 目标层级容量不足 | 拒绝搬运，返回失败通知，由ag-mem-38调整晋升策略 | 目标层级容量释放 |
| 目标层级写入失败（存储满/IO错误） | 该条目标记失败，其他条目继续搬运，若为存储故障则触发全量回滚 | 存储恢复正常 |
| 源层级删除失败 | 条目在两级同时存在，计为成功，由归并单元后续清理 | — |
| 搬运过程中系统断电 | 重启后检测未完成的搬运任务，已写入目标层级的条目保留，源层级条目保留（两级同时存在） | 系统重启后由归并单元清理重复 |
| L4→L5搬运未携带安全令牌 | 拒绝搬运，返回“L5写入需安全令牌” | 获取有效令牌后重试 |
| 紧急熔断 | 中断搬运，保留当前已完成的操作状态，未完成的条目保留在源层级 | 紧急解除后恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| T-01 | 编译期硬编码合法晋升路径（L1→L2→L3→L4→L5），禁止任何形式的回退或越级搬运 |
| T-02 | 搬运过程严格遵循“先写目标，确认成功后，再删源”的顺序，确保经验数据不丢失 |
| T-03 | 目标层级写入失败时，已部分完成的操作必须完整回滚（已写入目标层级的条目全部删除） |
| T-04 | L4→L5晋升必须携带 ag-mem-29 签发的有效安全令牌，否则拒绝搬运 |
| T-05 | 每条搬运操作必须记录完整的晋升事件日志，包括源层级、目标层级、I值与晋升原因 |
