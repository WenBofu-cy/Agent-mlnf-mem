# ag-mem-37-重要度增量定时刷新单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-37 |
| 模块名称 | 重要度增量定时刷新单元 |
| 所属分区 | 三、漏斗二：任务经验漏斗 / 三维重要度计算引擎 |
| 核心职责 | 作为漏斗二中重要度维护的定时驱动器，每24小时对全量经验条目执行一次I值增量刷新。协调 ag-mem-33（C值统计单元）重新计算所有条目的复用频次C值（因时间衰减持续变化），再协调 ag-mem-36（I值聚合单元）基于最新的C值及其他维度分值重新聚合I值。通过定期刷新，确保经验的重要度始终反映最新的调用状态与时间衰减趋势，防止长期未调用但仍有潜在价值的经验因C值自然衰减而被过早遗忘。不参与各维度分值计算，仅执行刷新任务的编排、分批调度与进度监控 |
| 依赖模块 | ag-mem-03（漏斗二专属调度单元，查询漏斗二全量经验条目列表）、ag-mem-33（C值统计单元，执行C值全量刷新）、ag-mem-36（I值聚合单元，执行I值全量重算）、ag-mem-35（三维权重系数配置单元，获取刷新周期等配置参数） |
| 被依赖模块 | ag-mem-03（接收刷新完成回执与周期性状态上报）、ag-mem-33（接收C值全量刷新指令）、ag-mem-36（接收I值全量重算指令） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 无刷新任务，等待定时或手动触发 | 系统初始化完成，距上次刷新不足24小时 |
| 扫描条目中 | `SCANNING` | 正在向 ag-mem-03 查询全量经验条目列表 | 定时触发或收到手动刷新指令 |
| C值刷新中 | `C_REFRESHING` | 已将全量条目列表分批发送至 ag-mem-33，等待C值全量更新完成 | 条目列表扫描完成 |
| I值重算中 | `I_RECALC` | C值刷新已全部完成，正在分批触发 ag-mem-36 进行I值重算 | ag-mem-33 返回C值全量刷新完成回执 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 定时刷新触发信号 | 内部定时器信号 | 本模块内部定时器 | 距上次刷新完成 ≥ 24小时 | 普通 |
| 手动刷新指令 | Struct（刷新范围 + 优先级 + 是否强制执行） | 系统管理接口 | 管理员手动触发维护时 | 普通 |
| 全量条目列表查询回执 | List（条目ID + 当前层级 + 来源分槽编号 + 当前I值 + 当前C值） | ag-mem-03 漏斗二专属调度单元 | 响应扫描请求时 | **高** |
| C值全量刷新完成回执 | Struct（刷新条目数 + 刷新耗时 + 异常条目列表） | ag-mem-33 C值统计单元 | C值全量更新完成时 | **高** |
| I值全量重算完成回执 | Struct（重算条目数 + 重算耗时 + I值变化统计） | ag-mem-36 I值聚合单元 | 每批I值重算完成时 | **高** |
| 刷新周期配置 | Struct（刷新间隔_h + 每批最大条目数 + 批次间隔_ms） | ag-mem-35 三维权重系数配置单元 | 系统初始化加载 | 普通 |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-mem-01 总控漏斗F0 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| C值全量刷新指令 | Struct（刷新范围 + 条目ID列表 + 触发原因 + 批次号） | ag-mem-33 C值统计单元 | 条目列表扫描完成，分批发送 | **高** |
| I值全量重算指令 | Struct（重算范围 + 条目ID列表 + 触发原因 + 批次号） | ag-mem-36 I值聚合单元 | C值刷新完成，分批发送 | **高** |
| 刷新完成回执 | Struct（刷新总条目数 + C值更新数 + I值变化数 + 异常条目数 + 总耗时） | ag-mem-03（调度单元）、管理接口 | 全量刷新全部完成时 | 普通 |
| 刷新进度上报 | Struct（当前状态 + 已完成批次数/总批次数 + 当前阶段 + 预计剩余时间） | ag-mem-03 漏斗二专属调度单元 | 每完成一批或每30秒周期上报 | 普通 |


## 刷新调度配置

| 配置项 | 默认值 | 说明 |
|--------|:---:|------|
| 定时刷新间隔 | **24小时** | 每24小时自动触发一次全量刷新 |
| 每批最大条目数 | **500条** | 每批发送给计算模块的条目数上限 |
| 批次间隔时间 | **200ms** | 每批之间的等待间隔，避免冲击下游模块 |
| C值刷新超时 | **300秒** | 等待ag-mem-33完成全量C值刷新的最长时间 |
| I值重算超时 | **600秒** | 等待ag-mem-36完成全量I值重算的最长时间 |
| 手动刷新冷却时间 | **1小时** | 两次手动刷新之间的最小间隔，防止频繁操作 |


## 核心处理逻辑

```
FUNCTION i_refresh_scheduler_main_loop():
    STATE_IDLE = IDLE
    STATE_SCAN = SCANNING
    STATE_C = C_REFRESHING
    STATE_I = I_RECALC
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_IDLE
    加载刷新周期配置（刷新间隔、每批条目数、批次间隔）
    初始化上次刷新完成时间 = NOW()
    初始化手动刷新冷却计时器 = 0

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_IDLE

        // 第2步：检测定时刷新触发
        IF internal_state == STATE_IDLE:
            距上次刷新 = NOW() - 上次刷新完成时间
            IF 距上次刷新 >= 24小时:
                SET internal_state = STATE_SCAN
                触发原因 = "定时刷新"
                开始刷新流程(触发原因)

        // 第3步：检测手动刷新触发
        IF 收到手动刷新指令 AND internal_state == STATE_IDLE:
            IF NOW() - 手动刷新冷却计时器 < 1小时:
                向管理接口返回拒绝通知("手动刷新冷却中，距上次手动刷新不足1小时")
                CONTINUE

            SET internal_state = STATE_SCAN
            触发原因 = "手动刷新"
            手动刷新冷却计时器 = NOW()
            开始刷新流程(触发原因)

        // 第4步：扫描条目列表阶段
        IF internal_state == STATE_SCAN:
            向 ag-mem-03 发送全量条目列表查询请求
            等待条目列表返回
            IF 返回超时 OR 条目列表为空:
                向 ag-mem-03 上报刷新失败("条目列表查询失败")
                SET internal_state = STATE_IDLE
                CONTINUE

            全量条目列表 = 返回的条目列表
            总分批数 = CEIL(全量条目列表.size / 500)

            // 进入C值刷新阶段
            SET internal_state = STATE_C
            当前批次号 = 0
            C值刷新开始时间 = NOW()

            // 发送第一批C值刷新指令
            发送下一批C值刷新指令(全量条目列表, 当前批次号)

        // 第5步：C值刷新阶段（逐批发送）
        IF internal_state == STATE_C:
            // 收到上一批C值刷新完成回执
            IF 收到 ag-mem-33 C值全量刷新完成回执:
                IF 回执.批次号 == 当前批次号:
                    当前批次号++
                    IF 当前批次号 < 总分批数:
                        // 发送下一批
                        等待批次间隔(200ms)
                        发送下一批C值刷新指令(全量条目列表, 当前批次号)
                    ELSE:
                        // C值刷新全部完成
                        SET internal_state = STATE_I
                        当前批次号 = 0
                        I值重算开始时间 = NOW()
                        发送下一批I值重算指令(全量条目列表, 当前批次号)

            // C值刷新超时检测
            IF NOW() - C值刷新开始时间 > 300秒:
                向 ag-mem-03 上报刷新失败("C值刷新超时")
                SET internal_state = STATE_IDLE

        // 第6步：I值重算阶段（逐批发送）
        IF internal_state == STATE_I:
            IF 收到 ag-mem-36 I值全量重算完成回执:
                IF 回执.批次号 == 当前批次号:
                    当前批次号++
                    IF 当前批次号 < 总分批数:
                        等待批次间隔(200ms)
                        发送下一批I值重算指令(全量条目列表, 当前批次号)
                    ELSE:
                        // I值重算全部完成
                        上次刷新完成时间 = NOW()
                        总耗时 = NOW() - (C值刷新开始时间)
                        向 ag-mem-03 发送刷新完成回执(
                            刷新总条目数=全量条目列表.size,
                            总耗时=总耗时
                        )
                        SET internal_state = STATE_IDLE

            // I值重算超时检测
            IF NOW() - I值重算开始时间 > 600秒:
                向 ag-mem-03 上报刷新失败("I值重算超时")
                SET internal_state = STATE_IDLE

        // 第7步：进度上报
        IF internal_state IN [STATE_C, STATE_I]:
            IF 距上次进度上报 >= 30秒:
                向 ag-mem-03 上报刷新进度(
                    当前状态=internal_state,
                    已完成批次数=当前批次号,
                    总批次数=总分批数,
                    当前阶段=IF internal_state == STATE_C THEN "C值刷新" ELSE "I值重算"
                )

        SLEEP 100ms


FUNCTION 开始刷新流程(触发原因):
    记录刷新开始日志(触发原因, NOW())
    向 ag-mem-03 上报刷新状态("开始", 触发原因)


FUNCTION 发送下一批C值刷新指令(全量条目列表, 批次号):
    起始索引 = 批次号 × 500
    结束索引 = MIN(起始索引 + 500, 全量条目列表.size)
    当前批条目 = 全量条目列表[起始索引:结束索引]

    向 ag-mem-33 发送C值全量刷新指令(
        刷新范围="全量",
        条目ID列表=当前批条目,
        触发原因=触发原因,
        批次号=批次号
    )


FUNCTION 发送下一批I值重算指令(全量条目列表, 批次号):
    起始索引 = 批次号 × 500
    结束索引 = MIN(起始索引 + 500, 全量条目列表.size)
    当前批条目 = 全量条目列表[起始索引:结束索引]

    向 ag-mem-36 发送I值全量重算指令(
        重算范围="全量",
        条目ID列表=当前批条目,
        触发原因=触发原因,
        批次号=批次号
    )
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 全量条目列表查询超时或返回空 | 上报刷新失败，等待下一个刷新周期 | 下次定时触发或手动触发 |
| C值刷新某批次失败 | 记录失败批次与条目，跳过该批继续下一批，最终回执中标记异常条目数 | — |
| I值重算某批次失败 | 同上 | — |
| C值刷新总耗时超过300秒 | 强制进入I值重算阶段（已完成批次有效，未完成批次标记为超时） | 下次刷新周期补全 |
| I值重算总耗时超过600秒 | 结束刷新，上报部分完成结果 | 下次刷新周期补全 |
| 刷新过程中系统资源紧张 | 增大批次间隔至500ms，降低系统负载 | 资源恢复正常 |
| 手动刷新冷却期内再次触发 | 拒绝手动刷新请求，提示冷却剩余时间 | 冷却期结束 |
| 紧急熔断 | 中断当前刷新流程，已完成的批次有效，未完成的批次等待下次刷新 | 紧急解除后恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| R-01 | 刷新过程不得修改经验条目的任何内容数据，仅触发数值更新（C值与I值） |
| R-02 | 每批条目数硬限制为500条，批次间隔默认200ms，防止批量操作冲击下游存储与计算模块 |
| R-03 | 刷新过程中不得跳过L4/L5层条目，确保长期与核心经验的重要度也得到及时更新 |
| R-04 | 手动刷新必须有1小时冷却时间，防止管理员误操作频繁触发全量刷新导致系统负载过高 |
| R-05 | 刷新超时后已完成的批次结果有效，未完成批次在下次刷新周期自动补全 |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-M37-01 | `IDLE`，距上次刷新>=24小时 | 定时触发信号 | 进入SCANNING→查询条目列表→分批C值刷新→分批I值重算→完成回执 |
| TC-M37-02 | `IDLE`，管理员手动触发 | 手动刷新指令 | 同上流程，记录手动触发标记，开始冷却计时 |
| TC-M37-03 | `IDLE`，手动刷新冷却期内 | 手动刷新指令（距上次手动刷新<1小时） | 拒绝请求，返回冷却剩余时间 |
| TC-M37-04 | `C_REFRESHING`，某批次C值刷新失败 | ag-mem-33返回某批次失败回执 | 记录失败批次与条目，跳过该批，继续下一批 |
| TC-M37-05 | `I_RECALC`，I值重算超时600秒 | 重算耗时超过600秒 | 结束刷新，上报部分完成结果（已完成批次有效） |
| TC-M37-06 | `C_REFRESHING`，收到紧急熔断 | 紧急熔断指令 | 中断刷新，已完成批次有效，进入暂停状态 |
