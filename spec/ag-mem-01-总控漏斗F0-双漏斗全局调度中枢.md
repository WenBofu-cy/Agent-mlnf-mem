## V1.1 模块升级总说明
### V1.1 重大升级变更点
1. **废弃V1.0固定5分槽静态管理**，顶层新增**动态子漏斗全生命周期全局管控**，统一管理profile/experience两类漏斗新建、合并、配额上限、闲置回收，支撑MLNF-Mem V2.3宏观自收敛架构；
2. **新增全局哈希索引资源池调度能力**，统一分配独立索引桶、管控索引总容量、漏斗合并自动合并/销毁索引，保障全系统记忆检索性能；
3. 对外提供全局存量funnel_id查询接口，为ag-mem-14路由、ag-mem-38晋升判定、五层存储单元提供漏斗合法性校验数据源；
4. 统一管理各funnel专属分层晋升阈值，替换原有固定分槽静态阈值表，实现不同领域经验差异化沉淀；
5. 新增全局晋升统计汇总能力，接收ag-mem-38分层晋升、拦截数据，周期性上报ECC全局记忆沉淀健康度；
6. 扩展输入输出总线结构，新增funnel_id、index_bucket、hash_tag、配额占用、晋升统计核心传输字段，全记忆模块联动兼容；
7. 状态机、主循环逻辑完整保留原有ECC读写、熔断、容量维护流程，仅新增漏斗/索引调度分支，无原有业务能力丢失；
8. 新增全局资源安全约束，强制8/32漏斗配额上限、索引桶物理隔离，杜绝记忆无限膨胀、数据标签串读污染；
9. 自检、告警、维护周期扩展适配动态漏斗与哈希索引，自动合并闲置漏斗、清理低频索引标签，降低存储资源占用。

## V1.1 全局核心管控规则（动态子漏斗+哈希索引）
### 一、动态子漏斗配额全局约束（顶层固定，不可修改）
1. `profile`（用户画像类）最大子漏斗数量：8
2. `experience`（任务经验类）最大子漏斗数量：32
3. 配额预警阈值：大类数量达到上限80%时触发提前合并维护
4. 闲置判定规则：子漏斗连续30天无读写访问，维护周期自动合并至最相似漏斗，释放配额

### 二、全局哈希索引资源管控规则
1. 每个动态子漏斗绑定唯一独立`index_bucket_id`，索引物理隔离，禁止跨漏斗共享索引桶
2. 全局哈希索引总容量上限由ag-mem-48统一管控，单漏斗索引条目上限可配置
3. 漏斗合并时自动合并双方hash_tag集合、去重，合并后销毁原源漏斗索引桶，回收索引资源
4. 维护周期自动清理全系统低频访问哈希标签，释放索引存储空间

### 三、全局晋升阈值分发规则
1. 新建子漏斗自动加载通用基准阈值；长期高频使用的漏斗可动态微调阈值，存入ag-mem-35
2. ag-mem-38晋升单元每次判定前，必须向本模块拉取对应funnel_id专属阈值，禁止使用全局硬编码固定分槽阈值
3. 全局统一管控result_validated强制校验规则：L2及以上晋升必须携带有效执行结果标记，规则全局下发

## 核心处理逻辑（V1.1完整伪代码，新增动态漏斗+哈希索引全生命周期调度）
```
FUNCTION f0_dispatch_main_loop():
    STATE_INIT = INIT
    STATE_IDLE = IDLE
    STATE_NORMAL = NORMAL_DISPATCH
    STATE_MAINT = MAINTENANCE
    STATE_MELTDOWN = MELTDOWN

    SET internal_state = STATE_INIT
    // V1.1冷启动新增全局资源初始化
    初始化全局动态漏斗注册表 = {}  # key:funnel_id, value:漏斗元数据(分类、领域标签、索引桶ID、创建时间、活跃度)
    初始化全局哈希索引资源池 = {}   # key:index_bucket_id, value:索引容量、关联funnel_id、标签集合
    初始化请求队列 = []
    初始化全局晋升统计汇总表 = {}

    // 第0步：冷启动自检（V1.1新增漏斗/索引完整性校验）
    加载全局配额、晋升基准阈值、索引容量配置
    校验存储分区、哈希索引底层存储完整性
    向 ag-mem-02/03 发送初始化指令，同步全局漏斗注册表快照
    等待各子漏斗就绪回执
    IF 自检通过:
        SET internal_state = STATE_IDLE
    ELSE:
        向 ECC 上报告警，标记降级模块、损坏漏斗/索引桶

    WHILE 系统运行中:
        // ==================== 优先级1：紧急熔断 ====================
        IF 收到紧急熔断指令:
            SET internal_state = STATE_MELTDOWN
            清空写入/漏斗新建请求队列，只读查询保留
            向所有子模块广播熔断信号，冻结所有漏斗新建、索引变更、写入操作
            CONTINUE

        IF 收到恢复指令 AND internal_state == STATE_MELTDOWN:
            重新校验全局漏斗注册表、索引资源池完整性
            SET internal_state = STATE_IDLE

        // ==================== 优先级2：容量/索引/漏斗配额告警维护触发 ====================
        IF 收到容量告警 OR 索引容量告警 OR 漏斗配额预警:
            向 ag-mem-03 发送低重要度记忆清理指令
            IF 子漏斗数量达到上限80% OR 索引容量>90%:
                SET internal_state = STATE_MAINT
                // 触发闲置子漏斗合并、过期索引标签清理
                向 ag-mem-03、ag-mem-14 下发闲置漏斗合并调度指令
                暂停接收新漏斗创建申请，优先资源回收
            向 ECC 上报全局资源告警通知

        // ==================== 优先级3：处理动态漏斗&哈希索引生命周期申请（V1.1新增最高优先级业务） ====================
        IF 收到ag-mem-14动态漏斗生命周期申请:
            IF internal_state == STATE_MELTDOWN AND 操作类型 == "create":
                返回操作回执：熔断禁止新建漏斗
                CONTINUE

            分类 = 请求.funnel_category
            操作 = 请求.操作类型
            IF 操作 == "create":
                // 配额校验
                当前存量数量 = 统计注册表中分类下漏斗总数
                上限 = 8 if 分类=="profile" else 32
                IF 当前存量数量 >= 上限:
                    返回回执：配额已满，禁止新建，需先执行漏斗合并
                    CONTINUE
                // 分配全新funnel_id、独立索引桶ID
                new_funnel_id = 生成唯一funnel_id()
                new_index_bucket = 生成独立索引桶ID(new_funnel_id)
                // 写入全局注册表、索引资源池
                全局动态漏斗注册表[new_funnel_id] = {
                    category:分类, domain_tags:请求.领域标签, index_bucket:new_index_bucket, create_time:当前时间, active:True
                }
                全局哈希索引资源池[new_index_bucket] = {funnel_id:new_funnel_id, tag_list:[], used_size:0}
                // 下发新建回执至ag-mem-14
                回执数据 = {
                    request_id:请求.request_id, funnel_id:new_funnel_id, funnel_category:分类,
                    index_bucket_id:new_index_bucket, is_new_created:True,
                    quota_remain:上限 - 当前存量数量 - 1
                }
                向 ag-mem-14 返回动态漏斗操作回执(回执数据)

            ELIF 操作 == "merge":
                // 合并源漏斗至目标漏斗，回收源索引资源
                source_id = 请求.source_funnel_id
                target_id = 请求.target_funnel_id
                target_index = 全局动态漏斗注册表[target_id].index_bucket
                // 下发合并指令至存储层，合并哈希标签
                向 ag-mem-03、五层存储单元下发漏斗合并调度指令(source_id, target_id)
                // 销毁源漏斗、回收索引桶
                删除 全局动态漏斗注册表[source_id]
                回收索引桶至资源池，清空标签集合
                // 返回合并完成回执
                回执数据 = {request_id:请求.request_id, funnel_id:target_id, index_bucket_id:target_index, is_new_created:False}
                向 ag-mem-14 返回动态漏斗操作回执(回执数据)

            ELIF 操作 == "query":
                // 返回全量存量漏斗ID列表，供晋升、存储模块校验合法性
                all_funnel_ids = list(全局动态漏斗注册表.keys())
                回执数据 = {request_id:请求.request_id, funnel_list:all_funnel_ids}
                向 ag-mem-14、ag-mem-38 返回动态漏斗操作回执(回执数据)

        // ==================== 优先级4：接收 ECC 读写业务请求 ====================
        IF 收到 ECC 请求:
            IF internal_state IN [STATE_INIT, STATE_MELTDOWN]:
                向 ECC 返回拒绝回执（原因=当前状态不可服务/熔断禁止写入）
                CONTINUE
            SET internal_state = STATE_NORMAL_DISPATCH

            // 3a. 请求类型判定与路由
            CASE 请求.类型 OF:
                "经验查询":
                    IF 请求.用户ID 存在:
                        // 同时查询profile画像漏斗、experience经验漏斗，支持指定funnel_id过滤
                        向 ag-mem-02 转发用户偏好查询(携带可选funnel_id过滤)
                        向 ag-mem-03 转发任务经验查询(携带可选funnel_id过滤)
                        等待双漏斗返回结果，附带关联哈希索引桶信息
                        合并查询结果，按重要度倒序排序
                        向 ECC 返回经验查询回执(附带匹配funnel_id集合、索引桶)
                    ELSE:
                        向 ECC 返回空结果回执

                "经验写入":
                    // 交由ag-mem-14动态路由单元匹配funnel_id，本模块仅做顶层分发
                    IF 写入数据包含用户偏好标签:
                        向 ag-mem-02 转发写入请求(携带待匹配领域文本)
                    ELSE:
                        向 ag-mem-03 转发写入请求(携带待匹配领域文本、result_validated标记)
                    等待写入确认（携带分配funnel_id、hash_tag_list）
                    向 ECC 返回经验写入确认回执

                "用户画像查询":
                    向 ag-mem-02 转发画像查询(可选funnel_id过滤)
                    等待返回偏好数据，携带对应funnel_id
                    向 ECC 回传画像数据回执

                "知识库查询":
                    向 ag-mem-44 转发独立知识库查询
                    向 ag-mem-45 转发安全规则查询（如需要）
                    合并外挂模块查询结果，回传给ECC

        // ==================== 优先级5：接收晋升统计上报，全局汇总 ====================
        IF 收到ag-mem-38晋升统计上报:
            按funnel_id聚合晋升、拦截数据，更新全局晋升统计汇总表

        // ==================== 优先级6：周期性全局状态汇总上报 ====================
        IF 距上次状态上报 >= 1秒:
            // V1.1新增漏斗、索引全局统计
            profile_total = 统计分类profile下漏斗总数
            exp_total = 统计分类experience下漏斗总数
            索引总占用 = 求和全局哈希索引资源池全部used_size
            全局健康快照 = {
                profile_funnel_count:profile_total, profile_max:8,
                exp_funnel_count:exp_total, exp_max:32,
                total_storage_used:ag-mem-48获取总存储占用,
                total_hash_index_used:索引总占用,
                promotion_global_stat:全局晋升统计汇总表
            }
            向 ECC 发送双漏斗全局健康状态汇总(全局健康快照)

        // ==================== 优先级7：定时全系统维护扫描 ====================
        IF 距上次维护扫描 >= 24小时:
            SET internal_state = STATE_MAINT
            // 下发晋升、遗忘扫描指令
            向 ag-mem-03 发送晋升扫描指令（联动ag-mem-38）
            向 ag-mem-03 发送遗忘扫描指令
            // V1.1新增闲置漏斗合并、过期索引清理
            筛选连续30天无访问的闲置漏斗列表
            IF 闲置漏斗列表非空:
                向 ag-mem-14 下发批量漏斗合并调度指令(闲置漏斗)
            向 五层存储单元 下发哈希低频标签清理指令
            等待全量维护任务完成
            SET internal_state = STATE_IDLE

        SLEEP 10ms
```


## 约束与异常处理（V1.1新增动态漏斗、索引专属异常分支）
| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 冷启动自检发现损坏漏斗/索引桶 | 标记故障漏斗为只读，禁止写入、新建，上报告警 | 底层存储修复，重新执行全量自检 |
| ECC 请求超时未响应 | 等待超时（500ms）后返回部分结果/超时回执，不阻塞全局调度 | 下一请求正常处理 |
| 漏斗一/漏斗二调度单元不可用 | 仅使用可用漏斗返回数据，回执标记结果不完整 | 故障调度单元恢复自检通过 |
| 维护模式下收到写入/新建漏斗请求 | 写入请求暂存队列，漏斗新建直接拒绝；维护结束后批量处理队列 | 维护模式退出（闲置漏斗合并、索引清理完成） |
| 全局熔断触发 | 冻结所有漏斗新建、索引变更、记忆写入；仅开放只读查询、存量数据读取 | 收到恢复指令+全量漏斗/索引自检通过 |
| ag-mem-14申请新建漏斗但配额已满 | 拒绝新建申请，返回回执提示执行闲置漏斗合并释放配额 | 执行合并后配额出现空余 |
| 哈希索引总容量逼近上限 | 提前进入维护模式，自动清理低频标签释放索引空间 | 索引清理完成，占用率回落至阈值下 |
| 晋升单元查询不存在funnel_id | 返回空漏斗列表，晋升单元自动走通用兜底阈值 | 上层路由模块补全合法funnel_id写入记忆条目 |

## 安全边界（V1.1新增动态漏斗、索引隔离安全规则）
| 规则编号 | 内容 |
|:---:|------|
| S-01 | 本模块为 Agent-mlnf-mem 双漏斗记忆中枢唯一对外入口，所有 ECC 记忆请求必须经本模块接收路由，禁止任何底层存储/路由模块直连外部ECC |
| S-02 | 漏斗一profile用户画像动态子漏斗数据编译期物理隔离，禁止画像索引、记忆条目流入experience经验漏斗，跨漏斗读取必须经过权限校验 |
| S-03 | 熔断状态下仅开放存量数据只读查询，永久冻结所有漏斗新建、索引桶分配、记忆写入、漏斗合并变更操作 |
| S-04 | 全局动态漏斗注册表、哈希索引资源池仅本模块具备读写权限，其他模块仅可只读查询，禁止底层模块自行创建/销毁漏斗、索引桶 |
| S-05 | 冷启动自检未通过时，必须标记损坏漏斗、索引桶、降级模块并向ECC完整上报告警，禁止伪装全量正常对外服务 |
| S-06 | 子漏斗配额严格执行全局上限8/32，任何场景不得绕过本模块直接新建漏斗，杜绝无限膨胀破坏宏观自收敛机制 |
| S-07 | 每个动态子漏斗绑定独立哈希索引桶，禁止跨漏斗共享索引资源，防止标签串读、检索数据污染 |
| S-08 | 闲置漏斗合并、索引清理仅允许在MAINTENANCE维护周期执行，业务高峰NORMAL_DISPATCH状态禁止执行资源销毁操作 |
