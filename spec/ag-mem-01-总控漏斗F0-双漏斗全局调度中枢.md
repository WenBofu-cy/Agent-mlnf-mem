# ag-mem-01-总控漏斗F0-双漏斗全局调度中枢 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-01 |
| 模块名称 | 总控漏斗F0-双漏斗全局调度中枢 |
| 所属分区 | 一、顶层总控中枢 |
| 核心职责 | 作为 Agent-mlnf-mem 双漏斗记忆中枢的全局唯一调度入口，接收 ECC 认知大脑通过 MemoryBus 下发的经验查询与写入请求，完成意图解析、优先级仲裁、双漏斗路由分发。管理漏斗一（用户画像）与漏斗二（任务经验）的资源分配与模式切换，协调漏斗外挂扩展区的只读查询。汇总各子漏斗运行状态，统一向 ECC 回传查询回执与健康状态。不参与任何认知决策，仅执行记忆管理的调度与路由 |
| 依赖模块 | ag-mem-02（漏斗一专属调度单元，用户画像槽管理）、ag-mem-03（漏斗二专属调度单元，任务经验分槽路由）、ag-mem-44（独立知识库，外挂查询）、ag-mem-45（安全规则库，外挂查询）、ag-mem-48（全局容量配额管控单元，容量监控） |
| 被依赖模块 | ECC 认知大脑（通过 MemoryBus 下发查询/写入请求）、ag-mem-02 至 ag-mem-51（全部记忆模块统一调度） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 初始化 | `INIT` | 系统冷启动，加载配置，校验存储完整性 | 系统上电/重启 |
| 就绪空闲 | `IDLE` | 系统就绪，等待 ECC 查询请求 | 初始化完成，无待处理请求 |
| 正常调度 | `NORMAL_DISPATCH` | 接收并处理 ECC 请求，路由至对应漏斗 | 收到 ECC 经验查询或写入请求 |
| 维护模式 | `MAINTENANCE` | 执行记忆整理、晋升扫描、遗忘清理 | 定时触发或容量逼近上限 |
| 熔断 | `MELTDOWN` | 全局紧急熔断，拒绝所有请求 | 收到紧急熔断信号或存储故障 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| ECC 经验查询请求 | Struct（查询类型 + 用户ID + 任务场景标签 + 查询条件 + 最大返回条数） | ag-ecc-05 记忆查询模块（通过 MemoryBus） | ECC 需要检索历史经验时 | **高** |
| ECC 经验写入请求 | Struct（写入类型 + 用户ID + 任务场景标签 + 经验条目数据 + 重要度预估值） | ag-ecc-06 结果评估模块（通过 MemoryBus） | 任务执行完成，产生新经验时 | **高** |
| 用户画像查询请求 | Struct（用户ID + 查询维度 + 最大返回条数） | ag-ecc-01 意图解析模块（通过 MemoryBus） | 需要用户偏好辅助意图解析时 | **高** |
| 冷启动自检指令 | Struct（自检范围 + 深度） | ag-ecc-12 资源调度模块（通过 MemoryBus） | 系统上电初始化时 | **高** |
| 各子漏斗状态上报 | Struct（模块编号 + 当前状态 + 存储使用率 + 健康标记） | ag-mem-02/03/48 | 周期性（1Hz） | 普通 |
| 容量告警 | Struct（当前容量 + 阈值 + 建议动作） | ag-mem-48 全局容量配额管控单元 | 存储逼近上限时 | **高** |
| 紧急熔断指令 | Struct（熔断等级 + 原因 + 时间戳） | ag-ecc-12 资源调度模块（通过 MemoryBus） | 系统检测到严重异常时 | **最高** |

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 经验查询回执 | Struct（匹配经验列表 + 置信度 + 来源漏斗 + 查询耗时） | ag-ecc-05 记忆查询模块（通过 MemoryBus） | 查询完成时 | **高** |

| 冷启动自检回执 | Struct（自检结果 + 可用模块清单 + 存储完整性校验） | ag-ecc-12 资源调度模块（通过 MemoryBus） | 冷启动自检完成时 | **高** |


## 核心处理逻辑

```
FUNCTION f0_dispatch_main_loop():
    STATE_INIT = INIT
    STATE_IDLE = IDLE
    STATE_NORMAL = NORMAL_DISPATCH
    STATE_MAINT = MAINTENANCE
    STATE_MELTDOWN = MELTDOWN

    SET internal_state = STATE_INIT
    初始化双漏斗资源分配表
    初始化请求队列 = []

    // 第0步：冷启动自检
    加载配置参数
    校验存储分区完整性
    向 ag-mem-02/03 发送初始化指令
    等待各子漏斗就绪回执
    IF 自检通过:
        SET internal_state = STATE_IDLE
    ELSE:
        向 ECC 上报告警，标记降级模块

    WHILE 系统运行中:
        // ==================== 优先级1：紧急熔断 ====================
        IF 收到紧急熔断指令:
            SET internal_state = STATE_MELTDOWN
            清空请求队列
            向所有子模块广播熔断信号
            CONTINUE

        IF 收到恢复指令 AND internal_state == STATE_MELTDOWN:
            SET internal_state = STATE_IDLE

        // ==================== 优先级2：容量维护触发 ====================
        IF 收到容量告警:
            向 ag-mem-03 发送低重要度清理指令
            向 ECC 上报容量告警通知
            IF 容量使用率 > 95%:
                SET internal_state = STATE_MAINT
                暂停接收写入请求，优先清理

        // ==================== 优先级3：接收 ECC 请求 ====================
        IF 收到 ECC 请求:
            IF internal_state IN [STATE_INIT, STATE_MELTDOWN]:
                向 ECC 返回拒绝回执（原因=当前状态不可服务）
                CONTINUE

            // 3a. 请求类型判定与路由
            CASE 请求.类型 OF:
                "经验查询":
                    IF 请求.用户ID 存在:
                        // 同时查询漏斗一（用户偏好）和漏斗二（任务经验）
                        向 ag-mem-02 转发用户偏好查询
                        向 ag-mem-03 转发任务经验查询
                        等待双漏斗返回结果
                        合并查询结果，按重要度排序
                        向 ECC 返回经验查询回执
                    ELSE:
                        向 ECC 返回空结果

                "经验写入":
                    判定写入目标漏斗:
                        IF 写入数据包含用户偏好标签:
                            向 ag-mem-02 转发写入请求
                        ELSE:
                            向 ag-mem-03 转发写入请求
                    等待写入确认
                    向 ECC 返回经验写入确认

                "用户画像查询":
                    向 ag-mem-02 转发查询
                    等待返回
                    向 ECC 回传画像数据

                "知识库查询":
                    向 ag-mem-44 转发查询
                    向 ag-mem-45 转发安全规则查询（如需要）
                    合并外挂模块查询结果
                    向 ECC 回传

        // ==================== 优先级4：周期性状态汇总 ====================
        IF 距上次状态上报 >= 1秒:
            汇总 ag-mem-02/03/48 上报的各子漏斗状态
            向 ECC 发送双漏斗健康状态汇总

        // ==================== 优先级5：定时维护扫描 ====================
        IF 距上次维护扫描 >= 24小时:
            SET internal_state = STATE_MAINT
            向 ag-mem-03 发送晋升扫描指令（ag-mem-38~43）
            向 ag-mem-03 发送遗忘扫描指令
            等待扫描完成
            SET internal_state = STATE_IDLE

        SLEEP 10ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 冷启动自检失败 | 标记故障模块为降级，部分功能不可用，上报告警 | 故障模块恢复并重新自检通过 |
| ECC 请求超时未响应 | 等待超时（500ms）后返回部分结果或超时回执 | 下一请求正常处理 |
| 漏斗一/漏斗二不可用 | 使用可用漏斗的数据返回，标记结果不完整 | 故障漏斗恢复 |
| 写入请求在维护模式 | 暂存写入队列，维护完成后批量写入 | 维护模式退出 |
| 紧急熔断 | 清空请求队列，拒绝所有请求，仅接收恢复信号 | 收到恢复指令且系统自检通过 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| MemoryBus | 读 | ECC 经验查询/写入请求 | 只读 | ag-ecc-05/06 下发 |
| MemoryBus | 读 | 用户画像查询请求 | 只读 | ag-ecc-01 下发 |
| MemoryBus | 读 | 冷启动自检指令 | 只读 | ag-ecc-12 下发 |
| MemoryBus | 读 | 紧急熔断指令 | 只读 | ag-ecc-12 下发 |
| MemoryBus | 写 | 经验查询回执 | 专属写入 | 向 ag-ecc-05 返回 |
| MemoryBus | 写 | 经验写入确认 | 专属写入 | 向 ag-ecc-06 返回 |
| MemoryBus | 写 | 双漏斗健康状态汇总 | 周期性写入 | 向 ag-ecc-12 返回 |
| 内部调度总线 | 写 | 全局调度指令 | 广播 | 向 ag-mem-02/03/48 发送 |
| 内部调度总线 | 读 | 各子漏斗状态上报 | 只读 | 各子模块周期性上报 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 本模块为 Agent-mlnf-mem 双漏斗记忆中枢的唯一对外入口，所有 ECC 记忆请求必须经本模块接收与路由，禁止任何子模块直接接收外部请求 |
| S-02 | 漏斗一（用户画像）数据仅向通过身份验证的查询请求开放，编译期禁止漏斗一数据流入漏斗二 |
| S-03 | 熔断状态下，本模块仅接收恢复信号，拒绝一切查询与写入请求 |
| S-04 | 本模块不参与认知决策，仅执行记忆存储的调度、路由与状态汇总 |
| S-05 | 冷启动自检未通过时，必须明确标记降级模块并向 ECC 上报告警，不得伪装正常服务 |
