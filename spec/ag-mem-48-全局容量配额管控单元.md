# ag-mem-48-全局容量配额管控单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-48 |
| 模块名称 | 全局容量配额管控单元 |
| 所属分区 | 五、存储与系统运维 |
| 核心职责 | 监控双漏斗记忆中枢（漏斗一 + 漏斗二）的全局存储使用量，实施分层级的容量配额管控。当存储逼近预置上限（默认 100MB，可在配置文件中自定义扩展至设备存储上限）时，触发分级响应：总量达 80% 时发出温和预警，90% 时启动低重要度条目清理（优先清理 L1/L2 可覆写条目），95% 时暂停非关键写入并强制清理。所有清理操作均通过 ag-mem-42（冗余记忆删除与归档单元）安全执行。同时向各层级存储单元及 ag-mem-01（总控漏斗 F₀）周期性上报容量使用状态。不参与经验内容的删除决策，仅负责容量监控与清理触发 |
| 依赖模块 | ag-mem-20~28（各层级存储单元，上报当前存储使用量）、ag-mem-42（冗余记忆删除与归档单元，接收清理指令） |
| 被依赖模块 | ag-mem-01（总控漏斗 F₀，接收容量告警与状态上报）、ag-mem-03（漏斗二专属调度单元，接收容量约束参数）、ag-mem-05（画像槽创建单元，查询可用配额）、ag-mem-20~28（各层级存储单元，接收配额分配与清理指令） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常服务 | `NORMAL` | 总存储使用率 < 80%，正常运行 | 系统初始化完成，使用率正常 |
| 容量预警 | `CAPACITY_WARNING` | 总存储使用率 ≥ 80%，启动温和预警与温和清理 | 使用率 ≥ 80% 且 < 90% |
| 容量紧急 | `CAPACITY_CRITICAL` | 总存储使用率 ≥ 90%，启动强制清理并暂停非关键写入 | 使用率 ≥ 90% |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 各层级存储使用量上报 | Struct（来源层级 + 当前条目数 + 当前使用量_bytes + 最大配额_bytes + 使用率） | ag-mem-20~28（各层级）、ag-mem-05（漏斗一画像槽） | 周期性（每30秒）或写入操作后 | **高** |
| 容量查询请求 | Struct（请求来源模块 + 查询层级 + 查询类型：当前使用率/可用空间/配额上限） | ag-mem-01/03/05/15~19 等 | 按需查询 | **高** |
| 清理完成回执 | Struct（清理条目数 + 释放空间量 + 剩余使用率） | ag-mem-42 冗余记忆删除与归档单元 | 清理操作完成后 | **高** |
| 全局容量配置 | Struct（总容量上限_bytes + 预警阈值_pct + 紧急阈值_pct + 各层级容量比例） | 系统配置文件（初始化加载） | 系统启动时 | 普通 |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-mem-01 总控漏斗 F₀ | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 容量状态上报 | Struct（总使用率 + 各层级使用率 + 当前状态 + 可用空间 + 最近清理统计） | ag-mem-01（总控）、ag-mem-03（调度） | 周期性（每30秒）或状态变更时 | 普通 |
| 容量告警 | Struct（告警等级 + 当前使用率 + 触发阈值 + 建议动作 + 时间戳） | ag-mem-01 总控漏斗 F₀ | 使用率越过预警或紧急阈值时 | **高** |
| 低重要度清理指令 | Struct（目标层级 + 清理策略 + 目标释放空间量 + 清理优先级） | ag-mem-42 冗余记忆删除与归档单元 | 触发容量预警或紧急时 | **高** |
| 写入暂停/恢复通知 | Struct（暂停标记 + 影响层级 + 暂停原因 + 预计恢复时间） | ag-mem-20~28（各层级）、ag-mem-03（调度） | 容量紧急触发或解除时 | **高** |
| 容量查询回执 | Struct（查询层级 + 当前使用率 + 可用空间 + 配额上限） | 请求来源模块 | 查询完成时 | 普通 |


## 容量配置参数

| 配置项 | 默认值 | 说明 |
|--------|:---:|------|
| 总容量上限 | **100MB**（可自定义扩展至设备存储上限） | 双漏斗记忆中枢的全局存储上限 |
| 容量预警阈值 | **80%** | 触发温和预警与温和清理 |
| 容量紧急阈值 | **90%** | 触发强制清理并暂停非关键写入 |
| 各层级容量比例 | L1(60%) + L2(25%) + L3(10%) + L4(4.5%) + L5(0.5%) | 按比例分配各层级配额 |
| 清理目标释放量 | 预警：释放至使用率 75%；紧急：释放至使用率 85% | 每次清理的目标 |
| 周期性上报间隔 | **30秒** | 向 ag-mem-01 上报容量状态 |
| 清理最大单次耗时 | **10秒** | 单次清理操作的最长执行时间 |


## 核心处理逻辑

```
FUNCTION global_quota_controller_main_loop():
    STATE_NORMAL = NORMAL
    STATE_WARN = CAPACITY_WARNING
    STATE_CRITICAL = CAPACITY_CRITICAL
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_NORMAL
    加载全局容量配置（总容量上限、各阈值、各层级比例）
    初始化各层级使用量记录表 = {}
    初始化写入暂停标记 = False

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_NORMAL
            重新加载各层级最新使用量()

        // 第2步：接收各层级存储使用量上报
        IF 收到存储使用量上报:
            更新各层级使用量记录表[上报.来源层级] = 上报
            // 更新总使用量
            总使用量 = SUM(各层级使用量记录表.values())
            总使用率 = 总使用量 / 总容量上限

            // 2a. 检查容量状态变化
            IF 总使用率 >= 容量紧急阈值 AND internal_state != STATE_CRITICAL:
                SET internal_state = STATE_CRITICAL
                触发容量紧急响应()
            ELSE IF 总使用率 >= 容量预警阈值 AND internal_state == STATE_NORMAL:
                SET internal_state = STATE_WARN
                触发容量预警响应()

        // 第3步：容量预警响应（温和清理）
        IF internal_state == STATE_WARN AND NOT 写入暂停标记:
            // 计算需要释放的空间
            目标使用率 = 0.75
            需释放空间 = 总使用量 - (总容量上限 × 目标使用率)
            IF 需释放空间 > 0:
                // 优先清理 L1 和 L2 中 I 值最低的条目
                向 ag-mem-42 发送低重要度清理指令(
                    目标层级=["L1", "L2"],
                    清理策略="清理I值最低的条目",
                    目标释放空间量=需释放空间,
                    清理优先级="温和"
                )

        // 第4步：容量紧急响应（强制清理 + 暂停写入）
        IF internal_state == STATE_CRITICAL:
            // 4a. 暂停非关键写入
            IF NOT 写入暂停标记:
                写入暂停标记 = True
                向 ag-mem-20~26 和 ag-mem-03 发送写入暂停通知(
                    影响层级=["L1", "L2"],
                    暂停原因="容量紧急",
                    预计恢复时间="清理完成后"
                )

            // 4b. 强制清理
            目标使用率 = 0.85
            需释放空间 = 总使用量 - (总容量上限 × 目标使用率)
            IF 需释放空间 > 0:
                向 ag-mem-42 发送低重要度清理指令(
                    目标层级=["L1", "L2", "L3"],
                    清理策略="强制清理低I值条目",
                    目标释放空间量=需释放空间,
                    清理优先级="紧急"
                )

        // 第5步：接收清理完成回执
        IF 收到 ag-mem-42 清理完成回执:
            总使用量 -= 回执.释放空间量
            总使用率 = 总使用量 / 总容量上限

            // 5a. 检查是否可解除紧急状态
            IF internal_state == STATE_CRITICAL AND 总使用率 < 容量预警阈值:
                SET internal_state = STATE_WARN
                // 恢复写入
                写入暂停标记 = False
                向 ag-mem-20~26 和 ag-mem-03 发送写入恢复通知()

            // 5b. 检查是否可解除预警状态
            IF internal_state == STATE_WARN AND 总使用率 < 容量预警阈值 - 5%:
                SET internal_state = STATE_NORMAL

        // 第6步：响应容量查询请求
        IF 收到容量查询请求:
            查询结果 = 构建容量查询回执(请求.查询层级)
            向请求来源返回容量查询回执(查询结果)

        // 第7步：周期性状态上报
        IF 距上次状态上报 >= 30秒:
            向 ag-mem-01 和 ag-mem-03 上报容量状态(
                当前状态=internal_state,
                总使用率=总使用率,
                各层级使用率=各层级使用率,
                可用空间=总容量上限 - 总使用量
            )

        SLEEP 100ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 某层级存储使用量上报超时（>60秒无更新） | 使用该层级最近一次有效值，标记“数据可能过时” | 上报恢复 |
| 清理操作未能释放足够空间（ag-mem-42 清理失败） | 扩大清理范围至 L3 层级，再次尝试 | 清理成功 |
| 全局容量配置加载失败 | 使用编译期内置默认值（100MB, 80%/90%），标记降级 | 配置文件恢复 |
| 清理过程中收到新的存储使用量上报 | 以最新数据重新计算，不中断当前清理 | — |
| 紧急熔断 | 暂停监控与清理，保持当前状态不变 | 紧急解除后恢复 |

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| 内部调度总线 | 读 | 各层级存储使用量上报 | 只读 | ag-mem-20~28/05 发送 |

| 内部调度总线 | 写 | 容量告警 | 事件触发写入 | 向 ag-mem-01 发送 |
| 内部调度总线 | 写 | 低重要度清理指令 | 专属写入 | 向 ag-mem-42 发送 |
| 内部调度总线 | 写 | 写入暂停/恢复通知 | 广播 | 向 ag-mem-20~26/03 发送 |
| 内部调度总线 | 写 | 容量查询回执 | 专属写入 | 向请求来源返回 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| Q-01 | 容量上限可在配置文件中自定义扩展至设备存储上限，但运行时不得低于最小安全容量 |
| Q-02 | 清理操作必须通过 ag-mem-42 执行，本模块不得直接操作任何存储层的数据删除 |
| Q-03 | L5 核心层条目永远不受容量管控驱动的清理操作影响 |
| Q-04 | 容量紧急时暂停写入的范围仅限于 L1/L2 层级，L3-L5 的晋升写入不受影响 |

| 用例编号 | 前置条件 | 输入 | 预期输出 |
