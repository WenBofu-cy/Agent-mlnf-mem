# ag-mem-32-风格匹配度V值计算单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-32 |
| 模块名称 | 风格匹配度V值计算单元 |
| 所属分区 | 三、漏斗二：任务经验漏斗 / 三维重要度计算引擎 |
| 核心职责 | 从任务经验条目中提取与用户价值相关的行为信号与反馈数据，计算用户价值分值V（0.0–1.0）。V值反映该任务经验对当前用户的个性化价值：用户显式正向反馈、高频复用行为、长时间查看、复制分享等操作将获得较高V值；偶发行为或负向反馈获得较低V值。V值作为三维重要度I值的关键组成部分，直接影响经验条目的留存优先级与个性化推荐的准确性。不参与认知决策，仅执行V值的客观量化计算 |
| 依赖模块 | ag-mem-15~19（各场景分槽，在写入新经验时附带用户行为数据供V值计算）、ag-mem-10（偏好累积统计单元，可查询用户历史偏好辅助V值判定） |
| 被依赖模块 | ag-mem-36（综合重要度I值聚合计算单元，消费V值）、ag-mem-35（三维权重系数配置单元，提供V值权重系数β） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 无计算请求，等待输入 | 系统初始化完成，无待处理经验 |
| 计算进行中 | `CALCULATING` | 正在基于经验数据和用户行为计算V值 | 收到各场景分槽的V值计算请求 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| V值计算请求 | Struct（条目ID + 任务类型 + 关联用户行为列表 + 显式反馈标记 + 结果标签 + 用户交互时长 + 行为参数） | ag-mem-15~19 各场景分槽（通过内部调度总线） | 每次新经验写入L1前 | **高** |
| 用户历史偏好查询回执（可选） | Struct（用户ID + 偏好关键词集合 + 高频工具列表 + 各场景交互频次分布） | ag-mem-10 偏好累积统计单元 | 当请求标记“需要用户偏好辅助”时 | 普通 |
| V值计算规则配置 | Struct（各类用户行为信号的权重系数 + 阈值表） | ag-mem-35 三维权重系数配置单元 | 系统初始化加载，配置更新时 | 普通 |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-mem-01 总控漏斗F0 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| V值计算结果 | Struct（条目ID + V值 + 触发的主要价值信号列表 + 用户偏好匹配标记） | 请求来源分槽（ag-mem-15~19）、ag-mem-36（重要度聚合） | 计算完成时 | **高** |
| 计算状态上报 | Struct（当前状态 + 最近V值分布 + 高V值事件计数） | ag-mem-03 漏斗二专属调度单元 | 周期性（每60秒）或状态变更时 | 普通 |


## V值计算规则

### 一、用户价值信号分类与权重

| 信号类别 | 信号示例 | 基准分值 | 权重系数 |
|----------|------|:---:|:---:|
| 显式正向反馈 | 用户主动点赞、收藏、标记有用、明确要求记住 | 0.90 | 0.40 |
| 显式负向反馈 | 用户主动点踩、删除、标记无用、明确要求忽略 | -0.50 | 0.20 |
| 高频复用 | 同类型任务7日内复用≥3次、同一工具7日内调用≥5次 | 0.70 | 0.25 |
| 深度交互 | 查看结果时长超过历史均值150%、复制结果内容、分享结果 | 0.55 | 0.10 |
| 任务完成质量 | 任务一次成功、结果完全匹配意图、用户无重试 | 0.40 | 0.05 |

### 二、V值计算公式

```
V值 = Σ (信号类别基准分 × 触发强度 × 权重系数) + 用户偏好修正值

V值 = CLAMP(计算值, 0.0, 1.0)   // 边界裁剪

其中:
- 触发强度: 0.0–1.0，根据信号的具体程度确定
  - 显式反馈：触发即1.0
  - 高频复用：根据频次归一化（如复用5次=1.0，复用3次=0.6）
  - 深度交互：根据查看时长比例归一化
- 用户偏好修正值: 如果任务关键词与用户历史偏好匹配，加0.05–0.15
```

### 三、快速判定规则

| 条件 | V值直接赋值 | 说明 |
|------|:---:|------|
| 用户显式点赞/收藏 + 任务成功 | 0.90 | 最高用户价值 |
| 用户显式点踩/删除 | 0.05 | 最低用户价值（负向反馈） |
| 首次偶发行为 + 无显式反馈 | 0.20 | 低价值（默认基线） |
| 高频复用（≥5次）+ 深度交互 | 0.80 | 高价值（隐性偏好） |

### 四、各场景分槽的V值权重调整

| 来源分槽 | V值权重调整 | 说明 |
|----------|:---:|------|
| ag-mem-15（对话交互） | ×1.2 | 上调20%，对话经验更重视用户价值 |
| ag-mem-16（工具调用） | 标准（×1.0） | 工具调用以S值为主要驱动 |
| ag-mem-17（信息检索） | 标准（×1.0） | 均衡 |
| ag-mem-18（创作生成） | ×1.1 | 上调10%，创作结果与用户偏好强相关 |
| ag-mem-19（通用任务） | 标准（×1.0） | 均衡 |


## 核心处理逻辑

```
FUNCTION v_value_calculator_main_loop():
    STATE_IDLE = IDLE
    STATE_CALC = CALCULATING
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_IDLE
    加载V值计算规则配置（信号权重、阈值表）

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_IDLE

        // 第2步：接收V值计算请求
        IF 收到来自 ag-mem-15~19 的V值计算请求:
            SET internal_state = STATE_CALC
            条目ID = 请求.条目ID
            来源分槽 = 请求.来源分槽编号
            用户行为列表 = 请求.关联用户行为列表
            显式反馈 = 请求.显式反馈标记
            结果标签 = 请求.结果标签
            交互时长 = 请求.用户交互时长
            行为参数 = 请求.行为参数

            // 2a. 快速判定：显式正向反馈 + 成功
            IF 显式反馈 IN ["点赞", "收藏", "标记有用", "要求记住"] AND 结果标签 == "成功":
                V值 = 0.90
                触发信号列表 = ["显式正向反馈+成功"]
                跳转到输出步骤

            // 2b. 快速判定：显式负向反馈
            IF 显式反馈 IN ["点踩", "删除", "标记无用", "要求忽略"]:
                V值 = 0.05
                触发信号列表 = ["显式负向反馈"]
                跳转到输出步骤

            // 2c. 快速判定：首次偶发+无显式反馈
            IF 用户行为列表为空 AND 显式反馈 == None:
                V值 = 0.20
                触发信号列表 = ["首次偶发行为"]
                跳转到输出步骤

            // 2d. 标准V值计算流程
            V值 = 0.0
            触发信号列表 = []

            // 信号1：显式正向反馈（权重0.40）
            IF 显式反馈 IN ["点赞", "收藏", "标记有用"]:
                V值 += 0.90 × 0.40
                触发信号列表.append("显式正向反馈:" + 显式反馈)

            // 信号2：显式负向反馈（权重0.20，负分）
            IF 显式反馈 IN ["点踩", "删除", "标记无用"]:
                V值 += (-0.50) × 0.20
                触发信号列表.append("显式负向反馈:" + 显式反馈)

            // 信号3：高频复用检测（权重0.25）
            复用次数 = 统计同类型任务复用次数(用户行为列表)
            IF 复用次数 >= 3:
                复用强度 = MIN(复用次数 / 5.0, 1.0)
                V值 += 0.70 × 复用强度 × 0.25
                触发信号列表.append("高频复用:" + 复用次数 + "次")

            // 信号4：深度交互检测（权重0.10）
            深度交互得分 = 0.0
            IF 行为参数.是否复制结果:
                深度交互得分 += 0.55 × 0.5
                触发信号列表.append("复制结果")
            IF 行为参数.是否分享结果:
                深度交互得分 += 0.55 × 0.5
                触发信号列表.append("分享结果")
            IF 交互时长 > 历史均值 × 1.5:
                深度交互得分 += 0.55 × 0.5
                触发信号列表.append("长时间查看")
            V值 += 深度交互得分 × 0.10

            // 信号5：任务完成质量（权重0.05）
            IF 结果标签 == "成功" AND 行为参数.重试次数 == 0:
                V值 += 0.40 × 0.05
                触发信号列表.append("一次成功")

            // 2e. 用户偏好修正（可选）
            IF 请求.需要用户偏好辅助:
                偏好摘要 = 向 ag-mem-10 查询用户偏好摘要(请求.用户ID)
                IF 偏好摘要 != None:
                    任务关键词 = 提取任务关键词(请求.任务类型, 行为参数)
                    偏好匹配度 = 计算关键词匹配度(任务关键词, 偏好摘要.偏好关键词集合)
                    IF 偏好匹配度 > 0.5:
                        修正值 = 0.05 + 偏好匹配度 × 0.10
                        V值 += 修正值
                        触发信号列表.append("用户偏好匹配:" + 偏好匹配度)

            // 2f. 应用分槽专属权重调整
            分槽调整系数 = 获取分槽V值调整系数(来源分槽)
            V值 = V值 × 分槽调整系数

            // 2g. 边界裁剪
            V值 = CLAMP(V值, 0.0, 1.0)

            // 输出步骤：
            // 3a. 构建V值计算结果
            计算结果 = {
                条目ID: 条目ID,
                V值: ROUND(V值, 2),
                触发的主要价值信号列表: 触发信号列表,
                用户偏好匹配标记: 偏好匹配度 > 0.5 IF 偏好摘要 != None ELSE False
            }

            // 3b. 发送至请求来源分槽和重要度聚合单元
            向请求来源分槽返回V值计算结果
            向 ag-mem-36 发送V值计算结果

            // 3c. 更新统计计数器
            IF V值 >= 0.70:
                高V值事件计数++

            SET internal_state = STATE_IDLE

        // 第4步：周期性状态上报
        IF 距上次状态上报 >= 60秒:
            向 ag-mem-03 上报计算状态(
                当前状态=internal_state,
                最近V值分布=统计分布,
                高V值事件计数=高V值事件计数
            )
            重置统计计数器

        SLEEP 10ms


FUNCTION 统计同类型任务复用次数(用户行为列表):
    复用计数 = 0
    当前任务类型 = 请求.任务类型
    FOR EACH 行为 IN 用户行为列表:
        IF 行为.任务类型 == 当前任务类型 AND 行为.时间戳在近7日内:
            复用计数++
    RETURN 复用计数


FUNCTION 提取任务关键词(任务类型, 行为参数):
    // 从任务类型和行为参数中提取代表任务意图的关键词
    关键词 = []
    关键词.append(任务类型)
    IF 行为参数.工具名称 != None:
        关键词.append(行为参数.工具名称)
    IF 行为参数.任务描述关键词 != None:
        关键词.merge(行为参数.任务描述关键词)
    RETURN 关键词
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| V值计算规则配置加载失败 | 使用编译期内置默认权重与阈值，标记降级 | 规则配置恢复 |
| 计算请求缺少用户行为数据 | 默认返回V值=0.20（低价值基线），标记“数据不完整” | 字段补充完整 |
| 显式反馈与隐式行为信号冲突（如点赞但从不复用） | 显式反馈优先，隐式行为作为参考但不抵消显式信号 | — |
| 用户偏好查询超时 | 跳过偏好修正步骤，仅使用行为信号计算 | 查询恢复正常 |
| V值计算结果异常（>1.0或<0.0） | 强制裁剪至[0.0, 1.0]，记录异常日志 | — |
| 紧急熔断 | 暂停计算，未完成的请求丢弃 | 紧急解除后恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| V-01 | V值计算仅基于用户行为元数据（行为类型、频次、反馈标记），不得解析用户原始交互内容 |
| V-02 | 用户偏好查询结果仅用于V值修正，不得将偏好数据写入经验条目本身 |
| V-03 | 显式负向反馈的V值扣减必须有上限（最低0.05），不得扣减至负数 |
| V-04 | V值计算结果必须可追溯，每个V值均附带触发的主要价值信号列表 |
| V-05 | 分槽专属权重调整系数为只读配置，运行时不得修改 |
