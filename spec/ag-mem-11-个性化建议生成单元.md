# ag-mem-11-个性化建议生成单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-11 |
| 模块名称 | 个性化建议生成单元 |
| 所属分区 | 二、漏斗一：用户画像漏斗 |
| 核心职责 | 基于 ag-mem-10 提供的当前活跃画像槽中的量化偏好统计数据，结合当前任务上下文场景标签，动态生成个性化交互建议。建议内容包括：回复风格适配、工具推荐优先级、主动推送时机、交互节奏调整等。输出至 ECC 社会心智模块（ag-ecc-10）或意图解析模块（ag-ecc-01），供其优化用户交互体验。不参与任何认知决策，仅提供基于用户画像的参考建议 |
| 依赖模块 | ag-mem-10（偏好累积统计单元，提供历史偏好统计摘要）、ag-mem-02（漏斗一专属调度单元，提供当前活跃槽位号）、ag-mem-06（画像槽数据隔离管控单元，获取读取令牌）、ag-mem-08（上下文场景标记单元，获取当前场景标签） |
| 被依赖模块 | ag-ecc-10（社会心智模块，接收交互风格建议）、ag-ecc-01（意图解析模块，可参考工具推荐优先级） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 无生成请求，等待触发 | 系统初始化完成，无待处理请求 |
| 数据获取 | `FETCHING_DATA` | 正在获取偏好统计摘要与当前场景标签 | 收到 ECC 的建议请求或触发自动推送条件 |
| 建议生成 | `GENERATING` | 正在基于规则生成个性化建议 | 数据获取完成 |
| 数据不足 | `INSUFFICIENT_DATA` | 画像数据稀疏，无法生成有效建议 | 偏好统计条目过少或场景标签缺失 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 建议生成请求 | Struct（请求来源模块 + 请求类型：风格/工具/时机/综合 + 可选：当前任务描述） | ag-ecc-10 社会心智模块 或 ag-ecc-01 意图解析模块（通过 MemoryBus） | ECC 需要个性化建议辅助决策时 | **高** |
| 当前活跃槽位绑定信息 | Struct（会话ID + 活跃槽位编号 + 用户ID） | ag-mem-02 漏斗一专属调度单元 | 槽位激活或切换时 | **高** |
| 历史偏好统计摘要 | Struct（槽位编号 + 各维度偏好强度 + 偏好关键词 + 显式/隐式分布 + 时间窗口） | ag-mem-10 偏好累积统计单元 | 响应数据获取请求时 | **高** |
| 当前上下文场景标签 | Struct（场景类别 + 任务类型 + 交互阶段 + 置信度） | ag-mem-08 上下文场景标记单元 | 每次生成建议时查询 | **高** |
| 读取许可令牌 | Struct（令牌ID + 授权槽位编号 + 有效期） | ag-mem-06 画像槽数据隔离管控单元 | 数据获取前申请 | **高** |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-mem-01 总控漏斗F0 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 个性化建议列表 | Struct（建议ID + 建议类型 + 建议内容 + 置信度 + 所依据的偏好数据引用） | ag-ecc-10 社会心智模块 或 ag-ecc-01 意图解析模块（通过 MemoryBus） | 建议生成完成时 | **高** |
| 数据不足通知 | Struct（原因 + 当前画像数据条目数 + 最小需求条目数） | 请求来源模块（通过 MemoryBus） | 画像数据不足以支持生成时 | 普通 |
| 建议生成状态上报 | Struct（当前状态 + 最近一次生成耗时 + 生成建议数 + 命中率反馈） | ag-mem-02 漏斗一专属调度单元 | 周期性（每120秒）或状态变更时 | 普通 |


## 建议生成规则

### 一、建议类型定义

| 建议类型 | 说明 | 输出目标模块 |
|----------|------|:---:|
| `STYLE_ADAPT` | 回复风格适配建议（简洁/详细、正式/随意、主动/被动） | ag-ecc-10 |
| `TOOL_PRIORITY` | 工具推荐优先级排序（基于历史工具调用偏好） | ag-ecc-01 |
| `TIMING_SUGGEST` | 主动推送时机建议（用户活跃时段、偏好交互间隔） | ag-ecc-10 |
| `PACE_ADJUST` | 交互节奏调整建议（快速响应 vs 缓慢引导） | ag-ecc-10 |
| `CONTENT_HINT` | 内容偏好提示（偏好主题、格式、语言风格） | ag-ecc-01/10 |
| `ENGAGEMENT_ALERT` | 用户兴趣衰减预警（当某维度偏好强度连续下降时） | ag-ecc-10 |

### 二、生成规则矩阵

| 规则ID | 触发条件 | 建议类型 | 建议内容模板 | 置信度计算 |
|--------|------|:---:|------|:---:|
| R01 | 用户显式偏好标记“简洁回复”且近期偏好强度>0.7 | STYLE_ADAPT | “用户偏好简洁回复，建议将平均回复长度控制在100字以内” | 基于显式偏好置信度×偏好强度 |
| R02 | TOOL_INVOKE 维度中某工具偏好强度排名前3 | TOOL_PRIORITY | “用户常用工具TOP3：{工具A}、{工具B}、{工具C}，建议在同类任务中优先推荐” | 基于工具调用频次占比 |
| R03 | 用户活跃时段集中在某2小时窗口内（30日内数据） | TIMING_SUGGEST | “用户在 {时间段} 活跃度最高，建议在此时段主动推送重要通知” | 基于时段集中度（>60%） |
| R04 | RESULT_VIEW 维度中用户平均查看时长 < 3秒 且 偏好强度>0.6 | PACE_ADJUST | “用户倾向于快速浏览，建议提高交互节奏，减少冗余确认” | 基于查看时长均值与偏差 |
| R05 | 偏好关键词集合中某主题关键词出现频次 > 阈值 | CONTENT_HINT | “用户对 {主题} 表现出持续兴趣，建议在相关任务中主动关联” | 基于关键词频次/总关键词数 |
| R06 | 某维度偏好强度在过去30日内连续下降超过30% | ENGAGEMENT_ALERT | “用户对 {维度} 的兴趣正在衰减，建议减少此类内容的推送频率” | 基于趋势斜率 |
| R07 | 用户隐式倾向标记中“复制”行为占比>20% | CONTENT_HINT | “用户经常复制结果内容，建议提供一键导出或批量复制功能” | 基于行为频次占比 |
| R08 | 新用户画像数据条目 < 50 | — | 不生成任何建议，返回数据不足 | — |

### 三、建议优先级排序

1. 显式偏好触发的建议（置信度最高）优先
2. 隐式倾向触发的建议按偏好强度降序排列
3. 同一类型建议最多输出3条，超出部分按置信度截断
4. 数据不足时返回空列表 + 数据不足通知


## 核心处理逻辑

```
FUNCTION suggestion_generator_main_loop():
    STATE_IDLE = IDLE
    STATE_FETCH = FETCHING_DATA
    STATE_GENERATE = GENERATING
    STATE_INSUFF = INSUFFICIENT_DATA
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_IDLE
    初始化建议生成计数器

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_IDLE

        // 第2步：更新活跃槽位
        IF 收到 ag-mem-02 活跃槽位绑定信息:
            当前活跃槽位 = 绑定信息.活跃槽位编号
            CONTINUE

        // 第3步：接收建议生成请求
        IF 收到 ECC 建议生成请求:
            SET internal_state = STATE_FETCH
            请求来源 = 请求.来源模块
            请求类型 = 请求.请求类型

            // 3a. 获取读取权限
            读取令牌 = 向 ag-mem-06 申请读取令牌(当前活跃槽位)
            IF 读取令牌 == None:
                向请求来源返回数据不足通知("无法获取画像槽读取权限")
                SET internal_state = STATE_IDLE
                CONTINUE

            // 3b. 获取偏好统计摘要
            统计摘要 = 向 ag-mem-10 查询偏好统计摘要(当前活跃槽位)
            IF 统计摘要 == None OR 统计摘要.总条目数 < 50:
                SET internal_state = STATE_INSUFF
                向请求来源返回数据不足通知(
                    原因="画像数据不足",
                    当前条目数=统计摘要.总条目数 IF 统计摘要 != None ELSE 0,
                    最小需求=50
                )
                SET internal_state = STATE_IDLE
                CONTINUE

            // 3c. 获取当前场景标签
            场景标签 = 向 ag-mem-08 查询当前上下文场景()
            IF 场景标签 == None:
                场景标签 = {场景类别: "通用任务", 置信度: 0.5}  // 默认

            // 第4步：生成建议
            SET internal_state = STATE_GENERATE
            建议列表 = []

            // 4a. 应用规则矩阵
            FOR EACH 规则 IN 建议生成规则矩阵:
                IF 规则.触发条件满足(统计摘要, 场景标签, 请求类型):
                    建议内容 = 填充建议模板(规则.建议内容模板, 统计摘要)
                    置信度 = 计算规则置信度(规则, 统计摘要)
                    建议列表.append({
                        建议ID: "SUG-" + 生成UUID(),
                        建议类型: 规则.建议类型,
                        建议内容: 建议内容,
                        置信度: 置信度,
                        依据数据: 规则.触发条件所引用的统计字段
                    })

            // 4b. 按优先级排序
            建议列表 = 按置信度降序排列(建议列表)

            // 4c. 截断超量建议
            IF 建议列表.size > 10:
                建议列表 = 建议列表[0:10]

            // 4d. 输出结果
            IF 建议列表为空:
                向请求来源返回数据不足通知("当前画像数据不触发任何建议规则")
            ELSE:
                向请求来源返回个性化建议列表(建议列表)

            更新生成计数器
            SET internal_state = STATE_IDLE

        // 第5步：周期性状态上报
        IF 距上次状态上报 >= 120秒:
            向 ag-mem-02 上报建议生成状态(
                当前状态=internal_state,
                最近生成耗时=上次生成耗时,
                累计生成建议数=生成计数器.总计,
                各类型建议分布=生成计数器.按类型统计
            )

        SLEEP 50ms


FUNCTION 计算规则置信度(规则, 统计摘要):
    // 置信度 = 所依据偏好数据的置信度 × 偏好强度
    SWITCH 规则.规则ID:
        CASE "R01":  // 显式偏好
            数据置信度 = 0.95  // 显式偏好置信度固定为0.95
            偏好强度 = 统计摘要.按行为维度["相关维度"].偏好强度
            RETURN 数据置信度 × 偏好强度
        CASE "R02":  // 高频工具调用
            频次占比 = 该工具调用次数 / 总工具调用次数
            RETURN 频次占比（归一化到0.5-0.9区间）
        CASE "R03":  // 时段集中度
            时段集中度 = 活跃时段占比
            RETURN IF 时段集中度 > 0.6 THEN 0.75 ELSE 0.50
        CASE "R06":  // 趋势下降
            下降幅度 = ABS(偏好强度变化量)
            RETURN 0.60 + 下降幅度 × 0.3  // 最大0.90
        DEFAULT:
            RETURN 0.55  // 其他规则默认中等置信度
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 读取令牌申请失败 | 返回数据不足通知，标记“权限不足” | 令牌申请成功 |
| 画像数据条目数 < 50（新用户） | 返回数据不足通知，明确告知最小需求 | 数据积累到50条以上 |
| 场景标签缺失 | 使用“通用任务”作为默认场景标签 | 场景标签正常获取 |
| 所有规则均不触发 | 返回空建议列表 + 通知，不强行生成 | 画像数据积累后可能触发新规则 |
| 紧急熔断 | 暂停服务，未完成的生成请求丢弃 | 紧急解除后恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 所有建议生成必须基于当前活跃画像槽的数据，禁止跨用户生成建议 |
| S-02 | 建议内容中不得暴露用户的原始行为数据（如具体输入文本、具体浏览内容），仅使用聚合统计信息 |
| S-03 | 数据不足时必须明确告知下游模块，不得使用系统默认值伪造建议 |
| S-04 | 建议仅作为参考输出，本模块不强制要求任何下游模块采纳 |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-M11-01 | `IDLE`，画像数据充足，用户偏好简洁回复 | 建议生成请求（类型=STYLE_ADAPT） | 返回建议“用户偏好简洁回复”，置信度>0.6 |
| TC-M11-02 | `IDLE`，高频工具调用数据可用 | 建议生成请求（类型=TOOL_PRIORITY） | 返回TOP3工具推荐列表，按置信度排序 |

| TC-M11-06 | `SYSTEM_PAUSED`，收到请求 | 建议生成请求 | 拒绝处理，保持暂停状态 |
