# ag-mem-20-L1临时层存储单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-20 |
| 模块名称 | L1临时层存储单元 |
| 所属分区 | 三、漏斗二：任务经验漏斗 / 五层存储 |
| 核心职责 | 作为漏斗二五层记忆存储架构的第一层，负责接收并存储本次会话或近期产生的瞬时任务经验片段。L1层是经验进入漏斗二存储系统的唯一入口，所有新经验（无论最终目标层级）均首先写入L1。管理L1层的容量上限（占漏斗二总容量的60%），当逼近容量上限时触发低重要度条目清理或向ag-mem-21请求衰减评估。不参与晋升判定或遗忘决策，仅执行经验的接收、暂存与基础管理 |
| 依赖模块 | ag-mem-15~19（各场景分槽，下发经验写入请求）、ag-mem-21（L1临时层时序衰减单元，接收衰减评估请求）、ag-mem-48（全局容量配额管控单元，查询L1当前容量） |
| 被依赖模块 | ag-mem-15~19（返回写入确认）、ag-mem-21（消费L1中待衰减评估的条目列表）、ag-mem-22（L2近期层存储单元，接收晋升条目） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常服务 | `NORMAL` | L1存储正常，接收新经验写入 | 系统初始化完成，存储分区就绪 |
| 容量预警 | `CAPACITY_WARNING` | L1存储使用率超过80%，触发温和清理 | L1使用率 ≥ 80% |
| 容量紧急 | `CAPACITY_CRITICAL` | L1存储使用率超过95%，暂停写入并强制清理 | L1使用率 ≥ 95% |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| L1经验写入请求 | Struct（经验条目数据 + 来源分槽编号 + 调整后重要度 + 时间戳） | ag-mem-15~19 各场景分槽（通过内部调度总线） | 任何场景产生新任务经验时 | **高** |
| L1清理完成确认 | Struct（清理条目数 + 释放空间量 + 剩余使用率） | ag-mem-21 L1衰减评估单元 | 衰减清理执行完成后 | **高** |
| L1容量查询回执 | Struct（当前条目数 + 当前使用率 + 可用空间） | ag-mem-48 全局容量配额管控单元 | 每次写入前查询 | **高** |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-mem-01 总控漏斗F0 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| L1写入确认回执 | Struct（条目ID + 写入状态 + L1当前使用率 + 分配存储位置） | 来源分槽（ag-mem-15~19） | 写入成功时 | **高** |
| L1写入拒绝通知 | Struct（条目ID + 拒绝原因 + L1当前状态 + 建议处理） | 来源分槽（ag-mem-15~19） | 写入被拒绝时（容量紧急且无法清理） | **高** |
| L1衰减评估请求 | Struct（待评估条目列表 + 触发原因 + L1使用率） | ag-mem-21 L1临时层时序衰减单元 | L1使用率≥80%或定时触发（每6小时） | **高** |
| L1状态上报 | Struct（当前状态 + 条目总数 + 使用率 + 各场景分槽写入量分布 + 最近写入时间） | ag-mem-48（容量管控）、ag-mem-03（调度单元） | 周期性（每30秒）或状态变更时 | 普通 |


## L1存储配置

| 配置项 | 默认值 | 说明 |
|--------|:---:|------|
| L1占漏斗二总容量比例 | **60%** | L1是最大的存储层，承载最活跃的近期经验 |
| L1最大条目数 | 10000条 | 硬上限，达到后强制触发清理 |
| L1单条目最大大小 | 10KB | 单条经验数据不得超过此值 |
| 写入超时 | 200ms | 单次写入操作的最大等待时间 |
| 容量预警阈值 | 80% | 触发温和清理 |
| 容量紧急阈值 | 95% | 触发强制清理并暂停写入 |
| 衰减评估触发间隔 | 6小时 | 定时触发衰减评估 |
| 最小保留条目数 | 500条 | 即使容量紧张，L1至少保留此数量的最近经验 |


## 核心处理逻辑

```
FUNCTION l1_storage_main_loop():
    STATE_NORMAL = NORMAL
    STATE_WARN = CAPACITY_WARNING
    STATE_CRITICAL = CAPACITY_CRITICAL
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_NORMAL
    初始化L1存储分区（内存索引 + 持久化存储）
    初始化条目计数器 = 0

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_NORMAL

        // 第2步：接收经验写入请求
        IF 收到来自 ag-mem-15~19 的L1经验写入请求:
            IF internal_state == STATE_PAUSED:
                向来源分槽返回L1写入拒绝通知("系统熔断中")
                CONTINUE

            // 2a. 查询当前L1容量
            容量回执 = 向 ag-mem-48 查询L1当前容量()
            L1使用率 = 容量回执.当前使用率

            // 2b. 容量状态更新
            IF L1使用率 >= 95%:
                SET internal_state = STATE_CRITICAL
            ELSE IF L1使用率 >= 80%:
                SET internal_state = STATE_WARN

            // 2c. 容量紧急处理
            IF internal_state == STATE_CRITICAL:
                // 检查是否已达硬上限
                IF 条目计数器 >= L1最大条目数:
                    // 强制触发衰减清理
                    向 ag-mem-21 发送L1衰减评估请求(L1中重要度最低的20%条目, "容量紧急")
                    等待清理结果
                    IF 清理后使用率仍 >= 95%:
                        向来源分槽返回L1写入拒绝通知("L1容量已满且无法清理")
                        CONTINUE

            // 2d. 容量预警处理（温和清理）
            IF internal_state == STATE_WARN:
                // 异步触发衰减评估，不阻塞当前写入
                向 ag-mem-21 发送L1衰减评估请求(L1中重要度最低的10%条目, "容量预警")

            // 2e. 校验条目大小
            IF 请求.经验条目数据.大小 > 10KB:
                向来源分槽返回L1写入拒绝通知("经验条目大小超过10KB上限")
                CONTINUE

            // 2f. 写入L1存储
            条目ID = 生成唯一ID("L1-" + 时间戳 + 序号)
            存储位置 = L1存储分区.追加写入(条目ID, 请求.经验条目数据, 请求.调整后重要度)
            
            IF 存储位置 == None:
                向来源分槽返回L1写入拒绝通知("存储写入异常")
                CONTINUE

            // 2g. 更新元数据
            条目计数器++
            更新L1索引(条目ID, 请求.来源分槽编号, 请求.调整后重要度, NOW())

            // 2h. 返回确认
            向来源分槽返回L1写入确认回执(
                条目ID=条目ID,
                写入状态="成功",
                L1当前使用率=L1使用率,
                分配存储位置=存储位置
            )

        // 第3步：定时衰减评估触发
        IF 距上次衰减评估 >= 6小时:
            IF 条目计数器 > 0:
                // 取L1中重要度最低的20%条目送入衰减评估
                待评估条目 = L1存储.按重要度升序排列().取前20%
                向 ag-mem-21 发送L1衰减评估请求(待评估条目, "定时衰减")
            重置衰减计时器

        // 第4步：接收清理完成确认
        IF 收到 ag-mem-21 L1清理完成确认:
            条目计数器 -= 确认.清理条目数
            从L1索引中移除已清理条目
            IF L1使用率 < 80%:
                SET internal_state = STATE_NORMAL

        // 第5步：周期性状态上报
        IF 距上次状态上报 >= 30秒:
            统计各场景分槽写入量分布
            向 ag-mem-48 和 ag-mem-03 上报L1状态(
                当前状态=internal_state,
                条目总数=条目计数器,
                使用率=L1使用率,
                写入量分布=各分槽写入统计
            )

        SLEEP 10ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| L1存储写入异常（磁盘满/IO错误） | 返回写入拒绝通知，上报告警 | 存储恢复正常 |
| 条目大小超过10KB上限 | 拒绝写入，返回错误提示 | — |
| L1使用率达95%且衰减清理后仍无法释放空间 | 拒绝新写入，保留最近500条，上报告警 | 衰减清理成功释放空间 |
| 同一经验条目重复写入（相同ID） | 覆盖更新（以新版本替换旧版本），更新时间戳 | — |
| 紧急熔断 | 暂停所有写入，保持L1数据完整性 | 紧急解除后恢复 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| 内部调度总线 | 读 | L1经验写入请求 | 只读 | ag-mem-15~19 发送 |
| 内部调度总线 | 读 | L1清理完成确认 | 只读 | ag-mem-21 返回 |
| 内部调度总线 | 读 | L1容量查询回执 | 只读 | ag-mem-48 返回 |
| 内部调度总线 | 写 | L1写入确认回执 | 专属写入 | 向来源分槽返回 |
| 内部调度总线 | 写 | L1写入拒绝通知 | 专属写入 | 向来源分槽返回 |
| 内部调度总线 | 写 | L1衰减评估请求 | 专属写入 | 向 ag-mem-21 发送 |
| 内部调度总线 | 写 | L1状态上报 | 周期性写入 | 向 ag-mem-48/03 发送 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | L1层仅为经验临时暂存区，不得在L1层对经验内容进行任何修改或加工 |
| S-02 | 写入经验时必须保留原始来源分槽编号，用于后续晋升时溯源 |
| S-03 | L1容量紧急时强制清理不得删除最近500条经验，确保当前会话的经验不丢失 |
| S-04 | L1存储的持久化写入必须保证原子性，写入中断时不产生损坏的半条记录 |
