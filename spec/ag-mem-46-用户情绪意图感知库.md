# ag-mem-46-用户情绪意图感知库 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-46 |
| 模块名称 | 用户情绪意图感知库 |
| 所属分区 | 四、漏斗外挂扩展区（物理隔离） |
| 核心职责 | 作为漏斗外挂扩展区的情绪感知服务库，提供用户实时情绪状态与交互意图的推断结果。基于多模态输入（文本语义、交互节奏、显式反馈、上下文行为序列），输出标准化的情绪标签（如：平静、焦虑、愤怒、满意、困惑）与意图倾向（如：寻求帮助、表达不满、探索尝试、确认信息）。为 ag-ecc-10（社会心智模块）提供情绪感知数据以适配交互风格，为 ag-ecc-01（意图解析模块）提供意图预判辅助，为 ag-mem-11（个性化建议生成单元）提供用户情绪背景以调整建议强度。本库完全独立于双漏斗记忆系统运行，不参与记忆沉淀、筛选、晋升与遗忘机制。情绪推断模型在系统部署前预置，支持定期离线更新。仅提供只读查询服务，不参与任何认知决策 |
| 依赖模块 | 无（作为独立情绪感知服务，不依赖记忆系统内部模块） |
| 被依赖模块 | ag-ecc-10（社会心智模块，获取情绪与意图数据）、ag-ecc-01（意图解析模块，获取意图预判辅助）、ag-mem-11（个性化建议生成单元，获取用户情绪背景）、ag-mem-01（总控漏斗 F₀，可转发查询请求） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常服务 | `NORMAL_SERVICE` | 情绪感知模型加载完成，正常响应查询 | 系统初始化完成，模型就绪 |
| 模型加载中 | `LOADING` | 正在加载或更新情绪推断模型 | 系统启动时或收到模型更新指令 |
| 模型降级 | `DEGRADED` | 情绪推断模型部分不可用，使用轻量规则引擎 | 模型加载失败或完整性校验异常 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 情绪意图查询请求 | Struct（请求来源模块 + 查询类型：情绪状态/意图倾向/综合感知 + 会话ID + 可选：用户最近交互文本序列 + 交互节奏数据 + 显式反馈标记） | ag-ecc-10/01、ag-mem-11、ag-mem-01（转发） | ECC 需要用户情绪数据辅助决策时 | **高** |
| 情绪推断模型更新指令 | Struct（更新数据包 + 模型版本号 + 校验签名 + 更新原因） | 系统管理接口或 OTA 升级系统 | 情绪模型需要更新时（低频操作） | 普通 |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-mem-01 总控漏斗 F₀ | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 情绪意图感知结果 | Struct（会话ID + 情绪标签 + 情绪置信度 + 意图倾向 + 意图置信度 + 感知依据摘要 + 模型版本号） | 请求来源模块 | 查询完成时 | **高** |
| 情绪感知状态上报 | Struct（当前状态 + 模型版本号 + 最近查询次数 + 平均响应耗时） | ag-mem-01（周期性）、ag-mem-03（调度单元） | 初始化完成或状态变更时 | 普通 |
| 模型更新确认回执 | Struct（更新结果 + 新模型版本号 + 生效时间） | 更新发起方（管理接口/OTA） | 模型更新完成时 | 普通 |


## 情绪与意图标签定义

### 一、情绪标签

| 情绪标签 | 标识 | 说明 | 典型触发信号 |
|----------|:---:|------|------|
| 平静 | `CALM` | 用户情绪稳定，正常交互 | 常规对话节奏、中性措辞 |
| 满意 | `SATISFIED` | 用户对当前交互或结果表示满意 | 正向反馈（点赞、感谢）、积极措辞 |
| 困惑 | `CONFUSED` | 用户对系统回复或操作不理解 | 反复询问相似问题、长时间停顿、求助措辞 |
| 焦虑 | `ANXIOUS` | 用户表现出急切或担忧 | 高频催促、多次重试、负面情绪词 |
| 愤怒 | `ANGRY` | 用户表现出不满或愤怒 | 严厉措辞、点踩、中断操作、投诉 |
| 失望 | `DISAPPOINTED` | 用户对结果质量表示失望 | 点踩但未愤怒、消极措辞、离开会话 |

### 二、意图倾向标签

| 意图倾向 | 标识 | 说明 | 典型行为信号 |
|----------|:---:|------|------|
| 寻求帮助 | `SEEK_HELP` | 用户遇到问题需要系统协助 | 提问句式、错误反馈、求助关键词 |
| 表达不满 | `EXPRESS_DISSAT` | 用户对系统服务表达不满 | 投诉措辞、点踩、要求转人工 |
| 探索尝试 | `EXPLORE` | 用户以探索心态使用系统功能 | 尝试不同工具、浏览式提问、无明确目标 |
| 确认信息 | `CONFIRM_INFO` | 用户需要确认或验证信息 | 反问句式、要求确认、重复提问 |
| 任务执行 | `TASK_EXEC` | 用户有明确的任务执行意图 | 直接下达指令、使用工具调用关键词 |
| 休闲闲聊 | `CASUAL_CHAT` | 用户无明确任务目标，以休闲交流为主 | 问候语、闲聊话题、情感表达 |


## 核心处理逻辑

```
FUNCTION emotion_intent_engine_main_loop():
    STATE_LOADING = LOADING
    STATE_NORMAL = NORMAL_SERVICE
    STATE_DEGRADED = DEGRADED
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_LOADING
    初始化情绪推断模型（可能是基于规则的引擎或轻量级分类模型）
    初始化意图推断模型

    // 系统启动时加载模型
    模型加载结果 = 加载情绪意图模型()
    IF 模型加载结果.成功:
        SET internal_state = STATE_NORMAL
    ELSE:
        SET internal_state = STATE_DEGRADED
        加载备用轻量规则引擎()

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_NORMAL

        // 第2步：接收情绪意图查询请求
        IF 收到情绪意图查询请求:
            IF internal_state NOT IN [STATE_NORMAL, STATE_DEGRADED]:
                向请求来源返回查询失败("情绪感知服务当前不可用")
                CONTINUE

            会话ID = 请求.会话ID
            查询类型 = 请求.查询类型
            交互文本序列 = 请求.交互文本序列
            交互节奏数据 = 请求.交互节奏数据
            显式反馈标记 = 请求.显式反馈标记

            // 2a. 快速路径：基于显式反馈直接判定
            IF 显式反馈标记 IN ["点赞", "满意"]:
                情绪标签 = "满意"
                情绪置信度 = 0.95
                意图倾向 = 推断意图(交互文本序列)  // 仍可推断意图
                跳转到输出结果

            IF 显式反馈标记 IN ["点踩", "投诉"]:
                情绪标签 = "愤怒"
                情绪置信度 = 0.90
                意图倾向 = "表达不满"
                跳转到输出结果

            // 2b. 基于交互节奏的快速判定
            IF 交互节奏数据.连续重试次数 >= 3 AND 交互节奏数据.平均响应间隔 < 2秒:
                情绪标签 = "焦虑"
                情绪置信度 = 0.75
                意图倾向 = "寻求帮助"
                跳转到输出结果

            // 2c. 标准推断流程（使用模型或规则引擎）
            IF internal_state == STATE_NORMAL AND 情绪推断模型 != None:
                情绪结果 = 情绪推断模型.推断(交互文本序列, 交互节奏数据)
                意图结果 = 意图推断模型.推断(交互文本序列, 交互节奏数据)
            ELSE:
                // 降级模式下使用轻量规则引擎
                情绪结果 = 轻量规则引擎.推断情绪(交互文本序列)
                意图结果 = 轻量规则引擎.推断意图(交互文本序列)

            // 输出结果：
            感知结果 = 构建情绪意图感知结果(
                会话ID=会话ID,
                情绪标签=情绪结果.标签,
                情绪置信度=情绪结果.置信度,
                意图倾向=意图结果.标签,
                意图置信度=意图结果.置信度,
                感知依据摘要=情绪结果.依据 + " | " + 意图结果.依据,
                模型版本号=当前模型版本号
            )

            向请求来源返回情绪意图感知结果(感知结果)

            // 记录查询统计
            更新查询统计()

        // 第3步：接收模型更新指令
        IF 收到模型更新指令:
            // 3a. 校验签名
            IF NOT 校验签名(指令.校验签名):
                向更新发起方返回模型更新确认回执(失败, "签名校验失败")
                CONTINUE

            SET internal_state = STATE_LOADING
            旧模型快照 = 当前模型数据.clone()

            // 3b. 执行更新
            更新结果 = 加载新模型(指令.更新数据包)
            IF 更新结果.成功:
                当前模型版本号 = 指令.模型版本号
                SET internal_state = STATE_NORMAL
            ELSE:
                恢复旧模型快照
                SET internal_state = STATE_NORMAL
                向更新发起方返回模型更新确认回执(失败, 更新结果.错误信息)
                CONTINUE

            向更新发起方返回模型更新确认回执(成功, 新模型版本号)

        // 第4步：周期性状态上报
        IF 距上次状态上报 >= 60秒:
            向 ag-mem-01 和 ag-mem-03 上报情绪感知状态(当前状态, 模型版本号, 查询统计)

        SLEEP 10ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 情绪推断模型加载失败 | 进入 DEGRADED 状态，使用备用轻量规则引擎 | 模型重新加载成功 |
| 交互文本序列为空或仅包含系统事件 | 返回默认情绪标签="平静"，意图倾向="未知"，置信度=0.3 | 用户产生有效交互 |
| 模型更新签名校验失败 | 拒绝更新，记录安全事件 | 使用正确签名重试 |
| 模型更新过程中系统断电 | 重启后回滚至更新前旧版本 | 重新尝试更新 |
| 紧急熔断 | 暂停服务，保持当前模型不变 | 紧急解除后恢复 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| 内部调度总线 / MemoryBus | 读 | 情绪意图查询请求 | 只读 | ag-ecc-10/01、ag-mem-11、ag-mem-01 发送 |
| 内部调度总线 / MemoryBus | 写 | 情绪意图感知结果 | 专属写入 | 向请求来源返回 |
| 管理接口 / OTA 总线 | 读 | 情绪推断模型更新指令 | 只读 | 管理接口 / OTA 系统 |
| 管理接口 / OTA 总线 | 写 | 模型更新确认回执 | 专属写入 | 向更新发起方返回 |
| 内部调度总线 | 写 | 情绪感知状态上报 | 周期性写入 | 向 ag-mem-01/03 发送 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| E-01 | 情绪意图推断仅基于会话内的交互元数据与文本语义特征，不得存储或分析用户的原始个人身份信息 |
| E-02 | 情绪推断结果仅供上游模块参考，不得直接触发任何不可逆的系统操作 |
| E-03 | 模型更新必须通过签名校验，防止恶意模型注入 |
| E-04 | 本库完全独立于漏斗记忆系统，不参与记忆沉淀、晋升、遗忘任何环节 |
