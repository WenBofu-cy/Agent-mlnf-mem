# ag-mem-49-存储压缩与冷归档单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-49 |
| 模块名称 | 存储压缩与冷归档单元 |
| 所属分区 | 五、存储与系统运维 |
| 核心职责 | 接收 ag-mem-42（冗余记忆删除与归档单元）发送的归档写入指令，对 L3 及以上层级的被遗忘经验条目执行压缩打包与冷归档存储。将经验数据及其元数据（来源层级、I值、遗忘原因、时间戳）序列化后，使用高效压缩算法压缩，写入冷存储分区，并返回归档存储位置与压缩后大小。同时管理冷存储分区的容量，当冷存储逼近上限时触发旧归档数据的清理（按归档时间从旧到新）。支持按需从冷存储中恢复归档条目（用于离线复盘或数据恢复），恢复时解压并重建原始数据结构。不参与遗忘判定或删除决策，仅执行压缩、归档与恢复的确定性操作 |
| 依赖模块 | ag-mem-42（冗余记忆删除与归档单元，发送归档写入指令）、ag-mem-48（全局容量配额管控单元，上报冷存储使用量） |
| 被依赖模块 | ag-mem-42（返回归档操作确认与存储位置）、ag-mem-48（接收冷存储使用量上报）、ag-mem-51（记忆变更日志追溯单元，记录归档事件） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常服务 | `NORMAL` | 冷存储正常运行，接收归档写入与恢复请求 | 系统初始化完成，冷存储分区就绪 |
| 压缩归档中 | `COMPRESSING` | 正在对一批条目执行压缩与冷存储写入 | 收到 ag-mem-42 的归档写入指令 |
| 恢复解压中 | `RESTORING` | 正在从冷存储中读取并解压恢复指定条目 | 收到归档恢复请求 |
| 冷存储容量预警 | `COLD_WARN` | 冷存储使用率 ≥ 80% | 冷存储使用量超过预警阈值 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 归档写入指令 | Struct（条目ID + 经验数据 + 元数据：来源层级/来源分槽/I值/遗忘原因/归档原因/时间戳） | ag-mem-42 冗余记忆删除与归档单元（通过内部调度总线） | L3/L4 条目被判定需归档时 | **高** |
| 归档恢复请求 | Struct（请求来源模块 + 恢复条件：条目ID列表 或 时间范围 + 来源层级 + 最大恢复条目数） | 系统管理接口、离线复盘系统、ag-mem-01（转发） | 需要从冷存储恢复数据时 | 普通 |
| 冷存储容量查询回执 | Struct（当前使用量 + 总容量 + 使用率 + 最早归档时间） | 冷存储分区（本地查询） | 每次归档写入前 | **高** |
| 冷存储清理完成确认 | Struct（清理条目数 + 释放空间量 + 剩余使用率） | 本模块内部（清理操作） | 清理操作完成后 | 普通 |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-mem-01 总控漏斗 F₀ | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 归档操作确认 | Struct（条目ID + 归档状态 + 归档存储位置 + 压缩前大小 + 压缩后大小 + 压缩比 + 归档时间戳） | ag-mem-42 冗余记忆删除与归档单元 | 归档写入完成时 | **高** |
| 归档恢复结果 | Struct（恢复条目列表 + 各条目经验数据与元数据 + 恢复耗时 + 恢复来源位置） | 请求来源模块 | 恢复操作完成时 | **高** |
| 冷存储状态上报 | Struct（当前状态 + 总容量 + 已用容量 + 使用率 + 归档条目总数 + 最早/最新归档时间） | ag-mem-48（容量管控）、ag-mem-03（调度单元） | 周期性（每120秒）或状态变更时 | 普通 |
| 归档事件日志 | Struct（事件类型="归档"/"恢复"/"清理" + 条目ID列表 + 操作结果 + 时间戳） | ag-mem-51 记忆变更日志追溯单元 | 每次归档/恢复/清理完成时 | 普通 |


## 冷存储配置

| 配置项 | 默认值 | 说明 |
|--------|:---:|------|
| 冷存储总容量 | **50MB** | 冷存储分区独立于热存储（五层存储） |
| 冷存储预警阈值 | **80%** | 触发旧归档数据清理 |
| 压缩算法 | **Zstandard (zstd)** | 高压缩比，低CPU开销 |
| 压缩级别 | **3**（平衡模式） | 范围 1-9，3 为速度与压缩比的平衡点 |
| 单条目归档超时 | **10秒** | 单条目压缩+写入的最长时间 |
| 恢复单条目超时 | **15秒** | 单条目读取+解压的最长时间 |
| 冷存储清理策略 | **按归档时间从旧到新清理** | 仅清理未被标记为“永久保留”的归档条目 |


## 归档条目结构

```
冷存储归档条目 = {
    归档ID: "ARCH-{原始条目ID}-{归档时间戳}",
    原始条目ID: 原始经验条目ID,
    压缩数据: 经Zstandard压缩的经验数据二进制块,
    压缩前大小: 压缩前字节数,
    压缩后大小: 压缩后字节数,
    压缩算法: "zstd-3",
    元数据: {
        来源层级: "L3 | L4",
        来源分槽编号: "ag-mem-15~19",
        归档时的I值: Float,
        遗忘原因: String,
        归档原因: String,
        归档时间戳: UTC时间戳,
        原始写入时间戳: UTC时间戳
    },
    校验和: SHA-256(压缩数据),
    永久保留标记: False（默认），True（人工标记）
}
```


## 核心处理逻辑

```
FUNCTION cold_storage_manager_main_loop():
    STATE_NORMAL = NORMAL
    STATE_COMPRESS = COMPRESSING
    STATE_RESTORE = RESTORING
    STATE_COLD_WARN = COLD_WARN
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_NORMAL
    初始化冷存储分区（文件系统路径或数据库）
    加载冷存储配置（容量、压缩参数、阈值）

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_NORMAL

        // 第2步：接收归档写入指令
        IF 收到 ag-mem-42 归档写入指令:
            SET internal_state = STATE_COMPRESS

            // 2a. 检查冷存储容量
            冷存储使用率 = 查询冷存储使用率()
            IF 冷存储使用率 >= 80%:
                SET internal_state = STATE_COLD_WARN
                触发冷存储清理(目标使用率=70%)

            IF 冷存储使用率 >= 95%:
                向 ag-mem-42 返回归档操作确认(
                    条目ID=指令.条目ID,
                    归档状态="失败",
                    归档存储位置=None,
                    压缩前大小=指令.经验数据.大小,
                    压缩后大小=0,
                    压缩比=0,
                    失败原因="冷存储容量已满"
                )
                SET internal_state = STATE_NORMAL
                CONTINUE

            // 2b. 执行压缩
            压缩开始时间 = NOW()
            压缩结果 = Zstandard压缩(指令.经验数据, 压缩级别=3)
            压缩耗时 = NOW() - 压缩开始时间

            IF 压缩结果.失败 OR 压缩耗时 > 10秒:
                向 ag-mem-42 返回归档操作确认(
                    条目ID=指令.条目ID,
                    归档状态="失败",
                    失败原因="压缩失败或超时"
                )
                SET internal_state = STATE_NORMAL
                CONTINUE

            // 2c. 构建归档条目
            归档条目 = 构建归档条目(
                原始条目ID=指令.条目ID,
                压缩数据=压缩结果.数据,
                压缩前大小=指令.经验数据.大小,
                压缩后大小=压缩结果.大小,
                元数据=指令.元数据
            )

            // 2d. 写入冷存储
            存储位置 = 冷存储分区.写入(归档条目)
            IF 存储位置 == None:
                向 ag-mem-42 返回归档操作确认(失败, "冷存储写入异常")
                SET internal_state = STATE_NORMAL
                CONTINUE

            // 2e. 返回确认
            压缩比 = ROUND(1 - 压缩结果.大小 / 指令.经验数据.大小, 2)
            向 ag-mem-42 返回归档操作确认(
                条目ID=指令.条目ID,
                归档状态="成功",
                归档存储位置=存储位置,
                压缩前大小=指令.经验数据.大小,
                压缩后大小=压缩结果.大小,
                压缩比=压缩比
            )

            // 2f. 记录归档事件日志
            向 ag-mem-51 发送归档事件日志("归档", [指令.条目ID], "成功")

            SET internal_state = STATE_NORMAL

        // 第3步：接收归档恢复请求
        IF 收到归档恢复请求:
            SET internal_state = STATE_RESTORE
            恢复条目列表 = []

            // 3a. 按条件检索冷存储
            IF 请求.条目ID列表 != None:
                检索范围 = 请求.条目ID列表
            ELSE IF 请求.时间范围 != None:
                检索范围 = 冷存储分区.按时间范围检索(请求.时间范围, 请求.来源层级)
            ELSE:
                向请求来源返回归档恢复结果(空, "查询条件无效")
                SET internal_state = STATE_NORMAL
                CONTINUE

            // 3b. 逐条恢复
            FOR EACH 检索条目 IN 检索范围:
                // 读取压缩数据
                归档数据 = 冷存储分区.读取(检索条目.归档ID)
                IF 归档数据 == None:
                    CONTINUE

                // 校验数据完整性
                校验和 = SHA256(归档数据.压缩数据)
                IF 校验和 != 归档数据.校验和:
                    记录异常日志("归档数据校验失败", 检索条目.归档ID)
                    CONTINUE

                // 解压
                解压结果 = Zstandard解压(归档数据.压缩数据)
                IF 解压结果.失败:
                    CONTINUE

                // 重建原始数据结构
                恢复条目 = 重建经验条目(解压结果.数据, 归档数据.元数据)
                恢复条目列表.append(恢复条目)

                IF 恢复条目列表.size >= 请求.最大恢复条目数:
                    BREAK

            // 3c. 返回恢复结果
            向请求来源返回归档恢复结果(恢复条目列表, 恢复耗时)
            SET internal_state = STATE_NORMAL

        // 第4步：冷存储容量管理
        IF internal_state == STATE_COLD_WARN:
            // 触发冷存储清理
            清理结果 = 冷存储分区.按归档时间清理(保留最近条目, 目标使用率=70%)
            向 ag-mem-51 发送归档事件日志("清理", 清理结果.清理条目列表, "容量预警触发")
            IF 冷存储使用率 < 70%:
                SET internal_state = STATE_NORMAL

        // 第5步：周期性状态上报
        IF 距上次状态上报 >= 120秒:
            向 ag-mem-48 和 ag-mem-03 上报冷存储状态(
                当前状态=internal_state,
                总容量=冷存储总容量,
                已用容量=冷存储当前使用量,
                使用率=冷存储使用率,
                归档条目总数=冷存储分区.条目总数
            )

        SLEEP 50ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 压缩操作超时（>10秒） | 标记该条目归档失败，跳过，继续处理下一条 | — |
| 冷存储写入异常（磁盘满/IO错误） | 返回归档失败，上报告警 | 存储恢复正常 |
| 归档数据校验失败（SHA-256不匹配） | 标记该条数据损坏，跳过恢复，记录异常日志 | 冷存储数据修复 |
| 冷存储使用率达95%且无旧数据可清理 | 拒绝新归档写入，上报告警，提示扩容 | 人工清理或扩容 |
| 恢复请求的条目ID在冷存储中不存在 | 跳过该ID，仅返回存在的条目 | — |
| 紧急熔断 | 完成当前正在处理的归档写入（保证数据一致性），暂停新操作 | 紧急解除后恢复 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| 内部调度总线 | 读 | 归档写入指令 | 只读 | ag-mem-42 发送 |
| 内部调度总线 / MemoryBus | 读 | 归档恢复请求 | 只读 | 管理接口、离线复盘系统、ag-mem-01 |
| 内部调度总线 | 写 | 归档操作确认 | 专属写入 | 向 ag-mem-42 返回 |
| 内部调度总线 / MemoryBus | 写 | 归档恢复结果 | 专属写入 | 向请求来源返回 |
| 内部调度总线 | 写 | 冷存储状态上报 | 周期性写入 | 向 ag-mem-48/03 发送 |
| 内部调度总线 | 写 | 归档事件日志 | 事件触发写入 | 向 ag-mem-51 发送 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| C-01 | 冷存储中的归档数据采用 SHA-256 校验和，恢复时必须验证完整性 |
| C-02 | 冷存储数据默认保留期限为归档后 1 年，超期数据在容量清理时优先删除（人工标记永久保留的除外） |
| C-03 | 归档操作必须保证原子性：压缩+写入全部成功才返回成功，任一步骤失败均标记为归档失败 |
| C-04 | 冷存储分区独立于热存储（五层存储），互不影响 |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-M49-01 | `NORMAL`，冷存储使用率50% | 归档写入指令（L3条目，经验数据大小=15KB） | 压缩成功，返回存储位置，压缩比 > 0.3 |
| TC-M49-02 | `NORMAL`，冷存储使用率85% | 归档写入指令 | 触发冷存储清理（目标70%），清理后写入成功 |
| TC-M49-03 | `NORMAL`，归档恢复请求 | 恢复请求（条目ID列表含3个已归档ID） | 读取→校验→解压→重建，返回3条恢复条目 |
| TC-M49-04 | `NORMAL`，归档恢复请求（部分ID不存在） | 恢复请求（条目ID列表含2个存在+1个不存在） | 返回2条恢复条目，跳过1条不存在 |
| TC-M49-05 | `NORMAL`，归档数据校验失败 | 恢复请求 + 归档数据SHA-256不匹配 | 跳过该校验失败条目，记录异常日志 |
| TC-M49-06 | `NORMAL`，冷存储容量达95% | 归档写入指令 | 拒绝写入，返回“冷存储容量已满” |
