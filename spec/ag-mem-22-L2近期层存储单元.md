# ag-mem-22-L2近期层存储单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-mem-22 |
| 模块名称 | L2近期层存储单元 |
| 所属分区 | 三、漏斗二：任务经验漏斗 / 五层存储 |
| 核心职责 | 作为漏斗二五层记忆存储架构的第二层，专门存储近7日内从L1晋升而来的频发场景经验。L2是经验从"临时暂存"进入"近期有效"的关键过渡层，占漏斗二总容量的25%。接收 ag-mem-21（L1时序衰减单元）的晋升条目，管理L2层的容量与条目生命周期，为L3中期层提供晋升候选。不参与晋升判定或遗忘决策，仅执行经验的接收、存储与基础管理 |
| 依赖模块 | ag-mem-21（L1临时层时序衰减单元，发送晋升条目列表）、ag-mem-23（L2近期层热度统计单元，消费条目访问记录）、ag-mem-24（L3中期层存储单元，接收晋升条目）、ag-mem-48（全局容量配额管控单元，查询L2当前容量） |
| 被依赖模块 | ag-mem-21（返回写入确认）、ag-mem-23（接收新条目通知用于热度统计）、ag-mem-15~19（各场景分槽，通过查询接口检索L2经验） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常服务 | `NORMAL` | L2存储正常，接收晋升条目 | 系统初始化完成，存储分区就绪 |
| 容量预警 | `CAPACITY_WARNING` | L2存储使用率超过80%，触发温和清理 | L2使用率 ≥ 80% |
| 容量紧急 | `CAPACITY_CRITICAL` | L2存储使用率超过95%，暂停写入并强制清理 | L2使用率 ≥ 95% |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| L2晋升条目列表 | List（条目ID + 经验数据 + 重要度I值 + 来源分槽编号 + 原L1写入时间戳） | ag-mem-21 L1临时层时序衰减单元（通过内部调度总线） | L1衰减评估判定晋升时 | **高** |
| L2经验查询请求 | Struct（查询条件 + 来源分槽编号 + 最大返回条数） | ag-mem-15~19 各场景分槽（通过内部调度总线） | 各分槽需要检索L2近期经验时 | **高** |
| L2清理完成确认 | Struct（清理条目数 + 释放空间量 + 剩余使用率） | ag-mem-23/40（通过内部调度总线） | 衰减清理执行完成后 | **高** |
| L2容量查询回执 | Struct（当前条目数 + 当前使用率 + 可用空间） | ag-mem-48 全局容量配额管控单元 | 每次写入前查询 | **高** |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-mem-01 总控漏斗F0 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| L2写入确认回执 | Struct（接收条目数 + 成功写入数 + L2当前使用率） | ag-mem-21 L1衰减评估单元 | 写入完成时 | **高** |
| L2查询结果列表 | Struct（匹配经验列表 + 各条目层级标记 + 重要度 + 最近访问时间） | 来源分槽（ag-mem-15~19） | 查询完成时 | **高** |
| L2新条目通知 | Struct（新增条目ID列表 + 来源分槽编号） | ag-mem-23 L2热度统计单元 | 每次成功接收晋升条目后 | 普通 |
| L2状态上报 | Struct（当前状态 + 条目总数 + 使用率 + 近7日写入量 + 近7日查询量） | ag-mem-48（容量管控）、ag-mem-03（调度单元） | 周期性（每30秒）或状态变更时 | 普通 |


## L2存储配置

| 配置项 | 默认值 | 说明 |
|--------|:---:|------|
| L2占漏斗二总容量比例 | **25%** | L2是第二大存储层，承载近期高频经验 |
| L2最大条目数 | 5000条 | 硬上限，达到后强制触发清理 |
| L2条目最大留存时长 | **7天**（168小时） | 超过此时长的条目必须晋升至L3或清除 |
| L2单条目最大大小 | 15KB | 略大于L1，允许更丰富的经验数据 |
| 写入超时 | 200ms | 单次写入操作的最大等待时间 |
| 容量预警阈值 | 80% | 触发温和清理 |
| 容量紧急阈值 | 95% | 触发强制清理并暂停写入 |


## 核心处理逻辑

```
FUNCTION l2_storage_main_loop():
    STATE_NORMAL = NORMAL
    STATE_WARN = CAPACITY_WARNING
    STATE_CRITICAL = CAPACITY_CRITICAL
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_NORMAL
    初始化L2存储分区（内存索引 + 持久化存储）
    初始化条目计数器 = 0

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_NORMAL

        // 第2步：接收L1晋升条目列表
        IF 收到 ag-mem-21 L2晋升条目列表:
            IF internal_state == STATE_PAUSED:
                向 ag-mem-21 返回L2写入确认回执(接收数=0, 成功数=0, "系统熔断中")
                CONTINUE

            // 2a. 查询当前L2容量
            容量回执 = 向 ag-mem-48 查询L2当前容量()
            L2使用率 = 容量回执.当前使用率

            // 2b. 容量状态更新
            IF L2使用率 >= 95%:
                SET internal_state = STATE_CRITICAL
            ELSE IF L2使用率 >= 80%:
                SET internal_state = STATE_WARN

            // 2c. 容量紧急处理
            IF internal_state == STATE_CRITICAL:
                // 检查是否已达硬上限
                IF 条目计数器 >= L2最大条目数:
                    // 强制清理：清除L2中重要度最低且留存超过7天的条目
                    过期条目 = L2存储.筛选(留存时长 > 7天).按重要度升序排列().取前20%
                    向 ag-mem-42 发送清除请求(过期条目)
                    等待清理结果
                    IF 清理后使用率仍 >= 95%:
                        // 仍不足，清除重要度最低的非过期条目
                        低重要度条目 = L2存储.按重要度升序排列().取前10%
                        向 ag-mem-42 发送清除请求(低重要度条目)
                        等待清理结果

            // 2d. 容量预警处理（温和清理）
            IF internal_state == STATE_WARN:
                // 异步清理超过7天留存且I值低于遗忘阈值的条目
                过期低价值条目 = L2存储.筛选(留存时长 > 7天 AND I值 < 0.20)
                IF 过期低价值条目 非空:
                    向 ag-mem-42 发送清除请求(过期低价值条目)

            // 2e. 逐条写入L2
            成功写入数 = 0
            新增条目ID列表 = []
            FOR EACH 晋升条目 IN 晋升条目列表:
                // 校验条目大小
                IF 晋升条目.经验数据.大小 > 15KB:
                    CONTINUE  // 跳过超大条目，保留在L1

                // 写入L2存储
                存储位置 = L2存储分区.追加写入(
                    条目ID=晋升条目.条目ID,
                    经验数据=晋升条目.经验数据,
                    重要度I值=晋升条目.重要度I值,
                    来源分槽=晋升条目.来源分槽编号,
                    原L1写入时间=晋升条目.原L1写入时间戳,
                    晋升时间=NOW()
                )
                IF 存储位置 != None:
                    成功写入数++
                    新增条目ID列表.append(晋升条目.条目ID)
                    条目计数器++

            // 2f. 通知L2热度统计单元
            IF 新增条目ID列表 非空:
                向 ag-mem-23 发送L2新条目通知(新增条目ID列表, 来源分槽编号)

            // 2g. 返回确认
            向 ag-mem-21 返回L2写入确认回执(
                接收条目数=晋升条目列表.size,
                成功写入数=成功写入数,
                L2当前使用率=L2使用率
            )

        // 第3步：响应经验查询请求
        IF 收到来自 ag-mem-15~19 的L2经验查询请求:
            查询条件 = 请求.查询条件
            来源分槽 = 请求.来源分槽编号

            // 3a. 检索匹配条目（仅返回来源分槽匹配的条目）
            匹配条目 = L2存储.查询(查询条件, 来源分槽, 请求.最大返回条数)
            // 更新条目的最近访问时间
            FOR EACH 条目 IN 匹配条目:
                L2存储.更新访问时间(条目.条目ID, NOW())

            // 3b. 返回结果
            向来源分槽返回L2查询结果列表(匹配条目, 层级="L2")

        // 第4步：定时清理超期条目（每小时）
        IF 距上次定时清理 >= 1小时:
            // 清除在L2留存超过7天的条目（无论I值高低）
            超期条目 = L2存储.筛选(留存时长 > 7天)
            IF 超期条目 非空:
                // 对于I值较高的超期条目，尝试晋升至L3而非直接清除
                FOR EACH 条目 IN 超期条目:
                    IF 条目.重要度I值 >= L2→L3晋升阈值:
                        向 ag-mem-24 发送晋升条目(条目)
                    ELSE:
                        向 ag-mem-42 发送清除条目(条目)
                条目计数器 -= 超期条目.size
            重置定时清理计时器

        // 第5步：周期性状态上报
        IF 距上次状态上报 >= 30秒:
            统计近7日写入量、查询量
            向 ag-mem-48 和 ag-mem-03 上报L2状态(
                当前状态=internal_state,
                条目总数=条目计数器,
                使用率=L2使用率,
                近7日写入量=写入统计,
                近7日查询量=查询统计
            )

        SLEEP 10ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| L2存储写入异常（磁盘满/IO错误） | 返回部分成功写入数，上报告警 | 存储恢复正常 |
| 条目大小超过15KB上限 | 跳过该条目，保留在L1，记录跳过日志 | — |
| L2使用率达95%且清理后仍无法释放空间 | 拒绝新写入，上报告警 | 清理成功或人工扩容 |
| L2中条目留存超过7天 | 高I值条目尝试晋升L3，低I值条目直接清除 | — |
| 紧急熔断 | 暂停所有写入，保持L2数据完整性 | 紧急解除后恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | L2层接收的晋升条目必须保留其原始来源分槽编号，用于分槽查询隔离 |
| S-02 | L2层条目在留存超过7天后必须处理（晋升或清除），不得无限期滞留 |
| S-03 | 分槽查询时，L2仅返回与请求来源分槽匹配的条目，不得跨槽返回其他分槽的经验 |
| S-04 | L2存储的持久化写入必须保证原子性，写入中断时不产生损坏的半条记录 |
